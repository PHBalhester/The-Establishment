# H098: SLOT_BROADCAST_INTERVAL_MS No Bounds

**Verdict:** CONFIRMED

## Summary

Environment variables controlling polling/broadcast intervals are parsed from `process.env` without min/max validation. Setting any interval to 0 or negative values causes event firehose behavior.

## Details

### Location
`app/lib/ws-subscriber.ts:251-254, 330-333, 369-372`

### Affected Intervals

1. **SLOT_BROADCAST_INTERVAL_MS** (line 251-254)
   ```typescript
   const BROADCAST_INTERVAL = parseInt(
     process.env.SLOT_BROADCAST_INTERVAL_MS ?? "5000",
     10,
   );
   ```
   Used in slot throttle at line 263: `if (now - state.lastSlotBroadcastAt >= BROADCAST_INTERVAL)`
   - If set to 0: broadcasts on every slot change (firehose)
   - If negative: always passes throttle check

2. **TOKEN_SUPPLY_POLL_INTERVAL_MS** (line 330-333)
   ```typescript
   const interval = parseInt(
     process.env.TOKEN_SUPPLY_POLL_INTERVAL_MS ?? "60000",
     10,
   );
   ```
   Passed directly to `setInterval()` at line 360

3. **STAKER_COUNT_POLL_INTERVAL_MS** (line 369-372)
   ```typescript
   const interval = parseInt(
     process.env.STAKER_COUNT_POLL_INTERVAL_MS ?? "30000",
     10,
   );
   ```
   Passed directly to `setInterval()` at line 430

### Impact

- **Slot broadcasts**: Setting `SLOT_BROADCAST_INTERVAL_MS=0` removes throttling entirely, flooding `protocolStore` with redundant slot updates on every received slot change
- **Supply polls**: `setInterval(pollSupply, 0)` creates tight loop of API calls, hammering RPC quota
- **Staker polls**: `setInterval(pollStakers, 0)` creates tight loop of gPA calls, hammering RPC quota

### Attack Surface

Environment variables are often injected at deployment time (Railway, Docker) and may be controlled by a deployer with less scrutiny than code review.

## Remediation

Add min/max bounds validation immediately after parsing:

```typescript
// For SLOT_BROADCAST_INTERVAL_MS
const BROADCAST_INTERVAL = Math.max(
  100,
  Math.min(
    parseInt(process.env.SLOT_BROADCAST_INTERVAL_MS ?? "5000", 10),
    300_000 // 5 min max
  )
);

// For poll intervals
const interval = Math.max(
  1000, // 1s minimum
  Math.min(
    parseInt(process.env.TOKEN_SUPPLY_POLL_INTERVAL_MS ?? "60000", 10),
    600_000 // 10 min max
  )
);
```

Log parsed vs. clamped values for visibility.

## CWE

- **CWE-20: Improper Input Validation**
