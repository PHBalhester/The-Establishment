**Status**: CONFIRMED
**Severity**: Tier 3 (Transitive)

## Impact Assessment

### Security Risk
- **glob@7.x**: Contains "widely publicized security vulnerabilities" per npm registry
- **inflight@1.0.6**: Memory leak vulnerability; not actively maintained

### Practical Risk
- These packages are used only in **development tooling** (react-native, babel, jest infrastructure)
- They are **NOT** bundled into production code or the Solana program binaries
- They are **NOT** used at runtime in the application
- Risk is primarily to the **build pipeline and developer machine security**

### Why They Persist
1. **Pulled in transitively**: `react-native@0.84.0` (from wallet adapter) specifies glob@7.x
2. **Not in direct dependencies**: Neither is directly required by the project
3. **Upstream constraint**: Only solution is for upstream packages to upgrade their constraints

## Remediation Path

### Option A: Update Upstream Dependencies (Recommended)
- Wait for `@solana-mobile/wallet-adapter-mobile` to upgrade `react-native` or remove it
- Once updated, `rimraf` and `test-exclude` will specify `glob@^10.x`
- This is the cleanest solution but depends on upstream maintainers

### Option B: Force Upgrade via npm overrides (Risky)
- Add to root `package.json`:
  ```json
  "overrides": {
    "glob": "^10.5.0",
    "inflight": null
  }
```
- May break `rimraf@3.0.2` and `test-exclude@6.0.0` if they're incompatible with glob@10
- Requires testing the build pipeline

### Option C: Remove React-Native Dependency (Nuclear)
- `react-native` is only required for the wallet adapter's mobile support
- If mobile wallet support is not critical, consider removing `@solana-mobile/wallet-adapter-mobile`
- Would eliminate the entire chain

## Recommendation
**Keep as-is for now** with the following reasoning:
1. **No production impact**: These are build-time only, not shipped to users
2. **Blocked upstream**: Can't fix without upstream action
3. **Breaking risk**: Forcing upgrades may break the React Native toolchain
4. **Low probability exploit**: These are tools used locally during development, not exposed to network attack

**Monitor for updates**: Watch for `@solana-mobile/wallet-adapter-mobile` to publish new versions that upgrade their React Native dependency.
