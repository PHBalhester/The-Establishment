**Severity**: Tier 3 (Design Risk)

## Summary
Graduation process is irreversible once `prepare_transition` executes. This is a design choice, not a bug — the state machine intentionally uses a terminal Graduated state with no recovery path.

Steps 1–12 are idempotent; step 2 (prepare_transition) is the irreversible gate:
1. Verify curves Filled
2. **[IRREVERSIBLE] prepare_transition** → Filled → Graduated
3–6. Withdraw SOL, close vaults
7–8. Create AMM pools
9. Whitelist pool vaults
10. Seed Conversion Vault
11–12. Distribute tax escrows
13. Skip whitelist authority burn (retained to Squads)

## Risk Assessment

### Accepted Because
- **Intentional Design**: Protocol by design graduates once to enable AMM phase
- **Admin Gated**: Manual operation — not autonomous
- **Documented**: Users warned via console output with red text formatting
- **Checkpoint/Resume**: Failures mid-graduation can resume from last completed step
- **Prerequisite Validation**: Bonding curves must be Filled before allowing transition

### Mitigation
- Admin runs on testnet first to validate curve state
- Graduation state persisted to JSON — recovery from step N without re-running 1 to N-1
- Explicit user confirmation required before executing (via console prompt or explicit flag)

## Verdict
**ACCEPTED_RISK** — graduation is operationally irreversible by design. No remediation required. This is a documented architectural decision with appropriate manual gatekeeping.
