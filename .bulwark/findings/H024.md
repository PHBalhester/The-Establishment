## Verdict

**PARTIALLY_FIXED with one live residual risk and one structural time-bomb.**

The original H002 finding — Helius API key baked into the browser bundle — has been substantially fixed. The RPC proxy pattern (`/api/rpc`) is implemented and working. The browser-facing `ConnectionProvider` endpoint and the wallet adapter both correctly point to `/api/rpc`. `DEVNET_RPC_URL` from `shared/programs.ts` is exported by `shared/index.ts` but **zero `app/` files import it**, confirming the fix holds.

However, two residual issues survive:

1. **`NEXT_PUBLIC_RPC_URL` is set in `app/.env.local` (devnet) and in `app/.env.mainnet` (template) with the full Helius API key embedded.** Because `NEXT_PUBLIC_` variables are inlined at Next.js build time, this bakes the key into the client-side JavaScript bundle. The key is the same devnet key across all committed/local config. For devnet this is low financial impact; for mainnet it is a **HIGH severity production exposure vector** when the Railway operator follows the template.

2. **`shared/programs.ts` still exports `DEVNET_RPC_URL` with the hardcoded key**, and `shared/index.ts:53` re-exports it. No `app/` code imports it today, but the export path remains live. A future developer importing this constant will bypass the proxy pattern with zero compiler warning.

### Evidence 6 — `NEXT_PUBLIC_RPC_URL` is server-only in practice, but Next.js bundles it client-side at build time

Because the variable has the `NEXT_PUBLIC_` prefix, Next.js **inlines its value into the JavaScript bundle** at build time regardless of whether any `typeof window` guard prevents it from being read at runtime. Any user who downloads the page source or runs a bundle analyzer will find the Helius API key embedded as a string literal in the production bundle.

This is a structural property of Next.js `NEXT_PUBLIC_` variables — they are compile-time substitutions, not runtime-filtered env vars.

## Attack Path

**Devnet (current):**
1. Attacker loads the deployed Railway app in a browser.
2. Opens DevTools → Network or runs `strings` on downloaded JS chunks.
3. Searches for `helius-rpc.com` → finds `?api-key=[REDACTED-DEVNET-KEY]-...`.
4. Uses the key to call `POST https://api.helius.xyz/v0/webhooks?api-key=<key>` to create, modify, or delete production webhooks — blinding the data pipeline or redirecting events to an attacker-controlled endpoint.

**Mainnet (if Railway operator follows the template):**
1. Operator copies `app/.env.mainnet` template into Railway env vars, setting `NEXT_PUBLIC_RPC_URL=https://mainnet.helius-rpc.com/?api-key=<REAL_KEY>`.
2. Railway triggers Next.js build; key is inlined into JS bundle.
3. Attacker extracts key from public JavaScript → exhausts Helius credit quota → blinds the data pipeline → or uses the key against the Helius API to modify webhook configuration.
4. Impact: complete loss of SSE data pipeline + RPC quota for every user of the production dApp.

## Recommendation

### R1 — Remove `NEXT_PUBLIC_RPC_URL` from both env files (HIGH PRIORITY before mainnet)

In `app/.env.local`, remove or blank the line:
```diff
-NEXT_PUBLIC_RPC_URL=https://devnet.helius-rpc.com/?api-key=[REDACTED-DEVNET-KEY]-...
```

In `app/.env.mainnet`, remove the variable entirely:
```diff
-# RPC URL for browser-side Solana connections (legacy, now proxied through /api/rpc).
-# Still referenced by some wallet adapter config. Use mainnet Helius URL.
-NEXT_PUBLIC_RPC_URL=https://mainnet.helius-rpc.com/?api-key=CHANGE_ME_MAINNET
```

The comment in `.env.mainnet` is also wrong ("Still referenced by some wallet adapter config") — `providers.tsx` hardcodes `/api/rpc`. This incorrect guidance will cause the next Railway operator to set a dangerous variable.

### R2 — Remove `NEXT_PUBLIC_RPC_URL` from the server-side fallback chains

In `app/lib/connection.ts:41`, remove the fallback:
```diff
-const serverUrl = process.env.HELIUS_RPC_URL ?? process.env.NEXT_PUBLIC_RPC_URL;
+const serverUrl = process.env.HELIUS_RPC_URL;
```

In `app/app/api/rpc/route.ts:131`, remove from the endpoint array:
```diff
 const endpoints = [
   process.env.HELIUS_RPC_URL,
   process.env.HELIUS_RPC_URL_FALLBACK,
-  process.env.NEXT_PUBLIC_RPC_URL,
 ].filter(Boolean) as string[];
```

If `NEXT_PUBLIC_RPC_URL` is never set, these fallbacks are harmless but send a confusing signal. Removing them eliminates the template instruction to set the variable.

### R3 — Remove or rename `DEVNET_RPC_URL` from `shared/programs.ts`

Either delete the constant or replace the value with a runtime env var read so the key cannot be statically bundled:
```diff
-export const DEVNET_RPC_URL =
-  "https://devnet.helius-rpc.com/?api-key=[REDACTED-DEVNET-HELIUS-KEY]";
+// DEVNET_RPC_URL removed -- use HELIUS_RPC_URL env var. Never export a credential as a constant.
```

Also remove the corresponding export from `shared/index.ts:53`.

### R4 — Rotate the devnet Helius API key

The key `[REDACTED-DEVNET-HELIUS-KEY]` is committed in git history (`.env.devnet`), in the working tree (`.env.local`, `shared/programs.ts`), and is baked into the devnet bundle on Railway. Rotation does not remove it from git history but limits ongoing exposure from the running bundle.
