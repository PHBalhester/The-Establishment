# H127: DB Without TLS in Staging (Reference H093)

**Severity**: Tier 3 (Medium)
**Category**: Data/Encryption (DATA-05)
**Status**: CONFIRMED
**Target**: `app/db/connection.ts:51-52`

## Summary

Database connections enforce TLS **only in production** environments. Staging and development may transmit database credentials and queries in plaintext if connected to remote hosts.

## Verification against H093

This finding confirms the analysis documented in H093 (DB TLS Only in Production):

### Quick Reference

**Code (connection.ts:51-52)**:
```typescript
const isProductionDb = process.env.NODE_ENV === "production";
const sslConfig = isProductionDb ? { ssl: "require" as const } : {};
```

**Finding**: TLS gated on `NODE_ENV === "production"` check. Non-production environments (staging, preview) skip TLS encryption.

**Existing Mitigation** (lines 59-74):
- Advisory console warning (`VH-L002`) logs when non-production connects to remote hosts
- Warning is **informational only** — connection still proceeds unencrypted
- Localhost connections (127.0.0.1, ::1) bypass warning

### Secondary Issue (migrate.ts)

Migration runner at `app/db/migrate.ts:42` applies **no TLS options at all**:
```typescript
const client = postgres(databaseUrl, { max: 1 });
```

Runs unencrypted regardless of NODE_ENV or target host.

## Recommendation

**Adopt Option 1 from H093**: Always require TLS in both connection.ts and migrate.ts. Railway Postgres supports TLS by default; zero operational burden.
