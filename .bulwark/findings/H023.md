const webhookSecret = process.env.HELIUS_WEBHOOK_SECRET;
const isProduction = process.env.NODE_ENV === "production";

if (!webhookSecret && isProduction) {
  console.error("[webhook] CRITICAL: HELIUS_WEBHOOK_SECRET is not set in production. ...");
  captureException(new Error("[webhook] HELIUS_WEBHOOK_SECRET not set in production"));
  return NextResponse.json(

    { status: 500 },
  );
}

if (webhookSecret) {
  const authHeader = req.headers.get("authorization") ?? "";
  // H001: Constant-time comparison prevents timing side-channel attacks.
  const secretBuf = Buffer.from(webhookSecret, "utf-8");
  const headerBuf = Buffer.from(authHeader, "utf-8");
  const lengthMatch = secretBuf.length === headerBuf.length;
  const compareBuf = lengthMatch ? headerBuf : secretBuf;
  if (!lengthMatch || !timingSafeEqual(secretBuf, compareBuf)) {
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
  }
}

## Verdict

**RESOLVED** — No regression. The webhook authentication is intact post-DBS refactor.

All three pillars of the H001 fix survive:
1. **Fail-closed production guard**: Missing `HELIUS_WEBHOOK_SECRET` in production returns 500 and fires Sentry, blocking all unauthenticated processing.
2. **Constant-time comparison**: `timingSafeEqual` is used with correct length-mismatch path that avoids timing leakage.
3. **No new bypass paths**: DBS refactor added zero new HTTP method handlers and only modified decode logic inside the post-auth `handleAccountChanges()` function.

---

## Attack Path

Not exploitable. There is no path to reach payload processing without passing the auth gate. The only way to bypass authentication is if `NODE_ENV !== "production"` AND `HELIUS_WEBHOOK_SECRET` is unset — the intentional local development bypass documented in the file header. In production (Railway sets `NODE_ENV=production`), both conditions of the fail-closed guard prevent unauthenticated access.

## Recommendation

No action required for this finding. The existing auth is correctly implemented and survived the DBS refactor untouched.

**Standing observation carried from H001 / AUTH-01 auditor (low severity, not a regression):** The webhook secret is a static bearer token in the `Authorization` header, not an HMAC-SHA256 body signature. This is Helius's protocol design, not a code defect. Body integrity in transit is covered by HTTPS. The primary residual risk (enhanced webhook replay) is a separate finding tracked under the API-04 audit area.
