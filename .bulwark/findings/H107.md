# H107: Smoke Test Logs Raw CLUSTER_URL

**Severity**: Tier 3 (Information Disclosure)
**Status**: CONFIRMED
**Location**: `scripts/e2e/smoke-test.ts:36`

## Vulnerability Description

The smoke test directly logs the raw `CLUSTER_URL` environment variable to stdout without masking or sanitizing it:

```typescript
console.log(`Cluster: ${process.env.CLUSTER_URL}`);
```

If the RPC endpoint URL contains credentials (API keys, authentication tokens), they will be exposed in:
- CI/CD logs (GitHub Actions, etc.)
- Build server logs
- Local console output
- Any log aggregation system capturing stdout

## Impact

- **Information Disclosure**: API keys or RPC credentials exposed in logs
- **Unauthorized Access**: Attackers can use exposed keys to hit rate limits, drain account balances, or impersonate the service
- **Supply Chain Risk**: Third-party log viewers or archive systems may retain credentials

## Proof of Concept

1. Set `CLUSTER_URL="https://devnet.helius-rpc.com/?api-key=secret123"`
2. Run `npm run smoke` (or `npx tsx scripts/e2e/smoke-test.ts`)
3. Output contains full URL with API key:
   ```
   Cluster: https://devnet.helius-rpc.com/?api-key=secret123
   ```

## Remediation

Extract and mask the URL before logging. Example:

```typescript
const clusterUrl = process.env.CLUSTER_URL || "";
const maskedUrl = maskRpcUrl(clusterUrl);
console.log(`Cluster: ${maskedUrl}`);

function maskRpcUrl(url: string): string {
  if (!url) return "(not set)";
  try {
    const parsed = new URL(url);
    // Mask query parameters
    if (parsed.search) {
      return `${parsed.protocol}//${parsed.hostname}/?...`;
    }
    return `${parsed.protocol}//${parsed.hostname}`;
  } catch {
    return "(invalid URL)";
  }
}
```

Alternatively, strip the URL entirely and log only the hostname:

```typescript
const clusterUrl = process.env.CLUSTER_URL || "";
const hostname = new URL(clusterUrl).hostname;
console.log(`Cluster: ${hostname}`);
```

## Recommendation

- Apply masking to all logging of `CLUSTER_URL` and similar credential-bearing env vars
- Audit other E2E scripts (e.g., `scripts/e2e/*.ts`) for similar patterns
- Consider adding a security lint rule to catch `process.env.*` logging
