# H001: Private Key Extraction from .mcp.json in Git History

## Verdict: CONFIRMED

## Evidence

**The private key is real, is the primary devnet wallet key, and is in the pushed remote git history.**

1. **File exists and contains a real private key:**
   - `.mcp.json` line 8: `"SOLANA_PRIVATE_KEY": "2zJgKnGrwgkcDLks6uJYcMWThWYHUCp1RuShDGajeutHvp7uWALoAwtSJEnXb2sx2GeizxrApX4QZcXoRwUC2FSS"`
   - Key decodes to a valid 64-byte Solana keypair (verified by `bs58.decode` + `Keypair.fromSecretKey`)
   - Derived public key: `8kPzhQoUPx7LYM18f9TzskW4ZgvGyq4jMPYZikqmHMH4`

2. **This is the primary devnet wallet, currently holding 59.5 SOL:**
   - `CLAUDE.md` documents: `"Devnet wallet": 8kPzhQoUPx7LYM18f9TzskW4ZgvGyq4jMPYZikqmHMH4 (keypairs/devnet-wallet.json)`
   - The committed key in `.mcp.json` is identical to the devnet wallet keypair referenced throughout the project
   - Balance confirmed via RPC: **59.586 SOL** on devnet at time of audit

3. **Key is in git history since at least commit `53ca01b`, and that commit is pushed to `origin/main`:**
   ```
   git log --all --oneline -- .mcp.json:
   53ca01b feat: add 24/7 crank runner for Railway + SVK security tooling  ← real key introduced
   a5235b7 chore: clean slate for v4 rebuild                                ← real key persists
   c96d523 WIP: final state before v3 restart                               ← file absent
   551e9be refactor: consolidate fork into Dr Fraudsworth                   ← placeholder only
   ```
   - `git branch -r --contains 53ca01b` returns `origin/main` — the commit is on the remote

4. **`.mcp.json` is NOT in `.gitignore`:**
   - No entry found for `.mcp.json` or `*.mcp.json` in `.gitignore`
   - The file is actively tracked: `git ls-files .mcp.json` returns `.mcp.json`

5. **GitHub remote exists:** `https://github.com/MetalLegBob/dr-fraudsworth.git`
   - HTTP status 404 when accessed unauthenticated — consistent with a private repository
   - However: private repos can be leaked, collaborators can exfiltrate, and if the repo is ever made public (e.g., for open-source launch), the key is immediately exposed with no warning

6. **The key persists at HEAD:** The current working tree still contains the real private key in `.mcp.json`. This is not just a historical artifact — it is live in every `git clone` or `git checkout` of the current state.

## Attack Path

1. Attacker obtains a copy of the repository (via leaked collaborator access, accidental public exposure, or if repo goes public)
2. Runs: `git log -p .mcp.json` or `git show 53ca01b:.mcp.json`
3. Extracts: `2zJgKnGrwgkcDLks6uJYcMWThWYHUCp1RuShDGajeutHvp7uWALoAwtSJEnXb2sx2GeizxrApX4QZcXoRwUC2FSS`
4. Imports key into any Solana wallet (Phantom, Solflare, or CLI): `solana config set --keypair <(echo '[...]')`
5. Drains all SOL and tokens from `8kPzhQoUPx7LYM18f9TzskW4ZgvGyq4jMPYZikqmHMH4`
6. Transfers out any SPL tokens held by the wallet (CRIME, FRAUD, PROFIT, WSOL)
7. If this wallet holds upgrade authority over any programs, attacker can also deploy malicious program upgrades

Even after removing the key from HEAD, it remains permanently accessible via `git show 53ca01b:.mcp.json` unless git history is rewritten and all remote branches force-updated.

**Secondary attack vector — AI tool exfiltration:** `.mcp.json` is read by Claude Code and similar AI assistants at session start. Any AI tool that logs file contents to telemetry, sends context to a third-party LLM API, or stores conversation history on external servers has already received this private key. This is a novel, non-obvious exfiltration channel that traditional secret scanners would not detect.

## Impact

**CRITICAL.**

- The exposed wallet (`8kPzhQoUPx7LYM18f9TzskW4ZgvGyq4jMPYZikqmHMH4`) currently holds ~59.5 SOL on devnet
- This wallet is also the deployment authority for all devnet programs — compromise gives attacker program upgrade capability
- The wallet has been the active signing key for all deployment operations and may be reused in scripts that target mainnet indirectly
- If a single collaborator or CI/CD system has ever cloned this repo, the key is already outside owner control
- The GitHub remote exists and 1,922 commits of history are pushed — any future public disclosure of the repo exposes the key instantly
- Even removing the key from HEAD does not help without a full history rewrite and force-push — the key will remain in `git show 53ca01b` indefinitely otherwise

**On devnet:** Fund theft of ~59.5 SOL (direct monetary loss on devnet, but establishes attacker control of the deployment wallet).
**On transition to mainnet:** If the same keypair is reused anywhere near mainnet (as a seed, as a signer, as an intermediary), the impact is CRITICAL fund theft.

## Recommendation

This requires immediate action, not deferred remediation.

**Step 1 — Rotate the key immediately (URGENT):**
- Generate a new devnet wallet keypair: `solana-keygen new -o keypairs/devnet-wallet.json`
- Transfer all remaining funds from `8kPzhQoUPx7LYM18f9TzskW4ZgvGyq4jMPYZikqmHMH4` to the new wallet
- Revoke/rotate all program upgrade authorities currently held by the old key

**Step 2 — Remove `.mcp.json` from HEAD and add to `.gitignore`:**
- Add `.mcp.json` to `.gitignore` immediately
- Replace the private key value with `YOUR_DEVNET_PRIVATE_KEY_HERE` (the placeholder from commit `551e9be`)
- Commit: `git add .gitignore .mcp.json && git commit -m "security: remove private key from .mcp.json, add to .gitignore"`

**Step 3 — Purge git history (REQUIRED before any public repo access):**
- Use BFG Repo-Cleaner (recommended over `git filter-branch`):
  ```bash
  bfg --replace-text secrets.txt  # file contains the base58 key
  git reflog expire --expire=now --all
  git gc --prune=now --aggressive
  git push --force --all
  git push --force --tags
  ```
- All collaborators must delete local clones and re-clone after the force push
- GitHub's "cached views" may persist for up to 90 days — contact GitHub support to purge if needed

**Step 4 — Audit for key reuse:**
- Confirm the devnet wallet key is NOT reused as mainnet deployer, crank wallet, or any other mainnet identity
- The mainnet deployer (`23g7xmrtXA6LSWopQcAUgiptGUArSLEMakBKcY1S59YR`) and crank wallet (`F84XUxo5VM8FJZeGvC3CrHYwLzFod3ep57CULjZ4ZXc1`) are distinct — confirm neither is derived from or related to `8kPzhQoUPx7LYM18f9TzskW4ZgvGyq4jMPYZikqmHMH4`

**Step 5 — Long-term hygiene:**
- `.mcp.json` should use a reference to an external secret store or environment variable, not a hardcoded key
- Consider a pre-commit hook (e.g., `detect-secrets`, `truffleHog`, or `gitleaks`) to catch future secret commits

**Relationship to prior work:** Audit #1 finding H005 (Keypairs Committed to Git, PARTIALLY_FIXED, HIGH) covers the broader `keypairs/*.json` issue. H001 is distinct and more severe because: (a) it is a named `SOLANA_PRIVATE_KEY` field rather than a keypair JSON array (more accessible to attackers), (b) it controls a funded wallet rather than just program deploy authority, and (c) `.mcp.json` is not a recognized secret file type — it would bypass many automated scanners that focus on `.env` and `.json` keypair patterns.
