# H132: Railway Dashboard SPOA

**Severity:** Tier 3 (Informational)
**Category:** Infrastructure
**Status:** ACCEPTED_RISK

## Finding

Railway platform dashboard represents a single point of organizational access to production infrastructure. Compromise of Railway account credentials or dashboard session would grant an attacker:

- Full control of deployed services and environment variables
- Access to secrets, API keys, and wallet configuration
- Ability to modify, redeploy, or terminate production services
- Access to logs and observability data

## Risk Assessment

**Likelihood:** Low
- Railway enforces OAuth2 via GitHub
- Account security depends on GitHub credentials
- Dashboard requires active authentication session

**Impact:** Critical
- Complete production compromise
- Potential loss of funds if wallet credentials are stored in Railway env vars
- Service downtime

**Overall Risk:** Medium (Low likelihood × Critical impact)

## Mitigation

- Railway is trusted infrastructure operated by mature provider
- GitHub account protected by strong authentication
- Environment variables stored as Railway secrets (encrypted at rest)
- Access logged by Railway platform

## Decision

**ACCEPTED_RISK** — Railway dashboard compromise is a legitimate operational risk inherent to any cloud-hosted platform. The benefits of managed infrastructure, automated deployment, and observability outweigh the need for additional mitigation layers. This risk is acceptable as long as:

1. GitHub account remains secure (strong password + 2FA)
2. Production wallet secrets are rotated regularly
3. Railway session timeouts are enabled
4. Access is reviewed periodically
