# H077: No Compute Budget on BC TXs

**Severity**: Tier 3 (Medium)
**Status**: CONFIRMED
**Target**: `app/lib/curve/curve-tx-builder.ts` (buildPurchaseInstruction, buildSellInstruction)
**Related**: H041

---

## Finding

Bonding curve buy/sell transactions lack ComputeBudgetProgram instructions.

### Location
- **File**: `/app/lib/curve/curve-tx-builder.ts`
- **Functions**:
  - `buildPurchaseInstruction()` (lines 92-128)
  - `buildSellInstruction()` (lines 143-180)
  - Called by `BuyForm.tsx:174` and `SellForm.tsx` (equivalent)

### Evidence

**curve-tx-builder.ts** returns a single `TransactionInstruction` for each operation:

```typescript
return program.methods
  .purchase(...)
  .accountsStrict({...})
  .remainingAccounts(hookAccounts)
  .instruction();  // ← No ComputeBudgetProgram
```

When the caller (BuyForm.tsx:183-186) builds the transaction:

```typescript
const tx = new Transaction();
tx.add(instruction);  // ← Only the purchase instruction
tx.feePayer = publicKey;
```

**Zero ComputeBudgetProgram imports/calls** found in `curve-tx-builder.ts` (confirmed via grep).

### Contrast: Correct Pattern (swap-builders.ts)

Swap transactions properly include compute budget:

```typescript
const tx = new Transaction();
tx.add(ComputeBudgetProgram.setComputeUnitLimit({ units: computeUnits }));
if (priorityFeeMicroMicroLamports > 0) {
  tx.add(ComputeBudgetProgram.setComputeUnitPrice({ microLamports: priorityFeeMicroLamports }));
}
tx.add(swapInstruction);
```

### Impact

- BC transactions rely on default compute budget (~200K units)
- Purchase/sell instructions may exceed default budget under network congestion
- Transactions fail with "compute budget exceeded" without explicit budgeting
- No priority fee mechanism for BC operations (tx ordering/censorship risk on busy blocks)

### Fix Required

1. Import ComputeBudgetProgram from `@solana/web3.js`
2. Modify `buildPurchaseInstruction()` and `buildSellInstruction()` to return a **Transaction** (not just `TransactionInstruction`)
3. Or: Modify caller (BuyForm.tsx, SellForm.tsx) to inject ComputeBudgetProgram instructions before adding the curve instruction
4. Measure actual compute usage of purchase/sell via simulation to set realistic unit limit

---

**Verdict**: CONFIRMED — BC TX builders completely lack compute budget instructions. Requires fix before mainnet deployment.
