# H110: Shell Injection via WALLET Env Var

**Severity**: Tier 3 (Medium)
**Status**: CONFIRMED
**Target**: `scripts/deploy/verify-authority.ts:390-396`
**Category**: Command Injection

---

## Vulnerability Description

The `WALLET` environment variable is unsafely interpolated into a shell command via `execSync()`. While the path is wrapped in double quotes, shell metacharacters and command substitution patterns are still interpreted within double quotes, allowing an attacker to execute arbitrary shell commands.

---

## Proof of Concept

If an attacker sets:
```bash
export WALLET='$(whoami)/devnet-wallet.json'
# or
export WALLET='`id`/devnet-wallet.json'
# or
export WALLET='"; rm -rf / ; echo "x'
```

The command on line 390-396 becomes:
```bash
"..." program set-upgrade-authority ... --keypair "$(whoami)/devnet-wallet.json" ...
# Shell interprets $(whoami) even inside double quotes
```

---

## Root Cause

**Line 103**: Reads `WALLET` env var without validation
```typescript
const walletPath = process.env.WALLET || path.join(KEYPAIRS_DIR, "devnet-wallet.json");
```

**Lines 390-396**: Interpolates `walletPath` into `execSync()` command string
```typescript
execSync(
  `"${solanaCli}" program set-upgrade-authority ${cvProgramId} ` +
    `--new-upgrade-authority ${deployer.publicKey.toBase58()} ` +
    `--keypair "${walletPath}" ` +    // <-- walletPath from WALLET env var
    `-u ${cluster === "devnet" ? "devnet" : clusterUrl}`,
  { encoding: "utf8", stdio: "pipe", timeout: 30000 }
);
```

The path is quoted but double quotes do NOT prevent shell metacharacter interpretation for `` ` ``, `$()`, and some special characters.

---

## Impact

- **Confidentiality**: Attacker can read sensitive files, steal keypairs, exfiltrate environment variables
- **Integrity**: Attacker can modify files, corrupt deployment state, inject malicious authority transfers
- **Availability**: Attacker can crash the script, trigger DoS, consume resources

This is especially dangerous in CI/CD or automated deployment contexts where env vars may be partially controlled by untrusted input.

---

## Fix

Use `execFile()` with argument array instead of `execSync()` with shell string interpolation:

```typescript
import { execFileSync } from "child_process";

// Instead of execSync with string interpolation:
execFileSync(solanaCli, [
  "program",
  "set-upgrade-authority",
  cvProgramId,
  "--new-upgrade-authority",
  deployer.publicKey.toBase58(),
  "--keypair",
  walletPath,  // Passed as separate argument, NOT interpolated
  "-u",
  cluster === "devnet" ? "devnet" : clusterUrl,
], { encoding: "utf8", stdio: "pipe", timeout: 30000 });
```

Or validate the `WALLET` env var as a valid file path:
```typescript
const walletPath = process.env.WALLET || path.join(KEYPAIRS_DIR, "devnet-wallet.json");
const resolvedPath = path.resolve(walletPath);
const baseName = path.resolve(KEYPAIRS_DIR);

// Ensure path is within keypairs directory
if (!resolvedPath.startsWith(baseName)) {
  throw new Error(`WALLET path must be within ${KEYPAIRS_DIR}, got ${walletPath}`);
}

// Verify file exists
if (!fs.existsSync(resolvedPath)) {
  throw new Error(`Keypair file not found: ${resolvedPath}`);
}
```

## Remediation Priority

**High** — Apply `execFileSync()` pattern immediately (5-minute fix).
