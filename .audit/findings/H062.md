# H062: WhitelistAuthority burn_authority Idempotency

**Status:** CONFIRMED
**Severity:** LOW
**Category:** Access Control — Idempotency & Replay Safety

## Summary

The `burn_authority` instruction in Transfer Hook is deliberately idempotent: if authority is already `None`, the function returns `Ok(())` without error. While this is safe (no state corruption occurs), it creates a subtle behavioral asymmetry where any unauthorized signer can **call** the instruction and succeed silently, even though they are not the authority. The authority check is skipped if the authority is already burned.

## Code Evidence

**File:** `programs/transfer-hook/src/instructions/burn_authority.rs:26-29`

```rust
// IDEMPOTENT: If already burned, succeed silently (15-CONTEXT.md decision)
// This check MUST come before authority validation
if auth.authority.is_none() {
    msg!("Authority already burned, idempotent success");
    return Ok(());
}
```

Then at line 32-35, the authority check is ONLY performed if authority is not `None`:

```rust
// Now verify signer is the current authority
require!(
    auth.authority == Some(ctx.accounts.authority.key()),
    TransferHookError::Unauthorized
);
```

## Verdict

**Not Vulnerable.** The idempotent design is intentional and safe per line 11 comment ("This allows safe retry logic and scripts to be re-run without error"). The early return at line 26-29 is correct—once authority is `None`, no further state mutation is possible, so no further checks are needed.

However, the behavior is surprising: in normal Anchor patterns, an unauthorized signer would receive `Unauthorized` error. Here, they receive `Ok(())` because the check is skipped. This is documented behavior ("idempotent success") and does not violate the security model (burning authority is a one-way operation with no reversible state changes).

**Monitoring Note:** Off-chain watchers checking for authority burns should not expect an error if authority is already burned. The success response from an unknown signer implies the burn already occurred.
