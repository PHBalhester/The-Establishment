# S006: Fallback MEV + Carnage Suppression Alternation Strategy

**Status:** CONFIRMED
**Severity:** HIGH
**Category:** MEV, Economic, State Machine
**Inspired by:** H009, H010

## Summary

A sophisticated attacker can combine the Carnage suppression vector (H009) with the fallback MEV sandwich vector (H010) into a coordinated alternation strategy: suppress Carnage for multiple epochs to grow the SOL vault, then allow exactly one trigger to proceed through the fallback path and sandwich it, capturing a disproportionately large payout. The two individually-confirmed findings compound when operated together.

## Analysis

### The Compound Attack Flow

Each individual finding has been independently confirmed:

- **H009**: Any actor can suppress Carnage by submitting `consume_randomness` without `carnage_state`, costing ~0.000005 SOL per epoch, succeeding unconditionally.
- **H010**: When Carnage proceeds through the fallback path (slots 50–300 after trigger), the slippage floor is computed against live (manipulable) reserves, enabling a sandwich that extracts up to 25% of the Carnage swap value.

The compound strategy exploits the relationship between vault balance and extraction magnitude:

```
Phase 1 — Accumulation (N epochs):
  Each epoch where Carnage would have triggered:
    → Attacker submits consume_randomness with carnage_state=None
    → Carnage trigger is silently skipped
    → 24% of all swap taxes continue flowing into carnage_sol_vault
    → Vault balance grows without deployment

Phase 2 — Controlled Trigger (1 epoch):
  Attacker detects a Carnage-triggering VRF epoch (byte 5 < 11)
    → Attacker allows consume_randomness to include carnage_state
    → Carnage enters CARNAGE_PENDING state
    → Carnage locks for 50 slots (atomic window)
    [Atomic window: attacker cannot sandwich; they allow it to expire or fail]
    → After slot 50, fallback window opens (slots 50-300)
    → Attacker front-runs with a large buy of carnage_target token
    → Attacker submits execute_carnage (fallback)
    → Carnage executes against inflated reserves — fewer tokens bought
    → Attacker back-runs, profiting from Carnage's buy pressure

Phase 3 — Resume Suppression:
  → Repeat Phase 1 with remaining vault balance
```

### Why the Vault Balance Amplifies Extraction

The Carnage swap amount at `carnage_execution.rs:265` is:

```rust
let available_sol = sol_balance.saturating_sub(rent_exempt_min);
let swap_amount = std::cmp::min(available_sol, MAX_CARNAGE_SWAP_LAMPORTS);
```

The swap amount scales directly with vault balance, up to the 1000 SOL cap. A vault that would normally deploy 50 SOL becomes 200 SOL after 3 suppressed epochs (assuming 50 SOL/epoch net accumulation). The attacker's extraction scales proportionally — the 25% MEV tolerance extracts 50 SOL from the 200 SOL swing vs. 12.5 SOL from the 50 SOL swing.

### Optimal Suppression Window

The attacker's optimal strategy is to allow a trigger only when:
1. The vault has reached the MAX_CARNAGE_SWAP_LAMPORTS cap (1000 SOL), OR
2. The marginal profit from another suppressed epoch is less than the incremental extraction from the next sandwich

At 1000 SOL vault balance and 25% slippage tolerance: theoretical extraction = 250 SOL. At any lower balance, the attacker weighs suppression cost (~0.000005 SOL/epoch) vs. extraction gain.

For a vault accumulating 50 SOL/epoch (moderate protocol volume):
- 20 suppressed epochs → vault = 1000 SOL cap
- One fallback sandwich → ~250 SOL gross extraction
- Suppression cost: 20 × 0.000005 SOL = 0.0001 SOL (negligible)
- Net profit per cycle: ~250 SOL minus front-run spread cost

### Atomic Path Cannot Be Sandwiched

The atomic path bundles `consume_randomness` + `execute_carnage_atomic` in a single transaction. The target is unknown until execution, making atomic-path sandwiching impossible without precognition. The alternation strategy specifically targets the fallback path:

1. The attacker needs to ensure the atomic path fails for the chosen trigger epoch.
2. The atomic path fails when compute budget is exceeded or when the TX is too large. This is a pre-existing risk (CPI depth at 4/4 hard limit).
3. Alternatively: the attacker could ensure the atomic TX fails by front-running the `trigger_epoch_transition` with a pool state change that makes the atomic slippage floor fail immediately. Then the fallback path is the only remaining execution route.

### Economic Model: Suppression Value vs. Extraction Value

| Epochs Suppressed | Vault Growth (est.) | Sandbox Extraction (est.) | Suppression Cost |
|---|---|---|---|
| 5 | +250 SOL | 62.5 SOL | 0.000025 SOL |
| 10 | +500 SOL | 125 SOL | 0.00005 SOL |
| 20 (to cap) | 1000 SOL | 250 SOL | 0.0001 SOL |

These values assume 50 SOL/epoch vault accumulation and 25% extraction ceiling. Real extraction depends on pool depth and actual execution price.

### Relationship to H009 and H010

This strategy is not merely additive. The two findings interact structurally:

- H009 enables **controlled timing**: the attacker chooses when Carnage triggers.
- H010 enables **extraction at scale**: the attacker profits from the one permitted trigger.
- Without H009, an attacker can only sandwich naturally occurring fallback events (low frequency, H010 at base rate).
- Without H010, an attacker who omits `carnage_state` simply suppresses Carnage (H009 at base impact).
- Together, the attacker converts the 4.3% Carnage probability into a deterministic extraction mechanism at self-chosen timing.

## Verdict

**CONFIRMED — HIGH severity.**

This is a compounding attack confirmed by source code analysis of two independently confirmed findings (H009, H010). The alternation strategy is not a theoretical composition — it exploits a direct causal chain: suppression grows the vault, fallback MEV extracts at scale proportional to vault size. The cost of execution is near-zero (tx fees). The maximum extraction scales to 250 SOL per cycle at the 1000 SOL cap.

The severity matches H010 (HIGH) rather than escalating further because:
- It remains bounded by the vault cap (1000 SOL) and AMM pool depth
- The atomic path must be specifically defeated to force fallback execution
- Operational complexity (multi-epoch coordination, pool depth capital) increases the barrier

However, the _expected value_ of this compounded attack dominates the individual findings because it converts a stochastic ~4.3%-per-epoch event into a self-scheduled, optimally-timed extraction.

## Recommendations

The root-cause fixes from H009 and H010 individually break the compound strategy:

1. **Fix H009 first** (make `carnage_state` mandatory): This eliminates Phase 1 (suppression). An attacker who cannot suppress Carnage cannot control the timing of the trigger and thus cannot reliably force the fallback path.

2. **Fix H010** (TWAP-based or pre-committed reserve snapshot for fallback slippage): This eliminates Phase 2 extraction even if Carnage proceeds through fallback.

3. **Add a minimum-accumulation guard (Defense in Depth)**: Optionally, cap the swap amount at a rolling-average of recent Carnage swap amounts, reducing the benefit of vault accumulation via suppression. This is a design-level mitigation that does not depend on fixing H009/H010.

4. **Monitor vault balance vs. trigger frequency**: If the vault balance grows significantly faster than Carnage trigger probability would predict, this is a signal of active suppression. Alert on vault-to-epoch-count ratios above 2× expected.

Fixing H009 alone makes Phase 1 impossible and thereby breaks the entire compound strategy. It is the highest-leverage single fix.
