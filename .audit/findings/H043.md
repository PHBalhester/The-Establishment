# H043: BC close_token_vault Fix Verification

**Status:** CONFIRMED FIXED
**Severity:** N/A (RECHECK -- previously LOW)
**Category:** Access Control
**Origin:** RECHECK (H005)

## Summary

The `close_token_vault` instruction now properly enforces authorization via the `BcAdminConfig` PDA using a `has_one = authority` constraint. Combined with state guards (Graduated status + empty vault), the instruction is now properly protected against unauthorized rent extraction and fully addresses the prior H005 vulnerability.

## Verdict

**CONFIRMED FIXED.** The vulnerability has been addressed by implementing proper admin authority gating. The instruction now enforces three independent protections:

1. **Admin gating:** The `has_one = authority` constraint on `BcAdminConfig` ensures only the authorized admin can call this instruction.
2. **Graduated state guard:** The curve must be in the `Graduated` terminal state, preventing vault closure on active curves.
3. **Empty vault guard:** The vault balance must be exactly 0 (verified by Anchor's token account amount constraint).

The authorization fix eliminates the prior vulnerability where any signer could drain vault rent. The state guards ensure the instruction is only callable in the appropriate lifecycle phase.
