# H064: distribute_tax_escrow Timing Before Graduation

**Status:** CONFIRMED
**Severity:** MEDIUM
**Category:** Token Economic — State Preconditions

## Summary

The `distribute_tax_escrow` instruction enforces a strict status guard: the curve MUST be in `Graduated` state. However, this guard is enforced as a **runtime require** rather than at the Anchor account-level constraint. Additionally, the instruction is **permissionless**—any caller can trigger distribution once the curve has graduated. The timing is correct (only after graduation), but the permissionless nature creates a potential griefing vector if escrow distribution is called repeatedly before all tax proceeds are collected.

## Code Evidence

**File:** `programs/bonding_curve/src/instructions/distribute_tax_escrow.rs:74-77`

```rust
// Step 1: Only graduated curves can distribute escrow.
require!(
    ctx.accounts.curve_state.status == CurveStatus::Graduated,
    CurveError::CurveNotGraduated
);
```

**Status Definition:** `programs/bonding_curve/src/state.rs:54-69` defines `CurveStatus` enum:

```rust
pub enum CurveStatus {
    Initialized,
    Active,
    Filled,
    Failed,
    Graduated,  // ← Terminal state, only set by prepare_transition (admin-only)
}
```

**Transition:** `programs/bonding_curve/src/instructions/prepare_transition.rs:73-75` is admin-gated:

```rust
pub fn prepare_transition(ctx: Context<PrepareTransition>) -> Result<()> {
    // ... verify both curves are Filled ...
    // Step 3: Transition both to Graduated (terminal state).
    ctx.accounts.crime_curve_state.status = CurveStatus::Graduated;
    ctx.accounts.fraud_curve_state.status = CurveStatus::Graduated;
```

**Permissionlessness:** Line 1-11 of distribute_tax_escrow.rs states "Permissionless: anyone can call once the curve has graduated."

## Verdict

**CONFIRMED SAFE — No Vulnerability.** The status guard is correctly enforced at the require! level. The Graduated status is only set by the admin-gated `prepare_transition` instruction, making it impossible to call `distribute_tax_escrow` before graduation.

**Permissionlessness Design Decision:** The instruction is intentionally permissionless because tax escrow distribution is a deterministic, idempotent operation:
1. The first call transfers all available lamports (escrow - rent-exempt minimum)
2. Subsequent calls detect zero transferable lamports and fail with `EscrowAlreadyDistributed` (line 86)

**Potential Enhancement (not a finding):** The idempotent guard at line 86 (`require!(transferable > 0, CurveError::EscrowAlreadyDistributed)`) ensures only one successful distribution per curve. This is safe but could benefit from an optional `only_once` flag to fail fast on repeated attempts by monitoring.
