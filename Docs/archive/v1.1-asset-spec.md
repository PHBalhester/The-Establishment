## Global Conventions

### Naming Convention

```
{component}-{variant}[-{state}].webp
```

- **component**: What it is (e.g., `frame`, `button`, `input`, `toggle`)
- **variant**: Visual variant (e.g., `ornate`, `riveted`, `primary`, `secondary`)
- **state** (optional): Interaction state. Omit for normal/default state.
  - `-active` = pressed/active state (interactive components only)
  - `-on` / `-off` = toggle position states
  - `-focus` = focus-ring state (only if asset-based focus ring is needed)

**Examples:**
- `ornate-paper.webp` (ornate frame with parchment fill, normal)
- `ornate-paper-active.webp` (ornate frame, active)
- `button-primary.webp` (primary button, normal)
- `button-primary-active.webp` (primary button, active/pressed)
- `toggle-on.webp` (toggle in ON position)

**Frame naming exception:** Frame assets use `{variant}-paper.webp` (not `frame-{variant}.webp`) to match the CSS border-image references established in the research phase. The "-paper" suffix indicates the parchment center fill.

### State Strategy

| State | How It Is Achieved | Separate Asset? |
|-------|-------------------|-----------------|
| Normal | Default asset | Yes (base asset) |
| Hover | CSS-only: `filter: brightness(1.1)` + `box-shadow` glow + `translateY(-1px)` | No |
| Active/Pressed | Swap to Active asset with 200ms transition | Yes |
| Disabled | CSS-only: `opacity: 0.4` + `filter: saturate(0.5)` | No |
| Focus | CSS-only: `box-shadow` glow ring | No |

This means **every interactive component needs exactly 2 assets**: Normal + Active.
Decorative components (dividers, frames used as static containers) need only 1 asset.

### Resolution Strategy

All assets are created at **2x native resolution** for retina displays. The CSS renders them at half the Photoshop canvas size.

```
Photoshop canvas: 600 x 600 px  -->  CSS renders at: 300 x 300 px
Photoshop canvas: 400 x 80 px   -->  CSS renders at: 200 x 40 px
```

This prevents sub-pixel seam artifacts on high-DPI screens (the PRIMARY retina issue with 9-slice border images).

### Transparency

All assets must have **transparent backgrounds** outside their visible content. The factory's dark background and parchment fills provide context -- assets float on top.

## WebP Export Settings

For every asset, use these Photoshop "Save for Web" / "Export As" settings:

| Setting | Value | Why |
|---------|-------|-----|
| Format | WebP | Required by project spec -- universal browser support, alpha channel |
| Quality (lossy body) | 90 | Google's recommended balance of quality vs size for textured images |
| Alpha channel | Lossless (100) | Prevents edge fringing on transparent borders -- critical for 9-slice |
| Metadata | None | Strip EXIF/XMP to reduce file size |
| Color space | sRGB | Web standard -- ensures color consistency across displays |

**Photoshop-specific:** In "Export As", choose WebP format, set Quality to 90. Check "Transparency" is enabled. If using "Save for Web", select WebP lossy, quality 90.

## Phase 60: Component Kit Assets

### Frame Assets (9-Slice)

These are the foundation -- used by Frame component (`border-image`) to wrap all modal content, cards, panels, and interactive containers.

#### 9-Slice Anatomy Diagram

```
           border-image-slice: 80 (ornate) / 40 (riveted)
           All values are IMAGE PIXELS (unitless), not CSS px

Source image (600 x 600 at 2x for ornate):
+============+==============================+============+
|            |                              |            |
|  CORNER    |     TOP EDGE                 |  CORNER    |
|  80 x 80   |     (repeats horizontally)   |  80 x 80   |
|            |     440 x 80                 |            |
+============+==============================+============+
|            |                              |            |
|  LEFT EDGE |     CENTER REGION            |  RIGHT EDGE|
|  80 x 440  |     (fill: parchment)        |  80 x 440  |
|  (repeats  |     440 x 440               |  (repeats  |
|  vertically)|                             |  vertically)|
+============+==============================+============+
|            |                              |            |
|  CORNER    |     BOTTOM EDGE              |  CORNER    |
|  80 x 80   |     (repeats horizontally)   |  80 x 80   |
|            |     440 x 80                 |            |
+============+==============================+============+

CSS usage:
  border: 30px solid transparent;
  border-image-source: url('/frames/ornate-paper.webp');
  border-image-slice: 80 fill;    /* 80 = image pixels at 2x */
  border-image-width: 30px;       /* 30px CSS = 60px at 2x device */
  border-image-repeat: round;     /* round prevents fractional seams */

IMPORTANT: Add 1px overlap at ALL slice boundaries in the source image.
This means corners are actually 81x81 extending 1px into each adjacent edge
region. This prevents sub-pixel gap artifacts on retina displays.
```

## Photoshop Workflow Tips

1. **Start with the ornate frame** (Asset 1) -- it's the most complex and sets the visual tone for everything else
2. **Create a master brass texture swatch** from the reference images that you reuse across all assets
3. **Create a master parchment texture** at large size (1200x1200), then crop/reposition for each frame center
4. **Duplicate Normal assets to make Active variants** -- apply brightness/glow adjustments on the duplicate
5. **Use layer styles** (bevel/emboss, inner shadow, drop shadow) consistently across all brass elements
6. **Test a frame asset early** -- export `frame-ornate.webp`, drop into `public/frames/`, test with CSS `border-image` to verify 9-slice alignment before completing all 33 assets
7. **Export checklist:** For each asset, verify: transparent background, WebP format, quality 90, correct canvas size
