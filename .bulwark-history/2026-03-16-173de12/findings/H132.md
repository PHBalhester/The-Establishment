# Finding: H132 - Railway Dashboard Single Point of Access

## Status: POTENTIAL
## Confidence Score: 3

## Rationale
This hypothesis concerns Railway platform 2FA configuration, which is an external infrastructure concern that cannot be verified from source code analysis alone. The Railway dashboard controls deployment of the web frontend, crank runner, and docs services. It also stores sensitive environment variables including `WALLET_KEYPAIR` (crank admin wallet), `DATABASE_URL` (PostgreSQL connection), and `HELIUS_WEBHOOK_SECRET`. Compromise of the Railway account would grant full control over all three services and access to all secrets. Whether 2FA is enabled is a Railway account setting that cannot be determined from the codebase -- it requires checking the Railway dashboard directly.

## Code Evidence
No code evidence available -- this is an infrastructure configuration audit item. The `railway.toml` configuration file and deployment scripts confirm Railway hosts 3 services but contain no authentication settings.

## Severity Estimate
MEDIUM -- If Railway 2FA is not enabled, account compromise (via credential stuffing, phishing, or password reuse) would grant full access to the crank wallet keypair and database. This is a standard operational security recommendation that should be verified manually.
