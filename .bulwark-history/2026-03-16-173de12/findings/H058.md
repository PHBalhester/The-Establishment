# Finding: H058 - Unredacted RPC URL in Crank Logs

## Status: CONFIRMED
## Confidence Score: 9

## Executive Summary

The crank runner logs the raw `CLUSTER_URL` environment variable to stdout at `scripts/crank/crank-runner.ts:177`. On Railway, stdout is captured in logs accessible to anyone with Railway dashboard access. If `CLUSTER_URL` contains a Helius RPC URL with an API key (e.g., `https://devnet.helius-rpc.com/?api-key=[REDACTED-DEVNET-KEY]-...`), the key is exposed in plain text in Railway logs. This is confirmed by examining the `pda-manifest.json` which shows the exact format: `https://devnet.helius-rpc.com/?api-key=[REDACTED-DEVNET-HELIUS-KEY]`.

## Hypothesis Investigated

Crank logs CLUSTER_URL raw, exposing Helius API key.

## Investigation Path

### Step 1: Verify the logging line
**Location:** `scripts/crank/crank-runner.ts:177`
**Finding:** The crank runner logs the raw environment variable with no redaction.
**Evidence:**
```typescript
console.log(`  RPC: ${process.env.CLUSTER_URL || "localhost"}`);
```

### Step 2: Verify the URL format contains API key
**Location:** `scripts/deploy/pda-manifest.json:2-3`
**Finding:** The `clusterUrl` field contains the full Helius URL with API key in query string.
**Evidence:**
```json
"clusterUrl": "https://devnet.helius-rpc.com/?api-key=[REDACTED-DEVNET-HELIUS-KEY]"
```

### Step 3: Verify Railway captures stdout
**Location:** `railway.toml` configuration
**Finding:** Railway captures all stdout by default. The crank runner is designed to log to stdout (see file header comment: "All output goes to stdout (Railway captures logs automatically).").

## Invariant Analysis

### Enforcement Points

| `crank-runner.ts:177` | Logs raw URL | VIOLATES invariant |

## Blocking Factors
- **Railway access control**: Only team members can view logs (but this is still an exposure surface)

## Enabling Factors
- **Direct env var logging**: `crank-runner.ts:177` -- `process.env.CLUSTER_URL` logged with no filtering
- **API key in URL**: Helius uses query-string API keys, so the full URL IS the secret
- **Railway log retention**: Logs persist and are searchable

## Severity Assessment
- **Severity:** MEDIUM
- **Likelihood:** HIGH (happens on every crank startup)
- **Impact:** API key exposure enabling RPC abuse, webhook management, rate-limit exhaustion

### Attack Scenario
```
Step 1: Access Railway dashboard logs
  Target: Crank service logs
  Prerequisite: Railway team member access
  Result: See "RPC: https://devnet.helius-rpc.com/?api-key=[REDACTED-DEVNET-KEY]-..."

Step 2: Extract Helius API key
  Target: URL query parameter
  Result: Key "[REDACTED-DEVNET-HELIUS-KEY]"

Step 3: Abuse Helius API
  Target: Helius webhook management endpoints
  Payload: DELETE/modify webhooks, register new ones
  Result: Disruption of webhook data pipeline, potential data poisoning
```

### Recommended Fix
```typescript
// Before (vulnerable)
console.log(`  RPC: ${process.env.CLUSTER_URL || "localhost"}`);

// After (fixed)
function redactUrl(url: string): string {
  try {
    const parsed = new URL(url);
    parsed.searchParams.forEach((_, key) => {
      if (key.toLowerCase().includes('key') || key.toLowerCase().includes('token')) {
        parsed.searchParams.set(key, '***REDACTED***');
      }
    });
    return parsed.toString();
  } catch {
    return url.replace(/api-key=[^&]+/gi, 'api-key=***REDACTED***');
  }
}
console.log(`  RPC: ${redactUrl(process.env.CLUSTER_URL || "localhost")}`);
```
