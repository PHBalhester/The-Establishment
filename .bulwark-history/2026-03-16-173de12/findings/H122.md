# Finding: H122 - Dead Code in rewards.ts

## Status: NOT VULNERABLE
## Confidence Score: 9

## Rationale
The hypothesis claims `calculateRewardRate` is dead code. This is incorrect -- it is actively imported and called. `app/hooks/useStaking.ts:41` imports `calculateRewardRate` and calls it at line 413 to compute reward rate statistics for the staking UI display. The function itself at `app/lib/staking/rewards.ts:138-166` is live code. However, its `annualizedPct` field is documented as dead (line 94-95: "DEAD CODE -- APR/APY will never be displayed (legal reasons)") and the `solPricePerProfit` parameter is always passed as `undefined`, making the APR branch permanently zero. The function still serves its purpose for `perEpochLamports`, `totalStakedProfit`, and `userSharePct` which are actively displayed.

## Code Evidence
```typescript
// app/hooks/useStaking.ts:41 - Import (NOT dead)
import { calculateRewardRate } from "@/lib/staking/rewards";

// app/hooks/useStaking.ts:413 - Active call site
return calculateRewardRate(...)
```

The function is live. Only the `annualizedPct` output field is effectively dead (always 0). This is a documentation accuracy issue, not a vulnerability.
