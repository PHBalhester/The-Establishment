# Finding: H043 - WALLET Env Var Path Traversal

## Status: NOT VULNERABLE
## Confidence Score: 8

## Executive Summary

The `WALLET` env var is used as a file path in both `crank-provider.ts` and `scripts/deploy/lib/connection.ts`. The hypothesis suggests this could allow path traversal to read arbitrary files. After investigation, the file is read and parsed as a JSON byte array, then passed to `Keypair.fromSecretKey()`. Any file that is not a valid 64-byte Solana keypair JSON array will cause an immediate error, preventing meaningful data exfiltration. Additionally, both scripts run in trusted admin/operator contexts (local CLI or Railway), not as web-facing services accessible to attackers.

## Hypothesis Investigated

WALLET env var used as file path without directory containment.

## Investigation Path

### Step 1: Check crank-provider.ts path handling
**Location:** `scripts/crank/crank-provider.ts:60-73`
**Finding:** WALLET env var is resolved to an absolute path, checked for existence, read as JSON, and passed to `Keypair.fromSecretKey`. No directory containment check.
**Evidence:**
```typescript
const keyPath = process.env.WALLET || "keypairs/devnet-wallet.json";
const resolvedPath = path.isAbsolute(keyPath)
  ? keyPath
  : path.resolve(process.cwd(), keyPath);
// ...
const secretKey = JSON.parse(fs.readFileSync(resolvedPath, "utf8"));
wallet = new anchor.Wallet(Keypair.fromSecretKey(new Uint8Array(secretKey)));
```

### Step 2: Check connection.ts path handling
**Location:** `scripts/deploy/lib/connection.ts:69-84`
**Finding:** Same pattern. WALLET env var resolved and read.
**Evidence:**
```typescript
const keyPath = walletPath || process.env.WALLET || "keypairs/devnet-wallet.json";
const resolvedKeyPath = path.isAbsolute(keyPath)
  ? keyPath
  : path.resolve(process.cwd(), keyPath);
// ...
const secretKey = JSON.parse(fs.readFileSync(resolvedKeyPath, "utf8"));
```

### Step 3: Analyze exploitation potential
**Finding:** Both paths are used in CLI scripts (crank runner on Railway, deployment scripts locally). The WALLET env var is set by the operator -- not by external user input. On Railway, env vars are set via the dashboard. Locally, they're set in .env files.

### Step 4: Check if WALLET is ever set from external input
**Finding:** No code path allows external users (HTTP requests, webhooks, etc.) to influence the WALLET env var. It is strictly an operator-configured value.

File system reads should be restricted to expected directories to prevent path traversal.

## Blocking Factors
- Both scripts run in trusted operator contexts (CLI/Railway), not web-facing
- `crank-provider.ts:44`: On Railway, WALLET_KEYPAIR env var (JSON string) takes priority -- no file read at all
- `Keypair.fromSecretKey`: Rejects any input that isn't a valid 64-byte array
- `JSON.parse`: Rejects non-JSON files entirely

## Enabling Factors
- `crank-provider.ts:62-64`: No directory containment on resolved path
- `connection.ts:72-74`: Same -- arbitrary absolute paths accepted
