# Finding: H131 - Webhook URL Discoverable in Source

## Status: CONFIRMED
## Confidence Score: 9

## Rationale
At `scripts/webhook-manage.ts:43`, the default webhook URL is hardcoded: `https://dr-fraudsworth-production.up.railway.app/api/webhooks/helius`. This is committed to the Git repository and reveals the production webhook endpoint. Combined with the fail-open authentication model (webhook auth is optional when `HELIUS_WEBHOOK_SECRET` is unset), an attacker who discovers this URL can POST fabricated transaction payloads to inject fake swap events, epoch transitions, and carnage events into the PostgreSQL database. These would then be broadcast via SSE to all connected clients, poisoning chart data and real-time event feeds. The Helius API key is also hardcoded at line 28 and in `shared/constants.ts:474`, enabling webhook management (create/update/delete) by anyone with the source code.

## Severity Estimate
MEDIUM -- The webhook URL exposure alone is informational, but combined with the fail-open auth (tracked separately) it enables data poisoning of display-only chart/event data. No financial loss is possible since all enforcement is on-chain, but it degrades data integrity for the frontend.
