# Finding: H105 - Mainnet Pubkey::default() Placeholders

## Status: CONFIRMED
## Confidence Score: 10

## Rationale
Multiple programs contain `Pubkey::default()` placeholders in their `#[cfg(not(feature = "devnet"))]` (mainnet) code paths. These would compile to the all-zeros address (`11111111111111111111111111111111`) on mainnet, causing immediate failures or sending funds to an unrecoverable address. Identified placeholders across 3 programs:

- `bonding_curve/src/constants.rs:125,131,142,147,166,172` -- crime_mint(), fraud_mint(), epoch_program_id() (6 placeholders)
- `tax-program/src/constants.rs:144` -- treasury_pubkey() (1 placeholder, funds sent to burn address)
- `conversion-vault/src/constants.rs:33,43,53` -- crime_mint(), fraud_mint(), profit_mint() (3 placeholders)

Total: 10+ `Pubkey::default()` placeholders in mainnet code paths. Each is documented with "MAINNET PLACEHOLDER: Must be set before mainnet deployment" but there is no compile-time guard preventing a mainnet build without replacement.

## Code Evidence
```rust
// tax-program/src/constants.rs:140-144
#[cfg(not(feature = "devnet"))]
pub fn treasury_pubkey() -> Pubkey {
    // MAINNET PLACEHOLDER: Must be set before mainnet deployment.
    Pubkey::default()
}
```

## Severity Estimate
CRITICAL -- If accidentally deployed to mainnet without patching, treasury funds go to the system program address (unrecoverable), mint validations fail, and cross-program CPIs target wrong programs. A compile-time `compile_error!()` macro in the mainnet path would prevent this.
