# Finding: H094 - Crank Manifest Poisoning

## Status: POTENTIAL
## Confidence Score: 6

## Rationale
The crank loads its PDA manifest from the `PDA_MANIFEST` environment variable at `scripts/crank/crank-provider.ts:146-154`. The manifest is parsed as JSON and used directly to construct PublicKey objects for all protocol PDAs, pool addresses, mint addresses, and program IDs. If an attacker gains access to Railway's environment variable configuration, they could inject a poisoned manifest pointing to attacker-controlled accounts. The crank would then sign transactions against wrong PDAs. However, this requires Railway platform compromise (environment variable write access), which is a high-privilege attack vector. The `JSON.parse` call has no schema validation -- any valid JSON is accepted.

## Code Evidence
```typescript
// scripts/crank/crank-provider.ts:144-155
const envManifest = process.env.PDA_MANIFEST;
if (envManifest) {
  try {
    const manifest = JSON.parse(envManifest) as PDAManifest;
    // No schema validation, no signature verification
    return manifest;
  } catch (err) {
    throw new Error(`Failed to parse PDA_MANIFEST env var: ${String(err).slice(0, 100)}`);
  }
}
```
The `as PDAManifest` cast provides no runtime validation.

## Severity Estimate
MEDIUM -- If exploited, the crank would submit transactions to wrong accounts, potentially losing SOL from the crank wallet. However, on-chain programs validate account ownership/seeds, so most poisoned PDAs would cause transaction failures rather than fund loss. The attack requires Railway platform compromise.
