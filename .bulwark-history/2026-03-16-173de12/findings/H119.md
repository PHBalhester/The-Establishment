# Finding: H119 - Fee Calculation Zero for Dust Amounts

## Status: CONFIRMED
## Confidence Score: 9

## Rationale
The `calculateTax` function at `app/lib/swap/quote-engine.ts:74-76` uses `Math.floor(amountLamports * taxBps / 10_000)`. For dust amounts where `amountLamports * taxBps < 10_000`, the tax rounds to zero. For example, with a 400 bps (4%) tax rate, any amount below 25 lamports produces zero tax: `Math.floor(24 * 400 / 10_000) = Math.floor(0.96) = 0`. This mirrors the on-chain Rust behavior exactly (`programs/tax-program/src/helpers/tax_math.rs` uses integer division which also floors to zero). Since the client-side quote matches on-chain execution, there is no divergence -- but it does mean dust swaps genuinely pay zero tax on-chain as well.

## Code Evidence
```typescript
// app/lib/swap/quote-engine.ts:74-76
export function calculateTax(amountLamports: number, taxBps: number): number {
  return Math.floor(amountLamports * taxBps / BPS_DENOMINATOR);
}
```

At 400 bps: `Math.floor(24 * 400 / 10_000) = 0`. At 1400 bps: `Math.floor(7 * 1400 / 10_000) = 0`.
