# Finding: H101 - WSOL Intermediary DoS

## Status: NOT VULNERABLE
## Confidence Score: 8

## Rationale
The hypothesis posits that the swap_authority PDA could be drained below rent-exempt, causing a DoS on the sell flow. However, the sell flow in `swap_sol_sell.rs` (lines 292-425) uses a close-and-recreate pattern: the WSOL intermediary is closed to swap_authority (returning rent lamports), then immediately recreated from those retained lamports. The swap_authority PDA is a signing-only PDA -- it holds native SOL only transiently during the sell instruction's atomic execution. After the close_account (line 292-311), swap_authority holds the rent-exempt lamports plus the unwrapped WSOL. It then distributes the tax portions via system_program::transfer and uses the retained rent-exempt lamports to fund the recreated intermediary (lines 401-426). Since this all happens within a single transaction atomically, there is no window for an external actor to drain the swap_authority between close and recreate.

The atomic close-recreate pattern within a single instruction prevents any external manipulation of swap_authority's balance.
