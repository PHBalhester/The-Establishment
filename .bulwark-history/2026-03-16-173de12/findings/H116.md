# Finding: H116 - Privy Chain Configuration

## Status: NOT VULNERABLE
## Confidence Score: 7

## Rationale
The project has migrated from Privy to standard wallet-adapter (`@solana/wallet-adapter-react`), as evidenced by `app/hooks/useProtocolWallet.ts` which wraps `useWallet()` from wallet-adapter (line 28). The app's RPC connection (`app/lib/connection.ts`) defaults to `DEVNET_RPC_URL` from the shared package (line 16, 33), which is hardcoded to `https://devnet.helius-rpc.com/?api-key=...` in `shared/programs.ts:21-22`. For mainnet, this requires changing `NEXT_PUBLIC_RPC_URL` env var or updating `DEVNET_RPC_URL`. Searching for "solana:devnet" or CAIP-2 format strings in the app code yields zero results, confirming Privy's CAIP-2 chain config is no longer used. The mainnet switch points are tracked in `Docs/mainnet-checklist.md` per project memory. The wallet-adapter approach does not have chain-specific configuration issues -- it connects to whatever RPC URL is provided.
