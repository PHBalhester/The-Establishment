# Finding: H125 - Demo Mode BigInt via Number Intermediate

## Status: CONFIRMED
## Confidence Score: 8

## Rationale
At `app/hooks/useCurveState.ts:191-192`, the demo mode code computes `BigInt(Math.floor(Number(TOTAL_FOR_SALE) * progress))` and `BigInt(Math.floor(Number(TARGET_SOL) * progress))`. `TOTAL_FOR_SALE` is `460_000_000_000_000n` (460 trillion). `Number(460_000_000_000_000n)` = `460000000000000` which is `4.6e14` -- this is below `Number.MAX_SAFE_INTEGER` (9.007e15), so the conversion itself is safe. However, the multiplication `4.6e14 * progress` (where progress is 0-1) produces values up to `4.6e14`, still within safe range. Similarly, `Number(TARGET_SOL)` = `1e12`, and `1e12 * progress` maxes at `1e12`, also safe. The pattern is fragile but currently safe for these specific constant values.
