# Finding: H123 - Solana CLI v3 --keypair Flag Broken

## Status: NOT VULNERABLE
## Confidence Score: 9

## Rationale
The hypothesis references the known Solana CLI v3 bug where `--keypair` is broken for transfers. Examining `scripts/deploy/deploy.sh`, the script uses `--keypair` exclusively with `solana program deploy` (line 171-176), `solana balance` (line 83), `solana airdrop` (line 106), and `solana-keygen pubkey` (line 156). None of these are `solana transfer` commands. The `--keypair` flag works correctly for `program deploy`, `balance`, and `airdrop` operations in Solana CLI v3. The known bug only affects `solana transfer`. The deploy script does not perform any SOL transfers via the CLI -- it only deploys programs, checks balances, and requests airdrops.

## Code Evidence
```bash
# scripts/deploy/deploy.sh:171-176 - Uses --keypair for deploy (works fine)
solana program deploy \
  "target/deploy/${name}.so" \
  --program-id "$keypair" \
  --keypair "$WALLET" \
  --url "$CLUSTER_URL" \
  --with-compute-unit-price 1
```

The MEMORY.md note about `--keypair` being broken specifically says "for transfers" and recommends using `@solana/web3.js` directly for SOL transfers. The deploy script does not do transfers.
