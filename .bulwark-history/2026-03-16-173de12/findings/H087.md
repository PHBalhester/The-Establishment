# Finding: H087 - Shared Package Exports Raw TypeScript

## Status: CONFIRMED
## Confidence Score: 8

## Rationale
The `shared/package.json` sets `"main": "index.ts"` and `"types": "index.ts"` (line 5-6), exporting raw TypeScript directly without a build step. This means consumers must have a TypeScript-aware bundler configured to handle `.ts` imports from `node_modules`. The Next.js app likely handles this via `transpilePackages` or similar config, but any non-TypeScript consumer (plain Node.js script, different build tool) would fail. This is fragile because it creates a hidden dependency on the consumer's build configuration.

## Code Evidence
```json
// shared/package.json:5-6
"main": "index.ts",
"types": "index.ts",
```
No `dist/` directory, no `tsconfig.json` build step, no `"exports"` field with conditional `.js`/`.ts` resolution.

## Severity Estimate
LOW -- This works within the monorepo because all consumers (Next.js app, scripts via tsx) support TypeScript. The fragility would only manifest if a new consumer without TS support tries to import the package. No security impact; purely a build reliability concern.
