# Finding: H102 - Cross-Program Upgrade Cascade

## Status: CONFIRMED
## Confidence Score: 9

## Rationale
The deployment pipeline in `scripts/deploy/deploy-all.sh` and `scripts/deploy/build.sh` reveals tight cross-program coupling through hardcoded addresses. The build script (line 46-50) runs a "Step [0/3]: Patch mint addresses in constants.rs files" that reads keypairs from `scripts/deploy/mint-keypairs/` and patches `Pubkey::from_str("...")` values in Rust constants.rs files before building. Three programs (conversion-vault, tax-program, bonding_curve) hardcode mint addresses, epoch program IDs, staking program IDs, and AMM program IDs via `cfg(feature = "devnet")` blocks. Changing any single program ID requires rebuilding and redeploying all programs that reference it. The `deploy-all.sh` pipeline already acknowledges this with its Phase 0 (keypair generation) -> Phase 1 (build with patching) -> Phase 2 (deploy) -> Phase 3 (initialize) ordering. However, post-deployment upgrades to any single program would require a cascade rebuild of dependent programs.

## Severity Estimate
MEDIUM -- Operational complexity risk. A hotfix to one program could require rebuilding and redeploying 3-4 programs atomically, increasing deployment risk and downtime.
