# Finding: H100 - Dual Seed Registry Drift

## Status: NOT VULNERABLE
## Confidence Score: 9

## Rationale
The hypothesis suggests PDA seeds are defined in two places (`scripts/deploy/lib/pda-manifest.ts` and `shared/constants.ts`) that could drift. Inspection shows this is handled correctly. `pda-manifest.ts` (line 28-47) imports all PDA seed constants from `tests/integration/helpers/constants.ts` -- it does not define its own seeds. `shared/constants.ts` (lines 90-127) defines its own `SEEDS` object with identical values. While there ARE two separate definitions (test helpers and shared package), the actual byte values are identical strings that mirror on-chain `constants.rs`. Both derive from the same source of truth: the on-chain Rust constants. If any seed drifts, the PDA derivation would immediately fail with "account not found" during deployment verification or integration tests.

## Code Evidence
```typescript
// scripts/deploy/lib/pda-manifest.ts:29 - imports from test helpers
import { STAKE_POOL_SEED, ESCROW_VAULT_SEED, ... } from "../../../tests/integration/helpers/constants";

// shared/constants.ts:92 - defines independently
STAKE_POOL: Buffer.from("stake_pool"),
// tests/integration/helpers/constants.ts:29
export const STAKE_POOL_SEED = Buffer.from("stake_pool");
```
Same string values, different variable names. Drift would be caught by deployment verify step.

## Severity Estimate
LOW -- While having seeds in two places is a maintenance concern, the values are verified against on-chain state during every deployment (verify.ts), integration test run, and PDA derivation. Any drift would be caught immediately. No silent failure mode exists.
