# Finding: H115 - No CORS Configuration

## Status: NOT VULNERABLE
## Confidence Score: 8

## Rationale
The Next.js app (`app/next.config.ts`) does not set any explicit CORS headers, which means it relies on the browser's same-origin policy -- the correct default for a web application serving its own API routes. The API routes (candle data, SOL price proxy, health, webhook handler, SSE streaming) are consumed by the same-origin frontend, so no CORS headers are needed. The CSP header (`connect-src` directive, line 19) explicitly whitelists the allowed external connections (WalletConnect, Helius RPC, Sentry). The `X-Frame-Options: DENY` and `frame-ancestors 'none'` prevent clickjacking. The SOL price API route (`app/app/api/sol-price/route.ts`, line 6-9) explicitly documents it exists to avoid CORS issues by proxying server-side. No cross-origin API access is intended or needed.

## Code Evidence
```typescript
// next.config.ts:7-23 - CSP with restrictive connect-src
const cspHeader = `
  default-src 'self';
  connect-src 'self' wss://relay.walletconnect.com ...;
  frame-ancestors 'none';
`;
```

Same-origin policy is the correct choice here. No CORS headers needed.
