# Finding: H055 - No CI/CD Pipeline or Automated Testing on PR

## Status: CONFIRMED
## Confidence Score: 9

## Executive Summary

There is no `.github/workflows/` directory at the project root, no CI/CD pipeline configuration of any kind (GitHub Actions, Railway auto-tests, or any other CI system), and no automated testing, linting, or dependency scanning runs on pull requests or pushes. The only workflow YAML files that exist are inside `node_modules/` from third-party packages. This means code merges to `main` are entirely manual with zero automated quality or security gates.

## Hypothesis Investigated

No .github/workflows/, no automated tests on PR, no dependency scanning.

## Investigation Path

### Step 1: Check for CI/CD configuration files
**Location:** Project root and `.github/` directory
**Finding:** No `.github/workflows/` directory exists at the project root. A glob for `**/.github/workflows/*` only returned results inside `node_modules/` and `docs-site/node_modules/` (third-party packages).
**Evidence:**
```
$ ls .github/workflows/
NO .github/workflows/ directory
```

### Step 2: Check for alternative CI configurations
**Location:** Project root
**Finding:** No `.gitlab-ci.yml`, no `Jenkinsfile`, no `bitbucket-pipelines.yml`, no `.circleci/`, no `railway.json` with test phases. The `railway.toml` only defines build/deploy commands with no test step.
**Evidence:**
```toml
# railway.toml
[build]
builder = "NIXPACKS"
buildCommand = "npm run --workspace app build"

[deploy]
startCommand = "npm run --workspace app start"
preDeployCommand = "npx tsx app/db/migrate.ts"
```

### Step 3: Check for dependency scanning tools
**Location:** `package.json` (root and app)
**Finding:** No `npm audit`, `snyk`, `socket`, `retire.js`, or any security scanning tool in scripts or devDependencies. No pre-commit hooks (no `.husky/`, no `lint-staged` in package.json).

## Blocking Factors
- None identified. There are zero automated gates.

## Enabling Factors
- **No `.github/workflows/`**: Project root has no CI config at all
- **Railway auto-deploy**: `railway.toml` deploys on push with no test phase
- **No `.npmrc` with audit**: No `.npmrc` file exists anywhere in the project
- **`package-lock.json` gitignored**: `.gitignore:9` excludes the lockfile, so `npm ci` cannot be used

## Severity Assessment
- **Severity:** MEDIUM
- **Likelihood:** HIGH (this is the current state, not a hypothetical)
- **Impact:** Enables supply chain attacks (H056, H057), untested regressions, and accidental secret commits

### Attack Scenario
```
Step 1: Contributor opens PR with vulnerable dependency bump
  Target: package.json
  Payload: Dependency with known CVE
  Prerequisite: Push access to repo
  Result: No automated check flags the vulnerability

Step 2: PR merged to main
  Target: main branch
  Prerequisite: Manual review misses the issue
  Result: Vulnerable code on main

Step 3: Railway auto-deploys
  Target: Railway production service
  Prerequisite: Railway is connected to main branch
  Result: Vulnerable code running in production
```

### Recommended Fix
```yaml
# .github/workflows/ci.yml
name: CI
on: [pull_request, push]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 22 }
      - run: npm ci
      - run: npm audit --audit-level=moderate
      - run: npm run lint --workspace app
      - run: npm test
```
