# Finding: H079 - SOL Price Proxy 60s Cache Staleness

## Status: NOT VULNERABLE
## Confidence Score: 8

## Rationale
The SOL price proxy at `app/app/api/sol-price/route.ts` caches prices for 60 seconds (`CACHE_TTL_MS = 60_000`). The hypothesis is that stale prices could cause harm. However, the SOL price is used exclusively for display purposes -- converting lamport amounts to USD for the user. It does not feed into any on-chain transaction parameter (`minimumOutput`, slippage, etc.). Even with a 60s cache, SOL price moves <1% in a minute under normal conditions. The fallback chain (CoinGecko -> Binance -> stale cache) with 5s timeouts provides reasonable availability.

## Code Evidence
```typescript
// route.ts:33-34 -- 60s cache for display-only price
const CACHE_TTL_MS = 60_000;

// The price is used in BuyForm/SellForm PreviewBreakdown for USD display only,
// never for transaction parameters.
```

The stale-cache-as-fallback on line 102-111 is appropriate -- showing a slightly old price is better than showing nothing.
