# Finding: H120 - Borsh Deserialization of Untrusted Data

## Status: NOT VULNERABLE
## Confidence Score: 8

## Rationale
The hypothesis suggests malformed log messages could crash the event parser. Examining `app/lib/event-parser.ts`, the parser uses Anchor's `EventParser.parseLogs()` (line 248, 316, 362) which is a generator that iterates log lines, matching the `Program data:` prefix and attempting base64 decode + Borsh deserialization. Anchor's EventParser internally catches deserialization errors for individual events and skips them (it yields only successfully parsed events). The parser creates fresh `EventParser` instances per call (lines 221-229), preventing cross-call state leakage. The switch statements (lines 253-297) only process known event names and silently ignore unrecognized ones. Even if Helius delivers completely malformed transaction data, the worst outcome is zero parsed events (not a crash). The webhook handler's outer try-catch would catch any truly unexpected error.
