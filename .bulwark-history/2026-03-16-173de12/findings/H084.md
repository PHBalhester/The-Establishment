# Finding: H084 - Shared Constants Drift from On-Chain

## Status: POTENTIAL
## Confidence Score: 6

## Rationale
`shared/constants.ts` hardcodes protocol constants that must match on-chain programs. Cross-verification shows current values match: `SOL_POOL_FEE_BPS = 100` matches `programs/amm/src/constants.rs:15` (`pub const SOL_POOL_FEE_BPS: u16 = 100`), and `VAULT_CONVERSION_RATE = 100` matches `programs/conversion-vault/src/constants.rs:7` (`pub const CONVERSION_RATE: u64 = 100`). However, there is no automated verification that these values stay in sync. A future on-chain change that isn't propagated to `shared/constants.ts` would cause the frontend to compute incorrect quotes, fees, or slippage bounds.

## Code Evidence
```typescript
// shared/constants.ts:62-63 -- matches on-chain today
export const SOL_POOL_FEE_BPS = 100;
export const VAULT_CONVERSION_RATE = 100;
```

The project has `scripts/verify-program-ids.ts` for program ID verification but no equivalent for fee/rate constants.

## Severity Estimate
MEDIUM -- If constants drift, the frontend would display incorrect fee estimates and compute wrong `minimumOutput` values, potentially causing users to accept worse execution than expected. The risk is latent (no current drift) but has no automated guard.
