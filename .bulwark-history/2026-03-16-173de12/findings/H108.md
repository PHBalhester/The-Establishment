# Finding: H108 - Carnage VRF Predictability Window

## Status: NOT VULNERABLE
## Confidence Score: 8

## Rationale
The hypothesis suggests that VRF reveal bytes being public creates a predictability window for Carnage swaps. The crank runner (`crank-runner.ts`, line 15-17) uses atomic bundling: "reveal + consume + executeCarnageAtomic are bundled in a single v0 VersionedTransaction, closing the CARN-002 MEV gap." This means the VRF randomness is revealed and consumed in the same transaction as the Carnage execution. There is no intermediate on-chain state where VRF bytes are visible but Carnage has not yet executed. The `vrfBytes` logged at line 296 are only logged to stdout after the atomic transaction has already been confirmed on-chain. While the VRF bytes are technically visible on-chain in the transaction data after confirmation, the Carnage swap has already been atomically executed in the same transaction, so there is no front-running window.

The atomic bundling eliminates the predictability window.
