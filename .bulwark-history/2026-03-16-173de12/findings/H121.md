# Finding: H121 - No Source Maps in Production

## Status: NOT VULNERABLE
## Confidence Score: 9

## Rationale
The hypothesis asks to verify that source maps are not served in production. Examining `app/next.config.ts`, there is no `productionBrowserSourceMaps: true` setting anywhere in the config. Next.js defaults to NOT generating browser source maps in production builds. The config file contains no `devtool`, `sourceMap`, or related settings. Next.js 16's default behavior is to exclude source maps from production bundles, meaning the JavaScript served to browsers is minified without accompanying `.map` files. This is the correct security posture -- source maps would reveal internal code structure, variable names, and comments to attackers.

The absence of `productionBrowserSourceMaps: true` confirms source maps are not exposed. Next.js default = no production source maps.
