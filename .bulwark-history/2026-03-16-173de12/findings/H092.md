# Finding: H092 - SSE Single-Process Only

## Status: CONFIRMED
## Confidence Score: 9

## Rationale
The SSE manager at `app/lib/sse-manager.ts:8-30` uses an in-memory `Set<SSECallback>` for pub/sub. The file comment (lines 8-11) explicitly acknowledges this limitation: "Railway runs a single Next.js process for devnet (no horizontal scaling). An in-memory Set of subscriber callbacks is sufficient." If the application is ever scaled horizontally (multiple Next.js processes or Railway replicas), webhook events received by one process would not be broadcast to SSE clients connected to other processes. The `globalThis` singleton pattern (line 84-92) only survives HMR reloads within the same process, not cross-process communication.

## Code Evidence
```typescript
// app/lib/sse-manager.ts:29-30
export class SSEManager {
  private subscribers = new Set<SSECallback>();
```
Pure in-memory state, no Redis pub/sub, no shared message broker.

## Severity Estimate
LOW -- Explicitly documented as a known limitation with a clear upgrade path (Redis pub/sub). Currently safe because Railway runs a single process. This would become MEDIUM if horizontal scaling is added without addressing the SSE broadcast mechanism.
