# Finding: H085 - Health Endpoint Always Returns 200

## Status: CONFIRMED
## Confidence Score: 9

## Rationale
The health endpoint at `app/app/api/health/route.ts:49-55` always returns HTTP 200 regardless of dependency health. When Postgres or Solana RPC is down, the response body indicates `status: "degraded"` but the HTTP status code remains 200. The file header comment (lines 6-8) explicitly documents this as intentional for Railway container liveness. However, if Railway or any external monitoring tool relies on HTTP status codes (not response body parsing) to detect degraded state, the degradation will be invisible. Railway's default healthcheck only checks for a non-5xx response, so it will never restart a degraded container.

## Code Evidence
```typescript
// app/app/api/health/route.ts:49-55
const status = postgres && solanaRpc ? "ok" : "degraded";

return NextResponse.json({
  status,
  checks: { postgres, solanaRpc },
  timestamp: new Date().toISOString(),
});
```
HTTP status is always 200 -- no conditional `return NextResponse.json(..., { status: 503 })` for degraded state.

## Severity Estimate
LOW -- This is a deliberate design choice (documented in the file header) to keep Railway from killing containers during transient RPC outages. The risk is that monitoring tools that only check HTTP status codes will miss degraded state. Impact is operational visibility, not security.
