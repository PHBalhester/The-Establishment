# Finding: H113 - Split Route MEV Observability

## Status: POTENTIAL
## Confidence Score: 6

## Rationale
The split router (`app/lib/swap/split-router.ts`) computes an optimal split ratio client-side and returns `splitRatioA` / `splitRatioB` (1% granularity, lines 103-116). When a split route is executed, it results in two separate transactions (or two instructions in one TX) with specific amounts derived from this ratio. The split amounts are visible in the transaction data on-chain, revealing the user's routing strategy. However, the practical MEV risk is limited: (1) these are swaps through the protocol's own AMM pools, not external DEXes; (2) the split is computed from current pool reserves which change between computation and submission; (3) the on-chain slippage check (`minimumOutput`) protects against sandwich attacks regardless of split visibility; (4) the split-router is a pure function with no RPC calls -- the actual execution happens through the standard swap path with slippage protection.
