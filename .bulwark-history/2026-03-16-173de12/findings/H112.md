# Finding: H112 - Audio Manager Math.random()

## Status: NOT VULNERABLE
## Confidence Score: 10

## Rationale
The `Math.random()` usage in `app/lib/audio-manager.ts` (lines 343 and 353) is exclusively for shuffling a music playlist using the Fisher-Yates algorithm and preventing the last-played track from repeating. This is purely a UX feature -- it determines which background music track plays next. There is zero security impact: the randomness does not influence any financial computation, transaction construction, slippage calculation, or on-chain state. Using `Math.random()` for music shuffle is appropriate and standard.

## Code Evidence
```typescript
// audio-manager.ts:342-354
// Fisher-Yates shuffle
for (let i = shuffled.length - 1; i > 0; i--) {
  const j = Math.floor(Math.random() * (i + 1));
  [shuffled[i], shuffled[j]] = [shuffled[j], shuffled[i]];
}
// No-repeat-last constraint
const swapIdx = 1 + Math.floor(Math.random() * (shuffled.length - 1));
```

Both uses are for audio playlist randomization only. Not security-critical.
