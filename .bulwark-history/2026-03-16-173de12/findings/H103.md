# Finding: H103 - Bounty Rent-Exempt Gap

## Status: CONFIRMED
## Confidence Score: 9

## Rationale
The on-chain `trigger_epoch_transition` checks `vault_balance >= TRIGGER_BOUNTY_LAMPORTS` (1,000,000 lamports) without accounting for rent-exempt minimum (~890,880 lamports for a 0-data SystemAccount). If the vault balance is between 1M and ~1.89M lamports, the check passes but `system_program::transfer` fails because remaining balance would drop below rent-exempt. The crank runner (`scripts/crank/crank-runner.ts`, lines 62-75) implements an off-chain mitigation: it checks vault balance against `MIN_VAULT_BALANCE = TRIGGER_BOUNTY_LAMPORTS + RENT_EXEMPT_MINIMUM + 100_000` (~2M lamports) and tops up with 0.005 SOL when below threshold. This mitigation works as long as the crank runner is operational and has sufficient SOL. If the crank is down or its wallet is empty, the on-chain bug remains exploitable -- any permissionless caller could trigger an epoch transition that fails due to this gap.

## Code Evidence
```typescript
// crank-runner.ts:72-74
const TRIGGER_BOUNTY_LAMPORTS = 1_000_000; // Must match on-chain constant
const RENT_EXEMPT_MINIMUM = 890_880;       // 0-data SystemAccount
const MIN_VAULT_BALANCE = TRIGGER_BOUNTY_LAMPORTS + RENT_EXEMPT_MINIMUM + 100_000;
```

## Severity Estimate
MEDIUM -- The on-chain bug is real but mitigated by the crank. If the crank fails, epoch transitions stall until vault is manually topped up. Already documented as a known TODO in project memory.
