# Finding: H078 - No Structured Logging

## Status: CONFIRMED
## Confidence Score: 8

## Rationale
A grep across all TypeScript files reveals approximately 3,899 `console.log` occurrences. There is no structured logging library (e.g., pino, winston) in use. All logging is unstructured `console.log` with ad-hoc prefixes like `[crank]`. This makes log aggregation, filtering by severity, and alerting in Railway difficult. The crank runner is the most critical consumer -- without structured logging, distinguishing warnings from errors from info messages requires string parsing.

## Code Evidence
The crank runner alone has 36 `console.log` calls with manual `[crank]` prefixes. No log levels, no JSON output, no correlation IDs.

## Severity Estimate
LOW -- This is an operational quality issue, not a security vulnerability. It becomes MEDIUM at mainnet scale where log volume makes unstructured logs unmanageable for incident response.
