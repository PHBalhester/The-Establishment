# Finding: H060 - pda-manifest.json Contains API Key

## Status: CONFIRMED
## Confidence Score: 10

## Executive Summary

The file `scripts/deploy/pda-manifest.json` contains the full Helius RPC URL including the API key at line 3: `"clusterUrl": "https://devnet.helius-rpc.com/?api-key=[REDACTED-DEVNET-HELIUS-KEY]"`. While the file is gitignored (`.gitignore:12`), it is loaded into the `PDA_MANIFEST` environment variable for Railway deployment, and the crank provider's `loadManifest()` function reads it. The API key is the same one hardcoded elsewhere in the codebase (a known cross-cutting concern per ARCHITECTURE.md). The gitignore provides the right protection for the file itself, but the key has already been committed in other locations (`shared/constants.ts`, `shared/programs.ts`), making this particular file a secondary exposure vector rather than the primary one.

## Hypothesis Investigated

Manifest contains full Helius RPC URL with API key.

## Investigation Path

### Step 1: Verify manifest contents
**Location:** `scripts/deploy/pda-manifest.json:2-3`
**Finding:** Full Helius URL with API key is present.
**Evidence:**
```json
{
  "generatedAt": "2026-02-27T10:15:28.032Z",
  "clusterUrl": "https://devnet.helius-rpc.com/?api-key=[REDACTED-DEVNET-HELIUS-KEY]",
```

### Step 2: Check gitignore coverage
**Location:** `.gitignore:12`
**Finding:** The file IS gitignored, preventing accidental commits.
**Evidence:**
```
scripts/deploy/pda-manifest.json
```

### Step 3: Check how manifest is consumed
**Location:** `scripts/crank/crank-provider.ts:141-177`
**Finding:** `loadManifest()` reads the file locally or from `PDA_MANIFEST` env var. On Railway, the entire JSON is stored in the `PDA_MANIFEST` env var, which means the API key is in Railway's environment variable store.
**Evidence:**
```typescript
export function loadManifest(): PDAManifest {
  const envManifest = process.env.PDA_MANIFEST;
  if (envManifest) {
    const manifest = JSON.parse(envManifest) as PDAManifest;
    // ...
  }
```

### Step 4: Verify manifest is not read for clusterUrl
**Location:** `scripts/crank/crank-provider.ts:35`
**Finding:** The crank provider reads `CLUSTER_URL` from a SEPARATE env var, NOT from the manifest's `clusterUrl` field. The `clusterUrl` in the manifest is informational only (records which cluster was used during deploy).
**Evidence:**
```typescript
const url = process.env.CLUSTER_URL || "http://localhost:8899";
```

## Invariant Analysis

### Target Invariant
Secrets should not be stored in configuration files, even gitignored ones, if the same data is loaded into shared environment stores.

### Enforcement Points
| Location | What It Enforces | Complete? |
|----------|-----------------|-----------|
| `.gitignore:12` | Prevents git commit of manifest | Yes |
| `crank-provider.ts:35` | Uses separate CLUSTER_URL, not manifest clusterUrl | Partially mitigates |

## Blocking Factors
- **Gitignored**: `.gitignore:12` prevents manifest from being committed to git
- **Informational field**: `clusterUrl` in manifest is not used programmatically by the crank (it uses `CLUSTER_URL` env var instead)
- **Already exposed elsewhere**: The API key `[REDACTED-DEVNET-KEY]-...` is already hardcoded in `shared/constants.ts` and `shared/programs.ts` (per ARCHITECTURE.md)

## Enabling Factors
- **Railway env var**: Full JSON including `clusterUrl` stored in `PDA_MANIFEST` env var
- **Developer machine**: File exists on any machine that has run the deploy pipeline

## Severity Assessment
- **Severity:** LOW (secondary exposure vector -- key already exposed in committed source)
- **Likelihood:** MEDIUM (file exists on dev machines; env var on Railway)
- **Impact:** API key exposure (same as existing exposure from hardcoded key)

### Attack Scenario
```
Step 1: Access developer's machine or Railway env vars
  Target: pda-manifest.json or PDA_MANIFEST env var
  Prerequisite: Physical/remote access to dev machine OR Railway dashboard
  Result: Full Helius API key extracted

Step 2: Abuse Helius API
  Target: Helius RPC + webhook management
  Result: Rate exhaustion, webhook manipulation
```

### Recommended Fix
```typescript
// In the deploy script that generates pda-manifest.json:
// Strip the clusterUrl or redact the API key before writing

const manifest = {
  generatedAt: new Date().toISOString(),
  // Don't include clusterUrl at all -- it's informational noise
  programs: { ... },
  mints: { ... },
  pdas: { ... },
  pools: { ... },
};
```

Or strip it when loading into PDA_MANIFEST env var:
```bash
# When setting Railway env var, remove clusterUrl
cat pda-manifest.json | jq 'del(.clusterUrl)' | railway variables set PDA_MANIFEST
```
