# Finding: H072 - Price Impact Additive Not Multiplicative

## Status: CONFIRMED
## Confidence Score: 7

## Rationale
At `app/lib/swap/route-engine.ts:322-325`, `totalPriceImpactBps` is computed by summing per-step price impacts additively. The comment on line 320-321 acknowledges this is a simplification: "For multi-hop, impact compounds. Use max as a simple heuristic" -- but the actual code uses `sum`, not `max`. True compounding would be multiplicative: `1 - (1-impact_1) * (1-impact_2)`. For example, two hops with 2% and 3% impact should compound to ~4.94%, but additive gives 5%. The additive approach slightly *overstates* impact on multi-hop routes, which is the conservative (user-protective) direction.

## Code Evidence
```typescript
// line 322-325: Additive sum, not multiplicative compound
const totalPriceImpactBps = steps.reduce(
  (sum, s) => sum + s.priceImpactBps,
  0,
);
```

## Severity Estimate
LOW -- The additive approach overstates price impact (conservative direction), so users are warned more than necessary rather than less. This is a display-only metric; on-chain slippage protection uses `minimumOutput` computed from the actual chained quote output.
