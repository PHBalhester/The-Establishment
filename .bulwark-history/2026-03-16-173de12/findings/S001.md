# Finding: S001 - Chained Webhook + Supply Chain Attack

## Status: CONFIRMED
## Confidence Score: 9

## Executive Summary

This strategy chains three confirmed findings -- fail-open webhook auth (H001), hardcoded Helius API key (H002), and npm supply chain compromise (H003) -- into a coordinated attack that gives the attacker full control over the protocol's data pipeline. The chain is: (1) a compromised npm dependency exfiltrates the Helius API key from `shared/constants.ts:474` (trivially reachable at build time), (2) the attacker uses the Helius webhook management API to delete the legitimate webhook and register a malicious one pointing to their server, (3) they POST fabricated transaction data to the fail-open webhook endpoint, injecting false prices into PostgreSQL and broadcasting them to all SSE clients.

The combined severity is CRITICAL because the chain requires only one realistic entry point (npm supply chain via gitignored lockfile) and each subsequent step is fully automated. The attack is self-reinforcing: once the legitimate webhook is replaced, the protocol stops receiving real data AND starts receiving attacker-controlled data, creating a permanent information asymmetry. Users see manipulated charts, make bad trades, and have no way to detect the manipulation from within the UI.

## Hypothesis Investigated

npm dependency compromise (H003) exfiltrates Helius API key (H002), deletes legitimate webhook, registers malicious one. Combined with fail-open auth (H001), attacker controls entire data pipeline.

## Chain Component Analysis

### Link 1: npm Supply Chain Entry (H003)
**Location:** `.gitignore:9`, Railway build process
**Status:** CONFIRMED independently
**Role in chain:** Provides code execution in Railway build/runtime environment. `package-lock.json` is gitignored, so every Railway deploy resolves dependencies fresh from npm registry. A compromised patch version of any caret-ranged dependency executes in the server process with full env var access.
**Evidence:**
```
# .gitignore line 9
package-lock.json
```

### Link 2: API Key Exfiltration (H002)
**Location:** `shared/constants.ts:474`, `shared/programs.ts:22`
**Status:** CONFIRMED independently
**Role in chain:** The compromised dependency doesn't even need env var access -- the API key is hardcoded in source code imported by both the web app and crank runner. At build time, the attacker's code can `require('../../shared/constants')` or simply read the file from disk.
**Evidence:**
```typescript
// shared/constants.ts:474
export const HELIUS_API_KEY = "[REDACTED-DEVNET-HELIUS-KEY]";
```

### Link 3: Webhook Takeover via Helius API
**Target:** Helius webhook management API (`api.helius.xyz/v0/webhooks`)
**Status:** CONFIRMED (API key grants webhook CRUD, per ARCHITECTURE.md and H002)
**Role in chain:** With the API key, the attacker calls `GET /v0/webhooks?api-key=...` to list existing webhooks, `DELETE /v0/webhooks/{id}?api-key=...` to remove the legitimate one, and `POST /v0/webhooks?api-key=...` to register a new webhook pointing to either (a) their own server (to intercept real transaction data) or (b) nowhere (to blind the protocol). The attacker then replays modified transaction data to the real webhook endpoint.

### Link 4: Data Injection via Fail-Open Webhook (H001)
**Location:** `app/app/api/webhooks/helius/route.ts:131-141`
**Status:** CONFIRMED independently
**Role in chain:** Even if `HELIUS_WEBHOOK_SECRET` IS set, the attacker can bypass it because they now control which webhooks exist for the API key. They can register a new webhook WITHOUT an auth secret, or they can POST directly to the known endpoint (URL hardcoded in `scripts/webhook-manage.ts:43`) if the secret is unset.
**Evidence:**
```typescript
const webhookSecret = process.env.HELIUS_WEBHOOK_SECRET;
if (webhookSecret) {
  const authHeader = req.headers.get("authorization");
  if (authHeader !== webhookSecret) {
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
  }
}
```

## Enabling Factors
- **Gitignored lockfile:** `.gitignore:9` -- every deploy resolves fresh deps
- **Hardcoded API key:** `shared/constants.ts:474` -- no env var needed
- **Known webhook URL:** `scripts/webhook-manage.ts:43` -- endpoint discoverable
- **No install scripts guard:** Railway runs `npm install` without `--ignore-scripts`
- **No tx signature verification:** Webhook handler trusts payload content

## Blocking Factors
- None identified. Each link in the chain is independently confirmed viable.

## Severity Assessment
- **Severity:** CRITICAL
- **Likelihood:** MEDIUM (requires npm supply chain compromise, but lockfile gitignore makes this far easier than normal)
- **Impact:** Full data pipeline takeover -- manipulated charts, user trading decisions based on false prices, potential for coordinated pump-and-dump via chart manipulation

### Combined Severity Justification
Individual findings are HIGH (H001), HIGH (H002), HIGH (H003). The chain elevates to CRITICAL because:
1. Each link enables the next -- they are not merely additive, they are multiplicative
2. The attack is fully automated after initial npm compromise
3. Detection is extremely difficult (no alerting per H004, no webhook health monitoring)
4. The attack is persistent (once webhook is replaced, it stays replaced)
5. Impact extends beyond data display to user financial decisions

### Recommended Fix

```typescript
// 1. Commit package-lock.json (remove from .gitignore)
// .gitignore -- REMOVE line 9: package-lock.json

// 2. Move API key to env var (shared/constants.ts)
// Before (vulnerable)
export const HELIUS_API_KEY = "[REDACTED-DEVNET-HELIUS-KEY]";

// After (fixed)
export const HELIUS_API_KEY = process.env.HELIUS_API_KEY;
if (!HELIUS_API_KEY) throw new Error("HELIUS_API_KEY env var required");

// 3. Make webhook auth fail-closed (route.ts)
// Before (vulnerable)
const webhookSecret = process.env.HELIUS_WEBHOOK_SECRET;
if (webhookSecret) { /* optional check */ }

// After (fixed)
const webhookSecret = process.env.HELIUS_WEBHOOK_SECRET;
if (!webhookSecret) {
  console.error("FATAL: HELIUS_WEBHOOK_SECRET not set");
  return NextResponse.json({ error: "Server misconfiguration" }, { status: 500 });
}
const authHeader = req.headers.get("authorization");
if (!crypto.timingSafeEqual(
  Buffer.from(authHeader ?? ""),
  Buffer.from(webhookSecret)
)) {
  return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
}

// 4. Add --ignore-scripts to Railway build (railway.toml or nixpacks.toml)
// [build]
// buildCommand = "npm ci --ignore-scripts && npm run build"
```
