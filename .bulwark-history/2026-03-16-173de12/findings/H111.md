# Finding: H111 - RPC Fallback to localhost in Crank

## Status: CONFIRMED
## Confidence Score: 10

## Rationale
The crank provider (`scripts/crank/crank-provider.ts:35`) defaults the RPC URL to `http://localhost:8899` when `CLUSTER_URL` is not set: `const url = process.env.CLUSTER_URL || "http://localhost:8899"`. On Railway (the production crank host), if the `CLUSTER_URL` environment variable is accidentally deleted or misspelled, the crank would silently connect to localhost:8899. Since Railway containers don't run a local Solana validator, all RPC calls would fail with connection errors, halting epoch transitions. This is a liveness risk, not a security risk -- the crank would stop working rather than connecting to a wrong network. However, if a malicious process somehow listened on port 8899 inside the container, it could feed fake RPC responses.

## Code Evidence
```typescript
// crank-provider.ts:35
const url = process.env.CLUSTER_URL || "http://localhost:8899";
```

## Severity Estimate
LOW -- Liveness risk only. The crank would fail loudly (connection refused) rather than silently misbehaving. A startup validation that rejects localhost in production environments would eliminate this.
