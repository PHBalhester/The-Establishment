# Finding: H059 - COMMITMENT Env Var Unsafe Cast

## Status: POTENTIAL
## Confidence Score: 5

## Executive Summary

At `scripts/crank/crank-provider.ts:37`, the `COMMITMENT` environment variable is cast directly to `anchor.web3.Commitment` using TypeScript's `as` assertion with no runtime validation. If an invalid value (e.g., `COMMITMENT=banana`) is set, it propagates to the Solana RPC Connection constructor and all subsequent RPC calls. The Solana web3.js library does not validate the commitment string at connection creation time -- it passes it through to RPC requests, where the RPC node rejects it with an opaque error. This would cause the crank runner to fail on its first RPC call with a confusing error message, but would NOT cause silent data corruption or security compromise.

## Hypothesis Investigated

`process.env.COMMITMENT as Commitment` unsafe cast; invalid values propagate to RPC.

## Investigation Path

### Step 1: Verify the unsafe cast
**Location:** `scripts/crank/crank-provider.ts:36-37`
**Finding:** Direct `as` cast with no validation. Falls back to `"confirmed"` if env var is unset, but does not validate if env var IS set.
**Evidence:**
```typescript
const commitment =
  (process.env.COMMITMENT as anchor.web3.Commitment) || "confirmed";
```

### Step 2: Trace how commitment is used
**Location:** `scripts/crank/crank-provider.ts:82-86`
**Finding:** The commitment value is passed to `Connection` constructor and `AnchorProvider` options.
**Evidence:**
```typescript
const connection = new Connection(url, commitment);
return new anchor.AnchorProvider(connection, wallet, {
  preflightCommitment: commitment,
  commitment,
});
```

### Step 3: Check Solana web3.js behavior with invalid commitment
**Location:** `@solana/web3.js` Connection class
**Finding:** The `Connection` constructor accepts any string for commitment -- no runtime type check. Invalid values are sent to the RPC node in JSON-RPC requests, which responds with an error like `"Invalid param: unknown variant ..."`. The crank would crash on the first RPC call.

### Recommended Fix
```typescript
// Before (vulnerable)
const commitment =
  (process.env.COMMITMENT as anchor.web3.Commitment) || "confirmed";

// After (fixed)
const VALID_COMMITMENTS = new Set(["processed", "confirmed", "finalized"]);
const rawCommitment = process.env.COMMITMENT || "confirmed";
if (!VALID_COMMITMENTS.has(rawCommitment)) {
  throw new Error(
    `Invalid COMMITMENT="${rawCommitment}". Must be one of: ${[...VALID_COMMITMENTS].join(", ")}`
  );
}
const commitment = rawCommitment as anchor.web3.Commitment;
```
