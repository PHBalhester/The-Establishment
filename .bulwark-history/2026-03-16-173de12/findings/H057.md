# Finding: H057 - 11 Install Script Packages Without --ignore-scripts

## Status: POTENTIAL
## Confidence Score: 6

## Executive Summary

The `package-lock.json` identifies 11 packages with `hasInstallScript: true` that execute postinstall hooks during `npm install`. These include `esbuild` (3 instances), `protobufjs`, `sharp`, `bufferutil`, `utf-8-validate`, `bigint-buffer`, `fsevents`, `unrs-resolver`, and `vite/node_modules/esbuild`. No `.npmrc` file exists with `ignore-scripts=true`, and Railway's Nixpacks builder runs `npm install` without `--ignore-scripts`. If any of these packages (or their future versions) were compromised, arbitrary code would execute during the build phase with full access to Railway environment variables including `WALLET_KEYPAIR`, `DATABASE_URL`, and `HELIUS_WEBHOOK_SECRET`.

However, all 11 packages are well-known, heavily-used open source projects (esbuild, sharp, protobufjs, etc.) with native binary compilation as the legitimate reason for install scripts. The risk is supply chain compromise of these specific packages, which is a real but low-probability event.

## Hypothesis Investigated

11 packages run postinstall hooks; compromised one could execute arbitrary code.

## Investigation Path

### Step 1: Enumerate packages with install scripts
**Location:** `package-lock.json` -- `"hasInstallScript": true` entries
**Finding:** 11 packages identified:
1. `@esbuild-kit/core-utils/node_modules/esbuild` (native binary)
2. `bigint-buffer` (native addon)
3. `bufferutil` (native addon)
4. `esbuild` (native binary download)
5. `fsevents` (macOS native, dev only)
6. `protobufjs` (code generation)
7. `sharp` (native image processing)
8. `tsx/node_modules/esbuild` (native binary)
9. `unrs-resolver` (native binary)
10. `utf-8-validate` (native addon)
11. `vite/node_modules/esbuild` (native binary, dev)

### Step 2: Check for --ignore-scripts protection
**Location:** Project root, `.npmrc`
**Finding:** No `.npmrc` file exists. No `--ignore-scripts` flag in `railway.toml` build command. Railway runs `npm install` with default settings, allowing all install scripts to execute.
**Evidence:**
```toml
# railway.toml
[build]
builder = "NIXPACKS"
buildCommand = "npm run --workspace app build"
```
Nixpacks runs `npm install` before the build command with no `--ignore-scripts` flag.

### Step 3: Assess Railway build environment exposure
**Location:** Railway deployment configuration
**Finding:** During the build phase on Railway, environment variables are accessible. This means install scripts have access to `WALLET_KEYPAIR` (crank wallet private key), `DATABASE_URL`, `PDA_MANIFEST`, and other secrets.

## Blocking Factors
- **Package reputation**: All 11 are well-known, heavily-used projects (esbuild, sharp, protobufjs)
- **npm 2FA**: Major packages typically have 2FA on publish
- **Lockfile exists locally**: `package-lock.json` does exist (just gitignored), providing some version pinning on developer machines

## Enabling Factors
- **No `.npmrc`**: No `ignore-scripts=true` anywhere in the project
- **Lockfile gitignored**: `.gitignore:9` -- Railway builds resolve fresh from registry
- **Secrets in build env**: Railway makes all env vars available during build phase
- **No CI scanning**: H055 -- no automated checks on dependency changes

## Severity Assessment
- **Severity:** MEDIUM
- **Likelihood:** LOW (requires compromising a well-maintained package)
- **Impact:** CRITICAL if exploited (crank wallet private key exfiltration = fund loss)

### Recommended Fix
```ini
# .npmrc (create at project root)
ignore-scripts=true
```
Then explicitly allowlist packages that need install scripts:
```bash
# After npm install --ignore-scripts, run specific builds:
npx esbuild --version  # triggers binary download
cd node_modules/sharp && npm run install  # if needed
```

Or better: separate build-time and runtime secrets on Railway using build variables vs runtime variables.
