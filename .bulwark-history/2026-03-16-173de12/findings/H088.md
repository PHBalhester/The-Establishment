# Finding: H088 - Auto-Reset Timer in useSwap

## Status: NOT VULNERABLE
## Confidence Score: 8

## Rationale
The auto-reset timer at `app/hooks/useSwap.ts:774-776` fires 10 seconds after a swap is confirmed (status = "confirmed"), calling `resetForm()` which clears the input amounts and resets status to "idle". This is a UX convenience feature, not a state-corruption risk. The timer only triggers after successful confirmation (line 768: `setStatus("confirmed")`), and it only resets the form UI state -- it does not affect on-chain state, balances, or any pending transaction. The `autoResetTimerRef` pattern using `useRef` ensures the timer is properly scoped and can be cleaned up on unmount.

## Code Evidence
```typescript
// app/hooks/useSwap.ts:773-776
// 9. Auto-reset after 10 seconds
autoResetTimerRef.current = setTimeout(() => {
  resetForm();
}, AUTO_RESET_MS);
```
The timer only fires after `setStatus("confirmed")` at line 768, and `resetForm()` only modifies React state (input amounts, status). No transaction construction or submission occurs during reset.

## Severity Estimate
N/A -- Not vulnerable. The auto-reset is a standard UX pattern that only affects display state.
