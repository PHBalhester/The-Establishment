# Finding: H042 - Graduation Env Override Without Bounds Validation

## Status: CONFIRMED
## Confidence Score: 7

## Executive Summary

The graduation script (`graduate.ts:102-107`) uses `Number(process.env.SOL_POOL_SEED_SOL_OVERRIDE)` and `Number(process.env.SOL_POOL_SEED_TOKEN_OVERRIDE)` with the `||` fallback operator. If the env var is set to a non-numeric string (e.g., `"abc"`), `Number("abc")` returns `NaN`, and `NaN || fallback` correctly falls back to the default. However, if the env var is set to `"0"`, `Number("0")` returns `0`, and `0 || fallback` falls back to the default -- which may be surprising but is safe.

The real vulnerability is that there are **no bounds checks** on valid numeric values. Setting `SOL_POOL_SEED_SOL_OVERRIDE=1` (1 lamport) or `SOL_POOL_SEED_SOL_OVERRIDE=999999999999999` (absurdly large) would both be accepted. Since this is an IRREVERSIBLE operation (pools cannot be re-seeded), incorrect values permanently damage the protocol. This is the exact Phase 69 lesson mentioned in MEMORY.md.

The severity is reduced because this is an admin-only CLI script run manually, not an externally-facing surface.

## Hypothesis Investigated

`Number(process.env.SOL_POOL_SEED_SOL_OVERRIDE)` accepts any value including NaN.

## Investigation Path

### Step 1: Examine env override pattern
**Location:** `scripts/graduation/graduate.ts:102-107`
**Finding:** Uses `Number(env) || hardcoded_default` pattern. No range validation.
**Evidence:**
```typescript
const GRADUATION_POOL_SEED_TOKEN = Number(process.env.SOL_POOL_SEED_TOKEN_OVERRIDE)
  || 290_000_000_000_000; // 290M * 10^6

const GRADUATION_POOL_SEED_SOL = Number(process.env.SOL_POOL_SEED_SOL_OVERRIDE)
  || 1_000 * LAMPORTS_PER_SOL; // 1000 SOL
```

### Step 2: Analyze NaN behavior
**Finding:** `Number("abc")` = `NaN`. `NaN || default` = `default`. So NaN correctly falls back. `Number("")` = `0`. `0 || default` = `default`. Also falls back. The `||` operator treats `0`, `NaN`, `undefined`, `null`, and `""` as falsy, falling back to the default.

### Step 3: Analyze dangerous valid inputs
**Finding:** `Number("1")` = `1`. `1 || default` = `1`. This would seed a pool with 1 lamport of SOL -- creating an absurdly imbalanced pool. `Number("-1000000000000")` = `-1e12`. Negative values pass `||` (truthy) and would be sent to the AMM's `initializePool` instruction as a BN, potentially causing an underflow or on-chain rejection.

### Step 4: Check if on-chain validates pool seed amounts
**Location:** On-chain (out of scope for off-chain audit, but contextually relevant)
**Finding:** The AMM's `initializePool` instruction likely has minimum reserve checks, but the off-chain script should validate BEFORE the irreversible `prepareTransition` step.

### Step 5: Verify the amounts are logged before execution
**Location:** `scripts/graduation/graduate.ts:865-866`
**Finding:** Amounts are logged to console before execution, providing an opportunity for the admin to catch errors visually. However, there's no interactive confirmation prompt.
**Evidence:**
```typescript
console.log(`  Pool token seed:  ${GRADUATION_POOL_SEED_TOKEN / 10 ** TOKEN_DECIMALS}M per pool`);
console.log(`  Pool SOL seed:    ${GRADUATION_POOL_SEED_SOL / LAMPORTS_PER_SOL} SOL per pool`);
```

### Recommended Fix
```typescript
// Before (vulnerable)
const GRADUATION_POOL_SEED_SOL = Number(process.env.SOL_POOL_SEED_SOL_OVERRIDE)
  || 1_000 * LAMPORTS_PER_SOL;

// After (fixed)
function parseEnvOverride(envVar: string | undefined, defaultValue: number, label: string): number {
  if (!envVar) return defaultValue;
  const parsed = Number(envVar);
  if (isNaN(parsed) || parsed <= 0) {
    throw new Error(`Invalid ${label}: "${envVar}" (must be a positive number)`);
  }
  // Sanity bounds for SOL amounts (in lamports)
  if (label.includes('SOL') && (parsed < 0.1 * LAMPORTS_PER_SOL || parsed > 10_000 * LAMPORTS_PER_SOL)) {
    throw new Error(`${label}=${parsed} out of sane range [0.1 SOL, 10000 SOL]`);
  }
  return parsed;
}

const GRADUATION_POOL_SEED_SOL = parseEnvOverride(
  process.env.SOL_POOL_SEED_SOL_OVERRIDE, 1_000 * LAMPORTS_PER_SOL, "SOL_POOL_SEED_SOL_OVERRIDE"
);
```

Additionally, add an interactive confirmation before the irreversible `prepareTransition` step.
