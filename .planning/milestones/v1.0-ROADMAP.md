# Milestone v1.0: Frontend Design & Interactive Factory

**Status:** SHIPPED 2026-02-24
**Phases:** 53-59 (plus 57.1 inserted)
**Total Plans:** 30

## Overview

Transform the functional tech prototype into a polished, branded interactive experience -- an explorable steampunk factory scene where clickable machines open themed modal interfaces for all protocol features. The factory scene IS the navigation layer: 6 positioned overlay objects with hover glow effects open modals containing the existing SwapForm, StakingForm, CarnageCard, and other components re-parented without logic changes. Asset optimization is the critical path (19.5MB PNGs must become <2MB WebP before any scene work). Zero new npm dependencies -- the entire visual layer uses Next.js Image, Tailwind v4 @theme tokens, CSS-only animations, and a custom modal system extending the existing ConnectModal pattern.

## Milestone Summary

**Decimal Phases:**

- Phase 57.1: Helius RPC/WebSocket Optimization (inserted after Phase 57 for urgent credit consumption fix)

**Key Decisions:**

- Zero new npm dependencies for visual layer (CSS-only animations, no Framer Motion/GSAP)
- Re-parent existing components into modals, do NOT rewrite -- hook logic stays untouched
- Contain-fit scene scaling using min() CSS function (letterbox/pillarbox with bg-factory-bg)
- Iris-open clip-path animation with dialog-relative coordinate conversion
- Toast system uses Popover API (popover=manual) to render in top layer above dialog::backdrop
- DashboardGrid orchestrator pattern: one component calls hooks, children receive props
- SplashScreen replaces localStorage-gated WelcomeModal: animated brass gear intro gate, shows every load
- Tooltips removed per user preference: glow hover provides sufficient interactive affordance
- useVisibility(stationId?) pattern for data hook lifecycle management
- Mobile CSS-only responsive visibility (display swap via @media width < 64rem)

**Issues Resolved:**

- Viewport-to-dialog coordinate conversion for iris animation origin
- Toast popover positioning above dialog::backdrop (Popover API solution)
- Privy wallet confirmation blocked by dialog inertness (usePrivyTopLayer.ts fix)
- Cross-origin CSP for embedded docs iframe
- Market cap display fix (replaced dead Jupiter v6 price API with CoinGecko)
- Canonical mint ordering in SwapStatsBar price derivation
- RPC credit drain from aggressive polling (~2,800/hr reduced to ~332/hr)

**Issues Deferred:**

- Book-layout documentation (Nextra theme customization for two-page book spread)
- Ambient factory animations (steam, bubbling, gears, lights) -- FUTURE-10-ANIM
- 3 phases missing formal VERIFICATION.md (verified via human checkpoints)

**Technical Debt Incurred:**
