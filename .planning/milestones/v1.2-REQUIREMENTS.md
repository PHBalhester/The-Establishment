# Requirements: v1.2 Bonding Curves & Launch Page

**Defined:** 2026-03-03
**Core Value:** Real SOL yield from real trading friction -- not ponzinomics.

## v1.2 Requirements

Requirements for the bonding curve launch system (7th on-chain program) and dedicated frontend launch page.

### Specification

- [x] **SPEC-01**: Bonding_Curve_Spec.md updated with buy+sell mechanics, 15% sell tax escrow, coupled graduation, token-proportional refunds, sells disabled when Filled, and tax escrow refundable on failure

### On-Chain Program

- [x] **CURVE-01**: Two linear bonding curves (CRIME + FRAUD), 460M tokens each, deterministic linear pricing P_start=0.0000009 → P_end=0.00000345 SOL/token, ∫₀^460M P(x)dx = 1,000 SOL
- [x] **CURVE-02**: Buy instruction computing tokens from SOL via integral (quadratic formula), with minimum_tokens_out slippage protection
- [x] **CURVE-03**: Sell instruction computing SOL from tokens via reverse integral, 15% tax deducted and routed to separate tax escrow PDA, with minimum_sol_out slippage protection
- [x] **CURVE-04**: Sells disabled once curve reaches Filled status (all 460M tokens sold, SOL reserves = 1,000)
- [x] **CURVE-05**: Coupled graduation: both curves must fill within 48 hours for success; Filled curve locks and waits for partner
- [x] **CURVE-06**: Permissionless graduation trigger (anyone can call when both Filled) and permissionless failure trigger (anyone can call after 48hr deadline if not both graduated)
- [x] **CURVE-07**: On success: prepare_transition locks curves, assets staged for pool seeding, tax escrow routed to carnage fund
- [x] **CURVE-08**: On failure: SOL vault + tax escrow proportionally refunded based on token holdings; users keep tokens
- [x] **CURVE-09**: 20M token per-wallet cap per curve, enforced on buy instruction
- [x] **CURVE-10**: CurveState PDA (1 per curve) with full state tracking (sol_raised, tokens_sold, tax_escrow_balance, status, deadline_slot, participant_count). No ParticipantState PDA -- per-wallet cap uses ATA balance reads, per-user purchase history available via emitted events (ParticipantState eliminated in v1.2 spec update, Phase 70)

### Protocol Integration

- [x] **INTG-01**: Transfer Hook whitelist entries for curve token vaults (2 entries) created before whitelist authority burn
- [x] **INTG-02**: Client-side graduation orchestration: prepare_transition → AMM pool seeding → Conversion Vault seeding → tax escrow distribution → finalize_transition (multi-TX, no CPI depth issues)
- [x] **INTG-03**: AMM pool seeding at P_end price: 290M tokens + 1,000 SOL per pool (CRIME/SOL and FRAUD/SOL)
- [x] **INTG-04**: Tax escrow → carnage fund on success; Conversion Vault seeded with 250M CRIME + 250M FRAUD + 20M PROFIT during graduation
- [x] **INTG-05**: Deploy pipeline extended: build.sh (7th program with devnet feature), deploy-all.sh pipeline, initialize.ts (curve setup + whitelist + burn sequencing)
- [x] **INTG-06**: Feature-gated mint addresses for bonding curve program + ALT extension with curve addresses

### Security Testing

- [x] **SAFE-01**: Property testing buy/sell math (10K+ iterations): no overflow, precision loss bounded, buy/sell round-trip returns ≤ spent SOL after tax
- [x] **SAFE-02**: SOL vault solvency invariant verified at all state transitions (vault_balance ≥ expected_from_integral)
- [x] **SAFE-03**: Rent-exempt minimum accounted for in all SOL vault calculations (subtract before distributing)

### Launch Page Frontend

- [x] **PAGE-01**: Dedicated /launch page route with steampunk theming consistent with factory design language
- [x] **PAGE-02**: Two steampunk pressure gauges (SVG + CSS) showing % progress per curve toward 1,000 SOL / 460M tokens
- [x] **PAGE-03**: Swap function: buy (SOL input → token output preview → execute) and sell (token input → SOL output after 15% tax preview → execute)
- [x] **PAGE-04**: Real-time tax escrow counter showing total SOL collected in 15% sell tax per curve
- [x] **PAGE-05**: Inline Nextra documentation embed (iframe to docs site)
- [x] **PAGE-06**: 48-hour countdown timer, current price per token (SOL + USD), SOL raised / market cap display
- [x] **PAGE-07**: Refund interface (conditional on Failed status): show proportional refund amount, claim button per curve
- [x] **PAGE-08**: Wallet connection, error handling with pre-validation (cap, balance, deadline), transaction status feedback, mobile responsive layout

## Out of Scope

| Feature | Reason |
|---------|--------|
| Chat/comment section | Toxic, requires moderation infrastructure |
| Creator profile/editing | Not a launchpad -- this is a specific protocol launch |
| Pre-sale/insider access | Zero team allocation is core trust feature |
| Candlestick charts during curve | Misleading on deterministic curve (no market dynamics) |
| Leverage/margin/flash loans | Amplifies manipulation vectors (EP-061, EP-094) |
| Referral/affiliate system | Perverse incentives, legal exposure |
| Auto-buy/DCA | Meaningless on monotonically increasing curve |
| Whitelist/verification gate | User decided open access for v1.2; 20M cap is only sybil resistance |
| Independent graduation | Both must succeed or fail together for v1.2; independent deferred |
