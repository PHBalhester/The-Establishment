  <action>
    Create `consolidate_for_refund.rs` implementing spec Section 8.9.

    **Accounts struct `ConsolidateForRefund`:**
    - `curve_state: Account<'info, CurveState>` (mut, seeds = [CURVE_SEED, curve_state.token_mint.as_ref()], bump = curve_state.bump)
    - `partner_curve_state: Account<'info, CurveState>` (seeds = [CURVE_SEED, partner_curve_state.token_mint.as_ref()], bump = partner_curve_state.bump). Read-only -- used for is_refund_eligible check.
      - Add constraint: `partner_curve_state.key() != curve_state.key() @ CurveError::InvalidStatus` -- prevents passing the same curve as its own partner.
    - `tax_escrow: UncheckedAccount<'info>` (mut, seeds = [TAX_ESCROW_SEED, curve_state.token_mint.as_ref()], bump, constraint = tax_escrow.key() == curve_state.tax_escrow @ CurveError::InvalidStatus). CHECK: SOL-only PDA, validated by seeds + stored key.
    - `sol_vault: UncheckedAccount<'info>` (mut, seeds = [CURVE_SOL_VAULT_SEED, curve_state.token_mint.as_ref()], bump). CHECK: SOL-only PDA, validated by seeds.
    - No signer constraint (permissionless).

    **Handler logic:**
    1. Require `curve.is_refund_eligible(partner_curve_state.status)` -> `CurveError::NotRefundEligible`
    2. Require `!curve.escrow_consolidated` -> `CurveError::EscrowAlreadyConsolidated`
    3. Get `rent = Rent::get()?`, `rent_exempt = rent.minimum_balance(0)`
    4. Get `escrow_lamports = tax_escrow.lamports()`, `transferable = escrow_lamports.saturating_sub(rent_exempt)`
    5. If `transferable > 0`: direct lamport manipulation `**tax_escrow.try_borrow_mut_lamports()? -= transferable`, `**sol_vault.try_borrow_mut_lamports()? += transferable`
    6. Set `curve.escrow_consolidated = true`
    7. Emit `EscrowConsolidated { token: curve.token, escrow_amount: transferable, new_vault_balance: sol_vault.lamports() }`

    **Important nuance:** The consolidation transfers whatever is available (could be 0 if no sells happened). The flag is still set to true even if transferable was 0 -- the point is that consolidation has been performed, not that there were funds to move.

    **Imports:**
    ```rust

    use crate::constants::*;
    use crate::error::CurveError;
    use crate::events::EscrowConsolidated;
    use crate::state::{CurveState, CurveStatus};
    ```

    **Accounts struct `ClaimRefund`:**
    - `user: Signer<'info>` (mut -- receives SOL refund)
    - `curve_state: Account<'info, CurveState>` (mut, seeds = [CURVE_SEED, token_mint.key().as_ref()], bump = curve_state.bump)
    - `partner_curve_state: Account<'info, CurveState>` (seeds = [CURVE_SEED, partner_curve_state.token_mint.as_ref()], bump = partner_curve_state.bump). Read-only for is_refund_eligible.
      - Constraint: `partner_curve_state.key() != curve_state.key() @ CurveError::InvalidStatus`
    - `user_token_account: Box<InterfaceAccount<'info, TokenAccount>>` (mut, associated_token::mint = token_mint, associated_token::authority = user, associated_token::token_program = token_program)
    - `token_mint: Box<InterfaceAccount<'info, Mint>>` (mut -- burn reduces supply)
    - `sol_vault: UncheckedAccount<'info>` (mut, seeds = [CURVE_SOL_VAULT_SEED, token_mint.key().as_ref()], bump). CHECK: SOL-only PDA.
    - `token_program: Interface<'info, TokenInterface>`

    **Handler logic (spec Section 8.8):**
    1. Require `curve.is_refund_eligible(partner_curve_state.status)` -> `CurveError::NotRefundEligible`
    2. Require `curve.escrow_consolidated` -> `CurveError::EscrowNotConsolidated`
    3. Read `user_balance = user_token_account.amount`, require `user_balance > 0` -> `CurveError::NothingToBurn`
    4. Read `total_outstanding = curve.tokens_sold`, require `total_outstanding > 0` -> `CurveError::NoTokensOutstanding`
    5. Compute refund pool: `rent = Rent::get()?`, `rent_exempt = rent.minimum_balance(0)`, `refund_pool = sol_vault.lamports().checked_sub(rent_exempt).ok_or(CurveError::Overflow)?`
    6. Compute proportional refund (floor, protocol-favored): `refund_amount = ((user_balance as u128).checked_mul(refund_pool as u128).ok_or(CurveError::Overflow)? / (total_outstanding as u128)) as u64`
    7. **Burn ALL tokens** (Token-2022, user signs, no Transfer Hook):
       ```rust
       anchor_spl::token_interface::burn(
           CpiContext::new(
               ctx.accounts.token_program.to_account_info(),
               anchor_spl::token_interface::Burn {
                   mint: ctx.accounts.token_mint.to_account_info(),
                   from: ctx.accounts.user_token_account.to_account_info(),
                   authority: ctx.accounts.user.to_account_info(),
               },
           ),
           user_balance,
       )?;
       ```
    8. **Transfer refund SOL** (direct lamport manipulation): `**sol_vault.try_borrow_mut_lamports()? -= refund_amount`, `**user.try_borrow_mut_lamports()? += refund_amount`
    9. **Update tokens_sold** (denominator shrinks): `curve.tokens_sold = curve.tokens_sold.checked_sub(user_balance).ok_or(CurveError::Overflow)?`
    10. Emit `RefundClaimed { user: user.key(), token: curve.token, tokens_burned: user_balance, refund_amount, remaining_tokens_sold: curve.tokens_sold, remaining_vault_balance: sol_vault.lamports(), slot: Clock::get()?.slot }`

    **Critical: Read remaining_vault_balance AFTER the SOL transfer (step 10, not before step 8). Direct lamport reads reflect post-mutation state within the same instruction.**
