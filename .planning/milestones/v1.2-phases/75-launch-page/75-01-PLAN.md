  2. Create `app/lib/curve/curve-math.ts`:
     Port the THREE public functions from programs/bonding_curve/src/math.rs to TypeScript using BigInt:

     a) `bigintSqrt(n: bigint): bigint` -- Newton's method integer sqrt (JS has no built-in BigInt sqrt).

     b) `calculateTokensOut(solLamports: bigint, currentSold: bigint): bigint` -- Mirrors on-chain calculate_tokens_out exactly:
        - coef = P_START * TOTAL_FOR_SALE + (P_END - P_START) * currentSold
        - b_num = P_END - P_START = 2550n
        - b_den = TOTAL_FOR_SALE
        - discriminant = coef^2 + 2n * b_num * solLamports * TOKEN_DECIMAL_FACTOR * b_den
        - tokens = (bigintSqrt(discriminant) - coef) / b_num
        - Floor to ensure protocol-favored direction
        - Cap at remaining tokens (TOTAL_FOR_SALE - currentSold)

     c) `calculateSolForTokens(tokens: bigint, currentSold: bigint): bigint` -- Mirrors on-chain calculate_sol_for_tokens:
        - Linear integral: SOL = [P_START * N + (P_END - P_START) * N * (2*x1 + N) / (2 * TOTAL)] / TOKEN_DECIMAL_FACTOR
        - Where N = tokens, x1 = currentSold - tokens (position BEFORE selling)
        - For sell preview: pass currentSold = curve.tokensSold, tokens = amount to sell
        - Floor division (integer division in BigInt is naturally floor)

     d) `calculateSellTax(solGross: bigint): bigint` -- Ceil-rounded BPS matching on-chain:
        `(solGross * 1500n + 9999n) / 10000n`

     e) `getCurrentPrice(tokensSold: bigint): bigint` -- Price in lamports per human token at position:
        `P_START + (P_END - P_START) * tokensSold / TOTAL_FOR_SALE`

     CRITICAL: Match on-chain math.rs EXACTLY. Use identical variable names where possible. Every intermediate must use BigInt -- no Number conversion until final display formatting. The quadratic formula intermediate values reach ~2.5e36 which exceeds Number.MAX_SAFE_INTEGER.
