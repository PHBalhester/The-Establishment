  <files>app/lib/curve/hook-accounts.ts</files>
  <action>
  Create `app/lib/curve/hook-accounts.ts` -- derives the 4 Transfer Hook remaining_accounts needed for purchase and sell instructions.

  CRITICAL from MEMORY.md: Hook accounts per mint = 4. These are:
  1. ExtraAccountMetaList PDA: seeds = ["extra-account-metas", mint], program = TRANSFER_HOOK
  2. Whitelist entry for source: seeds = ["whitelist", source], program = TRANSFER_HOOK
  3. Whitelist entry for destination: seeds = ["whitelist", destination], program = TRANSFER_HOOK
  4. Hook program ID itself: PROGRAM_IDS.TRANSFER_HOOK (as non-signer, non-writable)

  Export function:
  ```
  getCurveHookAccounts(
    mint: PublicKey,
    source: PublicKey,    // token account sending tokens
    destination: PublicKey // token account receiving tokens
  ): AccountMeta[]
  ```

  Returns array of 4 AccountMeta objects, all with `isSigner: false, isWritable: false`.

  Use `PublicKey.findProgramAddressSync()` for PDA derivation.
  Import PROGRAM_IDS and SEEDS from `@dr-fraudsworth/shared`.

  For PURCHASE (vault -> user):
    source = curve token vault PDA
    destination = user's ATA for the token mint

  For SELL (user -> vault):
    source = user's ATA for the token mint
    destination = curve token vault PDA

  NOTE: Do NOT use spl-token's `createTransferCheckedWithTransferHookInstruction` -- it has browser Buffer polyfill issues (documented in MEMORY.md). Manual PDA derivation is the project standard.

<task type="auto">
  <name>Task 2: Bonding curve instruction builders (purchase, sell, claim_refund)</name>
  <files>app/lib/curve/curve-tx-builder.ts</files>
  <action>
  Create `app/lib/curve/curve-tx-builder.ts` with three instruction builders.

  All builders accept a `Program<BondingCurve>` instance (from getBondingCurveProgram in anchor.ts) and return `Promise<TransactionInstruction>`.

  1. **buildPurchaseInstruction**:
     ```
     buildPurchaseInstruction(
       program: Program<BondingCurve>,
       user: PublicKey,
       tokenMint: PublicKey,  // MINTS.CRIME or MINTS.FRAUD
       solAmount: bigint,
       minimumTokensOut: bigint,
     ): Promise<TransactionInstruction>
     ```
     - Derive PDAs: curveState (["curve", mint]), tokenVault (["curve_token_vault", mint]), solVault (["curve_sol_vault", mint]) using program.programId
     - Derive user's ATA for tokenMint using TOKEN_2022_PROGRAM_ID (getAssociatedTokenAddressSync with allowOwnerOffCurve=true, programId=TOKEN_2022_PROGRAM_ID)
     - Get hook accounts: getCurveHookAccounts(tokenMint, tokenVault, userAta) -- source is vault (sending tokens to user)
     - Call program.methods.purchase(new BN(solAmount.toString()), new BN(minimumTokensOut.toString()))
     - Pass accounts: user, curveState, userTokenAccount (the ATA), tokenVault, solVault, tokenMint, tokenProgram (TOKEN_2022_PROGRAM_ID), associatedTokenProgram (ASSOCIATED_TOKEN_PROGRAM_ID), systemProgram
     - Pass .remainingAccounts(hookAccounts)
     - Return .instruction()

  2. **buildSellInstruction**:
     ```
     buildSellInstruction(
       program: Program<BondingCurve>,
       user: PublicKey,
       tokenMint: PublicKey,
       tokensToSell: bigint,
       minimumSolOut: bigint,
     ): Promise<TransactionInstruction>
     ```
     - Same PDA derivation as purchase, plus taxEscrow (["tax_escrow", mint])
     - Hook accounts: getCurveHookAccounts(tokenMint, userAta, tokenVault) -- source is user (sending tokens to vault), NOTE REVERSED vs purchase
     - Call program.methods.sell(new BN(tokensToSell.toString()), new BN(minimumSolOut.toString()))
     - Accounts: user, curveState, userTokenAccount, tokenVault, solVault, taxEscrow, tokenMint, tokenProgram (TOKEN_2022_PROGRAM_ID), systemProgram
     - Pass .remainingAccounts(hookAccounts)
     - Return .instruction()

  3. **buildClaimRefundInstruction**:
     ```
     buildClaimRefundInstruction(
       program: Program<BondingCurve>,
       user: PublicKey,
       tokenMint: PublicKey,
       partnerTokenMint: PublicKey,
     ): Promise<TransactionInstruction>
     ```
     - Derive curveState for tokenMint, partnerCurveState for partnerTokenMint
     - Derive solVault for tokenMint, userTokenAccount ATA for tokenMint
     - NOTE: claim_refund does NOT need remaining_accounts -- the refund burns tokens which does not trigger Transfer Hooks
     - Call program.methods.claimRefund()
     - Accounts: user, curveState, partnerCurveState, userTokenAccount, tokenMint, solVault, tokenProgram (TOKEN_2022_PROGRAM_ID)
     - Return .instruction()

  Import types: PublicKey, TransactionInstruction, SystemProgram from @solana/web3.js. BN from @coral-xyz/anchor. TOKEN_2022_PROGRAM_ID, ASSOCIATED_TOKEN_PROGRAM_ID, getAssociatedTokenAddressSync from @solana/spl-token. BondingCurve type from @/idl/types/bonding_curve.

  IMPORTANT: Use `new BN(bigintValue.toString())` to convert BigInt to BN for Anchor method args. Anchor expects BN, not BigInt.
  </action>
