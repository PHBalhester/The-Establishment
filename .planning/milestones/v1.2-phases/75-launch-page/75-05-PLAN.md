  1. Create `app/components/launch/RefundPanel.tsx`:
     Props: `{ crime: CurveStateData, fraud: CurveStateData }`

     This panel REPLACES the BuySellPanel when the curve phase has failed.

     Per-curve refund display (render both CRIME and FRAUD sections):
     - Show user's token balance (from useTokenBalances hook): "Your CRIME tokens: X"
     - Calculate proportional refund:
       The on-chain claim_refund computes: `refund = (user_tokens / total_outstanding) * (sol_vault + tax_escrow)`
       Client-side approximation for display:
       - total_outstanding = tokensSold - tokensReturned (tokens still held by users)
       - total_refundable = solRaised - solReturned + taxCollected (sol_vault balance + tax_escrow balance after consolidation)
       - user_refund = (userBalance * total_refundable) / total_outstanding
       - Display: "Estimated refund: X.XXXX SOL (~$Y.YY USD)"
     - Claim button per curve: "Claim CRIME Refund" / "Claim FRAUD Refund"
     - Disabled if user has 0 balance for that token
     - On click: build and submit claimRefundInstruction
       - For CRIME: tokenMint = MINTS.CRIME, partnerTokenMint = MINTS.FRAUD
       - For FRAUD: tokenMint = MINTS.FRAUD, partnerTokenMint = MINTS.CRIME
       - Use sign-then-send pattern, confirm-transaction, error parsing
     - After successful claim: balance should update via useTokenBalances refresh, refund amount recalculates to 0
     - Show TX status feedback (submitting, confirmed, error)

     Steampunk styling: distressed/warning aesthetic (darker tones, amber warning accents) since this is a failure state. Clear messaging: "The curve has failed. You can claim your proportional refund."
