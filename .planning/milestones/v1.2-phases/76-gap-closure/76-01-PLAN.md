  <name>Task 2: Fix RefundPanel display bug</name>
  <files>app/components/launch/RefundPanel.tsx</files>
  <action>
Fix line 93 in RefundPanel.tsx. Change:
```
const totalOutstanding = curve.tokensSold - curve.tokensReturned;
```
to:
```
const totalOutstanding = curve.tokensSold;
```

Why: On-chain `tokens_sold` is already decremented during sells (sell returns tokens to vault and decrements `tokens_sold`). Subtracting `tokensReturned` double-counts the reduction, inflating the refund estimate by ~12.5% when sells have occurred. The on-chain `claim_refund` instruction uses `tokens_sold` directly as the denominator, so the client must match.

Also update the JSDoc comment near line 11 that says `total_outstanding = tokensSold - tokensReturned` to say `total_outstanding = tokensSold` to match the fix.

No other changes needed -- the audit confirms on-chain refund logic is correct.
