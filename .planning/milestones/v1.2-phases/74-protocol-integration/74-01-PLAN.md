**withdraw_graduated_sol.rs** -- Admin-only instruction to withdraw all SOL (minus rent-exempt minimum) from a graduated curve's SOL vault.

**Accounts struct (WithdrawGraduatedSol):**
```rust
#[derive(Accounts)]
pub struct WithdrawGraduatedSol<'info> {
    /// Protocol authority (deployer). Must sign.
    #[account(mut)]
    pub authority: Signer<'info>,

    /// CurveState PDA -- must be Graduated.
    #[account(
        seeds = [CURVE_SEED, curve_state.token_mint.as_ref()],
        bump = curve_state.bump,
        constraint = curve_state.status == CurveStatus::Graduated @ CurveError::CurveNotGraduated,
    )]
    pub curve_state: Account<'info, CurveState>,

    /// SOL vault PDA -- source of SOL. Validated by seeds.
    /// CHECK: SOL-only PDA, validated by seeds matching curve_state.sol_vault
    #[account(
        mut,
        seeds = [CURVE_SOL_VAULT_SEED, curve_state.token_mint.as_ref()],
        bump,
        constraint = sol_vault.key() == curve_state.sol_vault @ CurveError::InvalidStatus,
    )]
    pub sol_vault: UncheckedAccount<'info>,

    pub system_program: Program<'info, System>,
}
```

**Handler logic:**
1. Get rent-exempt minimum for 0-byte account: `let rent = Rent::get()?.minimum_balance(0);`
2. Calculate withdrawable: `let vault_balance = sol_vault.lamports(); let withdrawable = vault_balance.checked_sub(rent).unwrap_or(0);`
3. If withdrawable == 0, return Ok (idempotent -- already withdrawn or only rent remains)
4. Transfer lamports from sol_vault to authority using direct lamport manipulation (the sol_vault is a program-owned PDA, so we can modify lamports directly):
   ```rust
   **sol_vault.try_borrow_mut_lamports()? -= withdrawable;
   **authority.try_borrow_mut_lamports()? += withdrawable;
   ```
5. Emit a `SolWithdrawn` event with curve token_mint, withdrawable amount, and slot.

**Add SolWithdrawn event to events.rs** with fields: `token_mint: Pubkey`, `amount: u64`, `slot: u64`.

**Why direct lamport manipulation?** The sol_vault is a program-owned PDA (owned by the bonding_curve program). The Solana runtime allows programs to freely modify lamports of accounts they own. No CPI to system_program needed. This is the same pattern used in `claim_refund.rs` for refund payouts.

**Security:** The curve must be Graduated (terminal state). Only the authority signer can call. The sol_vault PDA seeds are verified. This instruction is safe because Graduated is irreversible -- no future operations depend on the SOL vault balance.

```rust
#[derive(Accounts)]
pub struct CloseTokenVault<'info> {
    /// Protocol authority (deployer). Must sign. Receives rent.
    #[account(mut)]

    /// CurveState PDA -- must be Graduated.
    #[account(

    )]

    /// Token vault PDA to close. Must be empty (0 token balance).
    /// Seeds validated against curve state.
    #[account(
        mut,
        seeds = [CURVE_TOKEN_VAULT_SEED, curve_state.token_mint.as_ref()],
        bump,
        constraint = token_vault.key() == curve_state.token_vault @ CurveError::InvalidStatus,
    )]
    pub token_vault: InterfaceAccount<'info, TokenAccount>,

    /// Token mint (for close_account CPI)
    pub token_mint: InterfaceAccount<'info, Mint>,

    /// Token-2022 program
    pub token_program: Interface<'info, TokenInterface>,
}
```

**Handler logic:**

Better: add no new error, just `require!(ctx.accounts.token_vault.amount == 0, CurveError::InvalidStatus);` since closing a non-empty vault is an invalid operation state.
2. Close the token account using `anchor_spl::token_2022::close_account` CPI. The authority for the vault is the CurveState PDA itself (check how the vault was initialized -- look at initialize_curve.rs to confirm the vault's authority). Actually, looking at the bonding curve structure, the token_vault's authority is likely set to the CurveState PDA or a separate vault authority PDA. Check initialize_curve.rs to see what authority was used. The vault is created with `authority = curve_state` (PDA as owner). So the close CPI needs PDA signer seeds: `[CURVE_SEED, token_mint.key().as_ref(), &[curve_state.bump]]`.
