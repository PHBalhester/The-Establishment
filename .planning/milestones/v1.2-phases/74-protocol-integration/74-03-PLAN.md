// Check if already funded
const crimeVaultInfo = await getAccount(connection, crimeTokenVault, "confirmed", TOKEN_2022_PROGRAM_ID).catch(() => null);

  const [extraMeta] = PublicKey.findProgramAddressSync(

  const hookAccounts = [
    { pubkey: extraMeta, isSigner: false, isWritable: false },
    { pubkey: wlSource, isSigner: false, isWritable: false },
    { pubkey: wlDest, isSigner: false, isWritable: false },
    { pubkey: hookProgramId, isSigner: false, isWritable: false },
  ];

  const sig = await programs.bondingCurve.methods
    .fundCurve()

    .rpc();
  log.step(stepNum, TOTAL_STEPS, "Fund CRIME curve (460M tokens)", "done", sig);
  completed++;
}

**CRITICAL: Vault whitelist MUST occur after init but BEFORE fund, because fund_curve transfers 460M tokens which triggers Transfer Hook validation. If the vault isn't whitelisted, Transfer Hook rejects the transfer with WhitelistCheckFailed (0x1770).** The curve vault is created during initialize_curve (Step 17). So the whitelist entry for the vault must exist before fund_curve. This creates a sequencing issue:

Correct order for each curve:
1. Initialize curve (creates vault PDA)
2. Whitelist the vault (so Transfer Hook allows transfers TO it)
3. Fund the curve (transfers 460M tokens via Transfer Hook)
4. Start the curve

So the actual step ordering should be:
- Step 17: Initialize CRIME curve
- Step 18: Initialize FRAUD curve
- Step 19: Whitelist CRIME curve token vault
- Step 20: Whitelist FRAUD curve token vault
- Step 21: Fund CRIME curve (460M tokens)
- Step 22: Fund FRAUD curve (460M tokens)
- Step 23: Start CRIME curve
- Step 24: Start FRAUD curve
- Step 25: Burn whitelist authority (IRREVERSIBLE -- LAST whitelist step)
- Step 26: Generate PDA manifest (moved from Step 17 to Step 26)

This means TOTAL_STEPS = 26 (16 existing + 9 new + 1 moved manifest step).
