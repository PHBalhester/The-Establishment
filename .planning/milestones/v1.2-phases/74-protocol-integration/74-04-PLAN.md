/**

 *

 *   8. Create FRAUD/SOL AMM pool (290M FRAUD + ~1000 SOL)

 *   10. Distribute CRIME tax escrow to carnage fund
 *   11. Distribute FRAUD tax escrow to carnage fund

 *   source .env && npx tsx scripts/graduation/graduate.ts

 *   - Both curves must be in Filled status

**Step implementations:**

**Step 1 (verify_curves_filled):** Fetch both CurveState accounts, check status == Filled (or Graduated if re-running after a partial graduation). If either is not Filled/Graduated, abort with clear error showing each curve's current status. This is a verification-only step (no TX), but still checkpointed.

**Step 2 (prepare_transition):** Call `programs.bondingCurve.methods.prepareTransition()` with both CurveState PDAs. Both curves transition from Filled to Graduated. If already Graduated (re-run), skip.

**Steps 3-4 (withdraw SOL):** Call `programs.bondingCurve.methods.withdrawGraduatedSol()` for each curve. This transfers ~1000 SOL (minus rent) from each curve's SOL vault to the admin wallet. Log the exact amount withdrawn.

**Steps 5-6 (close token vaults):** Call `programs.bondingCurve.methods.closeTokenVault()` for each curve. Recovers rent from empty token vaults. If vaults are not empty (shouldn't happen since 460M were sold), log a warning and skip.

**Steps 7-8 (create AMM pools):** Use the exact same `initPool` pattern from initialize.ts. Requires:
- Admin WSOL account (create wrapped native account with the withdrawn SOL)
- Admin CRIME/FRAUD token accounts (held since deploy time with 290M each)
- Pool creation with `programs.amm.methods.initializePool()` with Transfer Hook remaining_accounts
- Fee: SOL_POOL_FEE_BPS
- Amounts: 290M tokens (SOL_POOL_SEED_TOKEN or hardcoded 290_000_000_000_000) + 1000 SOL (SOL_POOL_SEED_SOL or 1_000_000_000_000 lamports)
- **CRITICAL:** Read SOL_POOL_SEED_SOL and SOL_POOL_SEED_TOKEN from env or use hardcoded graduation amounts. The Phase 69 lesson: missing .env sourcing uses test defaults. For graduation, hardcode the correct amounts as constants (290M tokens, 1000 SOL) with env override capability.

**Step 9 (seed Conversion Vault):** Transfer 250M CRIME, 250M FRAUD, and ~20M PROFIT to the conversion vault token accounts. Use the same pattern from initialize.ts Step 10 (createTransferCheckedInstruction with hook remaining_accounts).

**Steps 10-11 (distribute tax escrow):** Call `programs.bondingCurve.methods.distributeTaxEscrow()` for each curve. This sends the accumulated sell tax SOL to the carnage fund.

**Checkpoint+resume loop:**
```typescript
for (let i = 0; i < state.steps.length; i++) {
  const step = state.steps[i];
  if (step.completed) {
    console.log(`  SKIP: ${step.name} (completed at ${step.timestamp})`);
    continue;
  }

  console.log(`\n  EXEC: ${step.name}...`);
  try {
    const sig = await executeStep(step.name, /* ... */);
    step.completed = true;
    step.txSig = sig || "verification-only";
    step.timestamp = new Date().toISOString();
    saveState(state);
    console.log(`  DONE: ${step.name} (${sig?.slice(0, 12) || "ok"})`);
  } catch (err: any) {
    step.error = err.message;
    saveState(state);
    console.error(`\n  FAILED: ${step.name}`);
    console.error(`  Error: ${err.message}`);
    console.error(`\n  Re-run this script to resume from this step.`);
    process.exit(1);
  }
}
```
