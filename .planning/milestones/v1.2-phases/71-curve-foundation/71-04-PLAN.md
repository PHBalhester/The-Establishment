**Accounts struct (Purchase):**
```rust
#[derive(Accounts)]
pub struct Purchase<'info> {
    /// The buyer
    #[account(mut)]
    pub user: Signer<'info>,

    /// Curve state PDA -- must be Active
    #[account(
        mut,
        seeds = [CURVE_SEED, token_mint.key().as_ref()],
        bump = curve_state.bump,
        constraint = curve_state.status == CurveStatus::Active @ CurveError::CurveNotActive,
    )]
    pub curve_state: Account<'info, CurveState>,

    /// User's ATA for this token -- used for cap enforcement AND as token destination
    /// Must be init_if_needed because first-time buyers won't have an ATA yet
    #[account(
        init_if_needed,
        payer = user,
        associated_token::mint = token_mint,
        associated_token::authority = user,
        associated_token::token_program = token_program,
    )]
    pub user_token_account: InterfaceAccount<'info, TokenAccount>,

    /// Curve's token vault PDA (source of tokens)
    #[account(
        mut,
        seeds = [CURVE_TOKEN_VAULT_SEED, token_mint.key().as_ref()],
        bump,
    )]
    pub token_vault: InterfaceAccount<'info, TokenAccount>,

    /// Curve's SOL vault PDA (destination for SOL)
    /// CHECK: SOL-only PDA, validated by seeds
    #[account(
        mut,
        seeds = [CURVE_SOL_VAULT_SEED, token_mint.key().as_ref()],
        bump,
    )]
    pub sol_vault: UncheckedAccount<'info>,

    /// Token mint (CRIME or FRAUD)
    pub token_mint: InterfaceAccount<'info, Mint>,

    /// Token-2022 program
    pub token_program: Interface<'info, TokenInterface>,

    /// Associated Token Account program (for init_if_needed)
    pub associated_token_program: Program<'info, AssociatedToken>,

    /// System program (for SOL transfer)
    pub system_program: Program<'info, System>,
}
```

**Handler -- follow the spec Section 8.5 logic EXACTLY, step by step:**

1. **Deadline check:** `Clock::get()?.slot <= curve_state.deadline_slot` (error: DeadlinePassed)
2. **Minimum check:** `sol_amount >= MIN_PURCHASE_SOL` (error: BelowMinimum)
3. **Calculate tokens:** `let tokens_out = crate::math::calculate_tokens_out(sol_amount, curve.tokens_sold)?;`
4. **Check tokens > 0:** If tokens_out == 0, return `CurveError::InsufficientTokensOut` (dust buy protection)
5. **Wallet cap check:** `user_token_account.amount + tokens_out <= MAX_TOKENS_PER_WALLET` (error: WalletCapExceeded). Use checked_add to prevent overflow.
6. **Supply cap:** `let remaining = TARGET_TOKENS - curve.tokens_sold; let actual_tokens = std::cmp::min(tokens_out, remaining);`
7. **Recalculate SOL for partial fill:** If `actual_tokens < tokens_out`, compute `actual_sol = crate::math::calculate_sol_for_tokens(curve.tokens_sold, actual_tokens)?;` else `actual_sol = sol_amount;`
8. **Re-check wallet cap with actual_tokens** (may differ from tokens_out on partial fill)
9. **Transfer SOL from user to sol_vault:** Use `system_program::transfer` CPI. The user is the signer.
10. **Transfer tokens from token_vault to user:** This is a Token-2022 transfer from a PDA-owned vault. The curve_state PDA is the authority. Use `transfer_checked` with PDA signer seeds. CRITICAL: Token-2022 Transfer Hook will fire on this transfer. The remaining_accounts must include the hook accounts (ExtraAccountMetaList, whitelist source, whitelist dest, hook program -- 4 accounts per mint, see MEMORY.md). Accept remaining_accounts in the instruction context and pass them to the transfer CPI. If using `anchor_spl::token_2022::transfer_checked`, note the known issue that it does NOT forward remaining_accounts. Use manual `invoke_signed` with the transfer_checked instruction data instead (same pattern as conversion-vault hook_helper.rs). Alternatively, try `anchor_spl::token_2022::spl_token_2022::instruction::transfer_checked` to build the instruction, then `invoke_signed` with all accounts including remaining_accounts.
11. **Participant count:** If `user_ata_balance_before == 0`, increment `curve.participant_count += 1;` (read ATA balance BEFORE the token transfer)
12. **Update state:** `curve.tokens_sold += actual_tokens; curve.sol_raised += actual_sol;` (all checked arithmetic)
13. **Status transition:** If `curve.tokens_sold >= TARGET_TOKENS`, set `curve.status = CurveStatus::Filled;` and emit CurveFilled
14. **Emit TokensPurchased event** with all fields from spec Section 10
