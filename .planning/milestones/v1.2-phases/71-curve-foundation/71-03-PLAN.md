<objective>
Implement the three admin-only lifecycle instructions: `initialize_curve`, `fund_curve`, and `start_curve`. These set up each curve's on-chain state before users can buy.

Purpose: Before users can purchase tokens, each curve must be initialized (PDAs created), funded (460M tokens deposited), and started (status set to Active, deadline set). These are admin-only one-time operations that follow the same pattern as the existing `conversion-vault/initialize`. Getting the PDA derivations, account constraints, and status transitions right here prevents cascading errors in the purchase instruction.

Output: Three compiling instruction modules with Anchor account structs, handler functions, and event emissions matching the Bonding_Curve_Spec.md Sections 8.1-8.3 exactly.
</objective>

**events.rs** -- Define ALL event structs from spec Section 10. Include events needed for future phases (sell, refund, graduation) so the file doesn't need modification later. Each event uses `#[event]` attribute. Include: `CurveInitialized`, `CurveFunded`, `CurveStarted`, `CurveFilled`, `CurveFailed`, `TokensPurchased`, `TokensSold`, `TaxCollected`, `EscrowConsolidated`, `EscrowDistributed`, `RefundClaimed`, `TransitionPrepared`, `TransitionComplete`. Import `Token` from `crate::state`.

**initialize_curve.rs** -- Follow spec Section 8.1 exactly. Anchor accounts struct:
- `authority`: Signer, Mut (payer). Admin-only -- but since there's no stored authority PDA, use a simple signer for now (admin enforcement is via deploy script, not on-chain constraint -- consistent with conversion-vault pattern where "any signer" can init but it's called once from deploy).
- `curve_state`: Init PDA with seeds `[CURVE_SEED, token_mint.key().as_ref()]`, bump, space = CurveState::LEN, payer = authority
- `token_vault`: Init PDA Token-2022 account with seeds `[CURVE_TOKEN_VAULT_SEED, token_mint.key().as_ref()]`, mint = token_mint, authority = curve_state
- `sol_vault`: SOL-only PDA. Use `UncheckedAccount<'info>` type with `/// CHECK: SOL-only PDA, validated by seeds` safety comment. Constraint: `#[account(init, payer = authority, space = 0, seeds = [CURVE_SOL_VAULT_SEED, token_mint.key().as_ref()], bump)]`. Anchor's `init` with `space = 0` and `UncheckedAccount` creates a zero-data system-owned PDA that holds lamports only. This is the definitive pattern -- do not use SystemAccount (requires `space = 8`), do not use handler-level `create_account`.
- `tax_escrow`: Same pattern as sol_vault. Use `UncheckedAccount<'info>` with `/// CHECK: SOL-only PDA, validated by seeds` and `#[account(init, payer = authority, space = 0, seeds = [TAX_ESCROW_SEED, token_mint.key().as_ref()], bump)]`.
- `token_mint`: InterfaceAccount<Mint> with token_program constraint. Add feature-gated mint validation (same pattern as conversion-vault): `constraint = cfg!(feature = "localnet") || token_mint.key() == constants::crime_mint() || token_mint.key() == constants::fraud_mint()`
- `token_program`: Interface<TokenInterface>
- `system_program`: Program<System>

Handler: Set all CurveState fields per spec Section 8.1. Status = Initialized, all counters = 0, store all PDA keys, store bump. Emit CurveInitialized event.

**fund_curve.rs** -- Follow spec Section 8.2. Transfer TARGET_TOKENS from authority's token account to the curve's token vault. Key accounts:
- `authority`: Signer, Mut
- `curve_state`: Must be Initialized status (mut for reading, not changing)
- `authority_token_account`: The admin's ATA for this token (source of 460M tokens)
- `token_vault`: Curve's token vault PDA (destination)
- `token_mint`: For transfer_checked decimals validation
- `token_program`: Token-2022

Use `anchor_spl::token_2022::transfer_checked` for the CPI. The authority signs directly (not a PDA signer). Emit CurveFunded event.

IMPORTANT: The transfer from admin's ATA to curve vault involves Token-2022 tokens with Transfer Hook. The admin's wallet should be whitelisted, and the curve vault should be whitelisted (entries #11-12 per spec Section 16.2). For the fund instruction, include remaining_accounts passthrough for hook accounts, OR if the admin is whitelisted and the destination vault is a PDA controlled by the bonding curve program, check whether the transfer hook fires on this path. For safety, accept `remaining_accounts` and pass them through to the transfer CPI. Use the same invoke_signed pattern from conversion-vault/helpers/hook_helper.rs if needed. NOTE: Since the authority is signing (not a PDA), use `CpiContext::new()` not `CpiContext::new_with_signer()`.

**start_curve.rs** -- Follow spec Section 8.3. Admin-only. Key accounts:
- `authority`: Signer
- `curve_state`: Mut PDA, must be Initialized
- `token_vault`: Read-only to check balance (InterfaceAccount<TokenAccount> or just read amount)

Handler: Verify status == Initialized. Verify token_vault.amount >= TARGET_TOKENS. Set status = Active, start_slot = Clock::get()?.slot, deadline_slot = start_slot + DEADLINE_SLOTS. Emit CurveStarted event.
