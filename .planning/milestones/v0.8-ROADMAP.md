# Milestone v0.8: Frontend Tech Foundations

**Status:** SHIPPED 2026-02-18
**Phases:** 39-45
**Total Plans:** 18

## Overview

This milestone built the complete frontend technology layer for the Dr. Fraudsworth protocol -- wallet connection, swap execution, staking, real-time protocol data, and historical price charts -- as a functional desktop-only prototype deployed on Railway. The on-chain protocol is live on Solana devnet (5 programs, continuous runner); this milestone wrapped it in a web interface.

## Phases

### Phase 39: Foundation + Scaffolding

**Goal**: Users (developers) can run `npm run dev` in `app/` and see a working Next.js development server with Anchor programs loadable in the browser and shared constants imported from a single source of truth
**Depends on**: Nothing (first phase)
**Requirements**: INFR-01
**Plans**: 3 plans

Plans:
- [x] 39-01-PLAN.md -- Next.js scaffold + shared/ constants package + npm workspaces + Tailwind + polyfills
- [x] 39-02-PLAN.md -- IDL sync pipeline + Anchor program factory + devnet proof-of-life
- [x] 39-03-PLAN.md -- Drizzle ORM schema for 4 indexer tables (swap_events, candles, epoch_events, carnage_events)

**Success Criteria**:
  1. `app/` directory contains a working Next.js project with dev server, Tailwind CSS, and base layout
  2. `shared/` directory exports PDA seeds, program IDs, token decimals, and fee constants -- imported by both `app/` and `scripts/`
  3. IDL JSON files are copied from `target/idl/` into `app/` at build time via a pre-build script
  4. Anchor Program objects load in the browser (Buffer polyfill, webpack config verified) and can deserialize account data from devnet RPC
  5. Drizzle ORM schema is defined for candles, swap_events, epoch_events, and carnage_events tables (schema only, no deployed DB yet)

## Milestone Summary

**Key Decisions:**
- Monorepo with npm workspaces (IDLs, PDAs, shared constants in one repo)
- Railway + Postgres for backend (predictable pricing, single platform)
- Helius webhooks for OHLCV indexing (reliable, first-class Solana provider)
- Frontend tech before design (build functionality, plug in design later)
- Custom swap transaction builders (no existing DEX SDK maps to Tax+AMM+Hook chain)
- Privy in v0.8 before bonding curves (build integration early)
- Zero-dependency Sentry (all @sentry/* packages break Turbopack SSR)
- Turbopack over webpack (faster dev builds, future direction)

**Issues Resolved:**
- Turbopack SSR compatibility with Sentry (zero-dep reporter via fetch)
- Privy v3 RPC config requires CAIP-2 format ("solana:devnet")
- Browser Buffer polyfill lacks BigInt methods (manual PDA derivation instead)
- Token-2022 hook account resolution in browser (sync PDA derivation, no RPC)
- Privy signTransaction defaults to mainnet without explicit chain parameter
- Nixpacks auto-detects Rust from Cargo.toml (must use Nixpacks not Railpack)

**Issues Deferred:**
- Server-side Sentry (instrumentation.ts is no-op, Railway logs cover server errors)
- BalanceDisplay component orphaned (dead code, 107 lines)
- Mobile-responsive layout (desktop-only tech prototype)

**Technical Debt Incurred:**
- 37 pre-existing test failures from v0.7 (19 AMM + 10 Tax + 8 Epoch)
- Treasury pubkey hardcoded to devnet wallet
- VRF bounty payment emits 0
- REQUIREMENTS.md traceability table was stale (many "Pending" should be "Complete")
- npm audit: 3 high from bigint-buffer (Solana ecosystem transitive dep, not exploitable)
