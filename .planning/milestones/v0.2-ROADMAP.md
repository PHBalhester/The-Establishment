# Milestone v0.2: AMM Program

**Status:** ✅ SHIPPED 2026-02-04
**Phases:** 8-13
**Total Plans:** 12

## Overview

Build the AMM on-chain program — the foundational Anchor/Rust program that implements constant-product swaps across four fixed pools (CRIME/SOL, FRAUD/SOL, CRIME/PROFIT, FRAUD/PROFIT) with mixed Token-2022/SPL Token support, CPI-only access control, and transfer hook passthrough.

## Phases

### Phase 8: Foundation & Scaffolding

**Goal**: AMM program compiles, math is proven correct, and test infrastructure runs both unit and integration tests
**Depends on**: Nothing (first phase of v0.2)
**Requirements**: SCAF-01, SCAF-02, SCAF-03, SCAF-04, SCAF-05, MATH-01, MATH-02, MATH-03, MATH-04, TEST-01
**Success Criteria**:
  1. `anchor build` compiles the AMM program workspace without errors on Anchor 0.32.1
  2. Math module computes correct swap outputs for known input/output pairs (verified against manual calculations)
  3. Math module rejects overflow scenarios (u64::MAX inputs) with explicit errors instead of panics
  4. k-invariant check passes for valid swaps and fails for invalid ones across randomized inputs
  5. Math unit tests (hand-picked edge cases + 10,000 proptest iterations) execute via `cargo test` and pass without Solana VM dependency

### Phase 11: SOL Pool Swaps

**Goal**: Users can swap between protocol tokens (CRIME/FRAUD) and SOL through the mixed T22/SPL pool instruction with correct fee deduction, slippage protection, and invariant preservation
**Depends on**: Phase 10
**Requirements**: SWAP-01, SWAP-03, SWAP-04, SWAP-05, SWAP-06, SWAP-07, SWAP-08, SWAP-09

## Milestone Summary

**Key Decisions:**
- One program per milestone: Limits blast radius when specs meet reality
- Backend-first: On-chain programs are source of truth; frontend is just a client
- AMM first in build order: Everything else depends on AMM (pools, swaps)
- TAX_PROGRAM_ID set to Mock Tax Program ID for testing (update for production)
- litesvm with type bridge for Anchor 0.32.1 compatibility (Solana 2.x/3.x)
- PoolType uses 2 behavioral variants (MixedPool, PureT22Pool) instead of 4 protocol-specific
- SwapDirection enum shared via re-export between swap_sol_pool and swap_profit_pool
- Dual-hook remaining_accounts split at midpoint for PROFIT pools

**Issues Resolved:**
- Mixed T22/SPL pool token routing with transfer_checked
- Cross-program PDA validation via seeds::program constraint
- CPI chain depth verification through Mock Tax -> AMM -> Token-2022
- Reentrancy protection via locked field and CEI ordering

**Issues Deferred:**
- TAX_PROGRAM_ID update to real Tax Program ID (deferred to v0.4)
- LP token minting (not needed — protocol-owned liquidity)
- Compute unit optimization (deferred to v0.7 integration)

**Technical Debt Incurred:**
- TAX_PROGRAM_ID constant points to Mock Tax (must update for mainnet)
- Test helper code replicated across test files (acceptable for test independence)
