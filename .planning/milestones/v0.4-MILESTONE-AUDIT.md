## Security Verification

### Access Control Layers

1. **AMM Protection:** seeds::program = TAX_PROGRAM_ID on swap_authority
2. **Carnage Protection:** seeds::program = epoch_program_id() on carnage_authority
3. **PDA Signature:** Only owning program can sign via invoke_signed

### Attack Vectors Mitigated

- ✓ Direct AMM access (blocked by seeds::program)
- ✓ PDA spoofing (blocked by program derivation)
- ✓ Unauthorized Carnage swaps (blocked by Epoch PDA requirement)
- ✓ Tax evasion (all SOL swaps route through Tax Program)
- ✓ Reentrancy (AMM pool.locked guard)
