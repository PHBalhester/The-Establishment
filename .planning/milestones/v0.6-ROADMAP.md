# Milestone v0.6: Staking/Yield System

**Status:** SHIPPED 2026-02-09
**Phases:** 26-29
**Total Plans:** 17

## Overview

Build the full Staking/Yield Program — stake/unstake/claim instructions with epoch-based yield distribution using the battle-tested cumulative reward-per-token pattern. This milestone completes the yield loop. Stub staking from v0.5 gets replaced with full implementation. Stakers deposit PROFIT tokens and receive SOL yield proportional to their stake over epoch durations. The cumulative reward-per-token pattern (Synthetix/Quarry) ensures fair, gas-efficient distribution.

## Milestone Summary

**Key Decisions:**
- Synthetix-style cumulative reward-per-token (industry standard, gas-efficient proportional distribution)
- 1e18 PRECISION constant for DeFi math (no overflow with u128 intermediates)
- MINIMUM_STAKE dead stake at initialization (first-depositor attack prevention)
- seeds::program constraints for CPI gating (Tax/Epoch programs validated by program ID)
- Custom transfer_checked_with_hook helper (Anchor SPL CPI bug workaround)
- Separate validator per test file (StakePool PDA singleton isolation)
- EscrowVault as system account for native SOL (not token account)
- Scope block pattern for Rust borrow checker before CPI calls

**Issues Resolved:**
- Anchor SPL transfer_checked doesn't forward remaining_accounts for Token-2022 hooks (fixed with manual CPI)
- StakePool PDA singleton conflicts across test files (resolved with separate validators)
- proptest rejection rates >50% with prop_assume! (resolved with percentage-based derivation)

- None
