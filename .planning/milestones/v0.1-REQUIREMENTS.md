# Requirements: Dr. Fraudsworth Documentation Audit

**Defined:** 2026-02-01
**Core Value:** The documentation is the source of truth — if the specs are wrong or incomplete, the code will inherit those problems.

## v0.1 Requirements

Requirements for the documentation audit milestone. Each maps to roadmap phases.

### Preparation

- [x] **PREP-01**: Create document inventory (INDEX.md) with dependency graph showing which specs depend on which
- [x] **PREP-02**: Create conflict tracking document (CONFLICTS.md) for logging discovered conflicts
- [x] **PREP-03**: Create gap tracking document (GAPS.md) for logging discovered gaps
- [x] **PREP-04**: Create iteration log (ITERATIONS.md) for tracking audit passes toward convergence

### Token Program Audit

- [x] **TOKEN-01**: Validate token program matrix exists for all 4 pools (CRIME/SOL, FRAUD/SOL, CRIME/PROFIT, FRAUD/PROFIT)
- [x] **TOKEN-02**: Ensure WSOL explicitly documented as SPL Token (not Token-2022) in all relevant specs
- [x] **TOKEN-03**: Document transfer hook coverage per pool side (hooks fire on T22 side only)
- [x] **TOKEN-04**: Document ATA derivation differences per token program (T22 vs SPL use different derivations)

### Cross-Reference Audit

- [x] **XREF-01**: Extract concept inventory from all 12 spec documents (constants, entities, behaviors, constraints, formulas, terminology)
- [x] **XREF-02**: Build cross-reference matrix (concepts as rows, documents as columns, values in cells)
- [x] **XREF-03**: Detect and log all value conflicts (same parameter, different numeric values)
- [x] **XREF-04**: Detect and log all behavioral conflicts (same flow, different sequences or outcomes)
- [x] **XREF-05**: Detect and log all assumption conflicts (implicit dependencies, undocumented assumptions)

### Name Changes + Yield Edits (Phase 3.1 -- INSERTED)

- [x] **NAME-01**: Replace placeholder token names (IPA->CRIME, IPB->FRAUD, OP4->PROFIT) across all specs
- [x] **NAME-02**: Update cascading account names (ipa_vault->crime_vault, ipb_vault->fraud_vault, op4_vault->profit_vault)
- [x] **YIELD-01**: Archive old Yield_System_Spec.md with deprecation header
- [x] **YIELD-02**: Update all yield references to describe staking-based model (Staking Program, not Yield Program)

### Gap Analysis

- [x] **GAP-01**: Audit each document against 14-category coverage checklist from research
- [x] **GAP-02**: Identify and log all missing mathematical invariants
- [x] **GAP-03**: Identify and log all missing state machine transitions (especially "during wait" behaviors)
- [x] **GAP-04**: Identify and log all missing CPI depth analysis and compute budget documentation

### Resolution & Convergence

- [x] **CONV-01**: Resolve all logged conflicts with documentation updates (choose authoritative source, update dependents)
- [x] **CONV-02**: Fill all logged gaps with new or updated documentation
- [x] **CONV-03**: Iterate audit (re-run cross-reference and gap analysis) until clean pass with no new issues
- [x] **CONV-04**: Final verification pass confirms documentation stability

### VRF Documentation

- [x] **VRF-01**: Document Switchboard VRF implementation from archive-V3 branch
- [x] **VRF-02**: Capture lessons learned (outdated approach vs current Switchboard method)

### Post-Audit Validation

- [x] **VAL-01**: Re-run research phase on completed documentation set to catch new gaps
- [x] **VAL-02**: Verify no new conflicts or issues emerged from documentation changes
