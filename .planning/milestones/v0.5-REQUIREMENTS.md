# Requirements: v0.5 Epoch/VRF Program

**Defined:** 2026-02-06
**Core Value:** The randomness engine must be manipulation-proof — VRF determines tax rates and Carnage triggers with cryptographic unpredictability.

## v0.5 Requirements

Requirements for the Epoch/VRF Program milestone. Each maps to roadmap phases.

### Epoch Core

- [x] **EPO-01**: EpochState account stores genesis_slot, current_epoch, epoch_start_slot for slot-based timing
- [x] **EPO-02**: EpochState stores active tax configuration (cheap_side, low_tax_bps, high_tax_bps)
- [x] **EPO-03**: EpochState caches derived tax rates (crime_buy_tax_bps, crime_sell_tax_bps, fraud_buy_tax_bps, fraud_sell_tax_bps)
- [x] **EPO-04**: initialize_epoch_state instruction creates genesis epoch with CRIME cheap side, 300/1400 bps initial rates
- [x] **EPO-05**: Epoch calculation uses slot-based timing (SLOTS_PER_EPOCH = 4,500 ≈ 30 min)
- [x] **EPO-06**: "Cheap side" regime determines which token has low tax (1-4%) vs high tax (11-14%)
- [x] **EPO-07**: 75% flip probability (VRF byte 0 < 192) changes cheap side each epoch
- [x] **EPO-08**: Discrete tax bands with 100-400 bps (low) and 1100-1400 bps (high) ranges

### VRF Integration

- [x] **VRF-01**: trigger_epoch_transition instruction validates epoch boundary reached (current slot >= next_epoch_boundary)
- [x] **VRF-02**: trigger_epoch_transition validates randomness account freshness (seed_slot within 1 slot)
- [x] **VRF-03**: trigger_epoch_transition validates randomness not yet revealed (get_value returns error)
- [x] **VRF-04**: trigger_epoch_transition binds pending_randomness_account for anti-reroll protection
- [x] **VRF-05**: trigger_epoch_transition pays 0.01 SOL bounty to triggerer (treasury integration deferred — emits 0)
- [x] **VRF-06**: consume_randomness verifies pending_randomness_account matches bound account
- [x] **VRF-07**: consume_randomness reads VRF bytes and derives new tax rates (bytes 0-2)
- [x] **VRF-08**: consume_randomness updates EpochState with new epoch and tax configuration
- [x] **VRF-09**: retry_epoch_vrf allows re-commit after VRF_TIMEOUT_SLOTS (300 slots ≈ 2 min)
- [x] **VRF-10**: TypeScript crank bot implements three-transaction flow (create, commit, reveal+consume)

### Carnage Fund

- [x] **CRN-01**: CarnageFundState account stores vault addresses, held_token, statistics
- [x] **CRN-02**: initialize_carnage_fund creates SOL/CRIME/FRAUD vault PDAs
- [x] **CRN-03**: Carnage trigger check uses VRF byte 3 (< 11 triggers ≈ 4.3% probability)
- [x] **CRN-04**: Carnage action determined by VRF byte 4 (< 250 = burn 98%, >= 250 = sell 2%)
- [x] **CRN-05**: Carnage target determined by VRF byte 5 (< 128 = CRIME, >= 128 = FRAUD)
- [x] **CRN-06**: Atomic Carnage execution within consume_randomness (burn/sell held, buy target)
- [x] **CRN-07**: Carnage uses Tax::swap_exempt with carnage_signer PDA (0% tax, 1% LP fee)
- [x] **CRN-08**: carnage_signer PDA validated with seeds::program = EPOCH_PROGRAM_ID
- [x] **CRN-09**: Carnage swap capped at 1000 SOL per execution
- [x] **CRN-10**: Fallback execute_carnage instruction if atomic fails (100-slot deadline)
- [x] **CRN-11**: expire_carnage instruction clears pending state after deadline (SOL retained)
- [x] **CRN-12**: Two-instruction atomic bundle (revealIx + consume) if single-instruction CU exceeds limit

### Staking Integration

- [x] **STK-01**: consume_randomness CPIs to Staking::update_cumulative to finalize epoch yield
- [x] **STK-02**: Stub Staking Program with update_cumulative instruction for testing

### Tax Program Updates

- [x] **TAX-01**: Tax Program reads EpochState for dynamic tax rates (replace hardcoded 400 bps)
- [x] **TAX-02**: Tax Program swap_exempt validates carnage_signer PDA from Epoch Program
- [x] **TAX-03**: Tax Program passes EpochState account to swap instructions
