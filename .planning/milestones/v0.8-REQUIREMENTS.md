## v0.8 Requirements

Requirements for the frontend technology layer. Desktop-only functional prototype -- no final design/theming.

### Infrastructure

- [x] **INFR-01**: Next.js monorepo scaffolding with `app/` directory, Anchor client integration, IDL sync, and shared code extraction from existing scripts
- [x] **INFR-02**: Privy embedded wallet validation -- confirm Privy can sign Solana transactions on devnet as smoke test during wallet integration
- [x] **INFR-03**: Transaction status feedback -- pending/confirmed/failed states with human-readable error messages (not raw error codes)
- [x] **INFR-04**: Railway deployment -- Next.js + Postgres deployed and accessible via URL with Helius webhook endpoint

### Wallet

- [x] **WALL-01**: User can connect standard wallets (Phantom, Solflare, Backpack) via Privy external wallet connectors
- [x] **WALL-02**: User can connect via Privy embedded wallet with social login (email, Google -- SMS deferred to production)
- [x] **WALL-03**: Unified wallet abstraction -- single signing interface that works transparently with both standard and Privy wallets
- [x] **WALL-04**: User can see token balances (CRIME, FRAUD, PROFIT, SOL) for their connected wallet

### Swap

- [x] **SWAP-01**: User can buy CRIME with SOL (CRIME/SOL pool via Tax Program)
- [x] **SWAP-02**: User can sell CRIME for SOL (CRIME/SOL pool, legacy TX)
- [x] **SWAP-03**: User can buy FRAUD with SOL (FRAUD/SOL pool via Tax Program)
- [x] **SWAP-04**: User can sell FRAUD for SOL (FRAUD/SOL pool, legacy TX)
- [x] **SWAP-05**: User can swap via CRIME/PROFIT pool (untaxed, 50bps LP fee only)
- [x] **SWAP-06**: User can swap via FRAUD/PROFIT pool (untaxed, 50bps LP fee only)
- [x] **SWAP-07**: Price quote engine shows estimated output with LP fee and tax as separate line items, updating as user types
- [x] **SWAP-08**: User can configure slippage tolerance for swaps
- [x] **SWAP-09**: User can configure priority fee / compute budget for transactions
- [x] **SWAP-10**: Sell-path transactions use legacy TX format (20 named + 4 transfer hook remaining accounts)

### Staking

- [x] **STAK-01**: User can stake PROFIT tokens (deposit with Token-2022 transfer hook handling)
- [x] **STAK-02**: User can unstake PROFIT tokens (withdraw with auto-claim of pending rewards)
- [x] **STAK-03**: User can claim accumulated SOL rewards with one-click claim
- [x] **STAK-04**: User can see pending unclaimed SOL rewards, updating per-epoch
- [x] **STAK-05**: User can see APY/yield stats calculated from historical reward deposit data

### Protocol Data

- [x] **DATA-01**: Dashboard shows current epoch number and cheap side indicator
- [x] **DATA-02**: Dashboard shows epoch countdown timer (slot-based, time until next transition)
- [x] **DATA-03**: Dashboard shows all 4 tax rates (CRIME buy/sell, FRAUD buy/sell) with direction indicators
- [x] **DATA-04**: Dashboard shows Carnage fund SOL balance
- [x] **DATA-05**: Dashboard shows Carnage history (time since last trigger, lifetime burn statistics for CRIME and FRAUD)
- [x] **DATA-06**: Dashboard shows pool prices as market cap (default) with toggle to price-per-token view
- [x] **DATA-07**: Epoch boundary warning banner displayed when epoch is about to flip (within ~2 minutes / 300 slots)

### Charts

- [x] **CHRT-01**: Helius webhook indexer receives swap events and stores OHLCV candle data in Postgres
- [x] **CHRT-02**: TradingView candlestick charts display historical price data with time range selector
