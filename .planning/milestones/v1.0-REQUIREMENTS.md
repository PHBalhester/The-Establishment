# Requirements: v1.0 Frontend Design & Interactive Factory

**Defined:** 2026-02-22
**Core Value:** Transform the functional tech prototype into a polished, branded interactive experience where the factory scene IS the navigation layer and all protocol features live in themed modals.

## v1.0 Requirements

### Scene Foundation

- [x] **SCENE-01**: Factory scene displays full-screen background image (WebP, <2MB delivered) covering the viewport without distortion on desktop viewports 1280px-3840px
- [x] **SCENE-02**: 6 transparent overlay objects (SwapMachine, CarnageCauldron, RewardsVat, ConnectWallet, DocumentationTable, Settings) positioned over the background using percentage-based coordinates that scale proportionally with viewport
- [x] **SCENE-03**: Scene maintains correct aspect ratio and object alignment across desktop viewport sizes without layout shift
- [x] **SCENE-04**: Scene loading state shows blur placeholder or skeleton while high-res images download, with no blank white screen or layout jump
- [x] **SCENE-05**: Z-index layering system defined and enforced: background < overlays < hover effects < tooltips < modal backdrop < modal content
- [x] **SCENE-06**: All scene images (background + 6 overlays) optimized via Next.js Image component with WebP format, responsive srcset, and quality 80-85

### Interactive Objects

- [x] **INTERACT-01**: Each of the 6 scene objects shows pointer cursor and visible hover effect (glow via CSS drop-shadow + brightness increase) on mouse enter
- [x] **INTERACT-02**: Clicking any scene object immediately opens its corresponding themed modal
- [x] **INTERACT-03**: ~~Hover tooltip label appears near each object identifying its function~~ DEVIATED: Tooltips removed per user preference. Glow provides sufficient affordance.
- [x] **INTERACT-04**: Objects show brief scale-down (depression) on click for tactile feedback
- [x] **INTERACT-05**: Big Red Button for swap execution with 3D physical appearance (raised, glossy, rim lighting), press animation, and state-driven appearance (idle=red, loading=spinning, success=green flash, error=shake)

### Modal System

- [x] **MODAL-01**: Modal opens with backdrop blur overlay dimming the scene behind, and smooth entrance animation (scale+fade, 200-300ms)
- [x] **MODAL-02**: Modal closes via X button, Escape key, or backdrop click, with exit animation faster than entrance (150-200ms)
- [x] **MODAL-03**: Only one modal open at a time; opening a new station closes the previous one and returns to scene
- [x] **MODAL-04**: Modal chrome themed to steampunk aesthetic (brass borders, rivets, aged metal textures, parchment backgrounds) -- not generic white panels
- [x] **MODAL-05**: Modal body scrolls independently when content exceeds viewport height, with header/close button remaining fixed
- [x] **MODAL-06**: Focus trap active when modal is open: Tab cycles within modal only, Escape closes and returns focus to triggering object

### Station Content

- [x] **STATION-01**: Swap station modal contains TradingView chart, market cap display, current tax rates, token pair selector, swap form with Big Red Button, fee breakdown, and route selector -- re-parenting existing SwapForm/CandlestickChart components
- [x] **STATION-02**: Carnage Cauldron modal displays Carnage fund SOL balance, last trigger timestamp, lifetime burn statistics, and recent Carnage events -- re-parenting existing CarnageCard content
- [x] **STATION-03**: Rewards Vat modal provides stake/unstake/claim tabs, pending rewards display (SOL), staked balance, total protocol staked, and APY estimate -- re-parenting existing StakingForm
- [x] **STATION-04**: Connect Wallet modal offers two-path connection (browser wallets + social login via Privy) -- extending existing ConnectModal with themed chrome
- [x] **STATION-05**: Documentation Table modal presents protocol explanation: how the three-token system works, what Carnage is, how staking yields real SOL, and FAQ-style answers to common questions
- [x] **STATION-06**: Settings modal provides slippage tolerance, priority fee configuration, and RPC display -- extracting from existing SlippageConfig into standalone modal

### Brand System

- [x] **BRAND-01**: Steampunk color palette defined as Tailwind v4 @theme tokens (primary, secondary, accent, background, surface, text colors) and applied consistently across all components
- [x] **BRAND-02**: Typography system with custom font family (or curated web font), clear weight hierarchy (headings, body, detail), and consistent sizing scale applied across modals and navigation
- [x] **BRAND-03**: Existing UI components (buttons, inputs, cards, tabs) re-themed to match steampunk palette via className changes without breaking existing hook logic

### Mobile

- [x] **MOBILE-01**: Below 1024px viewport width, interactive scene is replaced with a steampunk-themed vertical navigation list showing all 6 stations with icons and labels
- [x] **MOBILE-02**: Mobile modals display as full-screen slide-up panels with themed headers and close button (minimum 48px tap targets)
- [x] **MOBILE-03**: All functionality has 100% feature parity between desktop scene navigation and mobile list navigation -- same modals, same content
- [x] **MOBILE-04**: Scene teaser (static decorative image or header illustration) shown at top of mobile navigation to establish the factory theme

### Onboarding

- [x] **ONBOARD-01**: ~~First-time visitors see a welcome modal with protocol description and action buttons, persisted in localStorage~~ DEVIATED: SplashScreen replaces WelcomeModal. Animated brass gear intro gate shows every load with "Push to Enter" button. More visceral than text dialog.

### Accessibility

- [x] **A11Y-01**: All 6 interactive scene objects are `<button>` elements with descriptive `aria-label` (e.g., "Open Swap Machine"), reachable via Tab key in logical order
- [x] **A11Y-02**: Visible focus indicators (`:focus-visible` styled to match glow aesthetic) on all interactive elements, never globally removed
- [x] **A11Y-03**: All text meets WCAG AA contrast ratio (4.5:1) against steampunk-themed backgrounds
- [x] **A11Y-04**: Background image has `role="img"` with descriptive `aria-label`; overlay objects described by their button labels

## Out of Scope

| Feature | Reason |
|---------|--------|
| WebGL / Three.js 3D scene | Massively increases complexity and device requirements; 2D scene with CSS achieves the goal |
| Full parallax scrolling | Conflicts with single-viewport scene concept; users click, not scroll |
| Canvas-based hit detection | Fragile, inaccessible; DOM buttons with aria-labels are superior |
| Custom cursor | Input lag, accessibility issues, browser inconsistency |
| Drag-and-drop interactions | Breaks mobile, slower than click-to-modal |
| Animated page transitions between stations | Modals are faster; users want speed not cinematics |
| Multi-language support | English-only crypto audience; structure for i18n later |
| Persistent scene state | Unnecessary complexity; scene resets each visit |
| Complex particle physics | Stylized CSS animations beat physically accurate simulations |
| Spatial audio | Zero functional value, significant complexity |
| Mainnet deployment | Deferred until after battle-testing and pre-mainnet hardening |
