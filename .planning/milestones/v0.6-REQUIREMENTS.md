# Requirements: Dr. Fraudsworth v0.6 Staking/Yield System

**Defined:** 2026-02-06
**Core Value:** Stakers earn real SOL yield from protocol taxes through a secure, battle-tested cumulative reward-per-token system.

## v0.6 Requirements

Requirements for the staking/yield system milestone.

### State Accounts

- [x] **ACCT-01**: StakePool global singleton stores total_staked, rewards_per_token_stored, pending_rewards, last_update_epoch
- [x] **ACCT-02**: UserStake per-user account stores owner, staked_balance, rewards_per_token_paid, rewards_earned
- [x] **ACCT-03**: EscrowVault native SOL PDA holds undistributed yield
- [x] **ACCT-04**: StakeVault Token-2022 PDA holds staked PROFIT tokens

### Core Instructions

- [x] **INST-01**: initialize_stake_pool creates global state, escrow vault, and stake vault PDAs
- [x] **INST-02**: stake transfers PROFIT from user to stake vault, updates balances and checkpoints
- [x] **INST-03**: unstake transfers PROFIT from stake vault to user, auto-claims pending rewards
- [x] **INST-04**: claim transfers pending SOL rewards from escrow to user without unstaking
- [x] **INST-05**: deposit_rewards receives SOL from Tax Program, adds to pending_rewards
- [x] **INST-06**: update_cumulative finalizes epoch rewards into cumulative, called by Epoch Program

### Core Math

- [x] **MATH-01**: update_rewards calculates user pending = (global_cumulative - user_checkpoint) * balance / PRECISION
- [x] **MATH-02**: add_to_cumulative adds pending_rewards to cumulative with u128 precision
- [x] **MATH-03**: All arithmetic uses checked_* methods (no overflow/underflow)
- [x] **MATH-04**: PRECISION constant = 1e18 for industry-standard DeFi math
- [x] **MATH-05**: Division truncates (floors) to favor protocol, preventing escrow insolvency

### Security

- [x] **SEC-01**: First-depositor attack prevented via MINIMUM_STAKE (1 PROFIT) staked at initialization
- [x] **SEC-02**: Flash loan attack prevented via checkpoint pattern (no same-epoch rewards)
- [x] **SEC-03**: deposit_rewards validates Tax Program authority via seeds::program constraint
- [x] **SEC-04**: update_cumulative validates Epoch Program authority via seeds::program constraint
- [x] **SEC-05**: claim/unstake validates user is owner of UserStake account
- [x] **SEC-06**: Escrow solvency invariant: balance >= sum(all rewards_earned) + pending_rewards
- [x] **SEC-07**: CEI pattern: state updates before external transfers

### Integration

- [x] **INTG-01**: Tax Program CPIs deposit_rewards with 75% of SOL taxes
- [x] **INTG-02**: Epoch Program CPIs update_cumulative in consume_randomness
- [x] **INTG-03**: StakeVault PDA added to Transfer Hook whitelist (entry #14)
- [x] **INTG-04**: Epoch Program's stub-staking replaced with real Staking Program

### Events

- [x] **EVNT-01**: StakePoolInitialized emitted on initialize_stake_pool
- [x] **EVNT-02**: Staked emitted with user, amount, new_balance, total_staked
- [x] **EVNT-03**: Unstaked emitted with user, amount, rewards_claimed, new_balance
- [x] **EVNT-04**: Claimed emitted with user, amount, staked_balance, total_claimed
- [x] **EVNT-05**: RewardsDeposited emitted with amount, new_pending
- [x] **EVNT-06**: CumulativeUpdated emitted with epoch, rewards_added, new_cumulative

### Error Handling

- [x] **ERR-01**: ZeroAmount error when stake/unstake/deposit amount is 0
- [x] **ERR-02**: InsufficientBalance error when unstake exceeds staked_balance
- [x] **ERR-03**: InsufficientEscrowBalance error when claim exceeds escrow SOL
- [x] **ERR-04**: NothingToClaim error when rewards_earned is 0
- [x] **ERR-05**: Unauthorized error when signer doesn't own UserStake
- [x] **ERR-06**: AlreadyUpdated error when update_cumulative called twice for same epoch

## Out of Scope

| Feature | Reason |
|---------|--------|
| Lockup periods | Design philosophy: instant liquidity, no friction |
| Slashing | Not applicable to yield staking (no validator risk) |
| Tier systems | Complexity adds attack surface, proportional is fair |
| Auto-compound | User should control claim timing for gas efficiency |
| Withdrawal fees | Protocol taxes already capture value |
| Frontend UI | Backend-first, frontend after devnet validation |
