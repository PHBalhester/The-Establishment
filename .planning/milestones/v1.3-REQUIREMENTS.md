# Requirements: Dr. Fraudsworth's Finance Factory

**Defined:** 2026-03-08
**Core Value:** Real SOL yield from real trading friction -- not ponzinomics.

## v1.3 Requirements

Requirements for Protocol Hardening & Polish. Each maps to roadmap phases.

### Authority Hardening (AUTH)

- [ ] **AUTH-01**: Bonding Curve program has BcAdminConfig PDA gated by ProgramData upgrade authority, all 6 admin instructions require `has_one = authority`
- [ ] **AUTH-02**: Transfer Hook `initialize_authority` requires ProgramData upgrade authority check (front-running prevention)
- [ ] **AUTH-03**: Staking `initialize_stake_pool` requires ProgramData upgrade authority check
- [ ] **AUTH-04**: Epoch `initialize_epoch_state` requires ProgramData upgrade authority check
- [ ] **AUTH-05**: Epoch `initialize_carnage_fund` requires ProgramData upgrade authority check
- [ ] **AUTH-06**: Conversion Vault `initialize` requires ProgramData upgrade authority check
- [ ] **AUTH-07**: Authority map and gradual burn timeline documented in PROJECT.md decisions

### Financial Safety (FIN)

- [ ] **FIN-01**: Staking `claim` instruction reserves rent-exempt minimum before transferring rewards (escrow PDA protection)
- [ ] **FIN-02**: Tax Program sell path passes computed gross minimum to AMM CPI (not `minimum_amount_out = 0`)
- [ ] **FIN-03**: Epoch `trigger_epoch_transition` bounty payment checks `vault_balance >= bounty + rent_exempt_min`
- [ ] **FIN-04**: Bonding Curve `purchase` partial fill has explicit `actual_sol <= sol_amount` assertion
- [ ] **FIN-05**: Bonding Curve `claim_refund` validates partner curve is actual CRIME/FRAUD counterpart (not arbitrary curve)

### Defense-in-Depth (DEF)

- [ ] **DEF-01**: `read_pool_reserves()` verifies pool account is owned by AMM program before reading bytes
- [ ] **DEF-02**: Tax Program pool_reader has `is_reversed` canonical mint detection (ported from epoch_program)
- [ ] **DEF-03**: EpochState has `reserved: [u8; 64]` padding bytes for future schema evolution
- [ ] **DEF-04**: All `u128 as u64` casts replaced with `u64::try_from()` in staking/tax/amm math
- [ ] **DEF-05**: Bonding Curve purchase validates `remaining_accounts` count matches expected hook schema
- [ ] **DEF-06**: Carnage pool accounts have owner constraint before pre-CPI slippage check
- [ ] **DEF-07**: `Token::from_u8_unchecked` replaced with checked conversion + error in `consume_randomness`
- [ ] **DEF-08**: EpochState and Tax mirror struct both have `#[repr(C)]` with unit test validating layout match

### Compile-Time Guards (CTG)

- [ ] **CTG-01**: Mainnet placeholder pubkeys (`Pubkey::default()`) in Tax, BC, Conv. Vault guarded by `compile_error!()`
- [ ] **CTG-02**: `force_carnage` guarded by `compile_error!()` when built without devnet feature
- [ ] **CTG-03**: `const _: () = assert!(P_END > P_START)` in bonding curve constants

### Carnage Refactor (CARN)

- [ ] **CARN-01**: Shared logic extracted from `execute_carnage.rs` and `execute_carnage_atomic.rs` into common module (~1800 lines deduplication)
- [ ] **CARN-02**: All 6 Carnage paths (BuyOnly+Burn+Sell x CRIME+FRAUD) pass after refactor

### VRF & Crank Hardening (VRF)

- [ ] **VRF-01**: `force_carnage` sets `carnage_lock_slot` matching `consume_randomness` behavior
- [ ] **VRF-02**: Epoch skip behavior documented or limited to max 1 per transition
- [ ] **VRF-03**: Legacy `low_tax_bps`/`high_tax_bps` fields removed or populated with meaningful values
- [ ] **VRF-04**: Stale-VRF recovery handles "already consumed" error gracefully (TOCTOU gap)
- [ ] **VRF-05**: Timeout recovery calculates remaining wait from VRF request slot (not current slot)
- [ ] **VRF-06**: Anti-reroll security test asserts specific `ConstraintAddress` error code
- [ ] **VRF-07**: `PublicKey.default` in carnage-flow.ts replaced with real carnageWsol pubkey
- [ ] **VRF-08**: `MIN_EPOCH_SLOTS` configurable via env var for mainnet (750 devnet / 4500 mainnet)
- [ ] **VRF-09**: Binary offsets for EpochState consolidated into shared constants with validation test
- [ ] **VRF-10**: `SWAP_EXEMPT_DISCRIMINATOR` has matching validation test
- [ ] **VRF-11**: `loadCarnageWsolPubkey` uses pubkey-only path (no secret key loading)
- [ ] **VRF-12**: Crank wallet balance has configurable warning threshold and alerting

### Frontend Hardening (FE)

- [ ] **FE-01**: Crank runner masks RPC URL in startup logs (API key protection)
- [ ] **FE-02**: Webhook secret validation is mandatory (fail-closed) in production
- [ ] **FE-03**: Token balance updates via webhook+SSE (replacing 30s polling)
- [ ] **FE-04**: Carnage data updates via existing webhook/SSE infrastructure
- [ ] **FE-05**: Priority fees use Helius Priority Fee API (dynamic, not hardcoded presets)
- [ ] **FE-06**: Frontend error map covers Tax codes 6014-6017 including MinimumOutputFloorViolation
- [ ] **FE-07**: Backend RPC proxy protects Helius API key from browser exposure
- [ ] **FE-08**: Server-side Sentry error reporting hardened and verified
- [ ] **FE-09**: Token metadata JSON structure prepared (logos provided externally, Arweave upload in v1.4)

### Frontend Mainnet-Readiness (MNR)

- [ ] **MNR-02**: Explorer links use dynamic cluster suffix (no hardcoded `?cluster=devnet`)
- [ ] **MNR-03**: Devnet faucet link conditionally hidden on mainnet
- [ ] **MNR-04**: `staking-builders.ts` TS error fixed (removed systemProgram reference)
