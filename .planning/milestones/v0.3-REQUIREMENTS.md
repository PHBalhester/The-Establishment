# Requirements: Transfer Hook Program (v0.3)

**Defined:** 2026-02-05
**Core Value:** Enforce pool-mediated transfers — users can only acquire/sell tokens through official protocol mechanisms

## v0.3 Requirements

Requirements for Transfer Hook Program. Each maps to roadmap phases.

### Hook Infrastructure

- [x] **HOOK-01**: Transfer hook instruction implemented with correct SPL interface discriminator (Execute)
- [x] **HOOK-02**: ExtraAccountMetaList initialized per mint (CRIME, FRAUD, PROFIT) with dynamic PDA resolution
- [x] **HOOK-03**: ExtraAccountMetaList uses correct account indices (0=source, 2=destination) for whitelist PDA derivation

### Whitelist System

- [x] **WHTE-01**: WhitelistAuthority PDA stores authority pubkey with Option<Pubkey> for burn support
- [x] **WHTE-02**: WhitelistEntry PDAs created via existence-based pattern (seeds = ["whitelist", address])
- [x] **WHTE-03**: initialize_authority instruction creates WhitelistAuthority with deployer as authority
- [x] **WHTE-04**: add_whitelist_entry instruction creates WhitelistEntry PDA for given address (authority-gated)
- [x] **WHTE-05**: burn_authority instruction sets authority to None permanently (irreversible)
- [x] **WHTE-06**: Transfer allowed if source OR destination is whitelisted (direction-aware validation)
- [x] **WHTE-07**: Transfer blocked if neither source nor destination is whitelisted

### Security Validation

- [x] **SECU-01**: Transferring flag validated on token accounts (prevents direct hook invocation attack)
- [x] **SECU-02**: Mint validated against allowed set (CRIME, FRAUD, PROFIT only)
- [x] **SECU-03**: Zero amount transfers rejected
- [x] **SECU-04**: Whitelist PDA derivation verified (prevents spoofed accounts)

### Events

- [x] **EVNT-01**: AuthorityBurned event emitted with burned_by and timestamp
- [x] **EVNT-02**: AddressWhitelisted event emitted with address, added_by, and timestamp

### Error Handling

- [x] **ERRH-01**: NoWhitelistedParty error when neither source nor destination is whitelisted
- [x] **ERRH-02**: ZeroAmountTransfer error when amount is 0
- [x] **ERRH-03**: Unauthorized error when signer is not the authority
- [x] **ERRH-04**: AuthorityAlreadyBurned error when attempting whitelist modification post-burn
- [x] **ERRH-05**: AlreadyWhitelisted error when adding duplicate entry
- [x] **ERRH-06**: InvalidWhitelistPDA error when PDA derivation fails validation

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Mutable whitelist post-burn | Defeats trust model; authority burn is permanent by design |
| Per-mint whitelist | Unnecessary complexity; all tokens share identical restrictions |
| Transfer fee collection in hook | Handled by Tax Program; would double-tax |
| TransferBlocked event | Adds compute cost to every rejected transfer; debug via error codes |
| Transfer amount limits | Can be gamed with multiple smaller transfers; not in spec |
| Upgradability | Security requirement; hook is immutable post-deployment |
