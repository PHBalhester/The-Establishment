<task type="auto">
  <name>Task 1: Edge case audit and gap report</name>
  <files>docs/edge-case-audit.md</files>
  <action>
Perform a systematic edge case audit across all 7 programs. For each program:

1. **Read all instruction handlers** (the `src/instructions/*.rs` files)
2. **Read all existing tests** (the `tests/*.rs` files)
3. **Identify untested edge cases** by analyzing:
   - Error paths: Which error codes are never triggered in tests?
   - Boundary conditions: Which numeric limits (u64::MAX, 0, overflow) are untested?
   - State transitions: Which state machine transitions lack test coverage?
   - Access control: Which unauthorized caller paths are untested?
   - Math edge cases: Which arithmetic operations lack extreme-value tests?
   - Cross-program interactions: Which CPI failure modes are untested?

4. **Classify each gap** as:
   - **HIGH**: Financial risk (loss of funds, incorrect payouts), security risk (unauthorized access), or correctness risk (wrong state transitions)
   - **MEDIUM**: Correctness with user impact (wrong error messages, edge case UI breakage), robustness (graceful handling of unusual inputs)
   - **LOW**: Cosmetic, documentation, or extremely unlikely scenarios

5. **Write `docs/edge-case-audit.md`** with structure:
   ```
   # Edge Case Audit Report
   Date: 2026-03-08
   Scope: All 7 on-chain programs

   ## Summary
   - X HIGH gaps identified (Y implemented)
   - X MEDIUM gaps identified (Y implemented)
   - X LOW gaps documented

   ## Program: AMM
   ### Existing Coverage
   [What tests already exist]
   ### Gaps Found
   | ID | Risk | Description | Status |

   ## Program: Tax Program
   [same structure]

   ... (all 7 programs)

   ## Recommendations for Mainnet
   [Any systemic observations]
   ```
