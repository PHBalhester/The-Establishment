    1. **Data structure:** `Map<string, { timestamps: number[], blockedUntil: number }>` keyed by IP.
    2. **Core function:** `checkRateLimit(ip: string, config: { windowMs: number, maxRequests: number }): { allowed: boolean, retryAfter?: number }`

    - `/api/sse/*`: Rate limiting not needed here (SSE is long-lived, connection cap handles it)

    **Integration:**

    **H008 — SSE connection cap:**

    Add connection tracking to both SSE route handlers (`protocol/route.ts` and `candles/route.ts`).

    1. **Shared connection tracker** (can be in rate-limit.ts or a new `app/lib/sse-connections.ts`):
    ```typescript
    const connections = new Map<string, number>(); // ip -> count
    let globalCount = 0;
    const MAX_PER_IP = 3;
    const MAX_GLOBAL = 100;

    export function acquireConnection(ip: string): boolean {
      if (globalCount >= MAX_GLOBAL) return false;
      const ipCount = connections.get(ip) ?? 0;
      if (ipCount >= MAX_PER_IP) return false;
      connections.set(ip, ipCount + 1);
      globalCount++;
      return true;
    }

    export function releaseConnection(ip: string): void {
      const ipCount = connections.get(ip) ?? 0;
      if (ipCount > 0) connections.set(ip, ipCount - 1);
      if (ipCount <= 1) connections.delete(ip);
      globalCount = Math.max(0, globalCount - 1);
    }
    ```

    2. In each SSE route's GET handler:
    - Extract IP via `getClientIp(request)`
    - Call `acquireConnection(ip)`. If false, return `429 Too Many Connections` immediately.
    - In the ReadableStream's cancel callback (already exists for SSE cleanup), call `releaseConnection(ip)`.
    - Also release on stream error / abort signal.

    3. **Robustness:** Use the request's `AbortSignal` (`request.signal`) to detect disconnection and ensure release is called. Add a safety timeout (e.g., 30 min max connection) to prevent zombie connections from exhausting the cap.
  </action>
