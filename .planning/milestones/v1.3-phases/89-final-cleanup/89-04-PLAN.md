<objective>
Convert quote-engine.ts from JavaScript Number to BigInt for all arithmetic to eliminate silent precision loss at mainnet-scale reserves.

Purpose: Close H014. At mainnet reserves (290T token base units x 10B SOL lamports), Number.MAX_SAFE_INTEGER (9e15) is exceeded in intermediate products. The constant-product formula `reserveOut * effectiveInput / (reserveIn + effectiveInput)` produces a numerator of ~2.75e24, which silently loses precision in JS Number. BigInt eliminates this entirely.

Output: All quote functions use BigInt arithmetic, all callers updated, Number conversion only at display layer.
</objective>

  <files>app/lib/swap/quote-engine.ts</files>
  <action>
    Convert ALL functions in quote-engine.ts from `number` to `bigint`:

    1. **Constants:** `const BPS_DENOMINATOR = 10_000n;` (BigInt literal)

    2. **All function signatures** change from `number` params/returns to `bigint`:
    - `calculateEffectiveInput(amountIn: bigint, feeBps: bigint): bigint`
    - `calculateSwapOutput(reserveIn: bigint, reserveOut: bigint, effectiveInput: bigint): bigint`
    - `calculateTax(amount: bigint, taxBps: bigint): bigint`
    - And ALL other exported functions (calculateSolBuyQuote, calculateSolSellQuote, bonding curve quotes, etc.)

    3. **Math operations:**
    - Replace `Math.floor(a * b / c)` with `a * b / c` (BigInt division truncates by default = floor)
    - Replace `Math.ceil(a * b / c)` with `(a * b + c - 1n) / c` (standard BigInt ceil pattern)
    - Remove all `Math.floor`, `Math.ceil`, `Math.max`, `Math.min` calls
    - Replace `Math.max(a, b)` with `a > b ? a : b`
    - Replace `Math.min(a, b)` with `a < b ? a : b`

    4. **Edge cases:**
    - `0` becomes `0n` everywhere
    - Comparisons with `0` become `0n`
    - Any string-to-number conversions become `BigInt(string)`
    - The `Number()` call for display purposes should be clearly marked with a comment: `// DISPLAY ONLY: convert to Number for formatting`

    5. **Bonding curve functions:** If quote-engine also has bonding curve quote functions (calculateTokensOut, calculateSolForTokens, etc.), convert those too. The context decision says "Include ALL functions (AMM quotes + bonding curve quotes) for consistency."

    6. **Keep JSDoc comments** updated to reflect BigInt types and note the migration rationale.

    **Do NOT change the file's export names or function names** — only types and internal math.
  </action>
