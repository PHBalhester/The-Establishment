  <action>
    **H019 — Circuit breaker:**
    Add a `consecutiveErrors` counter (starts at 0). On any error in the main crank loop:
    - Increment `consecutiveErrors`
    - If `consecutiveErrors >= 5`, log a CRITICAL message and break out of the main loop (process exits)
    - The existing graceful shutdown handler will clean up

    On ANY successful epoch advancement or crank cycle completion:
    - Reset `consecutiveErrors = 0`

    This prevents infinite error loops (e.g., if the RPC is down or a program is in a bad state).

    **H019 — SOL spending cap:**
    Track SOL spent per rolling hour:
    - Maintain an array of `{ lamports: number, timestamp: number }` entries
    - Before each transaction, sum entries within the last 3600 seconds
    - If sum >= 0.5 SOL (500_000_000 lamports), log a CRITICAL message and halt
    - After each transaction, push the transaction fee (estimated or actual) to the array
    - Periodically (every cycle), prune entries older than 1 hour

    Transaction cost tracking: After `sendAndConfirmTransaction` or `sendTransaction`, estimate cost as the base fee (5000 lamports per signature) plus any priority fee. For simplicity, use a conservative estimate of 10_000 lamports per transaction (covers base + moderate priority). This is imprecise but safe — the cap has 50x headroom over normal usage.

    **H013 — Vault top-up ceiling:**
    Find the existing vault balance top-up logic (where the crank sends SOL to the vault when balance is low). Add a maximum cap per top-up: `MAX_TOPUP_LAMPORTS = 100_000_000` (0.1 SOL). If the calculated top-up amount exceeds this, cap it. Log a warning if the cap is hit (indicates something unexpected). This prevents a bug from draining the crank wallet into the vault in a single transaction.

    **H019 — Health endpoint:**
    Add a minimal HTTP server using Node.js built-in `http` module (no dependencies):

    ```typescript
    import { createServer } from 'http';

    const HEALTH_PORT = parseInt(process.env.HEALTH_PORT || '8080', 10);

    function startHealthServer() {
      const server = createServer((req, res) => {
        if (req.method === 'GET' && req.url === '/health') {
          const status = {
            status: consecutiveErrors >= 5 ? 'halted' : 'running',
            consecutiveErrors,
            hourlySpendLamports: getCurrentHourlySpend(),
            uptime: process.uptime(),
            lastSuccessAt: lastSuccessTimestamp ?? null,
          };
          res.writeHead(200, { 'Content-Type': 'application/json' });
          res.end(JSON.stringify(status));
        } else {
          res.writeHead(404);
          res.end();
        }
      });

      server.listen(HEALTH_PORT, '0.0.0.0', () => {
        console.log(`[health] Listening on :${HEALTH_PORT}/health`);
      });

      return server;
    }
    ```

    Call `startHealthServer()` before the main crank loop starts. The server binds to 0.0.0.0 so Railway's internal health checks can reach it, but no public domain is assigned (Railway only exposes ports with an explicit domain config).

    Track `lastSuccessTimestamp` (set to `Date.now()` on each successful cycle) for the health response.

    Add the health server to the graceful shutdown handler (close the server on SIGTERM/SIGINT).
  </action>
