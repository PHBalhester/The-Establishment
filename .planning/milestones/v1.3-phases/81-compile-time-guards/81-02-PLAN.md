**Bonding Curve constants.rs -- add const assertions block at the bottom of the file (before any `#[cfg(test)] mod tests` if one exists, otherwise at EOF):**

```rust
// ---------------------------------------------------------------------------
// Compile-Time Invariant Assertions (CTG-03)
// ---------------------------------------------------------------------------

/// Curve must go up: end price strictly greater than start price.
const _: () = assert!(P_END > P_START, "P_END must be greater than P_START");

/// Non-zero supply: cannot sell zero tokens.
const _: () = assert!(TOTAL_FOR_SALE > 0, "TOTAL_FOR_SALE must be non-zero");

/// u128 and u64 versions of total supply must agree.
/// TOTAL_FOR_SALE is u128, TARGET_TOKENS is u64 -- they represent the same value.
const _: () = assert!(TOTAL_FOR_SALE as u64 as u128 == TOTAL_FOR_SALE, "TARGET_TOKENS must equal TOTAL_FOR_SALE (no truncation)");
```

Note: Use `assert!()` with string message (stable since Rust 1.73). The pattern `const _: () = assert!(expr)` is zero-cost -- evaluated entirely at compile time, produces no runtime code.

The third assertion checks round-trip: cast TOTAL_FOR_SALE to u64 and back to u128 -- if it equals the original, no truncation occurred. This is more robust than comparing TARGET_TOKENS directly since it validates the cast itself.

**Epoch Program -- IDL verification test (CTG-02):**

Add a `#[cfg(test)]` module at the bottom of `programs/epoch-program/src/lib.rs` (or append to existing test module if one exists).

The test should read the generated IDL JSON file and assert `force_carnage` is NOT present when built without devnet feature. Approach:

```rust
#[cfg(test)]
mod tests {
    #[test]
    fn force_carnage_excluded_from_default_build() {
        // The force_carnage module is conditionally compiled with #[cfg(feature = "devnet")].
        // This test runs under default features (no devnet), confirming the module
        // is excluded. If someone accidentally removes the cfg gate, this test
        // will fail because the force_carnage module would be compiled in.
        //
        // We verify by checking the instruction enum variants at the type level.
        // The Anchor-generated instruction enum includes force_carnage only when
        // the devnet feature is active.

        // If force_carnage were compiled in, this would be a valid path:
        // crate::instruction::ForceCarnage { ... }
        // Since it's cfg-gated out, we just assert we got here (the real guard
        // is the triple cfg gate on mod, use, and #[program] instruction).

        // Structural test: count instruction variants.
        // Without devnet: 8 instructions (trigger, consume, execute_carnage,
        //   execute_carnage_atomic, expire_carnage, init_carnage, init_epoch, retry_epoch_vrf)
        // With devnet: 9 instructions (+force_carnage)
        // We can't easily count enum variants in Rust without a proc macro,
        // so we verify the module path doesn't exist.

        // The definitive check: try to reference the force_carnage module.
        // This line would cause a compile error if force_carnage were included:
        #[cfg(not(feature = "devnet"))]
        {
            // If this compiles, force_carnage is correctly excluded.
            // The cfg gate on instructions/mod.rs prevents the module from existing.
            assert!(
                cfg!(not(feature = "devnet")),
                "This test must run without devnet feature"
            );
        }
    }
}
```

Actually, a simpler and more robust approach: parse the IDL JSON file from the target directory. But IDL files are generated by `anchor build` and may not exist in unit test context. Instead, use a compile-time approach:

The test should verify that `super::instruction::ForceCarnage` is NOT a valid type. Since we can't negative-test type existence in Rust easily, the best approach from CONTEXT.md is:

Create a test that reads the IDL JSON file (if it exists) from `target/idl/epoch_program.json` and asserts "force_carnage" is not in the instructions array. If the IDL file doesn't exist (not built yet), the test should skip gracefully with a message.

```rust
#[cfg(test)]
mod tests {
    #[test]
    fn force_carnage_excluded_from_non_devnet_idl() {
        // Read the IDL file generated by `anchor build` (without devnet feature).
        // If built with default features, force_carnage should NOT appear.
        let idl_path = concat!(env!("CARGO_MANIFEST_DIR"), "/../../target/idl/epoch_program.json");
        let Ok(idl_content) = std::fs::read_to_string(idl_path) else {
            eprintln!("IDL file not found at {idl_path} -- skipping (run `anchor build` first)");
            return;
        };

        // In non-devnet builds, force_carnage should not appear in instructions
        if cfg!(not(feature = "devnet")) {
            assert!(
                !idl_content.contains("forceCarnage") && !idl_content.contains("force_carnage"),
                "force_carnage found in non-devnet IDL! The cfg gate may have been removed."
            );
        }
    }
}
```

Note: Anchor IDL uses camelCase for instruction names, so check both `forceCarnage` and `force_carnage`. The test gracefully skips if IDL hasn't been generated yet.
