    Systematic code audit for mobile issues at 375px minimum viewport width. For EACH file listed:

    1. **Horizontal overflow**: Check for fixed-width elements (px widths > 375), `min-w-` that could exceed viewport, `whitespace-nowrap` on long text without overflow handling. Fix with `max-w-full`, `overflow-hidden`, `text-ellipsis`, or responsive widths.

    2. **Fixed positioning conflicts**: Check `fixed` or `absolute` elements that could overlap on small screens. Ensure z-index layering doesn't cause hidden content.

    3. **Tap targets**: Buttons and interactive elements must be >= 48px touch target (per Phase 58 pattern). Check padding on small buttons, links, tab switches.

    4. **Text sizing**: Ensure no text smaller than 12px on mobile. Check `text-xs` (12px) is the floor. Long numbers or addresses should truncate/ellipsis.

    5. **Spacing**: Check `gap-*`, `p-*`, `m-*` values that may be too large on mobile, consuming precious viewport space. Use responsive variants (`gap-2 lg:gap-4`).

    6. **Input sizing**: Form inputs in BuySellPanel (Buy/Sell forms) must be full-width on mobile, not constrained by `max-w-[380px]` if that causes side padding issues.

    7. **Launch page mobile layout** (`lg:hidden` section in page.tsx): Verify the stacked layout has proper safe-area insets, doesn't overflow, and scrolls correctly. The `min-h-screen` + `fixed` background pattern should work but verify no content gets trapped behind the fixed backdrop.

    Apply fixes directly. Prefer responsive Tailwind variants (`sm:`, `md:`, `lg:`) over CSS media queries. Follow existing 1024px (`lg:`) breakpoint convention.
