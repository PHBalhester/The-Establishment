**VRF-09: EpochState offset validation test in Rust**

Add a test to `constants.rs` (in the existing `#[cfg(test)] mod tests` block) that validates the EpochState struct field offsets match documented values. Use `std::mem::offset_of!` (stable since Rust 1.77) or `memoffset` crate. Since the struct has `#[repr(C)]`, offsets are deterministic.

Actually, `offset_of!` may not work with Anchor's `#[account]` macro (which adds a discriminator). The discriminator is NOT part of the struct itself -- it's prepended during serialization. So `offset_of!` gives offsets within the DATA portion (after discriminator).

Add this test:
```rust
#[test]
fn test_epoch_state_field_offsets() {
    use crate::state::EpochState;

    // Offsets within the data portion (discriminator is 8 bytes prepended during serialization).
    // To get on-chain byte offset, add 8 to each value.
    //
    // These must match TypeScript EPOCH_STATE_OFFSETS in tests/integration/helpers/mock-vrf.ts
    assert_eq!(std::mem::offset_of!(EpochState, genesis_slot), 0);       // on-chain: 8
    assert_eq!(std::mem::offset_of!(EpochState, current_epoch), 8);      // on-chain: 16
    assert_eq!(std::mem::offset_of!(EpochState, epoch_start_slot), 12);  // on-chain: 20
    assert_eq!(std::mem::offset_of!(EpochState, cheap_side), 20);        // on-chain: 28
    assert_eq!(std::mem::offset_of!(EpochState, low_tax_bps), 21);       // on-chain: 29
    assert_eq!(std::mem::offset_of!(EpochState, high_tax_bps), 23);      // on-chain: 31
    assert_eq!(std::mem::offset_of!(EpochState, vrf_request_slot), 33);  // on-chain: 41
    assert_eq!(std::mem::offset_of!(EpochState, vrf_pending), 41);       // on-chain: 49
    assert_eq!(std::mem::offset_of!(EpochState, taxes_confirmed), 42);   // on-chain: 50
    assert_eq!(std::mem::offset_of!(EpochState, pending_randomness_account), 43); // on-chain: 51
    assert_eq!(std::mem::offset_of!(EpochState, carnage_pending), 75);   // on-chain: 83
    assert_eq!(std::mem::offset_of!(EpochState, carnage_target), 76);    // on-chain: 84
    assert_eq!(std::mem::offset_of!(EpochState, carnage_action), 77);    // on-chain: 85
    assert_eq!(std::mem::offset_of!(EpochState, carnage_deadline_slot), 78); // on-chain: 86
```

**IMPORTANT**: The `#[repr(C)]` on EpochState means no reordering. BUT Anchor's `#[account]` macro wraps serialization with Borsh, which serializes fields in order without padding. For `#[repr(C)]`, `offset_of!` may include alignment padding that Borsh doesn't. Check carefully:
- If `offset_of!` works directly, great.
- If the struct uses Borsh serialization (no padding), the offsets correspond to the Borsh byte positions directly. In that case, manually compute expected offsets from field sizes and assert against them using a const table.
