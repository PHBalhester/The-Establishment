In `swap_sol_sell.rs` handler, replace the hardcoded `amm_minimum: u64 = 0` (line ~147) with a computed gross floor derived from the user's `minimum_output`.

The user's `minimum_output` is the net amount they expect AFTER tax. To protect at the AMM level, compute what gross output the AMM must produce so that after tax, the user still gets at least `minimum_output`:

```rust
// Compute gross floor: what AMM must output so user gets >= minimum_output after tax
// gross_floor = ceil(minimum_output * BPS_DENOM / (BPS_DENOM - tax_bps))
// Use u128 intermediates to prevent overflow
let bps_denom: u64 = 10_000;
let gross_floor = if minimum_output > 0 && tax_bps < bps_denom {
    let numerator = (minimum_output as u128)
        .checked_mul(bps_denom as u128)
        .ok_or(error!(TaxError::TaxOverflow))?;
    let denominator = (bps_denom as u128)
        .checked_sub(tax_bps as u128)
        .ok_or(error!(TaxError::TaxOverflow))?;
    // Ceil division: (numerator + denominator - 1) / denominator
    let result = numerator
        .checked_add(denominator - 1)
        .ok_or(error!(TaxError::TaxOverflow))?
        / denominator;
    u64::try_from(result).map_err(|_| error!(TaxError::TaxOverflow))?
} else {
    0
};

let amm_minimum: u64 = gross_floor;
```

Per CONTEXT.md: Only use user's `minimum_output` for floor derivation. Do NOT double-layer with the 50% protocol floor (MINIMUM_OUTPUT_FLOOR_BPS is enforced separately above in the `output_floor` check). Sell path only.

The formula: `gross_floor = ceil(minimum_output * 10000 / (10000 - sell_tax_bps))` ensures the AMM rejects swaps where the output would be too low to satisfy the user's net minimum after tax deduction.
