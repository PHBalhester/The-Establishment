  <action>
    **H024 — Rate limit /api/sol-price:**
    Add rate limiting to `app/app/api/sol-price/route.ts` using the existing `checkRateLimit` + `getClientIp` infrastructure from `app/lib/rate-limit.ts`. Create a `SOL_PRICE_RATE_LIMIT` config with `windowMs: 60_000, maxRequests: 30` (generous — each page load makes 1 request, and there's a 60s cache). Add the rate check at the top of the `GET()` handler, returning 429 + Retry-After header when exceeded. Import pattern matches `/api/rpc/route.ts` exactly.

    **H047 — RPC proxy failover:**
    Modify `app/app/api/rpc/route.ts` to support multiple RPC endpoints with failover:
    1. Parse `HELIUS_RPC_URL` as primary endpoint. Add `HELIUS_RPC_URL_FALLBACK` env var as secondary endpoint. Keep `DEVNET_RPC_URL` from shared as last resort (local dev only).
    2. Build an ordered endpoint list: `[primary, fallback, devnetFallback].filter(Boolean)`.
    3. In the POST handler, try each endpoint in order. On fetch failure (network error or HTTP 5xx from upstream), try the next. On success (any response from upstream, even RPC errors), return it.
    4. Track which endpoint last succeeded in a module-level variable. On subsequent requests, try the last-successful endpoint first (sticky routing), falling back to the full list on failure.
    5. If ALL endpoints fail, return the existing 502 error.
    6. Log which endpoint is being used (masked — show only the hostname, not the full URL with API key). Use `new URL(endpoint).hostname` for masking.

    **H049 — Webhook blockTime staleness:**
    In `app/app/api/webhooks/helius/route.ts`, after the authorization check and before processing raw transaction webhooks, add a blockTime staleness check:
    1. For raw transaction payloads (the array of tx objects), check the `blockTime` of each transaction.
    2. If `blockTime` is present and `Date.now() / 1000 - blockTime > 300` (older than 5 minutes), skip that transaction with a warning log: `[webhook] Skipping stale transaction (blockTime ${blockTime}, age ${age}s)`.
    3. Do NOT reject the entire payload — just skip individual stale transactions. Helius may batch recent and slightly-old transactions together.
    4. For enhanced account change webhooks, blockTime is not applicable (they're account state snapshots). Skip the check for those.
    5. Add the check right before the `for (const tx of rawTxPayload)` processing loop.
  </action>

    3. `grep "blockTime.*300\|5.*minute\|stale" app/app/api/webhooks/helius/route.ts` confirms replay protection
    4. `cd app && npx next build` succeeds
  </verify>
