<objective>
Add ProgramData upgrade authority checks to all 6 initialization instructions across 5 programs.

Purpose: Currently these init instructions accept any signer as deployer. A front-runner could call initialize before the real deployer, setting up protocol state with malicious parameters. Adding ProgramData upgrade authority verification ensures only the program deployer can initialize.

Output: All 6 init instructions require the caller to be the ProgramData upgrade authority, matching the pattern already established in the AMM program.
</objective>

    For each program, add two new accounts to the init instruction's Accounts struct. The pattern is identical across all — copy from AMM's initialize_admin.rs:

    **1. Transfer Hook — `initialize_authority.rs` (AUTH-02):**
    The signer is currently named `signer`. Keep the name but add:
    ```rust
    /// The Transfer Hook program — used to look up its ProgramData address.
    #[account(
        constraint = program.programdata_address()? == Some(program_data.key())
    )]
    pub program: Program<'info, crate::program::TransferHook>,

    /// ProgramData account — upgrade_authority must match signer.
    #[account(
        constraint = program_data.upgrade_authority_address == Some(signer.key())
    ```
