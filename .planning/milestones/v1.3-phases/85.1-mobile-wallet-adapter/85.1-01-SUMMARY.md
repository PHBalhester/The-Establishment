## Accomplishments
- Mobile Chrome users see 3 tappable wallet options (Phantom, Solflare, Backpack) in ConnectModal

## Decisions Made

1. **Manual deep links over @solana-mobile/wallet-adapter-mobile** -- The SMS adapter is designed for Solana Mobile Stack (Saga devices / dApp Store), not mobile Chrome deep links. Manual universal links are simpler, zero-dependency, and work across all mobile browsers.

2. **"Open App" badge instead of "Install"** -- The deep link opens the wallet app if installed, or redirects to the app store if not. "Open App" is more accurate and less confusing than "Install" for users who already have the wallet.

3. **Connect prompt benefits all users** -- The inline "Connect Wallet to Trade" prompt in BuySellPanel is shown for both mobile and desktop disconnected users, not just mobile. More discoverable than the floating wallet button alone.

4. **No providers.tsx changes needed** -- The empty wallets array works correctly for both desktop (wallet-standard auto-detect) and mobile in-app browsers (wallet also auto-detected). Deep links are handled at the ConnectModal UI level only.

## Deviations from Plan

### Pivot: Deep links instead of npm adapter (planned pivot)

The plan explicitly anticipated that `@solana-mobile/wallet-adapter-mobile` might not fit (it's for SMS/Saga, not mobile Chrome). The pivot path was pre-planned: manual deep-link URLs in ConnectModal. This was the correct choice. No npm package installed, no providers.tsx changes needed.

**Total deviations:** 1 planned pivot (deep links over SMS adapter)
**Impact on plan:** None -- pivot path was documented in plan. All must-have truths satisfied.
