  <name>Task 1: Install mobile wallet adapter + create isMobile utility + wire providers</name>
  <files>
    app/package.json
    app/lib/isMobile.ts
    app/providers/providers.tsx
  </files>
  <action>
    1. Install @solana-mobile/wallet-adapter-mobile:
       `cd app && npm install @solana-mobile/wallet-adapter-mobile`

    2. Create `app/lib/isMobile.ts` — a shared utility that checks navigator.userAgent for mobile devices:
       - Export a function `isMobile(): boolean` that returns true for Android/iPhone/iPad/iPod/mobile user agents
       - Must handle SSR (typeof window === 'undefined' -> return false)
       - Keep it simple — regex match against common mobile UA strings
       - This is what wallet-adapter-mobile uses internally, so the detection is consistent

    3. Update `app/providers/providers.tsx`:
       - Import SolanaMobileWalletAdapter from @solana-mobile/wallet-adapter-mobile
       - Import isMobile from @/lib/isMobile
       - In the wallets useMemo, conditionally include SolanaMobileWalletAdapter instances for mobile:
         ```
         const wallets = useMemo(() => {
           if (typeof window !== 'undefined' && isMobile()) {
             return [new SolanaMobileWalletAdapter({
               appIdentity: { name: "Dr. Fraudsworth's Finance Factory" },
               authorizationResultCache: /* use createDefaultAuthorizationResultCache() or similar */,
               cluster: 'devnet', // TODO: make env-aware for mainnet
             })];
           }
           return []; // desktop: auto-detect wallet-standard extensions
         }, []);
         ```
       - IMPORTANT: The SolanaMobileWalletAdapter handles the deep-link/redirect flow automatically for Phantom/Solflare/Backpack. It registers as a single adapter that manages all mobile wallet connections.
       - Note: If SolanaMobileWalletAdapter doesn't fit well (it's designed for Saga/dApp Store, not mobile browser deep links), pivot to a manual deep-link approach in ConnectModal instead (see Task 2 action notes).

    IMPORTANT PIVOT NOTE: @solana-mobile/wallet-adapter-mobile may be designed for Solana Mobile Stack (SMS) / Saga devices, NOT mobile browser deep links. If during implementation you discover this:
    - Skip the npm install and provider changes
    - Instead, implement manual deep-link URLs directly in ConnectModal (Task 2)
    - Deep link patterns:
      - Phantom: `https://phantom.app/ul/browse/${encodeURIComponent(window.location.href)}?ref=${encodeURIComponent(window.location.origin)}`
      - Solflare: `https://solflare.com/ul/v1/browse/${encodeURIComponent(window.location.href)}?ref=${encodeURIComponent(window.location.origin)}`
      - Backpack: `https://backpack.app/ul/browse/${encodeURIComponent(window.location.href)}`
    - This approach requires NO npm package — just anchor tags with deep-link URLs
    - The wallet's in-app browser opens our dApp URL, where wallet-standard auto-detect works normally
  </action>

    - `cd app && npx tsc --noEmit` passes (no type errors)
    - `app/lib/isMobile.ts` exports isMobile function
    - providers.tsx compiles without errors
    - If manual deep-link approach used: no new npm dependency needed
  </verify>
