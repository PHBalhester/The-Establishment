# Phase 85.1: Mobile Wallet Adapter - Context

**Gathered:** 2026-03-08
**Status:** Ready for planning

<domain>
## Phase Boundary

Mobile Chrome users can connect Phantom/Solflare/Backpack via deep links using @solana-mobile/wallet-adapter-mobile. ConnectModal shows mobile-compatible wallets and handles redirect flow. Requirement: MOB-02.

</domain>

<decisions>
## Implementation Decisions

### Wallet Detection & Display
- Show all 3 target wallets (Phantom, Solflare, Backpack) on mobile as tappable deep-link options, regardless of installation status
- Visually distinguish with an "Install" badge on wallets that aren't detected as installed
- On mobile, replace "No wallets detected" empty state entirely with the deep-link wallet list — empty state only applies to desktop
- Deep-link wallets ONLY on mobile — no hybrid merge with detected extension wallets. If user is in Phantom's in-app browser, autoConnect handles it
- Only Phantom, Solflare, Backpack — no other wallets

### Redirect Flow UX
- Auto-detect connection when page regains focus after redirect — wallet-adapter handles this, show existing "Wallet connected" toast
- Auto-close ConnectModal immediately when user taps a wallet on mobile (they're leaving the browser)
- No special handling for uninstalled wallet taps — app store opens, user installs, comes back, taps again
- No special handling for failed redirect returns — autoConnect=true is self-healing on next page load

### Desktop vs Mobile Detection
- User-agent detection (navigator.userAgent) to determine mobile vs desktop — standard approach, wallet-adapter-mobile uses this internally
- No QR codes on desktop — out of scope (would require WalletConnect protocol, significant complexity)
- Desktop keeps current extension-only behavior unchanged
- Desktop empty state keeps current "Install Phantom, Solflare, or Backpack" message

### Modal UI
- Same header "Connect to Dr. Fraudsworth" on both platforms
- Different subtitle: desktop = "Select a wallet to connect", mobile = "Choose your wallet app" (or similar)

### Launch Page Integration
- LaunchWalletButton uses the same enhanced ConnectModal — no special handling needed (it already renders ConnectModal)
- Add inline "Connect Wallet to Trade" prompt inside BuySellPanel when wallet not connected on mobile — more discoverable than floating button alone
- No auto-scroll to BuySellPanel after connection
- All wallet connection points (WalletButton header + LaunchWalletButton) automatically benefit from ConnectModal enhancement

### Claude's Discretion
- Exact mobile user-agent detection regex/logic
- Install badge visual design (icon, color, positioning)
- Mobile subtitle copy
- How to structure the isMobile utility (shared helper vs inline)
- wallet-adapter-mobile integration details (adapter configuration in providers.tsx)

</decisions>

<specifics>
## Specific Ideas

- ConnectModal.tsx is the single modification target — both WalletButton and LaunchWalletButton render it, so one change covers all connection points
- providers.tsx currently passes empty wallets array (auto-detect wallet-standard). Need to add mobile wallet adapters to this array conditionally
- autoConnect=true in WalletProvider handles reconnection after redirect, including browser tab death — no extra work needed
- BuySellPanel inline connect prompt: a centered button that opens ConnectModal, shown only when !connected on mobile

</specifics>
