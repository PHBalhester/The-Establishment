  <name>Task 1: Dynamic Priority Fees</name>
  <files>
    app/providers/SettingsProvider.tsx
    app/hooks/useSettings.ts
  </files>
  <action>
    1. **FE-05**: Add dynamic priority fee fetching to SettingsProvider:
       - Create a `fetchPriorityFee` function that calls `getPriorityFeeEstimate` through the RPC proxy (/api/rpc). This is a Helius-specific RPC method (already in the Plan 01 allowlist).
       - The function should request priority fee estimates and return the result with percentile breakdowns
       - Map user's tier selection to Helius percentiles:
         - Low = "economy" or 25th percentile (priorityLevel: "LOW" in Helius API)
         - Medium = 50th percentile (priorityLevel: "MEDIUM", Helius default)
         - High = 75th percentile (priorityLevel: "HIGH")
       - Store the user's selected tier in SettingsProvider state (persisted to localStorage alongside existing settings)
       - Export a `getRecommendedFee(tier?: PriorityFeePreset): Promise<number>` function from useSettings that:
         - Calls fetchPriorityFee
         - Returns the micro-lamport fee for the selected tier
         - On failure (API unreachable), returns a hardcoded fallback of 50,000 micro-lamports and logs the fallback
       - The existing PriorityFeePreset type (migrated from useStaking in Plan 02) should be extended or adapted to represent Low/Medium/High tiers instead of raw numbers
       - Transaction builders (useSwap, useStaking, curve-tx-builder) should call `getRecommendedFee()` before each TX. Don't change them in this plan -- just ensure the hook API is clean enough for them to consume. A follow-up integration is straightforward.

    2. Read the existing SettingsProvider and useSettings carefully. Preserve all existing settings (slippage, etc.). The priority fee tier is a new setting alongside existing ones.
  </action>
  <verify>
    - `getRecommendedFee('Medium')` returns a number (micro-lamports) from Helius API
    - `getRecommendedFee()` returns 50000 fallback when RPC is unreachable
    - Settings persist across page reloads (localStorage)
    - `npx tsc --noEmit` passes
  </verify>
  <done>
    Priority fees are dynamic. Users select a tier (Low/Medium/High), and the actual micro-lamport fee is fetched from Helius before each transaction with a sensible fallback.
  </done>
