  <action>
    1. Create `app/app/api/rpc/route.ts` -- a Next.js API route that:
       - Accepts POST requests with JSON-RPC payloads
       - Validates the `method` field against an allowlist of RPC methods the frontend uses. Audit all hooks and transaction builders to discover the exact list. At minimum: getAccountInfo, getBalance, getMultipleAccounts, getTokenAccountsByOwner, getLatestBlockhash, sendTransaction, simulateTransaction, getSignatureStatuses, confirmTransaction, getSlot, getProgramAccounts, getPriorityFeeEstimate (Helius-specific). Block everything else.
       - Forwards allowed requests to the Helius RPC URL (from server-only env var `HELIUS_RPC_URL`, NOT any NEXT_PUBLIC_ var)
       - Returns the Helius response to the client
       - Returns 400 for disallowed methods with a clear error message
       - Use `export const runtime = "nodejs"` and `export const dynamic = "force-dynamic"`
       - Log disallowed method attempts (method name only, not full payload)

    2. Add cluster-keyed config to `shared/constants.ts`:
       - Create a `CLUSTER_CONFIG` object keyed by cluster name ('devnet' | 'mainnet-beta')
       - Devnet key contains current hardcoded values (PROGRAM_IDS, MINTS, etc.)
       - Mainnet key contains placeholder values (Pubkey.default() or clearly marked TODOs) -- filled in during v1.4
       - Export a `getClusterConfig(cluster: string)` helper function
       - Keep all existing exports working (backward compatible) -- they continue to export devnet values as before. The new CLUSTER_CONFIG is additive.
       - Move `DEVNET_RPC_URL` from programs.ts to be cluster-aware. The RPC URL is now a server-only concern (not exported to browser). Add a comment noting the API key should come from HELIUS_RPC_URL env var on the server.

    3. Update `app/lib/connection.ts`:
       - Detect browser environment: `typeof window !== 'undefined'`
       - In browser: use `/api/rpc` as RPC URL (relative path, works on any domain)
       - On server (API routes, SSR, crank): use `HELIUS_RPC_URL` env var directly (falls back to DEVNET_RPC_URL from shared for backward compat)
       - The singleton cache key should include whether we're in browser or server
       - Remove `NEXT_PUBLIC_RPC_URL` usage (the proxy replaces it). Add a migration comment.

    4. Update `shared/index.ts` to export new CLUSTER_CONFIG and getClusterConfig.
  </action>
  <verify>
    - `npx tsc --noEmit` passes in the app directory (no type errors)
    - `curl -X POST http://localhost:3000/api/rpc -H "Content-Type: application/json" -d '{"jsonrpc":"2.0","method":"getSlot","params":[],"id":1}'` returns a valid slot number
    - `curl -X POST http://localhost:3000/api/rpc -H "Content-Type: application/json" -d '{"jsonrpc":"2.0","method":"getBlockProduction","params":[],"id":1}'` returns 400 (blocked method)
    - Browser console shows no Helius API key in network requests (all go to /api/rpc)
  </verify>
  <done>
