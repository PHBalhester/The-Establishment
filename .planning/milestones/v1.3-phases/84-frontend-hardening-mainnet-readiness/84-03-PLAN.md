<task type="auto">
  <name>Task 1: Webhook Auth Hardening + Protocol Account Store</name>
  <files>
    app/app/api/webhooks/helius/route.ts
    app/lib/protocol-store.ts
  </files>
  <action>
    1. **FE-02**: Harden webhook auth in `app/app/api/webhooks/helius/route.ts`:
       - Current behavior: if HELIUS_WEBHOOK_SECRET is unset, auth is skipped (allows local testing)
       - New behavior: in production (`NODE_ENV === 'production'`), if HELIUS_WEBHOOK_SECRET is NOT set, log a critical warning and reject ALL requests with 500 (fail-closed). This prevents accidentally deploying without a webhook secret.
       - In non-production, keep current behavior (skip auth if unset, for local dev)
       - When secret IS set (any environment), existing auth check remains unchanged

    2. **FE-03/FE-04**: Create `app/lib/protocol-store.ts` -- an in-memory store for protocol account state:
       - Use a module-level `Map<string, unknown>` with globalThis singleton pattern (same as sse-manager.ts)
       - Keys are account public keys (base58 strings)
       - Values are parsed account data objects
       - Export functions: `setAccountState(pubkey, data)`, `getAccountState(pubkey)`, `getAllAccountStates()`
       - When state is updated via `setAccountState`, also call `sseManager.broadcast('protocol-update', { account: pubkey, data })` to push to SSE clients

    3. Extend the webhook handler to process Helius "account change" webhook payloads:
       - Helius Enhanced Webhooks with `accountAddresses` filter deliver account data changes
       - The existing handler processes raw transaction webhooks. Add a second code path that detects if the payload contains account data (Helius sends different shapes for raw vs enhanced)
       - For account updates: parse the account data, identify which protocol account it is (EpochState, PoolState, StakePool, CarnageFundState, CurveState) by matching the pubkey against known PDAs from shared/constants.ts
       - Store the raw account data in protocolStore
       - The existing raw transaction handler remains unchanged (candle/swap/epoch/carnage event processing)
       - NOTE: The actual Helius webhook registration (calling Helius API to set up the account-watching webhook) is a deployment task -- just ensure the handler code can process the payloads. Document the registration command in a code comment.
  </action>
  <verify>
    - In production mode with no HELIUS_WEBHOOK_SECRET, webhook returns 500
    - In production mode with HELIUS_WEBHOOK_SECRET set, webhook requires matching Authorization header
    - In dev mode with no HELIUS_WEBHOOK_SECRET, webhook accepts requests (local testing)
    - `protocolStore.setAccountState(key, data)` triggers SSE broadcast
  </verify>
  <done>
    Webhook is fail-closed in production. Protocol account changes are stored in memory and broadcast via SSE.
  </done>
</task>

<task type="auto">
  <name>Task 2: SSE Protocol Endpoint + React Hook</name>
  <files>
    app/app/api/sse/protocol/route.ts

  <action>
    1. Create `app/app/api/sse/protocol/route.ts` -- SSE endpoint for protocol state:
       - Follow the same pattern as `app/app/api/sse/candles/route.ts` (read that file first)
       - GET handler returns a ReadableStream with SSE events
       - Subscribe to sseManager for 'protocol-update' events
       - On connect, immediately send current state from protocolStore.getAllAccountStates() as initial payload (so new clients don't start with stale data)
       - Use `export const runtime = "nodejs"` and `export const dynamic = "force-dynamic"`
       - Clean up subscription on client disconnect

    2. Create `app/hooks/useProtocolState.ts` -- React hook for consuming protocol SSE:
       - Opens an EventSource to `/api/sse/protocol`
       - Parses 'protocol-update' events and maintains state via useState
       - Auto-reconnect on EventSource error (EventSource has built-in reconnect, but add explicit reconnection with exponential backoff if onerror fires)
       - If SSE connection is down for >30 seconds, fall back to polling via RPC every 60 seconds (use the protocol account PDAs from shared/constants.ts to fetch getAccountInfo)
       - When SSE reconnects, cancel the polling fallback
       - Export typed state for each protocol account type: epochState, poolStates, stakePool, carnageFund
       - No connection status indicator (per CONTEXT.md decision)
       - Use the existing visibility pattern (useVisibility) -- only maintain SSE when tab is visible

    3. Document in a code comment at the top of the hook: "To register the Helius account-watching webhook, use the Helius Dashboard or API to create an Enhanced Webhook monitoring [list of PDAs]. The webhook URL should be {deployment_url}/api/webhooks/helius with the HELIUS_WEBHOOK_SECRET as Authorization header."
  </action>
  <verify>

    - `npx tsc --noEmit` passes
    - Hook connects to SSE, receives initial state snapshot, and updates on new events
  </verify>
