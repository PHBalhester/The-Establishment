# Requirements: Dr. Fraudsworth's Finance Factory -- v0.2 AMM Program

**Defined:** 2026-02-03
**Core Value:** The documentation is the source of truth -- AMM must implement specs exactly, no silent decisions.

## v0.2 Requirements

Requirements for the AMM on-chain program. Each maps to roadmap phases.

### Scaffolding

- [x] **SCAF-01**: Fork `arrayappy/solana-uniswap-v2` and upgrade to Anchor 0.32.1 multi-program workspace
- [x] **SCAF-02**: Configure Cargo.toml with anchor-spl (token, token_2022, associated_token features), no external math crates
- [x] **SCAF-03**: Set up litesvm + anchor-litesvm for Rust unit tests
- [x] **SCAF-04**: Set up solana-test-validator integration test harness for CPI chain and hook tests
- [x] **SCAF-05**: Establish AMM program file structure (state/, instructions/, helpers/, constants.rs, errors.rs, events.rs)

### Math

- [x] **MATH-01**: Implement LP fee calculation using u128 checked arithmetic (effective_input = amount x (10,000 - fee_bps) / 10,000)
- [x] **MATH-02**: Implement constant-product output calculation (amount_out = reserve_out x effective_input / (reserve_in + effective_input))
- [x] **MATH-03**: Implement k-invariant verification (k_after >= k_before) with u128 multiplication
- [x] **MATH-04**: All arithmetic uses checked_add/checked_mul/checked_div -- overflow returns error, never panics

### Pool Initialization

- [x] **POOL-01**: `initialize_pool` creates pool state PDA with canonical mint ordering (mint_a < mint_b)
- [x] **POOL-02**: Pool vaults created as PDAs with correct token program per side (T22 for CRIME/FRAUD/PROFIT, SPL Token for WSOL)
- [x] **POOL-03**: Pool authority is the pool PDA -- vaults have pool PDA as token account authority
- [x] **POOL-04**: LP fee rates set per pool type (100 bps for SOL pools, 50 bps for PROFIT pools)
- [x] **POOL-05**: Initial reserves seeded and cached in pool state
- [x] **POOL-06**: Pool marked initialized -- swaps rejected on uninitialized pools
- [x] **POOL-07**: Deployer-only access on `initialize_pool` -- rejects non-deployer callers
- [x] **POOL-08**: Duplicate pool prevention -- same mint pair cannot create two pools (enforced by PDA seeds)

### Token Transfers

- [x] **XFER-01**: Token-2022 tokens (CRIME/FRAUD/PROFIT) use `transfer_checked` exclusively -- never plain `transfer`
- [x] **XFER-02**: SPL Token (WSOL) uses standard `transfer` instruction
- [x] **XFER-03**: Mixed T22/SPL pool swaps pass both token programs as instruction accounts
- [x] **XFER-04**: Transfer hook ExtraAccountMetaList and hook program accounts passed through for T22 transfers
- [x] **XFER-05**: PROFIT pool swaps handle hook accounts on BOTH sides (dual-hook invocation)
- [x] **XFER-06**: Vault-to-user transfers use PDA signing (pool PDA as authority with stored bump)

### Swap Instructions

- [x] **SWAP-01**: `swap_sol_pool` handles CRIME/SOL and FRAUD/SOL swaps (mixed T22/SPL)
- [x] **SWAP-02**: `swap_profit_pool` handles CRIME/PROFIT and FRAUD/PROFIT swaps (pure T22)
- [x] **SWAP-03**: Both directions supported per instruction (token A->B and B->A)
- [x] **SWAP-04**: LP fee deducted before output calculation -- fee compounds into pool reserves
- [x] **SWAP-05**: Slippage protection via `minimum_amount_out` parameter -- reverts if output too low
- [x] **SWAP-06**: Cached reserves updated atomically after transfers (checks-effects-interactions)
- [x] **SWAP-07**: k-invariant verified post-swap -- revert if k decreases
- [x] **SWAP-08**: Zero-amount swaps rejected
- [x] **SWAP-09**: SwapEvent emitted with pool, user, mints, amounts, fee, post-swap reserves
- [x] **SWAP-10**: PoolInitializedEvent emitted on pool creation

### Access Control

- [x] **AUTH-01**: All swap instructions require `swap_authority` PDA signed by Tax Program
- [x] **AUTH-02**: `swap_authority` validated with `seeds::program = TAX_PROGRAM_ID` (not AMM program)
- [x] **AUTH-03**: Direct user calls to swap instructions fail (no valid `swap_authority` signature)
- [x] **AUTH-04**: Mock Tax Program created for testing that produces valid `swap_authority` CPI signatures
- [x] **AUTH-05**: Negative test: swap with fake `swap_authority` (wrong program PDA) is rejected

### Testing

- [x] **TEST-01**: Math unit tests -- LP fee, output calculation, k-invariant for normal values, edge cases (0, 1, u64::MAX), overflow
- [x] **TEST-02**: Pool initialization tests -- all 4 pool types created with correct token programs, vaults, fee rates
- [x] **TEST-03**: Swap tests -- all 4 pools, both directions, correct output amounts, fee compounding, reserve updates
- [x] **TEST-04**: Slippage tests -- swap reverts when output below minimum
- [x] **TEST-05**: Access control tests -- mock Tax Program CPI succeeds, direct calls fail, fake PDA fails
- [x] **TEST-06**: Edge case tests -- minimum viable swap (1 token), heavily imbalanced reserves, near-empty pool
- [x] **TEST-07**: Mixed T22/SPL transfer tests -- correct program used per side, hook accounts passed correctly
- [x] **TEST-08**: CPI chain test -- Mock Tax Program -> AMM -> Token-2022 transfer_checked (verify full chain works)
