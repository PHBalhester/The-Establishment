## v0.4 Requirements

Requirements for Tax Program milestone. Each maps to roadmap phases.

### Swap Instructions (TAX)

- [x] **TAX-01**: User can swap SOL for CRIME via swap_sol_buy with buy tax applied
- [x] **TAX-02**: User can swap SOL for FRAUD via swap_sol_buy with buy tax applied
- [x] **TAX-03**: User can swap CRIME for SOL via swap_sol_sell with sell tax applied
- [x] **TAX-04**: User can swap FRAUD for SOL via swap_sol_sell with sell tax applied
- [x] **TAX-05**: User can swap CRIME for PROFIT via swap_profit_buy (no tax, LP fee only)
- [x] **TAX-06**: User can swap FRAUD for PROFIT via swap_profit_buy (no tax, LP fee only)
- [x] **TAX-07**: User can swap PROFIT for CRIME via swap_profit_sell (no tax, LP fee only)
- [x] **TAX-08**: User can swap PROFIT for FRAUD via swap_profit_sell (no tax, LP fee only)
- [x] **TAX-09**: Carnage can swap via swap_exempt without tax (Carnage PDA signer required)

### Tax Calculation (CALC)

- [x] **CALC-01**: Tax rates read from EpochState account owned by Epoch Program **(Partial: hardcoded until v0.5)**
- [x] **CALC-02**: Buy tax deducted from SOL input before swap execution
- [x] **CALC-03**: Sell tax deducted from SOL output after swap execution
- [x] **CALC-04**: Tax calculation uses u128 intermediates to prevent overflow
- [x] **CALC-05**: Basis point calculation: `amount * tax_bps / 10_000`

### Tax Distribution (DIST)

- [x] **DIST-01**: 75% of collected tax transferred to staking_escrow (Staking Program)
- [x] **DIST-02**: 24% of collected tax transferred to carnage_vault (Epoch Program)
- [x] **DIST-03**: 1% of collected tax transferred to treasury
- [x] **DIST-04**: Treasury portion calculated as remainder to eliminate rounding dust
- [x] **DIST-05**: Distribution occurs atomically within swap instruction (no accumulation)

### CPI Integration (CPI)

- [x] **CPI-01**: Tax Program owns swap_authority PDA with seeds `["swap_authority"]`
- [x] **CPI-02**: All swaps invoke AMM via CPI signed by swap_authority PDA
- [x] **CPI-03**: Slippage protection via minimum_output parameter passed to AMM
- [x] **CPI-04**: SOL pool swaps handle mixed SPL Token (WSOL) and Token-2022 (CRIME/FRAUD)
- [x] **CPI-05**: PROFIT pool swaps handle dual Token-2022 (both sides)
- [x] **CPI-06**: Transfer hook remaining_accounts passed through to AMM for T22 transfers

### Events (EVNT)

- [x] **EVNT-01**: TaxedSwap event emitted on successful SOL pool swaps
- [x] **EVNT-02**: UntaxedSwap event emitted on successful PROFIT pool swaps
- [x] **EVNT-03**: Events include user, pool_type, direction, amounts, tax breakdown

### Access Control (AUTH)

- [x] **AUTH-01**: swap_exempt only accepts Carnage PDA signer from Epoch Program
- [x] **AUTH-02**: swap_exempt rejects calls without valid Carnage PDA signature
- [x] **AUTH-03**: AMM verifies swap_authority PDA is owned by Tax Program (verification)
- [x] **AUTH-04**: User cannot bypass Tax Program by calling AMM directly (verification)

### Error Handling (ERR)

- [x] **ERR-01**: InvalidPoolType error when pool doesn't match instruction
- [x] **ERR-02**: SlippageExceeded error when output below minimum_output
- [x] **ERR-03**: InvalidEpochState error when EpochState account is invalid
- [x] **ERR-04**: UnauthorizedCarnageCall error when swap_exempt called without Carnage PDA
