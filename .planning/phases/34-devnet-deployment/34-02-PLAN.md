<task type="auto">
  <name>Task 1: Add priority fee to deploy.sh</name>
  <files>scripts/deploy/deploy.sh</files>
  <action>
  In `scripts/deploy/deploy.sh`, add `--with-compute-unit-price 1` to the `solana program deploy` command (around line 164). This adds a minimal priority fee (1 microlamport per compute unit) that helps transactions land reliably on devnet's Helius free tier (which rate-limits sendTransaction to 1/sec).

  Change the deploy command from:
  ```bash
  solana program deploy \
    "target/deploy/${name}.so" \
    --program-id "$keypair" \
    --keypair "$WALLET" \
    --url "$CLUSTER_URL"
  ```

  To:
  ```bash
  solana program deploy \
    "target/deploy/${name}.so" \
    --program-id "$keypair" \
    --keypair "$WALLET" \
    --url "$CLUSTER_URL" \
    --with-compute-unit-price 1
  ```

  **Why priority fee?** The Helius free tier rate-limits sendTransaction to 1/second. Program deployment sends hundreds of write-buffer transactions per program. Without priority fees, transactions compete with other devnet traffic and may expire before landing ("blockhash expired"). A 1 microlamport priority fee costs negligible SOL (~0.0003 SOL per program) but significantly improves transaction landing rate.
  </action>
  <verify>
  1. `grep "with-compute-unit-price" scripts/deploy/deploy.sh` finds the priority fee flag
  2. Syntax check: `bash -n scripts/deploy/deploy.sh` exits 0
  </verify>
  <done>
  deploy.sh includes `--with-compute-unit-price 1` on the `solana program deploy` command. Script syntax is valid.
  </done>

  **If deployment fails (e.g., "blockhash expired"):**
  1. Check for orphaned buffers: `solana program show --buffers -k keypairs/devnet-wallet.json --url "$CLUSTER_URL"`

  4. initialize.ts is idempotent -- it skips already-created accounts

  **If initialization fails:**
  1. Re-run just initialization: `npx tsx scripts/deploy/initialize.ts`
  2. The idempotent check-before-init design means it picks up where it left off
