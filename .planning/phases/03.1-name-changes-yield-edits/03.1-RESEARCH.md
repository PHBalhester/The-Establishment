## Summary

1. **Token/Pool Naming Overhaul**: Complete replacement of placeholder names (IPA, IPB, OP4) with thematic names (CRIME, FRAUD, PROFIT) across all 12 specification documents, the cross-reference matrices, and all audit tracking files.

The scope is well-defined: 595 occurrences of old token names across 11 docs, plus cascading changes to account names, vault names, PDA seeds, and concept inventory entries.

**Primary recommendation:** Execute as a systematic find-replace operation with manual verification, followed by a complete re-run of Phase 2-3 analysis to update all matrices and confirm no conflicts.

### Pattern 1: Systematic Find-Replace

**What:** Use grep to find all occurrences, then edit each file systematically
**When to use:** When making consistent terminology changes across many files
**Example:**
```bash
# Find all occurrences of "IPA" in docs
grep -rn "\bIPA\b" Docs/

# For each file, use Edit tool with replace_all=true for simple cases
# Use targeted Edit for context-sensitive replacements
```

### Pattern 2: Archive, Don't Delete

**What:** Move deprecated documentation to archive folder rather than deleting
**When to use:** When old documentation may have historical or reference value
**Example:**
```
Docs/Yield_System_Spec.md → Docs/archive/Yield_System_Spec_OLD.md
```
Add header to archived file:
```markdown
> **ARCHIVED:** This document describes the OLD yield model (passive yield to all OP4 holders).
> The current yield model is staking-based. See `Docs/New_Yield_System_Spec.md`.
```

### Anti-Patterns to Avoid

- **Partial replacement:** Don't leave IPA/IPB/OP4 in some places. The grep verification must show 0 hits.
- **Context-blind replacement:** Don't blindly replace "IPA" if it appears in a different context (though unlikely in this codebase).
- **Forgetting cascades:** Account names like `ipa_vault` must also change to `crime_vault`.
- **Skipping matrices:** The cross-reference matrices contain token names and must be updated.

## Don't Hand-Roll

Problems that look simple but have existing solutions:

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Mass text replacement | Custom scripts | Edit tool with replace_all | Built-in, atomic, auditable |
| Finding all occurrences | Manual search | Grep tool | Systematic, complete coverage |
| Tracking changes | Mental notes | Git commits per file/task | Recoverable checkpoints |

**Key insight:** The Edit tool with `replace_all=true` handles most cases. Only use targeted Edit when context-specific handling is needed (e.g., updating code comments vs. prose).

## Common Pitfalls

### Pitfall 1: Incomplete Replacement

**What goes wrong:** Old names remain in some files, causing inconsistency
**Why it happens:** Missing files, forgetting matrices, or partial grep coverage
**How to avoid:** Final verification with grep across entire project, not just Docs/
**Warning signs:** Any grep hit for IPA, IPB, or OP4 after completion

### Pitfall 2: Breaking Code Examples

**What goes wrong:** Rust code blocks have variable names that should change but break syntax highlighting or are copy-pasteable
**Why it happens:** Code examples use old token names in struct fields, enums, etc.
**How to avoid:** Review each code block carefully; update Token enum from `IPA/IPB` to `CRIME/FRAUD` consistently
**Warning signs:** Inconsistent naming between prose and code examples

### Pitfall 3: Missing Yield Model References

**What goes wrong:** Old yield mechanics mentioned in docs that don't obviously reference yield
**Why it happens:** Yield is mentioned in Overview, Epoch, Tax, Carnage specs - not just Yield spec
**How to avoid:** Search for "yield", "OP4 holders earn", "75%", "passive" across all docs
**Warning signs:** Documentation still implies holding OP4 earns yield without staking

### Pitfall 4: Cross-Reference Staleness

**What goes wrong:** Matrices and concept inventory still reference old names
**Why it happens:** Forgetting that Phase 3 artifacts contain token names
**How to avoid:** Include cross-reference files in scope explicitly
**Warning signs:** CONFLICTS.md or INDEX.md still mention IPA/IPB

### Pitfall 5: PDA Seed String Updates

**What goes wrong:** PDA derivations use string literals that may reference token names
**Why it happens:** Seeds like ["user_yield", user_pubkey] don't contain token names, but other seeds might
**How to avoid:** Review all seed derivations; most use generic names but verify
**Warning signs:** Code examples showing ["ipa_pool", ...] instead of ["crime_pool", ...]

## Yield Model Changes

### Old Model (Yield_System_Spec.md) - TO BE ARCHIVED

Key characteristics:
- Passive yield to ALL OP4 holders (no staking required)
- Checkpoint model tracking cumulative yield-per-OP4
- Auto-claim on every OP4 balance change (transfer hook integration)
- `UserYieldAccount` created on first OP4 purchase
- CPI integration: Tax Program calls yield program on every swap
- Circulating supply = Total - Pool Vaults

### New Model (New_Yield_System_Spec.md) - AUTHORITATIVE

Key characteristics:
- **Staking required** - only staked PROFIT earns yield
- Cumulative reward-per-token pattern (Synthetix/Quarry style)
- Separate `stake`, `unstake`, `claim` instructions
- `StakePool` (global) and `UserStake` (per-user) accounts
- **No CPI on swaps** - staking program is isolated
- Tax Program deposits 75% to escrow vault
- Epoch Program triggers cumulative update during VRF callback

### Documents Needing Yield Updates

These docs reference yield mechanics and need updates:

1. **DrFraudsworth_Overview.md** - Section "OP4 Yield"
   - OLD: "OP4 is the yield-bearing asset... Yield is distributed via Merkle claims"
   - NEW: Should reference staking requirement

2. **Tax_Pool_Logic_Spec.md** - Section 4 (Tax Distribution)
   - OLD: "75% → Yield Escrow Vault" (correct)
   - Needs: Update integration section to remove yield CPI on swaps

3. **Epoch_State_Machine_Spec.md** - VRF callback integration
   - Needs: Update cumulative update CPI to staking program

4. **Carnage_Fund_Spec.md** - Tax split reference
   - OK: Just references 75% split, no yield mechanics

5. **Protocol_Initialzation_and_Launch_Flow.md**
   - Needs: Update yield system initialization (Staking Program, not Yield Program)
