}

**Test cases (5 minimum, focus on security):**

1. `test_swap_exempt_direct_user_call_fails`
   - User creates their own keypair and tries to call swap_exempt
   - User signs transaction with their keypair (not Carnage PDA)
   - Verify: Transaction fails with ConstraintSeeds error
   - This proves: random users cannot call swap_exempt

2. `test_swap_exempt_wrong_pda_signer_fails`
   - Create a fake PDA with different seeds or different program
   - Try to pass it as carnage_authority
   - Verify: Transaction fails because PDA derivation doesn't match
   - This proves: only the real Carnage PDA from Epoch Program works

3. `test_swap_exempt_zero_amount_fails`
   - Construct transaction with valid accounts but amount_in = 0
   - Verify: Transaction fails with InsufficientInput error

4. `test_swap_exempt_invalid_direction_fails`
   - Construct transaction with direction = 2 (neither buy nor sell)
   - Verify: Transaction fails with InvalidDirection error (or similar)

5. `test_swap_exempt_authorized_carnage_succeeds` (marked #[ignore])
   - This test demonstrates what a happy path WOULD look like
   - Mark with #[ignore] and comment: "Requires mock Epoch Program - deferred to Phase v0.5+"
   - Include commented pseudocode showing:
     - Mock Epoch Program deployment
     - Carnage PDA signing via invoke_signed
     - Expected success outcome
   - This serves as documentation for future implementation

**Implementation notes:**

Since LiteSVM doesn't have a real Epoch Program, we CANNOT fully test the happy path in this phase. The seeds::program constraint requires the actual Epoch Program to exist and sign.

What we CAN test (and is MORE important for security):
- That unauthorized calls FAIL
- That the constraint is correctly configured

The security model is proven by:
1. Verifying direct calls fail (user cannot bypass)
2. Verifying wrong PDAs fail (other programs cannot bypass)
3. Building the #[ignore] test to document expected happy path behavior

This is the correct approach for testing cross-program PDA security before the calling program exists.
