```rust
//! swap_exempt: Tax-exempt bidirectional swap for Carnage Fund.

//! Supports both:
//! - BUY: SOL -> CRIME/FRAUD (direction = 0, AtoB)

//!
//! ARCHITECTURAL CONSTRAINT: This instruction adds CPI depth 1.
//! The full Carnage CPI chain is:

//!   -> AMM::swap_sol_pool (depth 2) -> Token-2022::transfer_checked (depth 3)
//!   -> Transfer Hook::execute (depth 4 -- SOLANA LIMIT)
//!
//! DO NOT add any CPI calls to this instruction path beyond AMM.
//!
//! Source: Carnage_Fund_Spec.md Section 2, 8.4, 16.1
```

**SwapExempt accounts struct:**
Required accounts (order matters for CPI):
1. `carnage_authority: Signer<'info>` - Carnage PDA from Epoch Program
   - Use seeds::program constraint: `seeds = [CARNAGE_SIGNER_SEED], bump, seeds::program = EPOCH_PROGRAM_ID`
   - This is the CRITICAL security constraint
2. `swap_authority: AccountInfo<'info>` - Tax Program's swap_authority PDA (signs AMM CPI)
   - Same as swap_sol_buy: `seeds = [SWAP_AUTHORITY_SEED], bump`
3. Pool accounts (same as swap_sol_buy): pool, pool_vault_a, pool_vault_b, mint_a, mint_b
4. Carnage token accounts: user_token_a, user_token_b (same field names, passed by Epoch Program)
5. Programs: amm_program, token_program_a, token_program_b, system_program

**DO NOT INCLUDE** (vs swap_sol_buy/swap_sol_sell):
- `user: Signer` - Carnage authority is the signer, not a user
- `staking_escrow`, `carnage_vault`, `treasury` - no tax distribution
- `epoch_state` - not reading tax rates

**handler function:**
```rust
pub fn handler<'info>(
    ctx: Context<'_, '_, 'info, 'info, SwapExempt<'info>>,
    amount_in: u64,
    direction: u8,  // 0 = AtoB (buy: SOL->Token), 1 = BtoA (sell: Token->SOL)
    is_crime: bool,
) -> Result<()>
```

Key implementation points:
- NO `minimum_output` parameter (per Carnage_Fund_Spec.md Section 9.3)
- Validate amount_in > 0 (same as swap_sol_buy line 69)
- Validate direction is 0 or 1
- Build AMM CPI exactly like swap_sol_buy/swap_sol_sell patterns
- Pass the direction byte directly to AMM (0=AtoB for buy, 1=BtoA for sell)
- Pass 0 for minimum_output in AMM CPI data (no slippage protection)
- NO tax calculation, NO distribution, NO event emission

**Wiring:**
