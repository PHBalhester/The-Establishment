<objective>
Build integration tests that verify transfer routing behavior in isolation using litesvm, exercising T22 transfers with hook account passthrough, SPL transfers, mixed pool routing, and defense-in-depth error cases.

Purpose: These tests prove the transfer behavior works correctly before swap instructions consume the helpers in Phases 11-13. Catching transfer routing bugs here prevents cascading failures in swap tests.

Output: `test_transfer_routing.rs` with tests covering XFER-01 through XFER-05 requirements and TEST-07, any needed Cargo.toml additions. All tests pass via `cargo test -p amm`.

Note on XFER-06 (PDA-signed vault-to-user): The transfer helpers SUPPORT PDA signing via the `signer_seeds` parameter (built in Plan 10-01). However, testing PDA-signed vault-to-user transfers requires an AMM instruction that calls the helpers from within the BPF runtime (invoke_signed). This cannot be done with raw token program instructions from the test client. The actual PDA-signed vault-to-user flow will be fully tested in Phase 11 when swap instructions exist. Phase 10 confirms the helpers accept signer_seeds and that pool PDA authority is correctly configured (proven by pool initialization tests).
</objective>
