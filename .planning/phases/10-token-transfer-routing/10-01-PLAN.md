  <name>Task 2: Create transfer routing helpers and wire module</name>
  <files>programs/amm/src/helpers/transfers.rs, programs/amm/src/helpers/mod.rs</files>
  <action>
Create `programs/amm/src/helpers/transfers.rs` with two public functions per CONTEXT.md and RESEARCH.md decisions:

**`transfer_t22_checked()`** -- Token-2022 transfer with hook account passthrough:
- Parameters: `token_program: &AccountInfo`, `from: &AccountInfo`, `mint: &AccountInfo`, `to: &AccountInfo`, `authority: &AccountInfo`, `amount: u64`, `decimals: u8`, `signer_seeds: &[&[&[u8]]]`, `hook_accounts: &[AccountInfo]`
- All parameters use lifetime `'info`
- Defense-in-depth: `require!(*token_program.key == anchor_spl::token_2022::ID, AmmError::InvalidTokenProgram)`
- Defense-in-depth: `require!(amount > 0, AmmError::ZeroAmount)`
- Build `TransferChecked` CPI accounts from parameters
- If `signer_seeds` is empty, use `CpiContext::new()`, otherwise `CpiContext::new_with_signer()`
- Forward hook accounts via `.with_remaining_accounts(hook_accounts.to_vec())` if non-empty
- Call `token_interface::transfer_checked(cpi_ctx, amount, decimals)`
- Return `Result<()>`

**`transfer_spl()`** -- Standard SPL Token transfer (no hooks):
- Parameters: Same as above MINUS `hook_accounts` (SPL Token has no hooks)
- Defense-in-depth: `require!(*token_program.key == anchor_spl::token::ID, AmmError::InvalidTokenProgram)`
- Defense-in-depth: `require!(amount > 0, AmmError::ZeroAmount)`
- Same CPI pattern but NO `with_remaining_accounts` call
- Call `token_interface::transfer_checked(cpi_ctx, amount, decimals)`

**Key implementation details from RESEARCH.md:**
- Use `anchor_spl::token_interface::{self, TransferChecked}` for CPI types
- Use `anchor_lang::prelude::*` for Result, require!, AccountInfo, CpiContext
- Import `crate::errors::AmmError` for error variants
- Both functions use `transfer_checked` (never plain `transfer`) -- this is critical for T22 hook invocation
- The `signer_seeds` parameter handles both directions: empty for user-signed (user-to-vault), populated for PDA-signed (vault-to-user)
- Document each function with rustdoc explaining when to use it, what it validates, and the security rationale

**Update `programs/amm/src/helpers/mod.rs`:**
Add `pub mod transfers;` after the existing `pub mod math;` line.

**What NOT to do:**
- Do NOT create a single routing function that dispatches based on token program -- CONTEXT.md locks separate explicit functions
- Do NOT hardcode any hook program ID -- the AMM is decoupled from specific hook programs
- Do NOT use plain `transfer` (non-checked) -- this silently bypasses T22 hooks (CRITICAL security)
- Do NOT read hook extension data from the mint on-chain -- caller pre-resolves off-chain per CONTEXT.md
