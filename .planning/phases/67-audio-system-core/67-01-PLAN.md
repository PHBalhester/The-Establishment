<task type="auto">
  <name>Task 2: Build AudioManager singleton class</name>
  <files>app/lib/audio-manager.ts</files>
  <action>
    Create `app/lib/audio-manager.ts` — a plain TypeScript class (NO React, NO 'use client') that implements the full audio engine. Export both the class and a pre-created singleton instance.

    **Architecture: HTMLAudioElement + MediaElementAudioSourceNode (NOT AudioBufferSourceNode)**
    Research confirmed that decodeAudioData would expand ~6.8MB MP3 into ~118MB raw PCM. Use HTMLAudioElement streaming with MediaElementAudioSourceNode for Web Audio API gain control. Near-zero JS memory overhead.

    **Type definition at top of file:**
    ```typescript
    interface AudioSlot {
      audio: HTMLAudioElement;
      source: MediaElementAudioSourceNode;
      gain: GainNode;
    }
    ```

    **Class: AudioManager**

    Private state:
    - `ctx: AudioContext | null` — created lazily on init()
    - `masterGain: GainNode | null` — master volume node
    - `slots: [AudioSlot | null, AudioSlot | null]` — two permanent crossfade slots
    - `activeSlot: 0 | 1` — which slot is currently playing
    - `playlist: string[]` — current shuffled track order
    - `playlistIndex: number` — current position in playlist
    - `lastTrack: string | null` — last played track URL (for no-repeat)
    - `isPlaying: boolean` — playback state
    - `volume: number` — 0.0 to 1.0 (NOT 0-100, that's SettingsProvider's domain)
    - `isMuted: boolean`
    - `wasPlayingBeforeHidden: boolean` — for tab visibility
    - `crossfadeDuration: number` — 2.5 seconds
    - `handleVisibility: (() => void) | null` — stored listener ref for cleanup

    Track list constant:
    ```typescript
    const TRACKS = ['/music/music-1.mp3', '/music/music-2.mp3', '/music/music-3.mp3'];
    ```

    **Public methods:**

    1. `init(): void`
       - Guard: if `this.ctx` already exists, just call `this.ctx.resume()` and return
       - Create `new AudioContext()`
       - If state is 'suspended', call `ctx.resume()`
       - Call `unlockiOS()` (private method)
       - Create `masterGain` connected to `ctx.destination`
       - Set `masterGain` gain to current volume (respecting muted state)
       - Create both slots via `createSlot()` (private)
       - Set up tab visibility handler via `setupVisibilityHandler()` (private)
       - Shuffle playlist and start playback if not muted

    2. `play(): void`
       - Guard: if no ctx or already playing, return
       - Resume AudioContext if suspended
       - Set `isPlaying = true`
       - Start playback: load current track into active slot, fade in

    3. `pause(): void`
       - Set `isPlaying = false`
       - Fade out active slot gain to 0 over 50ms
       - Do NOT suspend AudioContext (keep it warm for quick resume)

    4. `setVolume(vol: number): void` — vol is 0.0 to 1.0
       - Store `this.volume = vol`
       - If ctx and masterGain exist, do smooth 50ms ramp:
         ```
         masterGain.gain.cancelScheduledValues(now)
         masterGain.gain.setValueAtTime(masterGain.gain.value, now)
         masterGain.gain.linearRampToValueAtTime(vol, now + 0.05)
         ```
       - Do NOT set gain.value directly (causes clicks — Pitfall 8)

    5. `setMuted(muted: boolean): void`
       - Store `this.isMuted = muted`
       - If muted: smooth ramp masterGain to 0 over 50ms
       - If unmuted: smooth ramp masterGain to `this.volume` over 50ms
       - If unmuted and not playing, call `play()`

    6. `destroy(): void`
       - Pause both slots' Audio elements
       - Disconnect both slots' source and gain nodes
       - Remove visibility event listener
       - Close AudioContext
       - Null out all references

    **Private methods:**

    7. `createSlot(): AudioSlot`
       - Create `new Audio()` with `preload = 'auto'`
       - Do NOT set `crossOrigin` (same-origin files, setting it can cause CORS preflight issues)
       - Create MediaElementAudioSourceNode from audio element
       - Create GainNode, set gain to 0
       - Connect: source -> gain -> masterGain
       - Attach `ended` event listener on audio element that calls `onTrackEnded()`
       - Return `{ audio, source, gain }`

    8. `unlockiOS(): void`
       - Create a 1-sample silent buffer: `ctx.createBuffer(1, 1, 22050)`
       - Create BufferSourceNode, connect to destination, start(0)
       - The source auto-completes after 1 sample (no cleanup needed for 1-sample buffer)
       - Also create a temporary `new Audio()` and call `.play().catch(() => {})` for mute switch bypass
       - This is the feross/unmute-ios-audio pattern (Research: Pitfall 5)

    9. `shufflePlaylist(): string[]`
       - Fisher-Yates shuffle on copy of TRACKS array
       - If first track equals `this.lastTrack` and array length > 1, swap with random other position
       - Return shuffled array

    10. `onTrackEnded(): void`
        - If not playing, return
        - Increment `playlistIndex`
        - If past end of playlist, reshuffle and reset index to 0
        - Store current track URL as `lastTrack`
        - Crossfade to next track

    11. `crossfadeTo(url: string): void`
        - Determine next slot (opposite of activeSlot)
        - Set next slot's audio.src = url
        - Call next slot's audio.play()
        - Get `now = ctx.currentTime`
        - Fade in next slot: `setValueAtTime(0, now)` + `linearRampToValueAtTime(1, now + crossfadeDuration)`
        - Fade out active slot: `cancelScheduledValues(now)` + `setValueAtTime(current, now)` + `linearRampToValueAtTime(0, now + crossfadeDuration)`
        - Swap `activeSlot` to next slot

    12. `playTrack(url: string): void`
        - Load URL into active slot's audio element
        - Play it
        - Fade in active slot gain from 0 to 1 over crossfadeDuration

    13. `setupVisibilityHandler(): void`
        - Create handler function:
          - If `document.hidden`: store `wasPlayingBeforeHidden = isPlaying`, suspend ctx
          - If visible: if wasPlayingBeforeHidden, resume ctx
        - Add event listener for 'visibilitychange'
        - Store handler reference for cleanup in destroy()

    **Singleton export:**
    ```typescript
    export const audioManager = new AudioManager();
    ```

    **CRITICAL anti-patterns to avoid (from RESEARCH.md):**
    - Do NOT create AudioContext at module scope (only in init(), which is called from user gesture)
    - Do NOT use decodeAudioData for these tracks (118MB memory)
    - Do NOT set gain.value directly (use scheduling methods)
    - Do NOT use exponentialRampToValueAtTime to fade to zero (math error)
    - Do NOT create new Audio elements per track play — reuse 2 slots by changing .src
    - Do NOT set crossOrigin on Audio elements for same-origin files

    **Important edge case:** The `ended` event on HTMLAudioElement fires when playback reaches the end. For crossfade, we want to start the crossfade BEFORE the track ends (so there's overlap). Add a `timeupdate` listener that checks if `audio.currentTime >= audio.duration - crossfadeDuration` and triggers crossfade early. Use a boolean flag `crossfadeStarted` per play cycle to prevent multiple triggers. Reset the flag when starting a new track.
