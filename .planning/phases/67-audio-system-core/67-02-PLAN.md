    **File 1: `app/providers/AudioProvider.tsx`**

    Create a 'use client' React context provider that wraps AudioManager and syncs with SettingsProvider.

    ```typescript
    'use client';
    ```

    **Context value interface:**
    ```typescript
    interface AudioContextValue {
      /** Whether audio system has been initialized (gesture gate passed) */
      isInitialized: boolean;
      /** Whether music is currently playing */
      isPlaying: boolean;
      /** Initialize audio system — call from user gesture handler */
      initAudio: () => void;
      /** Start music playback */
      play: () => void;
      /** Pause music playback */
      pause: () => void;
    }
    ```

    NOTE: The context name `AudioContext` will shadow the Web Audio API's `AudioContext`. Name the React context `AudioCtx` internally to avoid confusion:
    ```typescript
    const AudioCtx = createContext<AudioContextValue | null>(null);
    ```
    Export `AudioCtx` for the useAudio hook.

    **Provider component: `AudioProvider`**

    Props: `{ children: ReactNode }`

    Implementation:
    1. Import `audioManager` singleton from `@/lib/audio-manager`
    2. Import `useSettings` from `@/hooks/useSettings`
    3. Track `isInitialized` with `useState(false)` — flipped to true after init()
    4. Track `isPlaying` with `useState(false)` — mirrors AudioManager state

    5. `initAudio` callback (wrapped in useCallback):
       - Call `audioManager.init()`
       - Set `isInitialized(true)`
       - Read current settings — if NOT muted, call `audioManager.play()` and `setIsPlaying(true)`
       - If muted, do NOT play (respect user preference)
       - Apply current volume: `audioManager.setVolume(settings.muted ? 0 : settings.volume / 100)`
       - Apply muted state: `audioManager.setMuted(settings.muted)`

    6. **Settings sync via useEffect:**
       - Dependencies: `[settings.muted, settings.volume, isInitialized]`
       - Guard: if not initialized, return
       - Call `audioManager.setMuted(settings.muted)`
       - Call `audioManager.setVolume(settings.muted ? 0 : settings.volume / 100)`
       - If unmuted and not playing: call `audioManager.play()`, `setIsPlaying(true)`
       - If muted and playing: `setIsPlaying(false)` (AudioManager.setMuted already handles fade-out)
       - NOTE: volume is 0-100 in Settings, 0.0-1.0 in AudioManager. Divide by 100.

    7. `play` callback:
       - `audioManager.play()`
       - `setIsPlaying(true)`

    8. `pause` callback:
       - `audioManager.pause()`
       - `setIsPlaying(false)`

    9. Cleanup useEffect (empty deps, return cleanup function):
       - On unmount: `audioManager.destroy()`
       - This handles page navigation / hot reload cleanup

    10. Memoize context value with useMemo to prevent unnecessary re-renders:
        ```typescript
        const value = useMemo(() => ({
          isInitialized, isPlaying, initAudio, play, pause
        }), [isInitialized, isPlaying, initAudio, play, pause]);
        ```

    Return: `<AudioCtx.Provider value={value}>{children}</AudioCtx.Provider>`

    **CRITICAL: Do NOT duplicate localStorage persistence.** SettingsProvider owns muted/volume persistence. AudioProvider only READS from SettingsProvider and pushes values to AudioManager. No localStorage access in AudioProvider.

    **File 2: `app/hooks/useAudio.ts`**

    Create a 'use client' convenience hook, following the exact pattern of `useSettings.ts`:

    ```typescript
    'use client';

    import { useContext } from 'react';
    import { AudioCtx } from '@/providers/AudioProvider';
    import type { AudioContextValue } from '@/providers/AudioProvider';

    export function useAudio(): AudioContextValue {
      const context = useContext(AudioCtx);
      if (!context) {
        throw new Error(
          'useAudio must be used within an AudioProvider. ' +
          'Wrap your component tree with <AudioProvider> in providers.tsx.',
        );
      }
      return context;
    }
    ```

    Also export the `AudioContextValue` type from `AudioProvider.tsx` so useAudio.ts can import it.

    **File 2: `app/components/onboarding/SplashScreen.tsx`**

    Wire the splash screen button click to initialize the audio system. The click is a user gesture — this is the only reliable place to create an AudioContext.

    Changes:
    1. Add import: `import { useAudio } from '@/hooks/useAudio';`
    2. Inside the component, call: `const { initAudio } = useAudio();`
    3. In the `handleEnter` callback, add `initAudio()` as the FIRST line (before setting phase to 'thanking'):
       ```typescript
       const handleEnter = useCallback(() => {
         if (phase !== 'ready') return;
         initAudio(); // Gesture-gated AudioContext creation + music start
         setPhase('thanking');
         // ... rest of existing setTimeout logic unchanged
       }, [phase, initAudio]);
       ```

    The `initAudio()` call is synchronous — it creates the AudioContext, unlocks iOS Safari, and starts music. The browser processes this within the click event's activation context.

    **Why initAudio must be FIRST:** Browsers have a narrow window during the click event where AudioContext creation is allowed. Any async work (setTimeout, setState flush) could close this window. Call initAudio synchronously at the top of the handler.

    Do NOT add `initAudio` to the useCallback deps if it causes re-render issues — but it should be stable (memoized in AudioProvider). If the linter complains about missing deps, add it.
