- All routes accessible normally
- `/launch` remains accessible (shows completed state -- handled by page.tsx)

Use the `config.matcher` negative lookahead pattern to exclude static assets at the matcher level (more efficient than checking in the function body).

Export the function as `proxy` (NOT `middleware` -- Next.js 16 convention per RESEARCH.md).

IMPORTANT: The `NEXT_PUBLIC_SITE_MODE` env var works in proxy because proxy runs server-side, AND the NEXT_PUBLIC_ prefix makes it available client-side too (needed for launch page graduated-state display).
