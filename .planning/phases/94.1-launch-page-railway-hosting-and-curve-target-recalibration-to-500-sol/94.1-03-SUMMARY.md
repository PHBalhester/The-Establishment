## Decisions Made

1. **MATHEMATICAL_FULL_CURVE_SOL computed dynamically** - Instead of hardcoding 500_250_000_000, compute from `TOTAL_FOR_SALE * (P_START + P_END) / (2 * TOKEN_DECIMAL_FACTOR)`. This ensures tests work with both devnet (5 SOL) and localnet/mainnet (500 SOL) feature flags.

2. **Boundary test reserve updated** - The `test_reversed_mint_floor_calculation_identical` test uses pool reserves to verify floor price calculation. Since it represents post-graduation pool state, updated from 1000 SOL to 500 SOL.

3. **LiteSVM injection tests left unchanged** - refund_clock_test and dual_curve_test inject arbitrary curve state values (1000 SOL raised). These test instruction logic, not curve math -- they work regardless of what constants say.
