  <name>Task 1: Refactor graduate.ts for dynamic pool seeding</name>
  <files>scripts/graduation/graduate.ts</files>
  <action>
Refactor `scripts/graduation/graduate.ts` to dynamically determine the SOL amount for pool seeding:

**Current problem:** `GRADUATION_POOL_SEED_SOL` defaults to `1_000 * LAMPORTS_PER_SOL` (1000 SOL). After recalibration to 500 SOL, this default is wrong. Even changing the default to 500 would be fragile -- the actual amount should come from the vault.

**Fix approach (from RESEARCH.md):**
1. Track admin wallet SOL balance before and after each `withdrawGraduatedSol` call
2. The balance delta IS the pool seed amount for that curve
3. Store the withdrawn amount per curve (CRIME and FRAUD get separate amounts)
4. Pass the actual withdrawn amount to `createAmmPool` instead of `GRADUATION_POOL_SEED_SOL`

**Specific changes:**

1. Remove the `GRADUATION_POOL_SEED_SOL` constant entirely (or keep it only as a sanity-check lower bound).

2. Modify `withdrawSol()` to return the withdrawn amount:
   - Read `connection.getBalance(authority.publicKey)` before the withdrawal TX
   - Execute withdrawal
   - Read balance after
   - Return `{ sig, withdrawn: balanceAfter - balanceBefore }`
   - Note: TX fee (~5000 lamports) is negligible vs 500 SOL and is acceptable

3. In the main execution loop, capture the withdrawn amounts:
   - `crimeWithdrawn` from step "withdraw_crime_sol"
   - `fraudWithdrawn` from step "withdraw_fraud_sol"
   - Store these in the graduation state JSON for checkpoint/resume

4. Modify `createAmmPool()` to accept `poolSeedSol: number` parameter instead of using the global constant.

5. In the pool creation steps, pass the corresponding withdrawn amount.

6. Keep `GRADUATION_POOL_SEED_TOKEN` (290M tokens) unchanged -- this is the fixed unsold allocation per CONTEXT.md decision.

7. Update the banner output that logs "Pool SOL seed: X SOL per pool" to show "dynamic (from vault withdrawal)".

8. For checkpoint/resume: if the script resumes after withdrawal but before pool creation, the withdrawn amounts must be recoverable. Store them in `graduation-state.json` alongside the step data. Add `crimeWithdrawn` and `fraudWithdrawn` fields to `GraduationState`.

**Keep the env override capability:** `SOL_POOL_SEED_SOL_OVERRIDE` should still work as an emergency override, but the default behavior is now dynamic. Logic: `if env override set, use it; else use withdrawn amount`.

Update all comments referencing "~1000 SOL" to "~500 SOL" throughout the file (header comments, step descriptions, etc).
