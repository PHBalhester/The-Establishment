**Gathered:** 2026-03-13
**Status:** Ready for planning

<domain>
## Phase Boundary

Two urgent deliverables:
1. **Site mode toggle** — Next.js middleware that locks fraudsworth.fun to `/launch` during bonding curve period, then switches to full site after graduation. Controlled by Railway env var.
2. **Curve target recalibration** — Change mainnet TARGET_SOL from 1000 to 500 SOL per curve, cascading through on-chain constants, client-side constants, graduation script, documentation, slippage analysis, and all test suites. Devnet stays at 5 SOL.

Also includes: fixing graduation script to read actual vault balance instead of hardcoded SOL amount, running full test suite, and fixing any CI/GitHub Actions failures.

</domain>

<decisions>
## Implementation Decisions

### Site Mode Toggle (Railway Hosting)
- `NEXT_PUBLIC_SITE_MODE` env var: `launch` or `live`
- In `launch` mode: ALL routes redirect to `/launch` (Next.js middleware redirect)
- In `live` mode: full site accessible, `/launch` shows "Curves graduated — trading is live!" completed state with link to main app
- Toggle by changing env var in Railway dashboard + redeploy — no code deploy needed
- fraudsworth.fun DNS needs to be configured (CNAME/A record → Railway custom domain)

### Curve Target Recalibration
- Mainnet TARGET_SOL: 1000 SOL → 500 SOL per curve
- Devnet TARGET_SOL: stays at 5 SOL (unchanged)
- Localnet TARGET_SOL: tracks mainnet → also changes to 500 SOL
- Preserve curve shape ratio (~3.83x): P_START=450, P_END=1725 for mainnet/localnet
- MIN_PURCHASE_SOL: stays at 0.05 SOL mainnet, 0.001 SOL devnet (unchanged)
- All dependent constants (TOTAL_FOR_SALE, MAX_TOKENS_PER_WALLET) remain unchanged

### Graduation Script Fix
- graduate.ts currently uses hardcoded `SOL_POOL_SEED_SOL_OVERRIDE` env var for pool seeding amount
- Fix: after `withdraw_graduated_sol`, read actual SOL balance delta and seed pool with exactly that amount
- No more hardcoded pool seed SOL amount — pool gets seeded with whatever the curve actually raised
- Token seed amount (290M) remains fixed (it's the unsold allocation, not variable)

### Pool Seeding
- Devnet SOL_POOL_SEED_SOL_OVERRIDE stays at 2.5 SOL (devnet target unchanged)
- 290M token allocation per pool stays fixed across all environments
- Mainnet pool seed is now dynamic (vault balance after graduation)

### Slippage Analysis
- Recalculate Docs/liquidity-slippage-analysis.md for 500 SOL pools
- Half liquidity = roughly double slippage on same-size trades — users need accurate numbers

### Documentation Cascade

### Test Suite
