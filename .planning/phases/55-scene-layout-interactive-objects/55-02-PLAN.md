**The scene compositor structure:**

```tsx
'use client';

import { FactoryBackground } from '@/components/scene/FactoryBackground';
import { SceneStation } from '@/components/scene/SceneStation';
import { STATIONS } from '@/components/scene/scene-data';

export default function Home() {
  return (
    <>
      {/* Desktop scene: visible at lg (1024px) and above */}
      <main className="hidden lg:block">
        <FactoryBackground>
          {/* Safe zone: maintains 1.81:1 aspect ratio within viewport */}
          <div className="absolute inset-0 flex items-center justify-center pointer-events-none">
            <div
              className="relative w-full h-full pointer-events-auto"
              style={{
                maxWidth: 'calc(100vh * 1.81)',
                maxHeight: 'calc(100vw / 1.81)',
              }}
            >
              {/* 6 stations rendered in tab-priority order */}
              {STATIONS.map((station) => (
                <SceneStation key={station.stationId} station={station} />
              ))}
            </div>
          </div>
        </FactoryBackground>
      </main>

      {/* Below 1024px: simple fallback message (Phase 59 replaces with mobile nav) */}
      <main className="lg:hidden min-h-screen flex items-center justify-center bg-factory-bg text-factory-text p-6">
        <div className="text-center max-w-md">
          <h1 className="text-2xl font-heading text-factory-accent mb-4">
            Dr. Fraudsworth&apos;s Finance Factory
          </h1>
          <p className="text-factory-text-secondary mb-2">
            The factory floor is best experienced on a larger screen.
          </p>
          <p className="text-factory-text-muted text-sm">
            Please visit on a desktop browser (1024px or wider) for the full interactive experience.
          </p>
        </div>
      </main>
    </>
  );
}
```

**Key implementation details:**

1. **Safe zone container:** The inner div uses `maxWidth: calc(100vh * 1.81)` and `maxHeight: calc(100vw / 1.81)` to maintain the original scene's 1.81:1 aspect ratio (5568/3072). On ultrawide monitors, the safe zone constrains width so overlays don't spread too far apart. On tall monitors, it constrains height. The background bleeds past the safe zone (cover mode in FactoryBackground) so there is never letterboxing.

2. **Pointer events:** The safe zone centering container uses `pointer-events-none` so clicks on the background outside the safe zone don't interfere. The inner safe zone div uses `pointer-events-auto` to re-enable clicks on station buttons.

3. **Tab order via DOM order:** The STATIONS array from scene-data.ts is already sorted by tab priority (wallet -> swap -> carnage -> staking -> docs -> settings). By mapping over this array in order, the DOM order matches the desired tab sequence. No positive tabIndex values needed.

4. **Below-1024px fallback:** Uses Tailwind `hidden lg:block` for the scene and `lg:hidden` for the fallback. The `lg:` breakpoint in Tailwind v4 defaults to `1024px`. The fallback is a simple centered message in factory theme styling. Phase 59 will replace this with a proper mobile navigation.

5. **Remove ALL old content:** Delete the entire old page content including:
   - `ModalDemoTrigger` component and `DEMO_STATIONS` constant (Phase 54 demo -- no longer needed)
   - Header with WalletButton (the scene IS the full-viewport experience; wallet is a station)
   - DashboardGrid import and rendering
   - Link import and swap nav link
   - All old imports that are no longer needed

6. **Keep `"use client"`** since SceneStation uses hooks (useModal) and the scene needs client-side interactivity.

7. **FactoryBackground already has `role="img"` and `aria-label`** from Phase 53 -- verify this is preserved. The background component wraps the scene and provides the viewport container with the factory background image.

**Optionally add to `app/app/globals.css`** if any scene-specific styles are needed that cannot be expressed as Tailwind utilities. Likely candidates:
