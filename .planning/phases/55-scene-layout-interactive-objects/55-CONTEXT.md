# Phase 55: Scene Layout + Interactive Objects - Context

**Gathered:** 2026-02-22

<domain>
## Phase Boundary

Assemble the factory scene as a full-viewport interactive experience. Position 6 clickable machine overlays over the steampunk background using pre-defined asset coordinates, wire hover glow, click feedback, tooltips, and keyboard navigation. The scene IS the navigation layer -- clicking machines opens modals (from Phase 54). Mobile navigation is Phase 59. Ambient animations are Phase 58.

</domain>

<decisions>
## Implementation Decisions

### Object Placement & Hierarchy
- All overlay positions and sizes come directly from the asset PNGs -- SCENE_DATA in `image-data.ts` is the source of truth for coordinates
- No manual repositioning or resizing of overlays; they render exactly where the image optimizer placed them
- Swap-station has no asset yet (`available: false`) -- render a CSS-only steampunk machine silhouette as placeholder until the real asset is provided
- Bounding boxes may overlap but visible content does not -- no special overlap handling needed
- Clickable area should be visible pixels only (not the entire bounding box) -- use CSS techniques to exclude transparent regions from pointer events

### Interaction Feel
- Hover glow: warm brass/amber color, medium intensity -- clear glow halo with noticeable brightness, obviously interactive but not dramatic
- CSS approach: `drop-shadow` with amber color + `brightness` filter boost on hover
- Click feedback: quick mechanical press -- fast scale-down (100-150ms), snappy and industrial
- Tooltips: simple station name label only (e.g., "Swap Machine", "Carnage Cauldron") -- no descriptions
- Cursor changes to pointer on hover (standard)

### Scene Scaling Behavior
- **Cover with safe zone**: background fills the viewport (cover mode, cropping edges), interactive objects positioned within an inner safe area that adjusts to never clip
- No letterboxing -- full-bleed visuals at all times
- Safe zone ensures all 6 stations remain reachable regardless of viewport aspect ratio (1280px to 3840px desktop range)

### Breakpoint Behavior
- Below 1024px: scene does not render (mobile navigation in Phase 59 handles this)
- Exact breakpoint transition behavior: Claude's discretion (hard cutoff vs graceful)

### Aspect Ratio Handling
- Claude's discretion on whether to maintain the 1.81:1 ratio or fill 100vh on extreme viewports
- Priority: preserve the art and keep objects accessible

### Navigation Order & Discovery
- **Tab order follows priority flow**: Connect Wallet -> Swap Machine -> Carnage Cauldron -> Rewards Vat -> Documentation Table -> Settings
- Most important user actions (connect, swap) reached first via keyboard
- Labels appear on hover/focus only -- clean scene at rest, tooltip appears on interaction
- No idle animations or discovery hints for this phase -- cursor change to pointer is sufficient discoverability signal
- Onboarding (Phase 60) handles first-time user orientation

### Focus Indicators
- Keyboard focus indicator uses the same warm brass glow as hover -- no distinct focus ring
- Focus and hover states are visually identical

### Claude's Discretion
- Breakpoint transition at 1024px (hard cutoff vs graceful fallback)
- Aspect ratio handling on extreme viewports (maintain ratio vs stretch)
- CSS technique for visible-pixels-only click areas (clip-path, alpha threshold, or manual shapes)
- Tooltip positioning (above vs below object)
- Tooltip appear/disappear timing
- Swap-station placeholder silhouette design

</decisions>
