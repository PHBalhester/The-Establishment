    **2. Z-Index Layering System (@theme block):**
    Use `--z-index-*` namespace. Values must be ascending with room for intermediate values.

    - `--z-index-background`: 0
    - `--z-index-overlays`: 10
    - `--z-index-hover`: 20
    - `--z-index-tooltip`: 30
    - `--z-index-modal-backdrop`: 40
    - `--z-index-modal`: 50
    - `--z-index-spinner`: 60

    This generates: `z-background`, `z-overlays`, `z-hover`, `z-tooltip`, `z-modal-backdrop`, `z-modal`, `z-spinner`

    **3. Custom Animation Keyframes (@theme block):**
    Define animations that will be used across phases:

    - `--animate-fade-in`: fade-in 0.6s ease-out (scene progressive reveal)
    - `--animate-gear-spin`: gear-spin 3s linear infinite (loading spinner, settings gears later)
    - `--animate-pulse-glow`: pulse-glow 2s ease-in-out infinite (subtle gaslight pulse)

    With corresponding `@keyframes` inside the @theme block:
    - `fade-in`: opacity 0 -> 1
    - `gear-spin`: rotate 0deg -> 360deg
    - `pulse-glow`: opacity oscillation (0.6 -> 1 -> 0.6)

    **4. Typography Scale (@theme block):**
    Use `--text-*` namespace for named font sizes:

    - `--text-display`: 3rem (page titles, scene headers -- 48px)
    - `--text-heading`: 2rem (modal headers -- 32px)
    - `--text-subheading`: 1.25rem (section titles -- 20px)
    - `--text-body`: 1rem (default body -- 16px)
    - `--text-detail`: 0.875rem (labels, captions -- 14px)
    - `--text-micro`: 0.75rem (fine print -- 12px)

    These generate `text-display`, `text-heading`, `text-subheading`, `text-body`, `text-detail`, `text-micro` utilities. NOTE: these may collide with Tailwind's default font-size scale -- if so, use `--text-size-*` namespace or `--font-size-*` to avoid conflicts.

    **5. Font Variable Bridges (@theme inline block):**
    SEPARATE `@theme inline` block (NOT inside the main @theme block):

    ```css
    @theme inline {
      --font-heading: var(--font-cinzel);
      --font-mono: var(--font-ibm-plex-mono);
    }
    ```

    The `inline` keyword is CRITICAL -- it inlines the variable reference so `.font-heading { font-family: var(--font-cinzel); }` works directly. Without `inline`, Tailwind wraps it in another layer of var() that may not resolve.

    **6. Reduced Motion:**
    Add a `@media (prefers-reduced-motion: reduce)` rule that disables all custom animations:
    ```css
    @media (prefers-reduced-motion: reduce) {
      *, *::before, *::after {
        animation-duration: 0.01ms !important;
        animation-iteration-count: 1 !important;
        transition-duration: 0.01ms !important;
      }
    }
    ```

  <action>
    **Create `app/app/fonts.ts`:**

    Define two Google Font families using `next/font/google`:

    ```typescript
    import { Cinzel } from 'next/font/google';
    import { IBM_Plex_Mono } from 'next/font/google';

    // Cinzel: Variable font (400-900), Roman inscriptional letterforms
    // Used for headings, titles, nameplate-style labels
    // Steampunk aesthetic: evokes engraved brass nameplates
    export const cinzel = Cinzel({
      subsets: ['latin'],
      display: 'swap',
      variable: '--font-cinzel',
    });

    // IBM Plex Mono: Industrial heritage, built-in tabular numerals
    // Used for financial data (prices, balances, percentages)
    // Steampunk aesthetic: instrument readout / gauge display feel
    export const ibmPlexMono = IBM_Plex_Mono({
      subsets: ['latin'],
      display: 'swap',
      variable: '--font-ibm-plex-mono',
      weight: ['400', '500', '700'],  // NOT a variable font, specify exact weights
    });
    ```

    **Why these fonts:**
    - Cinzel: Roman inscriptional origin matches "engraved nameplates" from context. Variable font = efficient loading. Designed for all-caps display.
    - IBM Plex Mono: IBM industrial heritage matches steampunk instrument aesthetic. Built-in tabular numerals for perfect column alignment of financial data.
    - Body text: Stays on system font stack (Tailwind default `font-sans`) for maximum readability per CONTEXT.md decision.

    **Update `app/app/layout.tsx`:**

    1. Import `cinzel` and `ibmPlexMono` from `./fonts`
    2. Apply font CSS variables to `<html>` element via className: `${cinzel.variable} ${ibmPlexMono.variable}`
    3. Apply steampunk base classes to `<body>`: `bg-factory-bg text-factory-text antialiased`
    4. Keep ALL existing structure: dynamic export, metadata, Providers wrapper

    The updated layout should look like:
    ```tsx
    import { cinzel, ibmPlexMono } from './fonts';
    // ... existing imports

    export default function RootLayout({ children }) {
      return (
        <html lang="en" className={`${cinzel.variable} ${ibmPlexMono.variable}`}>
          <body className="antialiased bg-factory-bg text-factory-text">
            <Providers>{children}</Providers>
          </body>
        </html>
      );
    }
    ```

    **Do NOT change:**
    - The `dynamic = "force-dynamic"` export
    - The metadata object
    - The `import "./globals.css"` line
    - The Providers wrapper structure
    - The React/Metadata type imports

    **Why this wiring works:**
    - next/font/google generates CSS variables (`--font-cinzel`, `--font-ibm-plex-mono`) at build time
    - Applying them to `<html>` makes them available to all descendants
    - `@theme inline` in globals.css maps these CSS vars to Tailwind's `--font-heading` and `--font-mono`
    - Result: `font-heading` class on any element applies Cinzel, `font-mono` applies IBM Plex Mono
