Purpose: Image optimization is the critical path for Phase 53 -- no scene work can happen until the 19.5MB PNGs are cropped, compressed, and have their position/dimension metadata extracted. This plan produces all optimized assets AND the metadata file that downstream components consume.

Output: Optimized WebP images in `app/public/scene/`, image metadata module at `app/lib/image-data.ts`, and updated `next.config.ts` with image optimization config.
</objective>

  <action>
    Create `scripts/optimize-images.mjs` -- a Node.js script using sharp (already available via Next.js) that:

    **Background processing:**
    1. Reads `WebsiteAssets/MainBackground.png` (12.9MB, 5568x3072 RGB)
    2. Generates three WebP variants at widths 1920, 2560, 3840 with quality 80
    3. Writes to `app/public/scene/background/factory-bg-{width}.webp`
    4. Generates a tiny blur placeholder (20px wide, quality 20) as base64 data URL

    **Overlay processing (for each of the 5 available overlays):**
    1. Reads each overlay PNG from `WebsiteAssets/` (CarnageCauldron.png, ConnectWallet.png, DocumentationTable.png, RewardsVat.png, Settings.png)
    2. Runs `sharp.trim({ threshold: 10 })` with `resolveWithObject: true` to crop transparent pixels to bounding box -- this returns `trimOffsetLeft` and `trimOffsetTop` which encode the object's position in the original 5568x3072 scene
    3. Converts trimmed buffer to WebP with quality 82
    4. Writes to `app/public/scene/overlays/{kebab-name}.webp`
    5. Generates a tiny blur placeholder (10px wide, quality 20) as base64 data URL
    6. Computes position metadata as percentages of scene dimensions (5568x3072):
       - `left`: `(trimOffsetLeft / 5568) * 100`
       - `top`: `(trimOffsetTop / 3072) * 100`
       - `widthPct`: `(trimmedWidth / 5568) * 100`
       - `heightPct`: `(trimmedHeight / 3072) * 100`

    **Swap Station placeholder:**
    - The SwapMachine asset is NOT yet available (user providing later)
    - Add a placeholder entry in the metadata with `available: false` and reasonable estimated dimensions/position
    - Do NOT create a placeholder image file -- just skip it in the build

    **Output metadata file:**
    After processing, the script auto-generates `app/lib/image-data.ts` containing:
    ```typescript
    export const SCENE_DATA = {
      background: {
        src: '/scene/background/factory-bg-1920.webp',
        blurDataURL: 'data:image/webp;base64,...',
        width: 5568,
        height: 3072,
      },
      overlays: {
        'carnage-cauldron': {
          src: '/scene/overlays/carnage-cauldron.webp',
          blurDataURL: 'data:image/webp;base64,...',
          width: NNN,   // actual trimmed width
          height: NNN,  // actual trimmed height
          left: NN.NN,  // percentage position
          top: NN.NN,
          widthPct: NN.NN,
          heightPct: NN.NN,
          available: true,
        },
        // ... other overlays
        'swap-station': {
          src: '/scene/overlays/swap-station.webp',
          blurDataURL: '',
          width: 0,
          height: 0,
          left: 50,
          top: 50,
          widthPct: 15,
          heightPct: 20,
          available: false,  // Asset not yet provided
        },
      },
    } as const;
    ```

    Add proper TypeScript types for the SCENE_DATA structure (OverlayData, BackgroundData, SceneData types).

    **Important implementation notes:**
    - Use `import sharp from 'sharp'` (ESM -- the script is .mjs)
    - Use `import { writeFileSync, mkdirSync } from 'fs'` and `import path from 'path'`
    - Create output directories (`app/public/scene/background/`, `app/public/scene/overlays/`) with `mkdirSync({ recursive: true })`
    - Handle the case where sharp trim() returns zero-dimension output (error handling with threshold fallback)
    - Log progress to stdout: which file is being processed, input/output sizes, trim offsets
    - All paths should be relative to the script's location (use `import.meta.url` to resolve)
    - Run the script with: `cd scripts && node optimize-images.mjs`

    **After writing the script, RUN IT** to generate all output files. Verify the generated files exist and check their sizes.

<task type="auto">
  <name>Task 2: Update next.config.ts with image optimization settings</name>
  <files>app/next.config.ts</files>
  <action>
    Add the `images` configuration block to the existing nextConfig object in `app/next.config.ts`:

    ```typescript
    images: {
      qualities: [75, 80, 82, 85],  // Allow all quality levels used by scene images (80 for bg, 82 for overlays)
      formats: ['image/webp'],       // WebP format for all optimized images
      deviceSizes: [1920, 2560, 3840], // Match our scene background breakpoints
    },
    ```

    This goes inside the `nextConfig` object, alongside `transpilePackages` and `turbopack`.

    **Why each field matters:**
    - `qualities`: REQUIRED in Next.js 16 -- without it, Image component quality prop may be rejected or default to [75] only
    - `formats`: Explicit WebP preference (Next.js default, but being explicit is good for clarity)
    - `deviceSizes`: Controls the srcset breakpoints for responsive images -- match our 3 background variants

    **Do NOT change:**
    - The CSP header (img-src 'self' data: blob: already covers our use case)
    - The turbopack config
    - The transpilePackages config
    - The headers() function
