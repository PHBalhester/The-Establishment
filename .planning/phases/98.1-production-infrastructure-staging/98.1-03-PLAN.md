<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Cloudflare CNAME + Railway domain binding + UptimeRobot</name>
  <action>
  Complete these dashboard actions. Order matters -- Railway domain first, then Cloudflare CNAME, then monitoring.

  **Step 1: Railway Custom Domain (5 min)**
  1. Go to Railway Dashboard -> mainnet-web service -> Settings -> Domains
  2. Click "Add Custom Domain"
  3. Enter `fraudsworth.fun`
  4. Railway will show a CNAME target (e.g., `mainnet-web-xxx.up.railway.app`) -- copy this
  5. Also add `www.fraudsworth.fun` as a second custom domain (Railway will handle redirect)
  6. Note: Railway will show "Waiting for DNS" until Cloudflare CNAME is added

  **Step 2: Cloudflare CNAME Records (5 min)**
  1. Go to Cloudflare Dashboard -> fraudsworth.fun -> DNS -> Records
  2. Add record:
     - Type: CNAME
     - Name: `@` (apex)
     - Target: Railway's CNAME target from Step 1
     - Proxy status: **Proxied** (orange cloud ON) -- required for CNAME flattening
     - TTL: Auto
  3. Add record:
     - Type: CNAME
     - Name: `www`
     - Target: `fraudsworth.fun`
     - Proxy status: Proxied
  4. Add Page Rule for www redirect:
     - URL: `www.fraudsworth.fun/*`
     - Setting: Forwarding URL -> 301 Permanent Redirect
     - Destination: `https://fraudsworth.fun/$1`

  **Step 3: Verify Domain (5 min)**
  Wait 2-3 minutes after adding CNAME records, then:
  1. Check Railway Dashboard -- should show domain as "Verified" (green check)
  2. Visit `https://fraudsworth.fun` in browser -- should see Railway's deployment (may show error page if service hasn't deployed yet, but SSL should work)
  3. Visit `https://www.fraudsworth.fun` -- should redirect to `https://fraudsworth.fun`

  **Step 4: UptimeRobot Monitoring (5 min)**
  1. Go to https://uptimerobot.com -> Create free account (or log in)
  2. Add Monitor 1:
     - Type: HTTP(s)
     - Friendly Name: "Fraudsworth Frontend"
     - URL: `https://fraudsworth.fun`
     - Monitoring Interval: 5 minutes
  3. Add Monitor 2:
     - Type: HTTP(s) - Keyword
     - Friendly Name: "Fraudsworth API Health"
     - URL: `https://fraudsworth.fun/api/health`
     - Keyword Type: Keyword Exists
     - Keyword: `ok`
     - Monitoring Interval: 5 minutes
  4. Add Monitor 3:
     - Type: HTTP(s) - Keyword
     - Friendly Name: "Fraudsworth Crank"
     - URL: `https://<mainnet-crank-railway-url>/health` (from Plan 02)
     - Keyword Type: Keyword Exists
     - Keyword: `running`
     - Monitoring Interval: 5 minutes
  5. Set up email alert contact (your email) for all 3 monitors
  </action>

<task type="auto">
  <name>Task 3: Register mainnet Helius webhook + final verification</name>
  <files></files>
  <action>
  Now that fraudsworth.fun is live with SSL, register the mainnet webhook.

  1. Set environment for mainnet webhook registration:
     ```
     export CLUSTER=mainnet
     export HELIUS_API_KEY=<mainnet-primary-key-from-Plan-02>
     export WEBHOOK_URL=https://fraudsworth.fun/api/webhooks/helius
     export HELIUS_WEBHOOK_SECRET=<secret-generated-in-Plan-02>
     ```

  2. First list any existing webhooks to avoid duplicates:
     ```
     npx tsx scripts/webhook-manage.ts list
     ```

  3. Create the mainnet webhook:
     ```
     npx tsx scripts/webhook-manage.ts create
     ```
     Verify output shows:
     - Cluster: mainnet (MAINNET)
     - API Base: https://api-mainnet.helius-rpc.com/v0
     - Type: raw (NOT rawDevnet)
     - URL: https://fraudsworth.fun/api/webhooks/helius
     - Webhook ID returned

  4. Save the webhook ID for future reference.

  5. Test webhook delivery with a curl to verify the endpoint accepts authenticated requests:
     ```
     curl -s -X POST https://fraudsworth.fun/api/webhooks/helius \
       -H "Content-Type: application/json" \
       -H "Authorization: <HELIUS_WEBHOOK_SECRET>" \
       -d '[{"test": true}]'
     ```
     Expected: 200 OK (handler processes and ignores unrecognized payload)

  6. Final infrastructure verification checklist:
     - `curl -s https://fraudsworth.fun` returns HTML (or Next.js error page if not deployed)
     - `curl -s https://fraudsworth.fun/api/health` returns health status
     - `dig A fraudsworth.fun +short` returns Cloudflare IPs
     - `dig CNAME www.fraudsworth.fun +short` returns fraudsworth.fun
     - Helius webhook registered with correct URL and type
  </action>
  <verify>
  - webhook-manage.ts list shows the newly created mainnet webhook
  - curl to fraudsworth.fun returns HTTP 200 (or 500 if app not deployed -- SSL works either way)
  - dig confirms Cloudflare is managing DNS
  </verify>
  <done>
  Mainnet Helius webhook registered pointing to fraudsworth.fun. Domain resolves correctly with SSL. UptimeRobot monitoring active. All production infrastructure is staged and ready for mainnet deployment.
  </done>
