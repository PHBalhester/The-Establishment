## Accomplishments
- CSP dynamically includes cluster-appropriate Helius RPC URLs (mainnet.helius-rpc.com vs devnet.helius-rpc.com) based on NEXT_PUBLIC_CLUSTER build-time env var
- CSP docs iframe sources switch between fraudsworth.fun (mainnet) and localhost/Railway (devnet)
- webhook-manage.ts selects correct Helius API base (api-mainnet vs api.helius.xyz) and webhook type (raw vs rawDevnet) based on cluster
- Both files default to devnet when env vars are unset -- safe default preserves existing behavior

- `scripts/webhook-manage.ts` - Added cluster detection, cluster-aware API base URL, webhook type constant, and startup logging

## Decisions Made
- CSP uses build-time NEXT_PUBLIC_CLUSTER (Next.js convention) -- this is the standard way Next.js exposes env vars to both server and client config
- webhook-manage.ts accepts both CLUSTER and NEXT_PUBLIC_CLUSTER for flexibility when sourcing different .env files
- Mainnet webhook type is 'raw' (not 'rawDevnet') per Helius API docs -- mainnet webhooks use different internal routing
- Both 'mainnet' and 'mainnet-beta' accepted as cluster values since Solana tooling uses both conventions
- Sentry ingest URLs remain in CSP for both clusters (same DSN, differentiated by cluster tag set in Phase 92)
