<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Cloudflare DNS migration + Helius mainnet keys + Railway mainnet services</name>
  <action>
  This task requires human dashboard actions across 4 services. Complete them in this order (DNS first because propagation is the longest wait):

  **Step 1: Cloudflare Setup (10 min)**
  1. Go to https://dash.cloudflare.com/sign-up (or log in if account exists)
  2. Click "Add a Site" -> enter `fraudsworth.fun`
  3. Select the **Free** plan
  4. Cloudflare will scan existing DNS records -- review and confirm they look correct
  5. Copy the two Cloudflare nameservers shown (e.g., `ns1.cloudflare.com`, `ns2.cloudflare.com`)
  6. Set SSL/TLS mode to **Full** (NOT Full Strict, NOT Flexible) -- Settings -> SSL/TLS -> Overview -> Full
  7. Do NOT add CNAME records yet -- that happens in Plan 03 after Railway services exist

  **Step 2: GoDaddy NS Change (5 min)**
  1. Go to GoDaddy -> My Products -> fraudsworth.fun -> DNS Management
  2. Find "Nameservers" section -> click "Change"
  3. Select "I'll use my own nameservers"
  4. Enter the two Cloudflare nameservers from Step 1
  5. Save. DNS propagation begins (typically 1-24 hours, up to 72 hours)

  **Step 3: Helius Mainnet API Keys (5 min)**
  1. Go to https://dashboard.helius.dev
  2. Create a new **mainnet** API key (label it "dr-fraudsworth-mainnet-primary")
  3. Copy the API key -- this becomes `HELIUS_API_KEY` and is embedded in `HELIUS_RPC_URL`
  4. Create a second **mainnet** API key (label it "dr-fraudsworth-mainnet-fallback")
  5. This becomes the fallback in `HELIUS_RPC_URL_FALLBACK`
  6. Also note the `HELIUS_API_KEY` value separately (used by webhook-manage.ts for REST API calls)

  **Step 4: Railway Mainnet Services (15 min)**
  1. Go to Railway Dashboard -> Dr Fraudsworth project
  2. Create new service: **mainnet-web**
     - Connect to the same GitHub repo
     - Set root directory to `/` (same as devnet)
     - Add ALL env vars from `app/.env.mainnet` template
     - Replace `CHANGE_ME_MAINNET` values with real Helius keys from Step 3
     - Set `NEXT_PUBLIC_CLUSTER=mainnet`
     - Set `NEXT_PUBLIC_SITE_MODE=launch`
     - Set `NEXT_PUBLIC_DOCS_URL=https://fraudsworth.fun/docs`
     - Leave program IDs / mints as `CHANGE_ME_MAINNET` (filled during mainnet deploy)
     - Generate a random `HELIUS_WEBHOOK_SECRET` (e.g., `openssl rand -hex 32`) -- save it for webhook registration
  3. Create new service: **mainnet-crank**
     - Do NOT connect to GitHub (manual deploy or Docker)
     - Set build command: `npm run build:crank` (or however crank is built)
     - Add crank env vars from `.env.mainnet` crank section
     - Set `CLUSTER_URL=https://mainnet.helius-rpc.com/?api-key=<primary-key>`
     - Leave `WALLET_KEYPAIR`, `CARNAGE_WSOL_PUBKEY`, `PDA_MANIFEST` as `CHANGE_ME_MAINNET`
     - Set `HEALTH_PORT=8080`
  4. Add **Postgres** plugin for mainnet
     - Railway Dashboard -> New -> Database -> Postgres
     - Railway auto-injects `DATABASE_URL` into services that reference it
     - Link `DATABASE_URL` to mainnet-web service

  **Do NOT deploy yet** -- services should be created with env vars but not actively serving traffic.
  </action>
  <resume-signal>
  Confirm all 4 steps complete:

  2. GoDaddy NS changed to Cloudflare
  3. Helius mainnet primary key: [first 8 chars...] and fallback key created
  4. Railway mainnet-web, mainnet-crank, mainnet-postgres services created with env vars
  5. HELIUS_WEBHOOK_SECRET generated and saved: [yes/no]
  </resume-signal>
