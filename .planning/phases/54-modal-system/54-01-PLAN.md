3. **Dialog animation rules** (add after the keyframes, in a new "Modal System" section):

   ```css
   /* Modal: iris open when dialog[open] */
   dialog.modal-shell[open] {
     animation: iris-open 280ms cubic-bezier(0.22, 1, 0.36, 1) forwards;
   }
   dialog.modal-shell[open]::backdrop {
     animation: backdrop-fade-in 280ms ease-out forwards;
   }
   /* Modal: closing state (added by JS, removed after animationend) */
   dialog.modal-shell.closing {
     animation: modal-close 180ms ease-in forwards;
   }
   dialog.modal-shell.closing::backdrop {
     animation: backdrop-fade-out 180ms ease-in forwards;
   }
   ```

4. **Body scroll lock** (add in the Modal System section):
   ```css

     overflow: hidden;
   }
   ```

5. **Station content crossfade classes**:

   }

- The cubic-bezier(0.22, 1, 0.36, 1) easing creates a satisfying deceleration curve (fast start, gentle stop)
- The content crossfade classes enable station switching without closing the dialog (RESEARCH.md Pitfall 7)
- Body scroll lock is needed because `<dialog>.showModal()` does NOT prevent body scroll in all browsers (RESEARCH.md Pitfall 4)

**Create `app/components/modal/ModalProvider.tsx`:**

This is the React Context that manages modal state globally. It enforces the single-modal policy (MODAL-03) and tracks click origin for iris animation.

```typescript
'use client';

import { createContext, useState, useCallback, useRef } from 'react';
import type { ReactNode, RefObject } from 'react';

// Station IDs matching the 6 factory stations
export type StationId = 'swap' | 'carnage' | 'staking' | 'wallet' | 'docs' | 'settings';

export interface IrisOrigin {
  x: number;
  y: number;
}

export interface ModalState {
  activeStation: StationId | null;
  irisOrigin: IrisOrigin | null;
}

export interface ModalContextValue {
  state: ModalState;
  openModal: (station: StationId, clickOrigin: IrisOrigin) => void;
  closeModal: () => void;
  /** Ref to the element that triggered the modal open -- for focus restoration on close (MODAL-06) */
  triggerRef: RefObject<HTMLElement | null>;
}

export const ModalContext = createContext<ModalContextValue | null>(null);
```

**Provider component:**
- Stores `activeStation` (StationId | null) and `irisOrigin` ({x, y} | null) in state
- `openModal(station, clickOrigin)`:
  - If a different station is already open, set activeStation to null briefly (for crossfade), then after a 0ms timeout (microtask), set the new station. This enables the content crossfade pattern from RESEARCH.md Pitfall 7.
  - Actually, the crossfade is handled at the ModalShell render layer (Plan 02). Here, just update the state synchronously. If the same station is requested, ignore. If different, update activeStation and irisOrigin.
  - Save `document.activeElement` to triggerRef for focus restoration on close.
  - Add `modal-open` class to body.
- `closeModal()`:
  - Set activeStation to null and irisOrigin to null.
  - Remove `modal-open` class from body.
  - Focus restoration happens in ModalShell (Plan 02) after close animation completes.

**Why a ref for the trigger element:** When the modal closes, MODAL-06 requires returning focus to the element that opened it. Storing the trigger element in a ref (not state) avoids unnecessary re-renders.

**Single-modal policy (MODAL-03):** The state only has ONE activeStation. There is no array, no stack, no queue. Opening a new station overwrites the previous one.

**Create `app/hooks/useModal.ts`:**

A thin hook that consumes ModalContext and throws if used outside the provider (developer safety).

```typescript
'use client';

import { useContext } from 'react';
import { ModalContext } from '@/components/modal/ModalProvider';
import type { ModalContextValue } from '@/components/modal/ModalProvider';

export function useModal(): ModalContextValue {
  const context = useContext(ModalContext);
  if (!context) {
    throw new Error('useModal must be used within a ModalProvider');
  }
  return context;
}
```

**Why a separate hook file instead of exporting from ModalProvider:**
- Follows the existing project pattern (hooks/ directory for all hooks)
- Keeps the provider focused on state management
- Allows components to import from `@/hooks/useModal` consistently

**Important implementation details:**
- The `body.modal-open` class toggle happens in the provider, NOT in ModalShell. This ensures scroll lock is applied immediately on state change, before the dialog animation begins.
- Do NOT use `useEffect` for the body class -- use it directly in openModal/closeModal callbacks for synchronous application.
- The provider does NOT render any dialog element -- it only manages state. The ModalShell (Plan 02) renders the `<dialog>`.
