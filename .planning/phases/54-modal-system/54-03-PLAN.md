**Update `app/providers/providers.tsx`:**

1. Import ModalProvider from `@/components/modal/ModalProvider` and ModalRoot from `@/components/modal/ModalShell`.
2. Wrap children with ModalProvider INSIDE the PrivyProvider (ModalProvider does not depend on Privy, but being inside Privy means modal content can use wallet hooks).
3. Render ModalRoot as a sibling to children inside ModalProvider. ModalRoot renders the singleton `<dialog>` element.

The provider tree becomes:
```
<PrivyProvider>
  <ModalProvider>
    {children}
    <ModalRoot />
  </ModalProvider>
</PrivyProvider>
```

**Why ModalProvider inside PrivyProvider:** Modal content (Phase 56) will need wallet hooks (useProtocolWallet, etc.). Privy must be above modal content in the tree.

**Why ModalRoot as sibling to children (not inside children):** The `<dialog>` element needs to be in the DOM always, not dependent on which page is rendered. As a sibling to `{children}` inside ModalProvider, it exists on every page.

**Update `app/app/page.tsx`:**

Add a TEMPORARY demo section with buttons to test each station modal. This will be removed in Phase 55 when the scene objects become the real triggers.

Add a small client component (either inline or a separate small component) that:

3. Each button's onClick calls `openModal(stationId, { x: rect.left + rect.width/2, y: rect.top + rect.height/2 })` using the button's own bounding rect as the iris origin
4. Style the buttons with basic factory theme classes so they're visible
