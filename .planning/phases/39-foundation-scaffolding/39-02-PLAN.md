    1. Create `app/scripts/sync-idl.mjs` following the pattern from 39-RESEARCH.md:
       - Define `PROGRAMS = ["amm", "epoch_program", "staking", "tax_program", "transfer_hook"]` (5 production programs only -- NOT fake_tax_program, mock_tax_program, stub_staking, epoch_program.json.bak)
       - Source: `../../target/idl/` (relative to script location)
       - Dest: `../idl/` (relative to script location, i.e. `app/idl/`)
       - `mkdirSync(DEST, { recursive: true })` to create idl/ dir
       - For each program: copy `${name}.json` from source to dest
       - Also attempt to copy type files from `../../target/types/${name}.ts` to `../idl/types/${name}.ts` -- these are the Anchor-generated TypeScript type definitions. If the types directory doesn't exist, create it. If a type file doesn't exist, skip it with a warning (don't fail).
       - Log each synced file to stdout
       - If source directory doesn't exist, print a warning "target/idl/ not found -- run 'anchor build' first" and exit 0 (don't fail the dev server for missing IDLs)

    2. Update `app/package.json` scripts to add predev and prebuild hooks:
       ```json
       "predev": "node scripts/sync-idl.mjs",
       "prebuild": "node scripts/sync-idl.mjs"
       ```
       These run automatically before `next dev` and `next build`, keeping IDLs fresh.

    3. Create `app/lib/connection.ts`:
       - Export `getConnection(rpcUrl?: string): Connection` function
       - Default RPC URL from `process.env.NEXT_PUBLIC_RPC_URL` or fall back to the devnet Helius URL from `@dr-fraudsworth/shared` (`DEVNET_RPC_URL`)
       - Use "confirmed" commitment
       - This is a simple factory, no caching/singleton needed yet

    4. Create `app/lib/anchor.ts`:
       - Import IDL JSON files using `@/idl/amm.json`, `@/idl/epoch_program.json`, etc.
       - Import type files as TypeScript types: `import type { Amm } from "@/idl/types/amm"` etc. If type imports cause build errors (the open question from RESEARCH.md about Anchor type file imports), fall back to using `any` type assertions on the JSON imports and add a comment explaining why.
       - Export factory functions: `getAmmProgram(connection?: Connection)`, `getEpochProgram(...)`, `getStakingProgram(...)`, `getTaxProgram(...)`, `getHookProgram(...)`
       - Each function creates a `new Program(idl, { connection })` using the read-only constructor (no wallet needed)
       - If connection not provided, call `getConnection()` from connection.ts
       - Import Connection type from `@solana/web3.js`

    5. Run the IDL sync script to populate `app/idl/`:
       ```bash
       cd app && node scripts/sync-idl.mjs
       ```
       Verify that 5 JSON files appear in `app/idl/` and type files appear in `app/idl/types/`.

    IMPORTANT: The `app/idl/` directory is gitignored (added in Plan 01). IDL files are build artifacts synced on demand. Additional resolveAlias stubs may be needed beyond fs/net/tls -- if Turbopack reports "Module not found" for other Node.js modules during dev server startup, add them to the resolveAlias config in next.config.ts following the same `{ browser: "./lib/empty.ts" }` pattern. This is the iterative process noted in 39-RESEARCH.md Open Question #1.
