**B. Make tryReveal() return instrumentation data:**
Change tryReveal() to return an object instead of just the instruction. New return type:
```typescript

  revealIx: any;
  attempts: number;
  durationMs: number;
}
```

- Track `const startMs = Date.now()` at function entry

- On success: return `{ revealIx, attempts, durationMs: Date.now() - startMs }`
- On exhaustion: return `null` (same as before)
- Update the function signature: `async function tryReveal(...): Promise<RevealResult | null>`
- Update ALL callers of tryReveal() in the file to destructure the result:
  - Happy path (line ~731): `const revealResult = await tryReveal(randomness, 10)` -> check for null, then use `revealResult.revealIx`

  - vrfPending timeout retry reveal (line ~557): accumulate attempts across stale + fresh calls
  - Happy-path timeout retry reveal (line ~823): same pattern

**C. Switch tryReveal() to exponential backoff:**
Replace the linear formula at line ~269:
```typescript
// OLD (linear — comment says "Exponential" but formula is linear):

// NEW (truly exponential):
const REVEAL_BASE_DELAY_MS = 1000;
const REVEAL_MAX_DELAY_MS = 16_000;
const delay = Math.min(REVEAL_BASE_DELAY_MS * Math.pow(2, i), REVEAL_MAX_DELAY_MS);
await sleep(delay);
```
Fix the misleading comment on line ~268 to accurately describe the formula.

**D. Change happy-path tryReveal from 10 attempts to 5:**
Per CONTEXT.md: "5 attempts, ~31s total" for happy path.
- Line ~731: `tryReveal(randomness, 10)` -> `tryReveal(randomness, 5)`

- Record `const commitSlot = await connection.getSlot()` right after TX2 confirmation
- After successful reveal: record `const revealSlot = await connection.getSlot()`
- Compute `commitToRevealSlots = revealSlot - commitSlot`
- For recovery paths: set `commitToRevealSlots` to 0 (no meaningful measurement since commit was in a prior cycle)

**F. Populate instrumentation fields in ALL return statements:**

- Happy path return (~line 911): gatewayMs from revealResult.durationMs, revealAttempts from revealResult.attempts, recoveryTimeMs = 0, commitToRevealSlots from slot delta
- vrfPending stale reveal return (~line 628): gatewayMs, revealAttempts, recoveryTimeMs = Date.now() - startMs (whole recovery), commitToRevealSlots = 0
- vrfPending timeout retry return (~line 628): same, with cumulative attempts from both reveal calls
- vrfPending TOCTOU returns (~line 489, ~line 589): gatewayMs = 0, revealAttempts = 0, recoveryTimeMs, commitToRevealSlots = 0
- Happy-path timeout recovery: populate from retry revealResult, recoveryTimeMs = recovery wall-clock
