Entry template format (from 03-RESEARCH.md Pattern 1):
```markdown

|-------|-------|
| Type | [Constant/Entity/Behavior/etc.] |
| Primary Document | [filename.md] |
| Value | [The actual value/definition] |
| Also Appears In | [List of other docs, or "Single-source"] |
| Location | [filename.md:Section X.X] |

```

For assumptions specifically, use extended format:
```markdown
### ASSUMP-[XXX]: [Assumption Description]

| Field | Value |
|-------|-------|
| Type | Assumption (inferred) |
| Inferred From | [filename.md] |
| Explicit? | No - implied by "[quote]" |
| Depends On | [What must be true] |
| If Wrong | [Consequence of invalid assumption] |

**Documents That Rely On This:**
- [List documents that depend on this assumption]
```

<task type="auto">
  <name>Task 2: Extract concepts from all 12 specification documents</name>
  <files>.planning/cross-reference/00-concept-inventory.md</files>
  <action>
Process all 12 documents in dependency order (from 03-RESEARCH.md Section "Document Processing Order"):

**Processing Order:**
1. Foundation: DrFraudsworth_Overview.md, Token_Program_Reference.md
2. Core: Epoch_State_Machine_Spec.md, Tax_Pool_Logic_Spec.md, AMM_Implementation.md
3. Dependent: Carnage_Fund_Spec.md, Yield_System_Spec.md, Soft_Peg_Arbitrage_Spec.md
4. Launch: Bonding_Curve_Spec.md, Protocol_Initialzation_and_Launch_Flow.md
5. Infrastructure: Transfer_Hook_Spec.md, SolanaSetup.md

**For each document, extract:**

1. **Constants** - Look for: numbers, percentages, "X bps", durations, slot counts
   Examples: LP_FEE_SOL_POOLS, EPOCH_LENGTH, TAX_LOW_RANGE, CARNAGE_PROBABILITY

2. **Entities** - Look for: proper nouns, PDA names, program references, account types
   Examples: IPA/IPB/OP4 tokens, pool names, EpochState, CarnageFund PDA

3. **Behaviors** - Look for: "when X, then Y", numbered steps, flowcharts, sequences
   Examples: Swap execution order, epoch transition sequence, carnage execution

4. **Constraints** - Look for: "must", "cannot", "only", "always", "never"
   Examples: "No direct wallet transfers", "Taxes apply ONLY to SOL pools"

5. **Formulas** - Look for: equations, code blocks with math, "calculated by"
   Examples: AMM pricing formula, tax calculation, ATA derivation

6. **Terminology** - Look for: definitions, glossary entries, "X means Y"
   Examples: "Cheap side", "Expensive side", "Epoch", "Carnage", "Regime flip"

**Critical for Assumptions:**
Apply these extraction prompts to each document:
- "What does this document ASSUME to be true but doesn't explicitly state?"
- "What must already exist for this to work?"
- "What would break if [X] wasn't true?"

**Numbering scheme:**
- CONST-001, CONST-002, etc. (sequential within type)
- Track "Also Appears In" for each concept - leave blank initially, will be populated during matrix building

**Quality check:** If you extract fewer than 50 total concepts across 12 documents, you're being too conservative. If you extract fewer than 5 assumptions, you're missing implicit dependencies (v3 lesson).
