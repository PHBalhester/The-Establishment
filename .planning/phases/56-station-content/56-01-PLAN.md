Create `app/components/toast/ToastProvider.tsx`:

This is a minimal, zero-dependency toast system for swap success/error feedback. Only one toast visible at a time (no queue/stacking -- D10 simplicity). Pattern mirrors ModalProvider.

1. Define `Toast` interface: `{ id: string; type: 'success' | 'error'; message: string; link?: { label: string; href: string } }`.
2. Create `ToastContext` with `createContext`.
3. Export `useToast()` hook returning `{ showToast(type, message, link?) }`.
4. Export `ToastProvider({ children })`:
   - State: `toast: Toast | null`, `exiting: boolean`.
   - `showToast`: sets toast with unique id (Date.now()), clears exiting. Starts auto-dismiss timer:
     - Success: 8 seconds (user may want to click the Solscan link).
     - Error: 5 seconds.
   - `dismissToast`: sets `exiting = true`, after 300ms animation sets `toast = null`.
   - Cleanup: clear timer on unmount or when new toast replaces old.
5. Export `ToastContainer()`:
   - Uses `useToast` context to read current toast.
   - If no toast, return null.
   - Renders a `position: fixed` div at bottom-right (`bottom: 1.5rem, right: 1.5rem`).
   - Uses `ReactDOM.createPortal(jsx, document.body)` to render OUTSIDE the dialog element (critical: dialog top-layer would trap toasts inside it otherwise).
   - Toast card: steampunk-themed (dark bg with brass border), includes dismiss X button.
   - Success type: green left-accent border, link rendered as `<a>` to Solscan.
   - Error type: red left-accent border.
   - CSS class `toast-enter` on mount, `toast-exit` when `exiting` is true.
   - Dismiss button on click calls `dismissToast()`.
