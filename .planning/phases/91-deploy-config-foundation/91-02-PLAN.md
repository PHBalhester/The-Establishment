<objective>
Build the code generator that reads deployment.json and produces a complete shared/constants.ts, replacing the hand-maintained version.

Purpose: Eliminates manual address copy-paste between deployment and frontend/scripts. Every protocol address in constants.ts is derived from the canonical deployment.json. Running the generator after any deploy ensures constants are always in sync.

Output: `scripts/deploy/generate-constants.ts` script and a regenerated `shared/constants.ts` that is API-compatible with the current version.
</objective>

<task type="auto">
  <name>Task 1: Catalog current constants.ts API surface and build the generator</name>
  <files>
    scripts/deploy/generate-constants.ts
  </files>
  <action>
  1. FIRST: Catalog every export from current `shared/constants.ts` and every import from consumer files. This is critical -- the generated output must be a drop-in replacement. Use grep to find all imports:
     ```
     grep -rn "from.*shared/constants" app/ scripts/ tests/ --include="*.ts" --include="*.tsx"
     ```
     Record every imported symbol. The generator must emit ALL of them.

  2. Create `scripts/deploy/generate-constants.ts` that:
     - Accepts a cluster argument (`devnet` or `mainnet`)
     - Reads `deployments/{cluster}.json`
     - Reads `scripts/deploy/lib/deployment-schema.ts` types (import DeploymentConfig)
     - Builds the entire `shared/constants.ts` file as a string
     - The generated file must include:
       a. `// AUTO-GENERATED from deployments/{cluster}.json -- DO NOT EDIT MANUALLY` header
       b. Imports: `PublicKey` from `@solana/web3.js`, `NATIVE_MINT`, `TOKEN_PROGRAM_ID`, `TOKEN_2022_PROGRAM_ID` from `@solana/spl-token`
       c. `PROGRAM_IDS` object with all 7 programs as `new PublicKey("...")` -- values from deployment.json
       d. `MINTS` object with CRIME, FRAUD, PROFIT -- values from deployment.json
       e. Static constants (hardcoded in generator template, NOT from deployment.json):
          - `TOKEN_DECIMALS = 6`
          - `MINIMUM_STAKE = 1_000_000`
          - `COOLDOWN_SECONDS = 43_200`
          - `SOL_POOL_FEE_BPS = 100`
          - `VAULT_CONVERSION_RATE = 100`
          - `VAULT_SEEDS` object
          - `SEEDS` object (all PDA seed buffers)
          - `SLOTS_PER_EPOCH = 750`
          - `MS_PER_SLOT = 400`
          - Bonding curve constants: `CURVE_TARGET_SOL`, `CURVE_TARGET_TOKENS`, `MAX_TOKENS_PER_WALLET`, `MIN_PURCHASE_SOL`, `CURVE_SELL_TAX_BPS`, `CURVE_DEADLINE_SLOTS`
       f. `DEVNET_CURVE_PDAS` -- from deployment.json curvePdas
       g. `DEVNET_PDAS` -- from deployment.json pdas
       h. `DEVNET_POOLS` -- from deployment.json pools
       i. `DEVNET_POOL_CONFIGS` -- array of PoolConfig objects built from pools + mints
       j. `TOKEN_PROGRAM_FOR_MINT` -- CRIME/FRAUD use TOKEN_2022, SOL uses TOKEN_PROGRAM_ID, PROFIT uses TOKEN_PROGRAM_ID
       k. Type exports: `TokenSymbol`, `SwapInstruction`, `PoolConfig`, `VaultConvertConfig`, `RouteConfig`, `ClusterName`, `ClusterConfig`
       l. Functions: `resolvePool()`, `resolveRoute()`, `getClusterConfig()`
       m. `DEVNET_PDAS_EXTENDED` -- from deployment.json pdas (extended set including carnage, staking, vault PDAs)
       n. `TREASURY_PUBKEY` -- from deployment.json treasury
       o. `CLUSTER_CONFIG` -- build from deployment.json addresses for devnet; mainnet uses placeholders or mainnet deployment.json if it exists
     - Writes the complete string to `shared/constants.ts`
     - Logs: "Generated shared/constants.ts from deployments/{cluster}.json ({N} lines)"

  3. IMPORTANT anti-patterns to avoid:
     - Do NOT use ts-morph or AST manipulation -- simple string concatenation is correct here
     - Do NOT import from the OLD constants.ts (circular dependency)
     - Do NOT change any export names, types, or function signatures
     - Static values (seeds, decimals, fees) are HARDCODED in the generator template -- they come from on-chain program code, not deployment.json
     - The `SEEDS` object must contain the exact same Buffer values as current constants.ts
  </action>
  <verify>
  - `npx tsx scripts/deploy/generate-constants.ts devnet` runs without errors
  - Generated `shared/constants.ts` starts with `// AUTO-GENERATED`
  - All exports from old constants.ts exist in new: `diff <(grep "^export" shared/constants.ts.bak) <(grep "^export" shared/constants.ts)` shows no removals
  </verify>
  <done>generate-constants.ts exists and can produce shared/constants.ts from deployment.json</done>
</task>
