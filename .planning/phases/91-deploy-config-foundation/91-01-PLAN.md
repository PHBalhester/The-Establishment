Create the canonical deployment.json schema, extend initialize.ts to write it, and split .env into cluster-specific files.

Purpose: deployment.json becomes the single source of truth for all protocol addresses. Every downstream tool (generate-constants, verify, crank runner) reads from this file instead of scattered sources. The .env split prevents wrong-cluster secrets from leaking.

Output: `deployments/devnet.json` generated from existing on-chain state, TypeScript schema types, `.env.devnet` and `.env.mainnet` files.
</objective>

<task type="auto">
  <name>Task 1: Create deployment.json schema and generate devnet config from existing state</name>
  <files>
    scripts/deploy/lib/deployment-schema.ts
    scripts/deploy/lib/pda-manifest.ts
    deployments/devnet.json
  </files>
  <action>
  1. Create `scripts/deploy/lib/deployment-schema.ts` with TypeScript interfaces for the deployment config:
     - `DeploymentConfig` interface with fields: `schemaVersion` (number), `generatedAt` (string), `cluster` (string), `programs` (Record mapping program names to address strings: amm, transferHook, taxProgram, epochProgram, staking, conversionVault, bondingCurve), `mints` (Record: crime, fraud, profit), `pdas` (Record of all singleton PDAs -- adminConfig, swapAuthority, taxAuthority, epochState, stakePool, escrowVault, stakeVault, whitelistAuthority, carnageFund, carnageSolVault, carnageCrimeVault, carnageFraudVault, carnageSigner, stakingAuthority, wsolIntermediary, vaultConfig, vaultCrime, vaultFraud, vaultProfit), `pools` (Record of pool configs with pool/vaultA/vaultB), `curvePdas` (Record of curve configs with curveState/tokenVault/solVault/taxEscrow per faction), `hookAccounts` (Record of hook extra_account_meta_list PDAs per mint), `alt` (string or null), `treasury` (string), `authority` (object with deployer/squadsVault/transferredAt), `metadata` (optional object with crime/fraud/profit URIs)
     - `validateDeploymentConfig()` function that checks all required fields are present and all address strings are valid base58 (32-44 chars)
     - Export the interface and validation function

  2. Extend `generateManifest()` in `scripts/deploy/lib/pda-manifest.ts` to also produce a `DeploymentConfig` object. Add a new exported function `generateDeploymentConfig(cluster: string, programIds: Record, mintAddresses: Record, altAddress: string | null, treasury: string)` that:
     - Derives all PDAs using existing seed constants and `PublicKey.findProgramAddressSync`
     - Derives all pool addresses using existing `derivePoolPDA` / `deriveVaultPDAs`
     - Derives bonding curve PDAs (curveState, tokenVault, solVault, taxEscrow per faction)
     - Derives hook extra_account_meta_list PDAs per mint
     - Returns a complete `DeploymentConfig` object
     - Keep existing `generateManifest()` working (don't break it -- it's still used by verify.ts until Plan 04 migrates it)

  3. Create a standalone script `scripts/deploy/generate-deployment-json.ts` that:
     - Reads program IDs from Anchor.toml `[programs.devnet]`
     - Reads mint addresses from `scripts/deploy/mint-keypairs/*.json` (or falls back to pda-manifest.json)
     - Reads ALT address from `scripts/deploy/alt-address.json`
     - Reads treasury from env var or defaults to deployer wallet
     - Calls `generateDeploymentConfig()` and writes `deployments/{cluster}.json`
     - Run this script to generate the initial `deployments/devnet.json` from current state

  4. Create `deployments/` directory and generate `deployments/devnet.json` by running the script. Verify it contains all addresses matching current pda-manifest.json plus the new fields (bondingCurve program, curvePdas, hookAccounts, authority section, schemaVersion).
  </action>
  <verify>
  - `npx tsx scripts/deploy/generate-deployment-json.ts devnet` succeeds without errors
  - `deployments/devnet.json` exists and contains valid JSON with schemaVersion: 1
  - All program IDs in deployment.json match Anchor.toml [programs.devnet]
  - All mint addresses match existing pda-manifest.json
  - All PDA addresses match existing pda-manifest.json PDAs
  - TypeScript compiles: `npx tsc --noEmit scripts/deploy/lib/deployment-schema.ts`
  </verify>
  <done>deployments/devnet.json exists with every protocol address (7 programs, 3 mints, 19+ PDAs, 2 pools, 2 curves, hook accounts, ALT, treasury, authority). Schema types exported from deployment-schema.ts.</done>
</task>

  2. Create `.env.mainnet` as a template with placeholder values. Add comment header: `# Mainnet environment -- sourced by deploy-all.sh mainnet`. Mark sensitive fields with comments like `# Get from Helius dashboard`.

  3. Update `.gitignore`: Add `.env.mainnet` (mainnet secrets must never be committed). Keep `.env` in gitignore (existing). Do NOT gitignore `.env.devnet` -- devnet credentials are non-sensitive and useful for onboarding. Do NOT gitignore `deployments/` -- addresses are public on-chain.

  4. Extend `initialize.ts` to write `deployments/{cluster}.json` at the END of its initialization run (after all mints, PDAs, pools are created):

     - Import `DeploymentConfig` from `./lib/deployment-schema`
     - After all initialization steps complete, gather all program IDs (from Anchor workspace or env), mint addresses (from created mints), ALT address (from alt-address.json if exists), treasury (from env)
     - Call `generateDeploymentConfig()` to produce the config
     - Write to `deployments/{cluster}.json` where cluster is derived from CLUSTER_URL (devnet if URL contains "devnet", mainnet if URL contains "mainnet")
     - Log: "Wrote deployments/{cluster}.json"
     - Keep the existing pda-manifest.json write for backward compatibility (will be removed in a later migration)
  </action>
