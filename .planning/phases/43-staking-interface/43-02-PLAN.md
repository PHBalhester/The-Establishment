Create `app/hooks/useStaking.ts` mirroring the `useSwap.ts` pattern.

**Must be a "use client" module.**

**Types:**

```typescript
export type StakingAction = "stake" | "unstake" | "claim";

export type StakingStatus =
  | "idle"
  | "building"
  | "signing"
  | "sending"
  | "confirming"
  | "confirmed"
  | "failed";

export interface StakingResult {
  action: StakingAction;
  unstakedAmount?: number;  // PROFIT base units (for unstake)
  claimedAmount?: number;   // SOL lamports (for unstake + claim)
}

export interface UseStakingReturn {
  // Tab state
  activeTab: StakingAction;
  setActiveTab: (tab: StakingAction) => void;

  // Form state
  amount: string;                     // User-entered amount string
  setAmount: (amount: string) => void;

  // On-chain data (polled every 30 seconds)
  stakePoolData: {
    totalStaked: number;              // PROFIT base units
    pendingRewards: number;           // SOL lamports
    totalDistributed: number;         // SOL lamports (lifetime)
    totalClaimed: number;             // SOL lamports (lifetime)
  } | null;
  userStakeData: {
    stakedBalance: number;            // PROFIT base units
    rewardsEarned: number;            // SOL lamports
    totalClaimed: number;             // SOL lamports (lifetime)
  } | null;
  pendingRewards: number;             // Client-calculated SOL lamports
  dataLoading: boolean;

  // Reward rate stats
  rewardRate: {
    perEpochLamports: number;
    annualizedPct: number;
    totalStakedProfit: number;
    userSharePct: number;
  } | null;

  // Execution
  execute: () => Promise<void>;
  status: StakingStatus;
  txSignature: string | null;
  errorMessage: string | null;
  lastResult: StakingResult | null;

  // Wallet & balance
  connected: boolean;
  profitBalance: number;              // PROFIT display units

  // Reset
  resetForm: () => void;
}
```

**Implementation:**

1. **Data hooks**: Use `useProtocolWallet()` for wallet, `useTokenBalances(wallet.publicKey)` for PROFIT balance (reuse the `profit` field). Use `usePoolPrices()` for SOL/PROFIT price if a PROFIT pool is available (for annualized calculation).

2. **On-chain account polling**: Use `useState` + `useEffect` with 30-second interval:
   - Fetch `StakePool` via `getStakingProgram().account.stakePool.fetch(DEVNET_PDAS_EXTENDED.StakePool)`
   - Fetch `UserStake` via `getStakingProgram().account.userStake.fetch(deriveUserStakePDA(userPubkey))` -- wrap in try/catch, return null if account doesn't exist (user hasn't staked yet)
   - Extract BN values to numbers for React state: `totalStaked: bn.toNumber()`, etc.
   - For u128 fields (rewardsPerTokenStored, rewardsPerTokenPaid): store as BigInt using `BigInt(bn.toString())`
   - Re-fetch on wallet publicKey change and on "confirmed" status (refresh after transaction)

3. **Pending reward calculation**: Use `useMemo` calling `calculatePendingRewards()` from rewards.ts whenever stakePool or userStake data changes. Pass BigInt values for u128 fields.

4. **Reward rate**: Use `useMemo` calling `calculateRewardRate()` from rewards.ts with stakePool data.

5. **Execute function**: Switch on `activeTab`:
   - **"stake"**: Parse amount, call `buildStakeTransaction()`, sign, send, confirm
   - **"unstake"**: Parse amount, call `buildUnstakeTransaction()`, sign, send, confirm. Set `lastResult` with `{ action: "unstake", unstakedAmount, claimedAmount: pendingRewards }`.
   - **"claim"**: Call `buildClaimTransaction()`, sign, send, confirm. Set `lastResult` with `{ action: "claim", claimedAmount: pendingRewards }`.

   Follow the same lifecycle pattern as useSwap:
   ```
   setStatus("building") -> build TX -> set blockhash/feePayer
   setStatus("signing") -> wallet.signTransaction(tx)
   setStatus("sending") -> connection.sendRawTransaction(signed.serialize(), { skipPreflight: false, maxRetries: 2 })
   setStatus("confirming") -> connection.confirmTransaction()
   check confirmation.value.err -> "failed" with parseStakingError
   setStatus("confirmed") -> refresh balances + refresh staking data
   Auto-reset after 10 seconds
   ```

6. **Priority fee**: Use a simple `priorityFeePreset` state (same presets as useSwap: none/low/medium/high/turbo). Or reuse the same PRIORITY_FEE_MAP.

7. **Minimum stake warning**: Compute in the hook: if `activeTab === "unstake"` and amount is entered, check if `stakedBalance - amountBaseUnits < MINIMUM_STAKE && amountBaseUnits < stakedBalance`. Export a `minimumStakeWarning: boolean` field. MINIMUM_STAKE = 1_000_000 (1 PROFIT in base units).

8. **Cleanup**: Clear timers on unmount (same pattern as useSwap).

Import from Plan 01 artifacts:
- `buildStakeTransaction, buildUnstakeTransaction, buildClaimTransaction, deriveUserStakePDA` from `@/lib/staking/staking-builders`
- `parseStakingError` from `@/lib/staking/error-map`
- `calculatePendingRewards, calculateRewardRate` from `@/lib/staking/rewards`
