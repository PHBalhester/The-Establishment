    **Step 2 — Replace ModalCloseButton.tsx:**
    Replace the entire component. Remove the SVG X mark. Use a plain `<img>` tag (NOT next/image — asset is <2KB, Image optimization is overhead). Keep the same props interface and `aria-label="Close"`. New component:

    ```tsx
    'use client';

    /**
     * ModalCloseButton — Brass valve close button with rotation animation.
     *
     * Uses ExitButton.png asset (64x64 source, 32x32 rendered = 2x retina).
     * Hover: clockwise valve rotation + brass glow (CSS in globals.css).
     * Active/click: quick snap rotation.
     * Hidden on mobile (CSS breakpoint); mobile uses back arrow instead.
     */

    interface ModalCloseButtonProps {
      onClick: () => void;
    }

    export function ModalCloseButton({ onClick }: ModalCloseButtonProps) {
      return (
        <button
          type="button"
          className="modal-close-btn"
          onClick={onClick}
          aria-label="Close"
        >
          <img
            src="/buttons/exit-button.png"
            alt=""
            width={32}
            height={32}
            draggable={false}
          />
        </button>
      );
    }
    ```

    **Step 3 — Update globals.css `.modal-close-btn` rules (lines 528-573):**
    Replace the existing close button CSS block. Key changes:
    - Remove `background: linear-gradient(...)` (asset has its own visual treatment)
    - Remove `border: 1px solid #8b6914` (asset includes border)
    - Remove `box-shadow` beveled appearance (asset is pre-rendered)
    - Set `background: transparent`
    - Set `border: none`
    - Set `padding: 0`
    - Change `transition` to `transform 200ms ease, filter 200ms ease`
    - Add `.modal-close-btn img` rule: `width: 32px; height: 32px; display: block;`

    **Hover state (.modal-close-btn:hover):**
    - `transform: translateY(-50%) rotate(20deg)` (subtle clockwise valve turn + preserve centering)
    - `filter: brightness(1.1) drop-shadow(0 0 6px rgba(240, 192, 80, 0.4))` (brass glow via drop-shadow, NOT box-shadow — respects alpha channel)

    **Active state (.modal-close-btn:active):**
    - `transform: translateY(-50%) rotate(45deg)` (quick snap rotation)
    - `transition-duration: 80ms` (fast snap feel)
    - Remove the old brightness(0.95) and pressed-in box-shadow

    **Focus-visible (dialog .modal-close-btn:focus-visible):**
    - Keep existing circular `box-shadow` glow ring — no changes needed. Remove the old beveled `box-shadow` from the rule if it's inherited; focus should only show the glow ring.

    **CRITICAL — Transform stacking pitfall:**
    The base rule uses `transform: translateY(-50%)` for vertical centering. ALL hover/active transforms MUST combine: `translateY(-50%) rotate(Xdeg)`. Setting just `rotate()` removes the translateY and the button jumps 16px.

    **CRITICAL — Do NOT add will-change: transform:**
    This creates a compositor layer that can conflict with the iris-open `clip-path` animation on the dialog. The rotation is simple enough for automatic browser optimization.

    **Step 4 — Add overscroll-behavior to .modal-body (line ~520-526):**
    Add `overscroll-behavior: contain;` to the existing `.modal-body` rule. This prevents scroll chaining — when modal content is scrolled to the end, the page behind won't scroll.

    Apply to `.modal-body` specifically (NOT the dialog element). The dialog has `overflow: visible` for the floating close button; only `.modal-body` is the scroll container.

    **Step 5 — Verify mobile breakpoint (lines 1193-1196):**
    The existing `display: none` rule on `.modal-close-btn` at `width < 64rem` should continue working since we keep the same class name. No changes needed, but verify the rule is still present and hasn't been accidentally removed.

    **What NOT to change:**
    - `.modal-floating-close .modal-close-btn` positioning (top: -20px, right: -36px) — keep as-is
    - `.modal-floating-close` container rules — keep as-is
    - `.modal-header` rules — keep as-is
    - Mobile breakpoint `display: none` — keep as-is
    - `dialog :focus-visible` generic rule — keep as-is
