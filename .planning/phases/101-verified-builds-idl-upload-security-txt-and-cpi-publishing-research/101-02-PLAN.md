<task type="auto">
  <name>Task 1: Create verified-build.yml workflow</name>
  <files>.github/workflows/verified-build.yml</files>
  <action>
    Create `.github/workflows/verified-build.yml` with these specifications:

    **Trigger:** `workflow_dispatch` only (manual). No auto-run on push/PR. Include a required input for selecting cluster (mainnet or devnet) with mainnet as default:

    ```yaml
    on:
      workflow_dispatch:
        inputs:
          cluster:
            description: 'Target cluster (mainnet or devnet)'
            required: true
            default: 'mainnet'
            type: choice
            options:
              - mainnet
              - devnet
    ```

    **Runner:** `ubuntu-latest` with `timeout-minutes: 120` (7 programs, ~10-15 min each in Docker).

    **Environment variables:**
    - SOLANA_VERSION: "3.0.13"
    - ANCHOR_VERSION: "0.32.1"

    **Steps:**

    1. **Free disk space** -- same pattern as ci.yml (remove dotnet, android, ghc; expand swap). Docker images are large.

    2. **Checkout** -- `actions/checkout@v4`

    3. **Install Rust 1.93.0** -- `dtolnay/rust-toolchain@1.93.0`

    4. **Install solana-verify** -- `cargo install solana-verify` (this installs the CLI that wraps Docker builds)

    5. **Install Solana CLI** -- needed for `solana-verify get-executable-hash`. Same cache pattern as ci.yml.

    6. **Add Solana to PATH** -- same as ci.yml

    7. **Setup Node.js 22** -- needed for npm ci (Anchor IDL generation needs node)

    8. **Install npm dependencies** -- `npm ci`

    9. **Install Anchor CLI** -- needed for `anchor build` (IDL generation step):
        ```bash
        cargo install --git https://github.com/coral-xyz/anchor --tag v0.32.1 anchor-cli
        ```

    10. **Configure Solana CLI** -- set URL to localhost, generate a throwaway keypair

    11. **Generate IDLs via anchor build FIRST** (before solana-verify overwrites .so files):
        ```bash
        # anchor build generates IDLs at target/idl/*.json
        # We need these IDLs, but anchor build produces NON-deterministic .so files
        # So we run anchor build first for IDLs, then solana-verify build overwrites with deterministic .so files
        if [ "${{ inputs.cluster }}" = "devnet" ]; then
          SAFE_PROGRAMS="amm transfer_hook staking"
          FLAGGED_PROGRAMS="tax_program epoch_program conversion_vault bonding_curve"
          for lib in $SAFE_PROGRAMS; do
            anchor build -p "$lib"
          done
          for lib in $FLAGGED_PROGRAMS; do
            anchor build -p "$lib" -- --features devnet
          done
        else
          for lib in amm transfer_hook tax_program epoch_program staking conversion_vault bonding_curve; do
            anchor build -p "$lib"
          done
        fi
        # Save IDLs before solana-verify overwrites target/deploy/
        cp -r target/idl target/idl-saved
        ```

    12. **Build all 7 programs sequentially with solana-verify:**

        For mainnet (default -- no features):
        ```bash
        for lib in amm transfer_hook tax_program epoch_program staking conversion_vault bonding_curve; do
          echo "=== Building $lib ==="
          solana-verify build --library-name "$lib"
          echo "Hash: $(solana-verify get-executable-hash target/deploy/${lib}.so)"
        done
        ```

        For devnet cluster input:
        ```bash
        SAFE_PROGRAMS="amm transfer_hook staking"
        FLAGGED_PROGRAMS="tax_program epoch_program conversion_vault bonding_curve"
        for lib in $SAFE_PROGRAMS; do
          solana-verify build --library-name "$lib"
        done
        for lib in $FLAGGED_PROGRAMS; do
          solana-verify build --library-name "$lib" -- --features devnet
        done
        ```

        CRITICAL: For mainnet builds, do NOT pass any features. The default (no features) produces mainnet binaries. All 7 programs compile cleanly without features (bonding_curve uses 3-way cfg pattern, NOT compile_error!).

    13. **Restore saved IDLs** -- copy back from the saved directory:
        ```bash
        cp -r target/idl-saved target/idl
        ```

    14. **Verify security.txt is embedded** -- install `query-security-txt` and check each .so:
        ```bash
        cargo install query-security-txt
        for so in target/deploy/{amm,transfer_hook,tax_program,epoch_program,staking,conversion_vault,bonding_curve}.so; do
          echo "=== $(basename $so) ==="
          query-security-txt "$so"
        done
        ```

    15. **Generate hash manifest** -- create a JSON file with all 7 binary hashes:
        ```bash
        echo "{" > verified-hashes.json
        for lib in amm transfer_hook tax_program epoch_program staking conversion_vault bonding_curve; do
          hash=$(solana-verify get-executable-hash "target/deploy/${lib}.so")
          echo "  \"${lib}\": \"${hash}\"," >> verified-hashes.json
        done
        echo "}" >> verified-hashes.json
        ```

    16. **Upload .so artifacts** -- `actions/upload-artifact@v4` with name `verified-binaries`, path `target/deploy/*.so`, retention 90 days.

    17. **Upload hash manifest** -- `actions/upload-artifact@v4` with name `verified-hashes`, path `verified-hashes.json`, retention 90 days.

    18. **Upload IDL artifacts** -- `actions/upload-artifact@v4` with name `verified-idls`, path `target/idl/*.json`, retention 90 days.

    Key design decisions:
    - Sequential builds (not parallel) because Docker is resource-heavy and they share the Cargo cache
    - `anchor build` runs FIRST to generate IDLs at `target/idl/`, saved to `target/idl-saved/`
    - Then `solana-verify build` runs, which overwrites `target/deploy/*.so` with deterministic binaries
    - Saved IDLs are restored after solana-verify completes
    - No `anchor build` runs AFTER `solana-verify build` -- this would overwrite deterministic binaries (Pitfall 1 from RESEARCH.md)
    - The hash manifest is a convenience artifact for deployment verification
  </action>

<task type="auto">
  <name>Task 2: Local build validation -- verify security.txt embeds correctly</name>
  <files>None (validation only)</files>
  <action>
    Run a local anchor build (NOT solana-verify -- that requires Docker/x86) to validate that the security_txt! macro compiles and embeds correctly. This is proxy validation only -- the actual deterministic builds happen in CI (Plan 03). This step catches embedding issues before committing.

    1. Build all 7 production programs locally:
       ```bash
       source "$HOME/.cargo/env" && export PATH="/Users/mlbob/.local/share/solana/install/active_release/bin:$PATH"
       anchor build -p amm
       anchor build -p transfer_hook
       anchor build -p staking
       anchor build -p epoch_program -- --features devnet
       anchor build -p tax_program -- --features devnet
       anchor build -p conversion_vault -- --features devnet
       anchor build -p bonding_curve -- --features devnet
       ```

    2. Install query-security-txt locally and verify each binary:
       ```bash
       cargo install query-security-txt
       for so in target/deploy/{amm,transfer_hook,tax_program,epoch_program,staking,conversion_vault,bonding_curve}.so; do
         echo "=== $(basename $so) ==="
         query-security-txt "$so"
         echo ""
       done
       ```

       Each should output the security.txt fields: name, project_url, contacts, policy, preferred_languages, auditors, expiry.

    3. Verify CPI builds do NOT include security.txt:
       ```bash
       # Build with no-entrypoint feature (CPI mode)
       cargo build --lib -p amm --features no-entrypoint --release 2>&1 | tail -5
       # The library should compile fine without security_txt references
       ```

    4. Run the full Rust test suite to confirm no regressions:
       ```bash
       cargo test --workspace --features devnet
       ```

    This is the pre-CI validation. If any of these fail, fix before pushing to trigger the GitHub Actions workflow.

    NOTE: This task does NOT produce the final mainnet binaries. The CI workflow (Plan 03) does that. Local builds are non-deterministic (Apple Silicon != x86 Linux Docker). The purpose here is purely to validate that security.txt embedding works correctly before committing.
  </action>

    - `cargo test --workspace --features devnet` passes with zero failures
  </verify>
  <done>
