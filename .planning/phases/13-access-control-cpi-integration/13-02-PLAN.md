For direct call tests:
- The AMM expects swap_authority with seeds::program = TAX_PROGRAM_ID
- If we try to call with a different PDA (wrong seeds::program), Anchor's constraint check fails
- If we try to call without swap_authority at all, we get a missing account error
- The specific error code depends on how Anchor validates - we should capture the actual error

Note: Direct calls will fail because:
1. If swap_authority account is missing entirely: "Account not found" or similar
2. If swap_authority is provided but not a valid PDA from TAX_PROGRAM_ID: Anchor's seeds constraint fails with "A]count seeds constraint was violated"
3. If swap_authority is valid PDA but not signed: "missing required signature"
