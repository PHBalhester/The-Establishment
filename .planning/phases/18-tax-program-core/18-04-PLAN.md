    Implement the handler following Tax_Pool_Logic_Spec.md Section 14.3 (SELL: IP -> SOL):

    **Key difference from buy:** Tax is applied to OUTPUT, not input.

    Flow:
    1. User sends full token amount to AMM (no deduction)
    2. AMM returns gross SOL output
    3. Tax Program calculates tax on gross SOL: `tax = sol_gross * sell_tax_bps / 10_000`
    4. Tax Program distributes tax
    5. User receives net SOL: `sol_net = sol_gross - tax`
    6. Slippage check: `sol_net >= minimum_output`

    **Challenge:** The AMM CPI executes the full swap including SOL transfer to user. We need to intercept and redirect.

    **Solution approach:** Per RESEARCH.md, the Tax Program needs to handle this differently:

    Option A (simpler): Have user's WSOL account receive gross amount, then Tax Program does system transfers to redirect tax portions. But WSOL is Token-2022 - we'd need to unwrap.

    Option B (per spec pattern): The Tax Program acts as an intermediary:
    1. User sends tokens
    2. AMM sends gross SOL to a Tax Program-controlled account (or directly to user)
    3. Tax Program transfers tax portions from... somewhere

    **Actually, looking at spec more carefully:**
    - Tax_Pool_Logic_Spec.md Section 4 says taxes are collected in native SOL
    - Section 10.3 says "Tax deducted from SOL output"
    - The pattern is: AMM outputs WSOL to user, Tax needs to intercept

    **Practical implementation:**
    Since AMM swap puts WSOL in user_wsol account, and we distribute native SOL:
    1. Call AMM CPI: user trades token for WSOL
    2. The user receives WSOL in their account
    3. Calculate tax on the received WSOL amount
    4. User needs to have SOL balance for tax distribution (or we need a holding pattern)

    **Simpler approach for Phase 18:**
    The user provides the tax upfront as native SOL. The AMM swap gives them WSOL.
    Tax = (expected_gross_output * sell_tax_bps / 10_000)
    User must have this much native SOL available.

    **BUT** this creates UX complexity. Let me re-read spec...

    Looking at Tax_Pool_Logic_Spec.md Section 4: "SELL (IP -> SOL): tax deducted from SOL output"

    The cleanest implementation:
    1. Execute AMM swap: user's token -> user's WSOL
    2. Calculate tax on the WSOL received
    3. Tax Program needs authority to transfer FROM user's WSOL to distribution targets
       - But distribution targets hold native SOL, not WSOL

    **Resolution:** The tax distribution must use native SOL. The user must have native SOL to pay the tax, OR we need to unwrap WSOL.

    For Phase 18, implement the straightforward pattern:
    - User pre-pays tax in native SOL (from their main SOL balance)
    - Swap executes
    - Net output = gross output - tax paid

    This matches RESEARCH.md pattern and avoids WSOL unwrapping complexity.

    pub fn handler<'info>(
        ctx: Context<'_, '_, 'info, 'info, SwapSolSell<'info>>,
        amount_in: u64,        // Token amount to sell
        minimum_output: u64,   // Minimum net SOL after tax
        is_crime: bool,
    ) -> Result<()> {
        // 1. Read sell tax rate
        let tax_bps = if is_crime {
            // epoch_state.crime_sell_tax_bps
            1400u16  // TODO: Read from EpochState
        } else {
            // epoch_state.fraud_sell_tax_bps
            1400u16  // TODO: Read from EpochState
        };

        // 2. We need to estimate gross output to calculate tax
        //    But we can't know exact output until AMM executes...
        //
        //    Solution: Execute swap first, then calculate tax on actual output.
        //    User must have enough native SOL for worst-case tax.

        // 3. Build AMM CPI for BtoA direction (Token -> SOL)
        //    Capture the output amount after CPI via account balance diff

        // Save user's WSOL balance before swap
        let wsol_before = ctx.accounts.user_wsol.amount;

        // Execute AMM CPI (direction = BtoA = 1)
        let swap_authority_seeds: &[&[u8]] = &[
            b"swap_authority",
            &[ctx.bumps.swap_authority],
        ];

        // ... build instruction same as swap_sol_buy but with direction = 1 (BtoA)
        // ... execute invoke_signed

        // Reload user_wsol to get updated balance
        ctx.accounts.user_wsol.reload()?;
        let wsol_after = ctx.accounts.user_wsol.amount;
        let gross_output = wsol_after.checked_sub(wsol_before)
            .ok_or(error!(TaxError::TaxOverflow))?;

        // 4. Calculate tax on gross output
        let tax_amount = crate::helpers::tax_math::calculate_tax(gross_output, tax_bps)
            .ok_or(error!(TaxError::TaxOverflow))?;

        // 5. Calculate net output and check slippage
        let net_output = gross_output.checked_sub(tax_amount)
            .ok_or(error!(TaxError::TaxOverflow))?;

        require!(
            net_output >= minimum_output,
            TaxError::SlippageExceeded
        );

        // 6. Split distribution
        let (staking_portion, carnage_portion, treasury_portion) =
            crate::helpers::tax_math::split_distribution(tax_amount)
                .ok_or(error!(TaxError::TaxOverflow))?;

        // 7. Execute native SOL transfers for tax distribution
        //    User pays tax from their native SOL balance
        if staking_portion > 0 {
            solana_program::program::invoke(
                &solana_program::system_instruction::transfer(
                    ctx.accounts.user.key,
                    ctx.accounts.staking_escrow.key,
                    staking_portion,
                ),
                &[
                    ctx.accounts.user.to_account_info(),
                    ctx.accounts.staking_escrow.to_account_info(),
                    ctx.accounts.system_program.to_account_info(),
                ],
            )?;
        }
        if carnage_portion > 0 {
            solana_program::program::invoke(
                &solana_program::system_instruction::transfer(
                    ctx.accounts.user.key,
                    ctx.accounts.carnage_vault.key,
                    carnage_portion,
                ),
                &[
                    ctx.accounts.user.to_account_info(),
                    ctx.accounts.carnage_vault.to_account_info(),
                    ctx.accounts.system_program.to_account_info(),
                ],
            )?;
        }
        if treasury_portion > 0 {
            solana_program::program::invoke(
                &solana_program::system_instruction::transfer(
                    ctx.accounts.user.key,
                    ctx.accounts.treasury.key,
                    treasury_portion,
                ),
                &[
                    ctx.accounts.user.to_account_info(),
                    ctx.accounts.treasury.to_account_info(),
                    ctx.accounts.system_program.to_account_info(),
                ],
            )?;
        }

        // 8. Emit TaxedSwap event
        let clock = Clock::get()?;
        emit!(TaxedSwap {
            user: ctx.accounts.user.key(),
            pool_type: if is_crime { PoolType::SolCrime } else { PoolType::SolFraud },
            direction: SwapDirection::Sell,
            input_amount: amount_in,
            output_amount: net_output,
            tax_amount,
            tax_rate_bps: tax_bps,
            staking_portion,
            carnage_portion,
            treasury_portion,
            epoch: 0,  // TODO: Read from EpochState
            slot: clock.slot,
        });

        Ok(())
    }
