  Both bonding curves are now LIVE on devnet. The bonding curve program was deployed and curves initialized via Stage 5.

  <how-to-verify>
  **Coordinate with your remote tester and test the following:**

  **Pre-test setup:**
  1. Share the Railway devnet frontend URL with your remote tester
  2. Ask them for their wallet address
  3. Send them 2-3 devnet SOL: `solana transfer <TESTER_ADDRESS> 2.5 --url devnet --keypair keypairs/devnet-wallet.json`
  4. Both of you connect Phantom/Solflare (set to devnet) to the frontend

  **Manual UX testing (both users):**
  1. Visit the launch page -- verify bonding curve UI is displayed for both CRIME and FRAUD
  2. Buy CRIME with a small amount (0.05-0.1 SOL) -- verify TX confirms, balance updates, gauge moves
  3. Buy FRAUD with a small amount (0.05-0.1 SOL) -- verify same
  4. Try SELLING some CRIME back -- verify sell works, SOL returns, gauge adjusts
  5. Try SELLING some FRAUD back -- verify same
  6. Watch pressure gauges update in real-time on BOTH screens
  7. Verify wallet balances update correctly after each transaction
  8. Test error states: try buying with insufficient balance, test max wallet cap if applicable

  **What to look for:**
  - Smooth transaction flow (connect -> approve -> confirm -> balance update)
  - Pressure gauges reflect purchases from both users
  - Sell proceeds return correct SOL amount
  - No stale data or ghost transactions
  - Error messages are clear and helpful

  **After manual testing is complete:** Claude will run the fill script to bring both curves to capacity quickly.
  </how-to-verify>
  <resume-signal>Type "approved" after manual UX testing is satisfactory, and Claude will run the fill script to complete both curves. Describe any issues if found.</resume-signal>
