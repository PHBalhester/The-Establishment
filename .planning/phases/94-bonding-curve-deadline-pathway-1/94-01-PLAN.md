**On-chain constants (constants.rs):**

Add three-way `#[cfg]` devnet variants for these constants, following the exact pattern already used for `crime_mint()`, `fraud_mint()`, `epoch_program_id()`:

1. **DEADLINE_SLOTS**: Add devnet variant = 4,500 (~30 min at 400ms/slot). Change existing `#[cfg(not(feature = "localnet"))]` to `#[cfg(not(any(feature = "devnet", feature = "localnet")))]`.

2. **P_START and P_END**: Derive devnet values for 5 SOL target. The integral formula: `TARGET_SOL = (P_START + P_END) / 2 * TOTAL_FOR_SALE / TOKEN_DECIMAL_FACTOR`. Working backwards:
   - 5_000_000_000 = (P_START + P_END) / 2 * 460_000_000
   - (P_START + P_END) = 5_000_000_000 * 2 / 460_000_000 = 21.739...
   - Maintaining ratio P_END/P_START = 3450/900 = 3.8333:
   - 4.8333 * P_START = 21.739 -> P_START = 4.497 -> round to 5
   - P_END = 5 * 3.8333 = 19.17 -> round to 17 to get close to 5 SOL
   - CHECK: (5 + 17) / 2 * 460_000_000 = 11 * 460_000_000 = 5,060,000,000 lamports = 5.06 SOL. Close enough.
   - Use P_START=5, P_END=17 for devnet.
   - Verify the compile-time assertion `P_END > P_START` still passes.

3. **TARGET_SOL**: Add devnet variant = 5_000_000_000 (5 SOL in lamports).

4. **MIN_PURCHASE_SOL**: Add devnet variant = 1_000_000 (0.001 SOL) for testability with small amounts.

For constants that DON'T change across environments (TOTAL_FOR_SALE, TARGET_TOKENS, MAX_TOKENS_PER_WALLET, SELL_TAX_BPS, BPS_DENOMINATOR, PRECISION, seeds, TOKEN_DECIMALS), leave them as-is (no cfg needed).

