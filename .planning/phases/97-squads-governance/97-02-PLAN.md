       a. Generates 3 signer keypairs (writes to keypairs/squads-signer-{1,2,3}.json if they don't exist, loads them if they do — idempotent)
       b. Funds each signer with 0.05 SOL from devnet wallet (enough for voting transactions)

       e. Derives vaultPda via `multisig.getVaultPda({ multisigPda, index: 0 })`
       f. Fetches Squads program config to get treasury address: `multisig.accounts.ProgramConfig.fromAccountAddress(connection, programConfigPda)`
       g. Calls `multisig.rpc.multisigCreateV2()` with:
          - threshold: 2
          - members: all 3 signers with Permissions.all()
          - timeLock: Number(process.env.SQUADS_TIMELOCK_SECONDS) || 300
          - configAuthority: null (autonomous — multisig governs itself)
          - rentCollector: null
          - sendOptions: { skipPreflight: true }
       h. Logs multisig PDA address and vault PDA address clearly
       i. Updates deployments/devnet.json: sets authority.squadsVault to the vault PDA address, adds squadsMultisig and squadsCreateKey fields
       j. If multisig already exists (idempotent check), skip creation and just log addresses

    c. **Transfers 7 program upgrade authorities** to vault PDA:
       - For each program (amm, transferHook, taxProgram, epochProgram, staking, conversionVault, bondingCurve):
         - Use Solana CLI: `solana program set-upgrade-authority <PROGRAM_ID> --new-upgrade-authority <VAULT_PDA> --skip-new-upgrade-authority-signer-check --keypair keypairs/devnet-wallet.json --url devnet`
         - OR construct the BPFLoaderUpgradeable SetAuthority instruction in TypeScript if CLI has path-with-spaces issues (see Research Pitfall 5)
         - Verify after each transfer by reading ProgramData account and confirming upgrade authority matches vault PDA
         - Log success/failure for each program
    d. **Transfers 3 admin PDA authorities** to vault PDA (using the NEW instructions from Plan 01):
       - AMM: call `transfer_admin(vaultPda)` with deployer as admin signer, adminConfig PDA
       - Transfer Hook: call `transfer_authority(vaultPda)` with deployer as authority signer, whitelistAuthority PDA
       - Bonding Curve: call `transfer_bc_admin(vaultPda)` with deployer as authority signer, bcAdminConfig PDA
       - Verify after each by fetching the PDA account and checking admin/authority field equals vault PDA
    e. Updates deployments/devnet.json: sets authority.transferredAt to ISO timestamp
    f. Script is idempotent: checks current authority before transferring, skips if already transferred

    a. Reads deployments/devnet.json for all addresses

       - For AMM AdminConfig PDA: fetch account, assert admin == vault PDA
       - For WhitelistAuthority PDA: fetch account, assert authority == Some(vault PDA)
       - For BcAdminConfig PDA: fetch account, assert authority == vault PDA
    c. **Negative check**: Attempt to deploy a dummy .so to the Conversion Vault program from the deployer keypair. Confirm it FAILS with authority mismatch. This proves the deployer can no longer upgrade.
    d. Print summary table: program name | authority holder | status (PASS/FAIL)
    e. Exit code 0 if all pass, 1 if any fail
