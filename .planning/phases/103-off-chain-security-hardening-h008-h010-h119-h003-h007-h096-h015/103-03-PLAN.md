<task type="auto">
  <name>Task 1: Switch Railway builds to npm ci and fix workspace dependency</name>
  <files>railway-crank.toml, railway.toml, app/package.json</files>
  <action>
  Three one-line changes:

  **1. railway-crank.toml (H003):**
  Change `buildCommand = "npm install"` to `buildCommand = "npm ci"`. This enforces the exact lockfile, preventing supply chain drift. The crank runs from workspace root, so npm ci correctly resolves all workspace dependencies.

  **2. railway.toml (H003 defense-in-depth):**
  The current buildCommand is `buildCommand = "npm run --workspace app build"`. Nixpacks may or may not auto-run `npm ci` before this -- it depends on Nixpacks version and detection logic. To be safe, prepend npm ci: `buildCommand = "npm ci && npm run --workspace app build"`. If Nixpacks already ran npm ci, a second run is nearly instant (no-op when node_modules matches lockfile). The cost is ~1s slower builds. The benefit is guaranteed lockfile enforcement regardless of Nixpacks behavior.

  **3. app/package.json (H007):**
  Change `"@dr-fraudsworth/shared": "0.0.1"` to `"@dr-fraudsworth/shared": "workspace:*"`. The workspace:* protocol hard-fails if workspace resolution fails (instead of silently falling back to npm registry). This prevents dependency confusion attacks where an attacker publishes a malicious @dr-fraudsworth/shared on npm.

  After making all 3 changes, regenerate the lockfile by running `export PATH="/opt/homebrew/bin:$PATH" && cd "/Users/mlbob/Projects/Dr Fraudsworth" && npm install` from the workspace root. This updates package-lock.json to reflect the workspace:* protocol. Verify the lockfile was updated by checking git diff on package-lock.json.
  </action>
  <verify>
  Grep for "npm ci" in railway-crank.toml and railway.toml. Grep for "workspace:\\*" in app/package.json. Confirm "0.0.1" no longer appears for @dr-fraudsworth/shared in app/package.json. Run `export PATH="/opt/homebrew/bin:$PATH" && cd "/Users/mlbob/Projects/Dr Fraudsworth" && npm ci` to verify the lockfile resolves correctly. Run `export PATH="/opt/homebrew/bin:$PATH" && cd "/Users/mlbob/Projects/Dr Fraudsworth/app" && npx next build 2>&1 | tail -20` to verify the app builds.
  </verify>
  <done>

  </done>
