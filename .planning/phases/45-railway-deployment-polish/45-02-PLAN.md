    **3. Modify `app/instrumentation-client.ts` -- APPEND Sentry init AFTER Buffer polyfills:**

    The existing file has Buffer polyfills and ends with `globalThis.Buffer = Buffer;`. APPEND (do not replace) Sentry initialization after the existing code:

    ```typescript
    // ── Sentry Client Init ──────────────────────────────────────────────
    // Appended AFTER Buffer polyfills. Order matters: Solana signing
    // libraries need Buffer.writeBigUInt64LE which is shimmed above.
    import * as Sentry from "@sentry/nextjs";

    if (process.env.NEXT_PUBLIC_SENTRY_DSN) {
      Sentry.init({
        dsn: process.env.NEXT_PUBLIC_SENTRY_DSN,
        tracesSampleRate: 0.1,  // 10% of transactions (cost control)
        replaysSessionSampleRate: 0,  // No session replay
        replaysOnErrorSampleRate: 1.0,  // 100% replay on errors
      });
    }
    ```

    Note the guard: only init Sentry if DSN is set. This allows local dev without Sentry configured.

    **7. Create `app/app/global-error.tsx`** (NEW file -- React error boundary):

    This is a Next.js App Router convention. It catches unhandled errors in client components and reports them to Sentry.

    ```tsx
    "use client";

    import * as Sentry from "@sentry/nextjs";
    import { useEffect } from "react";

    export default function GlobalError({
      error,
      reset,
    }: {
      error: Error & { digest?: string };
      reset: () => void;
    }) {
      useEffect(() => {
        Sentry.captureException(error);
      }, [error]);

      return (
        <html>
          <body>
            <div style={{ padding: "2rem", fontFamily: "monospace" }}>
              <h2>Something went wrong</h2>
              <p>An unexpected error occurred. The error has been reported.</p>
              <button
                onClick={() => reset()}
                style={{
                  marginTop: "1rem",
                  padding: "0.5rem 1rem",
                  cursor: "pointer",
                }}
              >
                Try again
              </button>
            </div>
          </body>
        </html>
      );
    }
    ```

    Minimal styling (inline) because this page renders when the app itself has errored -- Tailwind may not be available.

    **1. Pin all dependency versions in `app/package.json`:**

    Replace all `^` prefixes with exact versions in both `dependencies` and `devDependencies`. This prevents `npm install` from silently pulling newer (potentially compromised or breaking) versions.

    Process:
    - Read the current `app/package.json`
    - For each dependency, remove the `^` prefix (e.g., `"^0.32.1"` becomes `"0.32.1"`)
    - Write the updated package.json

    Why this matters for a DeFi app: Supply chain attacks via malicious package updates are a real threat. Pinned versions ensure deterministic builds. Updates are intentional, not automatic.

    **2. Run npm audit:**

    ```bash
    export PATH="/opt/homebrew/bin:$PATH"
    cd app && npm audit --audit-level=high
    ```

    If high/critical vulnerabilities are found:
    - For direct dependencies: update to patched version
    - For transitive dependencies: try `npm audit fix` (non-breaking only)
    - If unfixable: document the vulnerability and assess if it's exploitable in our context

    The goal is zero high/critical vulnerabilities. Low/moderate are acceptable for now (reviewed periodically).

    **3. Regenerate package-lock.json with pinned versions:**

    After pinning, run `npm install` to regenerate the lockfile:
    ```bash
    export PATH="/opt/homebrew/bin:$PATH"
    cd app && npm install
    ```

    The lockfile captures the exact dependency tree including transitive dependencies.
  </action>
