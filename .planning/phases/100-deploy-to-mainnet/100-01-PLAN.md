  <name>Task 2: Modify setup-squads.ts for mainnet signer architecture</name>
  <files>scripts/deploy/setup-squads.ts</files>
  <action>
    Modify setup-squads.ts to support two signer modes:

    **Devnet mode (existing behavior, preserved):** When no SQUADS_SIGNER_2_PUBKEY or SQUADS_SIGNER_3_PUBKEY env vars are set, the script auto-generates file keypairs at keypairs/squads-signer-{1,2,3}.json as it does today. This preserves backward compatibility.

    **Mainnet mode (new):** When SQUADS_SIGNER_2_PUBKEY and SQUADS_SIGNER_3_PUBKEY env vars are set:
    - Signer 1: Load from file keypair at keypairs/squads-signer-1.json (the script proposer -- needs private key for transaction signing)
    - Signer 2: Use `new PublicKey(process.env.SQUADS_SIGNER_2_PUBKEY)` -- the Phantom browser wallet pubkey
    - Signer 3: Use `new PublicKey(process.env.SQUADS_SIGNER_3_PUBKEY)` -- the Ledger hardware wallet pubkey
    - Do NOT fund signers 2 and 3 (they fund themselves via their own wallets)
    - Only fund signer 1 if needed

    The multisig creation call (`multisig.instructions.multisigCreateV2`) only needs public keys for member registration -- no private keys required for members 2 and 3. The `configAuthority` remains null (autonomous multisig).

    Add clear console output indicating which mode is active:
    - "Signer mode: DEVNET (all file keypairs)"
    - "Signer mode: MAINNET (signer 1 file + signers 2,3 pubkey-only)"

    Do NOT change the multisig threshold (2), timelock logic, deployment.json update, or idempotency behavior.
  </action>
  <verify>

    2. `grep -c "SQUADS_SIGNER_2_PUBKEY" scripts/deploy/setup-squads.ts` returns >= 2

    4. TypeScript compiles: `export PATH="/opt/homebrew/bin:$PATH" && npx tsc --noEmit scripts/deploy/setup-squads.ts 2>&1 | head -20` (may have ambient type issues but no new errors)
  </verify>
