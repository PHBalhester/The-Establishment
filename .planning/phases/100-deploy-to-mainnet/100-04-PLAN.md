  <files></files>
  <action>
    Execute Stage 7 on mainnet. This creates the Squads multisig and transfers all authorities.

    **Step 1: Create Squads multisig**

    Ensure .env.mainnet has correct signer pubkeys:
    - SQUADS_SIGNER_2_PUBKEY (Phantom wallet pubkey)
    - SQUADS_SIGNER_3_PUBKEY (Ledger wallet pubkey)
    - SQUADS_TIMELOCK_SECONDS=900 (15 minutes initial timelock)

    ```bash
    set -a && source .env.mainnet && set +a
    npx tsx scripts/deploy/setup-squads.ts
    ```

    This creates the 2-of-3 multisig and writes squadsVault, squadsMultisig, squadsCreateKey to deployments/mainnet.json.

    Verify: multisig exists on-chain, vault PDA correct, threshold = 2, timelock = 900s.

    **Step 2: Transfer all authorities**

    Execute stage-7-governance.sh which runs transfer-authority.ts:
    ```bash
    ./scripts/deploy/stage-7-governance.sh mainnet
    ```

    This transfers:
    - 7 program upgrade authorities (AMM, Hook, Tax, Epoch, Staking, Vault, BC) to Squads vault
    - AMM AdminConfig.admin to Squads vault
    - Transfer Hook WhitelistAuthority to Squads vault (NOT burned at graduation -- retained for future flexibility)
    - BcAdminConfig.admin to Squads vault
    - 3 token metadata update authorities (CRIME, FRAUD, PROFIT mints) to Squads vault

    The script requires typing "TRANSFER" to confirm.

    **CRITICAL PITFALL AWARENESS:**
    - Pitfall #15 (BorshCoder snake_case): Already fixed in transfer-authority.ts (Phase 97). The script uses snake_case field names. Do NOT revert to camelCase.
    - Pitfall #12 (TX creator must be member): Use signer 1 (file keypair) as creator, NOT the deployer wallet.
    - Verify each transfer individually -- do not batch blindly.

    **Step 3: Verify all authorities**
    ```bash
    npx tsx scripts/deploy/verify-authority.ts mainnet
    ```

    Every authority must show as held by the Squads vault PDA, not the deployer.
  </action>
