**Toggle component** (`app/components/kit/Toggle.tsx`):

```tsx
interface ToggleProps {
  /** Current state */
  checked: boolean;
  /** Change handler */
  onChange: (checked: boolean) => void;
  /** Optional label */
  label?: string;
  disabled?: boolean;
  className?: string;
}
```

Implementation:
- Renders a button element (NOT input checkbox -- button is simpler for custom toggle)
- aria-pressed={checked}, role="switch", aria-label from label prop
- Visual: horizontal track (40px x 22px) with circular knob (18px) that slides left/right
- Track background: off = var(--color-toggle-off-track), on = var(--color-toggle-on-track)
- Knob: brass gradient (matches close button pattern), slides via CSS transform translateX
- Transition: --duration-kit-toggle (300ms) for weighted mechanical feel
- If label provided, render label text next to toggle in a flex container
- Applies .kit-interactive for shared hover/disabled behavior
- The knob slides from left (off) to right (on) with translateX(18px)

Kit CSS for Toggle:
- `.kit-toggle`: button base, position relative, w-40px h-22px, border-radius 11px, border 1px solid factory-border
- `.kit-toggle-track`: background transition between off/on colors
- `.kit-toggle-knob`: absolute, w-18px h-18px, border-radius 50%, brass gradient, box-shadow bevel
- `.kit-toggle[aria-pressed="true"] .kit-toggle-knob`: translateX(18px)
- `.kit-toggle-wrapper`: flex container for toggle + label

**Slider component** (`app/components/kit/Slider.tsx`):

```tsx
interface SliderProps {
  /** Current value */
  value: number;
  /** Change handler */
  onChange: (value: number) => void;
  /** Minimum value */
  min?: number;
  /** Maximum value */
  max?: number;
  /** Step increment */
  step?: number;
  /** Optional label */
  label?: string;
  /** Show current value display */
  showValue?: boolean;
  /** Value display format function */
  formatValue?: (value: number) => string;
  disabled?: boolean;
  className?: string;
}
```

Implementation:
- Renders native `<input type="range">` for maximum accessibility (keyboard arrows, screen reader support)
- Wraps in container div with optional label and value display
- Label: same style as Input label (serif, micro, uppercase, ink-secondary)
- Value display: positioned right of slider, monospace font, shows formatted value
- Default formatValue: `(v) => String(v)`
- Styled via CSS appearance: none + custom track/thumb pseudo-elements

Kit CSS for Slider:
- `.kit-slider`: appearance none, width 100%, height 6px, border-radius 3px
- `.kit-slider::-webkit-slider-runnable-track`: background var(--color-slider-track), height 6px, border-radius 3px
- `.kit-slider::-webkit-slider-thumb`: appearance none, w-20px h-20px, border-radius 50%, brass gradient, box-shadow bevel, margin-top for centering
- `.kit-slider::-moz-range-track`: matching Firefox track styles
- `.kit-slider::-moz-range-thumb`: matching Firefox thumb styles
- Focus-visible on thumb: glow ring
- Active thumb: slightly compressed (scale(0.95))
- `.kit-slider-wrapper`: flex column, gap
- `.kit-slider-header`: flex row between label and value display

**WCAG AA contrast verification:**

Compute actual contrast ratios using the W3C relative luminance formula for the new parchment color pairs. Add results as a comment block at the top of kit.css (matching the existing WCAG matrix format in globals.css).

Pairs to verify:
- Ink (#2a1f0e) on parchment (#f5e6c8): must exceed 4.5:1
- Ink secondary (#5a4830) on parchment (#f5e6c8): must exceed 4.5:1
- Brass highlight (#f0c050) on parchment (#f5e6c8): for decorative only (document ratio)
- Error (#c04030) on parchment (#f5e6c8): must exceed 3:1 for UI components
- Success (#5da84a) on parchment (#f5e6c8): must exceed 3:1 for UI components
- Factory-text (#ecdcc4) on toggle off track (#3d2b1a): must exceed 4.5:1
- Factory-text (#ecdcc4) on slider track (#3d2b1a): must exceed 4.5:1

Use the formula: L = 0.2126 * R + 0.7152 * G + 0.0722 * B (where R/G/B are linearized sRGB).
Ratio = (L_lighter + 0.05) / (L_darker + 0.05).

If any pair fails WCAG AA, adjust the token color value until it passes.
