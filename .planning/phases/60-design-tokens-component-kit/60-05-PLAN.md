interface InputProps extends React.InputHTMLAttributes<HTMLInputElement> {

  /** Optional error message displayed below input */

}

**Tabs component** (`app/components/kit/Tabs.tsx`):

Compound component pattern for maximum flexibility:

```tsx
interface TabsProps {
  /** Currently active tab value */
  value: string;
  /** Callback when tab changes */
  onChange: (value: string) => void;
  children: React.ReactNode;
}

interface TabListProps {
  children: React.ReactNode;
  className?: string;
}

interface TabProps {
  /** Unique value for this tab */
  value: string;
  children: React.ReactNode;
  disabled?: boolean;
  className?: string;
}

interface TabPanelProps {
  /** Show this panel when Tabs value matches */
  value: string;
  children: React.ReactNode;
  className?: string;
}
```

Implementation:
- Tabs: React Context provider holding {value, onChange}
- TabList: flex container with gap, renders children (Tab buttons)
- Tab: button element, reads context to determine if active (value === context.value)
  - Applies `kit-tab` + `kit-interactive` classes
  - Active state via `data-state="active"` attribute (matches existing .lever-tab pattern)
  - Click calls context.onChange(value)
  - aria-selected, role="tab"
- TabPanel: conditional render based on value match
  - role="tabpanel"
  - Only renders children when active (no hidden panels in DOM)

Kit CSS for Tabs (formalization of .lever-tab):
- `.kit-tab`: matches the existing .lever-tab styling exactly -- raised lever appearance with gradient, border, box-shadow
- `.kit-tab[data-state="active"]`: pressed down (translateY(2px)), inverted shadow, accent border, accent text color
- `.kit-tab-list`: display flex, gap 0 (tabs touch each other), with first/last border-radius

NOTE: The existing .lever-tab class in globals.css should NOT be removed. Kit Tabs is the new formal path.
