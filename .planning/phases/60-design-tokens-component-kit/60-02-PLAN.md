**Step 1: Add component-level tokens to the existing @theme block in globals.css.**

Add a new section "Component Kit Tokens" to the existing @theme block (after the animation tokens, before the closing brace). These tokens follow the established namespace conventions (--color-*, --duration-*, --shadow-*) so Tailwind v4 auto-generates utility classes.

Tokens to add:
```
/* Frame system */
--color-frame-parchment: #f5e6c8;
--color-frame-parchment-dark: #e8d5a8;
--color-frame-ink: #2a1f0e;
--color-frame-ink-secondary: #5a4830;
--color-frame-brass: #c4956a;
--color-frame-brass-highlight: #f0c050;
--color-frame-brass-shadow: #5a4510;

/* Interactive timing (weighted 200-300ms per CONTEXT.md) */
--duration-kit-hover: 250ms;
--duration-kit-press: 200ms;
--duration-kit-toggle: 300ms;

/* Slider */
--color-slider-track: #3d2b1a;
--color-slider-fill: #c4956a;

/* Toggle */

/* Hover glow (shared across all kit components per CONTEXT.md) */

```

**Step 3: Add @import for kit.css with layer declaration.**

At the TOP of globals.css, BEFORE the existing `@import "tailwindcss"`, add:
1. A layer order pre-declaration: `@layer theme, base, kit, components, utilities;`
2. Keep the existing `@import "tailwindcss";`
3. Add `@import "./kit.css" layer(kit);` AFTER the tailwindcss import

The final import section should look like:
```css
@layer theme, base, kit, components, utilities;
@import "tailwindcss";
@import "./kit.css" layer(kit);
```

IMPORTANT: If the layer order pre-declaration causes issues with Turbopack (the existing `@import "tailwindcss"` already declares its own layers), fall back to putting kit.css import after tailwindcss without pre-declaration. Test both approaches.

**Step 4: Create kit.css with foundational shared styles.**

Create `app/app/kit.css` with:

1. Shared interactive behavior class `.kit-interactive`:
   - Transition: filter, transform, box-shadow using --duration-kit-hover
   - Hover: brightness(1.1), translateY(-1px), box-shadow: --shadow-kit-hover-glow
   - Active: brightness(0.95), translateY(0), faster transition (--duration-kit-press)
   - Disabled: opacity 0.4, saturate(0.5), cursor not-allowed
   - Focus-visible: inherits from existing `dialog :focus-visible` pattern (box-shadow glow)

2. Frame base class `.kit-frame`:
   - position: relative
   - color: var(--color-frame-ink) -- auto-applies dark text for parchment readability

3. CSS-only frame `.kit-frame-css`:
   - Background: linear-gradient parchment
   - Border: 3px solid var(--color-factory-accent)
   - Border-radius: 8px
   - Box-shadow: multi-layer (matches existing .modal-chrome pattern)

4. Asset-based frame `.kit-frame-asset`:
   - Border: 30px solid var(--color-factory-accent) (fallback)
   - border-image-source: url placeholder (actual path set by component)
   - border-image-slice: 80 fill (placeholder, calibrated after Photoshop assets)
   - border-image-width: 30px
   - border-image-repeat: round
   - border-radius: 0 (explicit -- border-image ignores it)
   - background: none (fill paints above background, avoid layering)
   - Active state via [data-active="true"]: swaps border-image-source

5. A `.kit-focus` utility for focus-visible glow (reusable across kit):
   - outline: none
   - box-shadow: 0 0 0 2px var(--color-factory-glow), 0 0 8px rgba(240, 192, 80, 0.4)

Add descriptive comments explaining each section, consistent with the commenting style in globals.css.
