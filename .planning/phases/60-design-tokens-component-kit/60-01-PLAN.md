Analyze the entire codebase and all three reference images to build the comprehensive asset specification. The document must cover:

**Phase 60 assets (Component Kit):**
- Frame assets: ornate-paper.webp, ornate-paper-active.webp, riveted-paper.webp, riveted-paper-active.webp
  - Native size: 600x600px (2x for retina at 300px CSS render)
  - 9-slice anatomy: 80px corners for ornate, 40px for riveted
  - Parchment texture fill in center region
  - Transparent outside frame borders
- Button assets: Analyze current usage of .brass-button (ConnectModal, WalletButton, StakeTab, UnstakeTab, SlippageConfig) and .big-red-button (BigRedButton.tsx) to determine variant needs
  - Primary (action): brass-highlighted, prominent
  - Secondary (utility): dark recessed, subtle
  - Ghost: transparent with border only
  - Each variant: normal + active state (2 assets per variant)
- Input field: recessed dark metal frame (normal + focus states)
- Toggle switch: ON and OFF positions (2 separate assets)
- Slider: track texture + brass knob (separate assets)
- Divider: ornate scrollwork horizontal rule (referencing Image 1 center dividers)
- Scrollbar: track + thumb textures

**Phase 61 assets (Charts):**
- Chart frame: dedicated frame variant optimized for chart wrapper dimensions
- Chart watermark: "Dr. Fraudsworth" branded watermark image

**Phase 62 assets (Swap Station):**
- Custom scene-specific decorative elements for swap modal
- Riveted panel header asset for stats bar

**Phase 63 assets (Other Stations):**
- Gauge/meter component assets (Carnage fund balance, epoch progress, staked amount)
- Event card frame variant

**Phase 64 assets (Modal Infrastructure):**
- Brass valve close button (replacing current CSS-only close button)
- Gear loading spinner (individual gear asset for CSS rotation)

**Phase 65 assets (Settings):**
- Section divider variant (gauge-style)

**Phase 66 assets (Documentation):**
- Callout frame variants (info, warning, danger)
- Table header decoration

**Phase 67-68 assets (Audio):**
- No visual assets needed (audio files are separate concern)

For each asset, specify:
1. **Component name** and CSS class it maps to
2. **Native Photoshop creation size** (always 2x for retina)
3. **State variants** needed (Normal + Active for interactive; single state for decorative)
4. **File format**: WebP (lossy body + lossless alpha, quality 90/100)
5. **Output file name** matching the naming convention in 60-RESEARCH.md
6. **Which phase** uses this asset
7. **Reference image**: Which inspiration image to reference (Image 1 = frames/dividers, Image 2 = polished game UI components, Image 3 = weathered industrial variants)
8. **Creation notes**: Specific guidance on blending polished brass (Image 2) with weathered patina (Image 3) per the CONTEXT.md decisions

Also include:
- WebP export settings (lossy body + lossless alpha, quality 90/100)
- 9-slice slice region specifications with diagrams
- Color palette reference (brass: #c4956a to #f0c050, parchment: #f5e6c8 to #e8d5a8, ink: #2a1f0e)
- Asset naming convention: `{component}-{variant}[-{state}].webp`
- Directory structure: `public/frames/` for frame 9-slice, `public/kit/` for component assets

The aesthetic direction per CONTEXT.md:
- Ornate Victorian brass style with decorative scrollwork and filigree corners
- Two frame types: ornate Victorian AND functional riveted
- All content areas: aged parchment texture
- Text: dark brown/sepia "ink on paper"
- Blend of polished brass (Image 2) and weathered patina (Image 3)
