  <files>app/components/kit/Frame.tsx, app/app/kit.css</files>
  <action>
Create `app/components/kit/Frame.tsx` with TypeScript:

```tsx
interface FrameProps {
  /** 'css' = box-shadow/border with border-radius (for rounded elements)
   *  'ornate' | 'riveted' = border-image 9-slice (rectangular steampunk frames) */
  mode?: 'css' | 'ornate' | 'riveted';
  /** Active/pressed state -- swaps to Active asset in asset modes */
  active?: boolean;
  /** Additional padding inside the frame */
  padding?: 'none' | 'sm' | 'md' | 'lg';
  children: React.ReactNode;
  className?: string;
}
```

Implementation details:
- Default mode: 'css' (works without any Photoshop assets)
- CSS-only mode: renders div with `kit-frame kit-frame-css` classes + padding class
- Asset modes ('ornate' | 'riveted'): renders div with `kit-frame kit-frame-asset` classes
  - Sets `data-active={active}` attribute for CSS state switching
  - Sets `data-variant={mode}` attribute for CSS variant switching
  - The border-image-source will be set via CSS using data-variant attribute (NOT inline style) so all frame paths are in one CSS file and easy to update when assets are created
- Padding maps: none=0, sm=0.75rem, md=1.25rem, lg=2rem (via CSS classes .kit-frame-pad-sm etc.)
- Use forwardRef so Frame can be composed with other elements
- className prop merges with kit classes

Add to kit.css:
- `.kit-frame-pad-sm`, `.kit-frame-pad-md`, `.kit-frame-pad-lg` padding utilities
- `.kit-frame-asset[data-variant="ornate"]` with border-image-source pointing to `/frames/ornate-paper.webp`
- `.kit-frame-asset[data-variant="riveted"]` with border-image-source pointing to `/frames/riveted-paper.webp`
- `.kit-frame-asset[data-variant="ornate"][data-active="true"]` with active asset
- `.kit-frame-asset[data-variant="riveted"][data-active="true"]` with active asset

NOTE: Frame assets now exist in public/frames/ (created by user in Plan 60-03). The CSS references will resolve to real WebP files. The CSS fallback border (solid var(--color-factory-accent)) still provides graceful degradation if assets fail to load.

**Card component** (`app/components/kit/Card.tsx`):

Card is a semantic wrapper around Frame that adds header and content structure.

```tsx
interface CardProps {
  /** Frame mode passed to inner Frame */
  frame?: 'css' | 'ornate' | 'riveted';
  /** Optional card header text */
  header?: string;
  children: React.ReactNode;
  className?: string;
}
```

Implementation:
- Renders `<Frame mode={frame} padding="md">` wrapping children
- If header is provided, renders a `.kit-card-header` div above children with:
  - Serif heading font (font-heading)
  - Dark ink color (var(--color-frame-ink))
  - Bottom border separator (2px solid var(--color-factory-accent) at 30% opacity)
  - Margin bottom for spacing
- Card body wraps children in `.kit-card-body` div
- Use forwardRef
- Default frame mode: 'css'

**Divider component** (`app/components/kit/Divider.tsx`):

```tsx
interface DividerProps {
  /** 'simple' = single line, 'ornate' = decorative scrollwork, 'riveted' = bolted plate */
  variant?: 'simple' | 'ornate' | 'riveted';
  className?: string;
}
```

Implementation:
- Renders an `<hr>` element with appropriate CSS class
- 'simple': thin horizontal line in brass color with subtle gradient
- 'ornate': thicker line with decorative CSS ornaments (using ::before and ::after pseudo-elements for scrollwork dots at center and ends -- pure CSS, no images needed)
- 'riveted': line with small circular pseudo-element dots at regular intervals (simulating rivets)
- All variants: margin-y 1rem, color from --color-factory-accent
- role="separator" and aria-hidden="true" for a11y (decorative elements)

Add to kit.css:
- `.kit-card-header` styles (font, color, border, spacing)
- `.kit-card-body` styles (color: var(--color-frame-ink) for ink-on-parchment readability)
- `.kit-divider` base styles
- `.kit-divider-simple`, `.kit-divider-ornate`, `.kit-divider-riveted` variant styles
- Divider pseudo-elements for ornate scrollwork and riveted dots
