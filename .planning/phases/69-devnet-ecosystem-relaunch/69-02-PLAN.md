<task type="auto">
  <name>Task 2: Deploy programs, initialize protocol, verify, and rebuild with new mints</name>
  <files></files>
  <action>
    This task runs the full deployment pipeline. The sequence is critical — each step depends on the previous.

    **Step 1: Deploy all 6 programs to devnet**
    ```bash
    source "$HOME/.cargo/env"
    export PATH="/Users/mlbob/.local/share/solana/install/active_release/bin:$PATH"
    export PATH="/opt/homebrew/bin:$PATH"

    ./scripts/deploy/deploy.sh https://api.devnet.solana.com
    ```
    This deploys using existing program keypairs (same program IDs as before). All 6 should deploy successfully. The conversion vault is a fresh deploy; the other 5 are upgrades.

    **Step 2: Run initialize.ts to create mints, pools, vault, whitelist**
    ```bash
    set -a && source .env && set +a
    CLUSTER_URL=https://devnet.helius-rpc.com/?api-key=[REDACTED-DEVNET-HELIUS-KEY] \
      npx tsx scripts/deploy/initialize.ts
    ```
    This runs the 17-step initialization:
    - Generates new mint keypairs (saved to scripts/deploy/mint-keypairs/)
    - Creates 3 Token-2022 mints with MetadataPointer + TransferHook extensions
    - Mints 1B tokens of each
    - Creates 2 SOL pools (CRIME/SOL, FRAUD/SOL) with 2.5 SOL + 290M tokens each
    - Initializes conversion vault with 250M CRIME + 250M FRAUD + 20M PROFIT
    - Creates all whitelist entries
    - Initializes epoch state, staking pool, carnage fund
    - Burns mint authorities
    - Generates pda-manifest.json

    **Step 3: Rebuild with new mint addresses compiled in**
    After initialize.ts generates new mint keypairs, we must rebuild with --devnet so the conversion vault and tax program have the correct devnet mint addresses:
    ```bash
    ./scripts/deploy/build.sh --devnet
    ```
    Step [0/3] will now find the new mint-keypairs and patch them into constants.rs files.

    **Step 4: Re-deploy the 3 feature-flagged programs**
    Only epoch_program, tax_program, and conversion_vault need re-deployment (they have devnet-specific compiled constants):
    ```bash
    WALLET="keypairs/devnet-wallet.json"
    URL="https://api.devnet.solana.com"
    solana program deploy target/deploy/tax_program.so --program-id keypairs/tax-program-keypair.json --keypair $WALLET --url $URL --with-compute-unit-price 1
    solana program deploy target/deploy/epoch_program.so --program-id keypairs/epoch-program.json --keypair $WALLET --url $URL --with-compute-unit-price 1
    solana program deploy target/deploy/conversion_vault.so --program-id keypairs/vault-keypair.json --keypair $WALLET --url $URL --with-compute-unit-price 1
    ```

    **Step 5: Run verify.ts to confirm everything**
    ```bash
    CLUSTER_URL=https://devnet.helius-rpc.com/?api-key=[REDACTED-DEVNET-HELIUS-KEY] \
      npx tsx scripts/deploy/verify.ts
    ```
    All 36+ checks should PASS. Pay special attention to:
    - "CRIME/FRAUD/PROFIT MintAuthority Burned" checks
    - "ConversionVault State" checks
    - Pool reserve checks (should show 2.5 SOL worth of reserves)

    **Step 6: Recreate ALT**
    ```bash
    CLUSTER_URL=https://devnet.helius-rpc.com/?api-key=[REDACTED-DEVNET-HELIUS-KEY] \
      npx tsx scripts/e2e/lib/alt-helper.ts
    ```
    This creates a new ALT with all current addresses and saves the address to scripts/deploy/alt-address.json.

    **IMPORTANT:** If initialize.ts fails partway through, it is idempotent — re-run it and it will skip already-completed steps.

    **IMPORTANT:** If the wallet runs low on SOL during deployment, request an airdrop:
    `solana airdrop 2 --url https://api.devnet.solana.com --keypair keypairs/devnet-wallet.json`
  </action>
