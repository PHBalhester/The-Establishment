**app/hooks/useCarnageEvents.ts** -- Fetches last 5 Carnage events by parsing transaction logs:

- Mark `"use client"` at top.
- This hook uses `getSignaturesForAddress` on the CarnageFund PDA to find recent transactions, then `getParsedTransaction` to parse program logs for CarnageExecuted event data.

**Approach:**
1. Call `connection.getSignaturesForAddress(DEVNET_PDAS.CarnageFund, { limit: 20 })` to get recent transaction signatures involving the CarnageFund account. We fetch 20 because not all transactions are Carnage events (some might be fund deposits).
2. For each signature, call `connection.getParsedTransaction(sig, { maxSupportedTransactionVersion: 0 })`.
3. Search the transaction's `meta.logMessages` for Anchor event logs. Anchor emits events as base64-encoded data prefixed with `"Program data: "` in log messages. The CarnageExecuted event will be emitted by the Epoch Program.
4. Parse the event data to extract: epoch number, which token was targeted, amount burned, amount sold, SOL spent.
5. Also extract the block time from `transaction.blockTime` for the event date.
6. Collect up to 5 successfully parsed events and return them.

**CarnageEvent type:**
```typescript
interface CarnageEvent {
  signature: string;          // Transaction signature
  timestamp: number;          // Unix timestamp (from blockTime)
  epoch: number;              // Which epoch triggered Carnage
  tokenBurned: string;        // "CRIME" or "FRAUD"
  amountBurned: number;       // Token base units burned
  solSpent: number;           // Lamports spent on buy-for-burn
}
```

**Important caveats:**
- Anchor event parsing requires knowing the event discriminator (first 8 bytes = sha256("event:CarnageExecuted")[0..8]).
- If parsing is too complex or fragile, fall back to a simpler approach: check log messages for known strings like "Carnage" or instruction names, and extract basic info (epoch from logs, token from instruction data).
- This hook should poll infrequently (every 60 seconds or on demand) since Carnage events are rare. Use `useEffect` with a 60-second interval.
- Return `{ events: CarnageEvent[]; loading: boolean; error: string | null }`.
- If parsing fails for a transaction, skip it (don't crash the whole hook). Log a warning to console.
- Handle the case where no Carnage events have occurred yet (empty array, no error).

**Fallback approach if Anchor event parsing is too complex:**
- Instead of decoding base64 Anchor events, parse the simpler `Program log:` messages. Carnage execution logs will contain identifiable patterns from the program's `msg!()` calls.
- At minimum, extract: the transaction signature (for Explorer link), the block time (for date), and whether it's a Carnage transaction (by checking if CarnageFund PDA is involved and the epoch program is the caller).
- Even if we can only show "Carnage triggered at [date] - [explorer link]" without detailed breakdown, that's better than nothing for the tech layer.
