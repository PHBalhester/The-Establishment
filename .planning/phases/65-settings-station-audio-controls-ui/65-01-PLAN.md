Create `app/providers/SettingsProvider.tsx` following the established ModalProvider.tsx pattern (createContext + useCallback + Provider). The provider manages a `Settings` interface:

```typescript
interface Settings {
  slippageBps: number;        // Default: 100 (1%)
  priorityFeePreset: PriorityFeePreset;  // Default: 'medium'
  muted: boolean;             // Default: false (or true if prefers-reduced-motion)
  volume: number;             // Default: 20 (0-100 range)
}
```

Implementation details:
- `'use client'` directive at top
- Import `PriorityFeePreset` type from `@/hooks/useSwap` (it's exported as a type from there)
- Storage key: `'dr-fraudsworth-settings'`
- `loadSettings()` function: reads from localStorage with `typeof window !== 'undefined'` SSR guard. Validates each field individually with type checks, falling back to defaults for missing/invalid fields. When no localStorage exists, check `window.matchMedia('(prefers-reduced-motion: reduce)').matches` to default `muted: true` for accessibility.
- `useState` with lazy initializer `() => loadSettings()` -- runs only on client during hydration
- `updateSettings` callback: takes `Partial<Settings>`, merges with previous state, writes to localStorage synchronously inside the setter (NOT in a useEffect -- avoids one-render-behind issues per RESEARCH.md anti-pattern)
- Individual setters: `setSlippageBps`, `setPriorityFeePreset`, `setMuted`, `setVolume` -- each wraps `updateSettings` with the relevant partial
- Export `SettingsContext` (for the hook) and `SettingsProvider` (for providers.tsx)

Create `app/hooks/useSettings.ts` following the useModal.ts pattern:
- `'use client'` directive
- `useContext(SettingsContext)` with null check throwing `'useSettings must be used within a SettingsProvider.'`
- Returns the full `SettingsContextValue` (settings object + individual setters)
