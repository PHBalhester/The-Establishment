Insert a new "Preflight" section in deploy-all.sh AFTER the mainnet confirmation prompt (line ~194)
and BEFORE Phase 0 (mint keypairs, line ~197). This section runs on BOTH devnet and mainnet deploys.
Label it "Preflight" (not "Phase -1") to avoid renumbering all existing phases.

The preflight section performs four checks in order:

**Check 1: No keypairs in git staging**
- Run `git diff --cached --name-only 2>/dev/null`
- Filter for files matching patterns: `keypair`, `wallet`, `mint`, `deployer`, `mainnet` AND ending in `.json`
- Also catch any `.json` file inside `keypairs/` directory
- If matches found: print clear error listing the staged files, suggest `git reset HEAD <file>`, exit 1
- This runs on ALL clusters (devnet keypairs shouldn't be committed either)

**Check 2: All required env vars present**
- Define REQUIRED_VARS for all clusters: HELIUS_API_KEY, CLUSTER_URL, COMMITMENT, SOL_POOL_SEED_SOL_OVERRIDE, SOL_POOL_SEED_TOKEN_OVERRIDE
- Define MAINNET_EXTRA_VARS (only checked on mainnet): DEPLOYER_KEYPAIR, TREASURY_PUBKEY, MAINNET_MIN_BALANCE
- For each var: check if empty (`-z`) OR still set to "CHANGE_ME" or "CHANGE_ME_MAINNET"
- Collect all missing/placeholder vars and report them all at once (don't exit on first miss)
- If any missing: print clear error listing all missing vars, exit 1

**Check 3: WALLET sanity (mainnet only)**
- After WALLET is set (line 151: `export WALLET="${WALLET:-keypairs/devnet-wallet.json}"`), check:
  - On mainnet: if WALLET contains "devnet", print error "Mainnet deploy using devnet wallet. Set WALLET in .env.mainnet to ~/mainnet-keys/deployer.json", exit 1
  - On mainnet: if WALLET file doesn't exist, print error with the path, exit 1
  - On mainnet: DEPLOYER_KEYPAIR file must also exist

**Check 4: Deployer balance (mainnet only)**
- Use `solana balance --keypair "$WALLET" --url "$CLUSTER_URL"` to get balance
- Compare against MAINNET_MIN_BALANCE (default: 10 SOL if unset)
- Use `bc` for float comparison (or awk if bc unavailable)
- If balance < min: print current balance, required balance, exit 1

**Check 5: Binary hash verification (mainnet only)**
- Check if `deployments/expected-hashes.${CLUSTER}.json` exists
- If not: print error suggesting `./scripts/deploy/generate-hashes.sh mainnet`, exit 1
- For each .so in target/deploy/: compute `shasum -a 256`, compare against expected hash from JSON
- Use jq to extract expected hash (jq already used elsewhere in deploy-all.sh)
- If any mismatch: print program name, expected hash, actual hash, exit 1
- If all match: print "All binary hashes verified. OK."

- Add a summary line "Preflight checks passed." after all checks complete
