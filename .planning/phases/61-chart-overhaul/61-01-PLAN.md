  <action>
Create two new files in app/components/chart/:

**chart-theme.ts** -- Pure constants module (NO lightweight-charts imports, safe for server-side):
- Export `FACTORY_CHART_THEME` object with layout colors: background (#1c120a), textColor (#bca88a), fontSize (11), gridVertColor (#4a3520), gridHorzColor (#4a3520), scaleBorderColor (#86644a), crosshairColor (#86644a), crosshairLabelBg (#2c1e12). Each value has a comment mapping to its CSS token name (e.g., `// --color-factory-bg`).
- Export `FACTORY_CANDLE_COLORS` object: upColor, downColor, borderUpColor, borderDownColor, wickUpColor, wickDownColor -- all using #5da84a (success) and #c04030 (error).
- Export `FACTORY_VOLUME_COLORS` object: up = 'rgba(93, 168, 74, 0.35)', down = 'rgba(192, 64, 48, 0.35)'.
- Export `FACTORY_LEGEND_COLORS` object: text (#ecdcc4 = factory-text), label (#daa520 = factory-accent), bg ('rgba(28, 18, 10, 0.85)' = factory-bg with opacity).
- All objects use `as const` for type safety.
- This file has NO 'use client' directive and NO lightweight-charts imports (pure data, SSR-safe).

**create-chart.ts** -- Factory function (imports lightweight-charts, must only be imported from 'use client' files):
- NO 'use client' directive on this file (it doesn't render JSX, but it imports lightweight-charts which is DOM-dependent -- it relies on being imported only from 'use client' consumer files).
- Import createChart, CandlestickSeries, HistogramSeries, ColorType, PriceScaleMode, IChartApi, ISeriesApi from lightweight-charts.
- Import FACTORY_CHART_THEME, FACTORY_CANDLE_COLORS from './chart-theme'.
- Define and export `CreateChartOptions` interface: { width?: number; height: number; priceFormatter?: (price: number) => string; logScale?: boolean }.
- Define and export `CreateChartResult` interface: { chart: IChartApi; candleSeries: ISeriesApi<'Candlestick'>; volumeSeries: ISeriesApi<'Histogram'> }.
- Export `createThemedChart(container: HTMLDivElement, opts: CreateChartOptions): CreateChartResult` function that:
  1. Creates chart with FACTORY_CHART_THEME colors applied to layout, grid, timeScale, rightPriceScale, crosshair.
  2. Sets timeScale.timeVisible=true, secondsVisible=false.
  3. Sets rightPriceScale.mode to PriceScaleMode.Logarithmic if opts.logScale is true (default false).
  4. Enables data conflation: `handleScroll: { pressedMouseMove: true }, kineticScroll: { touch: true }` -- actually, conflation is NOT a top-level option in v5.1.0 typings. Skip conflation for now.
  5. Adds CandlestickSeries with FACTORY_CANDLE_COLORS, applies priceFormatter if provided.
  6. Adds HistogramSeries with priceScaleId: '' (overlay), lastValueVisible: false, priceLineVisible: false, priceFormat: { type: 'volume' }.
  7. Calls volumeSeries.priceScale().applyOptions({ scaleMargins: { top: 0.8, bottom: 0 } }) to position volume at bottom 20%.
  8. Returns { chart, candleSeries, volumeSeries }.
- Export `toVolumeData(candles: CandleData[]): HistogramData[]` helper that converts CandleData[] to HistogramData[] with per-bar color based on close >= open direction using FACTORY_VOLUME_COLORS. Import CandleData type from '@/hooks/useChartData' and HistogramData/UTCTimestamp from lightweight-charts.

WHY this split: chart-theme.ts is pure data (no DOM dependencies, SSR-safe, importable anywhere). create-chart.ts wraps the DOM-dependent lightweight-charts library. Keeping them separate means the OhlcLegend component (Plan 02) can import theme colors without pulling in the chart library.
  </action>

  </verify>

Both compile without errors.

**API route modification** (app/app/api/candles/route.ts):
- Add a `gapfill` query parameter. Parse it from searchParams: `const gapfill = searchParams.get('gapfill') !== 'false';` (defaults to true for backward compatibility).
- When gapfill is false, skip the gapFillCandles() call and return `rows.map(formatCandle)` directly.
- When gapfill is true (default), preserve existing behavior unchanged.
- Update the JSDoc comment at top of file to document the new `gapfill` parameter: "gapfill (optional) -- Set to 'false' to skip gap-fill and return only real trade candles. Defaults to true."
- This is backward compatible: no existing consumer breaks because the default is true.

WHY: The CONTEXT.md decision says synthetic zero-volume flat candles from gap-fill create misleading long flat lines. The chart will pass gapfill=false to get only real candles. Other consumers (if any) still get gap-filled data by default.
