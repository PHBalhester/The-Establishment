  <action>
Create a new wrapper component that provides the Frame border, loading state, empty state, keyboard accessibility, and aria-live region around the CandlestickChart.

**ChartWrapper.tsx** ('use client' directive required -- uses state/effects and imports client components):

Props interface `ChartWrapperProps`:
- `children: React.ReactNode` -- the CandlestickChart goes here
- `loading: boolean` -- whether historical data is fetching
- `isEmpty: boolean` -- true when candles array is empty AND not loading
- `poolLabel: string` -- human-readable pool name for aria-label (e.g., "CRIME/SOL")
- `latestPrice?: number` -- for aria-live screen reader announcements
- `chartRef?: React.RefObject<IChartApi | null>` -- optional ref to chart instance for keyboard nav

Component behavior:

1. **Frame wrapper**: Wrap everything in `<Frame mode="css" padding="none">`. Use padding="none" because the chart itself needs to fill edge-to-edge inside the frame. The Frame provides the steampunk border treatment from Phase 60.

2. **Keyboard navigation**: Add an onKeyDown handler on the outer container div (NOT the Frame, because Frame may not forward event handlers). The container div gets `tabIndex={0}` and `role="figure"` and `aria-label={`Price chart for ${poolLabel}`}`.
   ```
   onKeyDown={(e) => {
     const chart = chartRef?.current;
     if (!chart) return;
     const ts = chart.timeScale();
     switch (e.key) {
       case 'ArrowLeft':
         e.preventDefault();
         ts.scrollToPosition(ts.scrollPosition() - 3, false);
         break;
       case 'ArrowRight':
         e.preventDefault();
         ts.scrollToPosition(ts.scrollPosition() + 3, false);
         break;
       case '+':
       case '=':
         e.preventDefault();
         { const current = chart.options().timeScale?.barSpacing ?? 6;
           ts.applyOptions({ barSpacing: current + 2 }); }
         break;
       case '-':
         e.preventDefault();
         { const current = chart.options().timeScale?.barSpacing ?? 6;
           ts.applyOptions({ barSpacing: Math.max(current - 2, 2) }); }
         break;
     }
   }
   ```
   If chartRef is not provided, keyboard nav is a no-op (graceful degradation).

3. **aria-live region**: Add a visually-hidden aria-live polite region that announces price changes:
   ```
   <div className="sr-only" aria-live="polite" aria-atomic="true">
     {latestPrice !== undefined ? `${poolLabel} price: ${latestPrice}` : ''}
   </div>
   ```
   Use `sr-only` (Tailwind's screen-reader-only class) so it's invisible but announced.

4. **Loading state**: When `loading` is true AND `isEmpty` is true (initial load, no cached data), show the spinning gear:
   ```
   <div className="absolute inset-0 flex items-center justify-center bg-factory-bg/80 z-20">
     <img
       src="/logo-icon.png"
       alt="Loading chart data..."
       className="w-12 h-12 animate-gear-spin"
     />
   </div>
   ```
   When `loading` is true but NOT isEmpty (background refetch after pool/range change), show a smaller subtle loading indicator in the corner instead of overlaying the entire chart. A small spinning gear in the top-right corner: `absolute top-2 right-2 z-20 w-6 h-6 animate-gear-spin opacity-60`.

5. **Empty state**: When `isEmpty` is true AND `loading` is false (pool genuinely has no trades), render:
   ```
   <div className="flex flex-col items-center justify-center h-[300px] text-factory-text-secondary">
     <p className="font-heading text-lg">No trades recorded in this factory</p>
     <p className="text-sm text-factory-text-muted mt-2">
       Trades will appear here once swaps occur in this pool
     </p>
   </div>
   ```
   This is inside the Frame, so it still has the steampunk border.

6. **Normal state**: When not loading-initial and not empty, render `{children}` (the CandlestickChart).

Structure:
```tsx
<Frame mode="css" padding="none">
  <div
    className="relative"
    tabIndex={0}
    role="figure"
    aria-label={`Price chart for ${poolLabel}`}
    onKeyDown={handleKeyDown}
  >
    {/* Screen reader price announcement */}
    <div className="sr-only" aria-live="polite" aria-atomic="true">...</div>

    {/* Content: chart, loading, or empty */}
    {isEmpty && !loading ? <EmptyState /> : children}

    {/* Loading overlays */}
    {loading && isEmpty && <FullLoadingOverlay />}
    {loading && !isEmpty && <CornerLoadingIndicator />}
  </div>
</Frame>
```

WHY this design:
- Frame provides the steampunk border matching all other modals (CONTEXT.md decision).
- Keyboard nav makes this more accessible than any memecoin chart platform (RESEARCH.md finding).
- aria-live announces prices for screen readers (accessibility edge).
- Two loading states: full overlay for initial load, subtle corner for refetch (no jarring flash on pool switch).
- Empty state is informative, not a call-to-action (CONTEXT.md decision).
