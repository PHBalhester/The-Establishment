**BigRedButton.tsx -- Rewrite for asset-based layout:**

The current component is a CSS-only circular button with text content (SWAP, Connect, loading spinner, checkmark, error indicator). The new version uses a rectangular image asset with CSS state overlays.

1. Structure:
   ```tsx
   <button
     type="button"
     className="big-red-button-asset kit-interactive kit-focus"
     data-state={dataState}
     disabled={disabled && status === 'idle'}
     onClick={handleClick}
     aria-label={ariaLabel}
   >
     {/* The button image or CSS placeholder */}
     <div className="big-red-button-image-wrapper">
       <img
         src="/buttons/big-red-button.png"
         alt=""
         className="big-red-button-image"
         draggable={false}
         onError={(e) => {
           // Hide the broken image -- CSS placeholder kicks in
           (e.target as HTMLImageElement).style.display = 'none';
         }}
       />
     </div>

     {/* State overlays render on TOP of the image */}
     {dataState === 'loading' && (
       <div className="big-red-button-overlay">
         <LoadingSpinner />
       </div>
     )}
     {dataState === 'success' && (
       <div className="big-red-button-overlay big-red-button-success" />
     )}
     {dataState === 'error' && (
       <div className="big-red-button-overlay big-red-button-error" />
     )}
   </button>
   ```

2. Keep the existing data-state management logic (loading, success, error with auto-reset timers). This is the same logic from the current BigRedButton.

3. Keep the existing click handler logic (onSwap on idle, onReset on confirmed/failed).

4. Keep the existing aria-label derivation (context-aware labels for each state).

5. Remove the inline text content (`SWAP`, `Connect`, checkmark SVG, `!`). The button text is baked into the Photoshop asset. The CSS placeholder provides a temporary text label.

6. The `connected` prop still determines the aria-label, but no visual difference on the button itself (the asset is the same for all states -- overlays handle visual changes).

When the user eventually places the Photoshop asset at that path, the img tag loads successfully and the placeholder hides automatically.
