**Verification checks (in order):**

1. **Keypair existence:** For each program, check that the keypair file exists at the expected path. Report MISSING if not found.

2. **Keypair -> Pubkey derivation:** Read each keypair JSON, parse the byte array, create a `Keypair` from `@solana/web3.js`, extract pubkey as base58. This is the **source of truth** ID for each program.

3. **declare_id! match:** Read each program's `lib.rs`, extract the `declare_id!("...")` value using regex `/declare_id!\("([A-Za-z0-9]+)"\)/`. Compare against keypair-derived pubkey. Report MATCH or MISMATCH.

4. **Anchor.toml [programs.localnet] match:** Parse Anchor.toml (use simple line-by-line parsing since the format is predictable -- no need for a full TOML parser for this structure). For each program, check that the ID in Anchor.toml matches the keypair-derived pubkey.

5. **Anchor.toml [programs.devnet] match:** Same check for the devnet section. Report MISSING SECTION if devnet section doesn't exist. For production programs only.

6. **Cross-program references match:** For each entry in CROSS_REFS, read the source file, find the function containing the reference name, extract the ID using the pattern regex, and compare against the expected program's keypair-derived pubkey.

7. **Placeholder detection:** Scan all programs/ Rust files for known placeholder patterns:
   - `EpochProgram1111111111111111111111111111111`
   - Any pubkey that is all 1s after a prefix
   Report as ERROR if found.

JSON object with structure:

```json
  "timestamp": "ISO date",
  "programs": { "<name>": { "keypairId": "...", "declareId": "...", "anchorLocalnet": "...", "anchorDevnet": "...", "allMatch": true } },
  "crossRefs": [ { "source": "...", "ref": "...", "expected": "...", "actual": "...", "match": true } ],
  "placeholders": [],
  "summary": { "total": 20, "passed": 20, "failed": 0 },

```

**Exit codes:**
- 0: All checks pass
- 1: Any check fails

**CLI parsing:** Use `process.argv` directly (no need for a CLI framework for one flag). Check for `--json` in argv.

**Dependencies:** Only use `@solana/web3.js` (already in package.json) and Node built-ins (`fs`, `path`). Do NOT add new npm dependencies. For ANSI colors, use raw escape codes (`\x1b[32m` for green, `\x1b[31m` for red, `\x1b[33m` for yellow, `\x1b[0m` for reset).

**NOTE about cross-program reference extraction:** The constants.rs files have multiple functions, each returning a different program ID. The script needs to parse carefully -- it should find the FUNCTION by name (e.g., find `fn epoch_program_id` or `fn staking_program_id`) and then extract the ID from within that function. A simple approach: read the file, find the line containing the function name, then scan forward for the pubkey pattern within the next ~10 lines.

**Error handling:** If a keypair file is missing, skip that program's checks gracefully and report it as a failure. If a lib.rs can't be read, report the error. The script should never crash -- always produce a report.
