  <action>
**Step 1: Full Anchor build**

Run `anchor build` to compile all 8 programs. This will:
- Read keypairs from `target/deploy/` (which Plan 01 synced from `keypairs/`)
- Compile each program with its `declare_id!` macro (which Plan 01 synced)
- Produce `.so` binaries in `target/deploy/`

If any program fails to build, investigate the error. Common issues:
- `declare_id!` was not synced properly (re-run `anchor keys sync`)
- Cross-program reference uses an ID that doesn't parse as a valid Pubkey (check constants.rs)
- Rust compilation errors unrelated to IDs (should not happen since no logic changed)

**Step 2: Run verification script**

Run `npm run verify-ids` (or `npx tsx scripts/verify-program-ids.ts`).

Expected outcome: ALL checks PASS with zero failures. If any fail:
- Read the failure details
- Fix the underlying issue (likely a missed ID update from Plan 01)
- Re-run verification until clean

Also run `npm run verify-ids:json` and confirm the JSON output shows `"success": true`.

**Step 3: Run existing Rust unit tests (cargo test)**

Run `cargo test` from the workspace root to verify that Rust-level unit tests and proptests still pass. These tests don't require a validator -- they test math, serialization, and other pure logic.

Expected: All existing tests pass. The only changes were to program ID strings, not to any logic.

**Step 4: Spot-check AMM integration tests**

The AMM LiteSVM integration tests (`test_cpi_access_control.rs`, `test_swap_sol_pool.rs`, etc.) had their hardcoded IDs updated in Plan 01. Run the AMM program's Rust integration tests:

```bash
cargo test -p amm --test test_pool_initialization
cargo test -p amm --test test_swap_sol_pool
cargo test -p amm --test test_cpi_access_control
```

These tests deploy .so files using LiteSVM. The test helper IDs must match the compiled .so's `declare_id!` for the tests to work. If tests fail with "program not found" or similar, the IDs in test helpers don't match the built .so.

**NOTE:** Do NOT run the full `anchor test` suite (which spins up a validator for TypeScript tests) -- that will be done in Phase 31 when integration test infrastructure is set up. For now, just verify that Rust-level tests pass and the build is clean.

**Step 5: Record verification output**

Capture the JSON verification output for the summary:
```bash
npx tsx scripts/verify-program-ids.ts --json > /tmp/verify-ids-output.json
```

Include the summary stats (total/passed/failed) in the plan summary.
