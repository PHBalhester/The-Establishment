<task type="auto">
  <name>Task 2: Convert useCurrentSlot to throttled WebSocket subscription</name>
  <files>app/hooks/useCurrentSlot.ts</files>
  <action>
Rewrite `useCurrentSlot.ts` to replace `setInterval` + `getSlot()` polling with `connection.onSlotChange()` WebSocket subscription, using a ref-based throttle to prevent slot-flood re-renders.

**CRITICAL PITFALL (from research):** `onSlotChange` fires on EVERY new slot (~2.5/sec = 150/min). If each notification triggers `setState`, the component re-renders 150 times per minute. This MUST be throttled.

**Key changes:**

1. Add imports:
   - `import { getConnection } from "@/lib/connection"`
   - `import { useVisibility } from "@/hooks/useVisibility"`

2. Remove `POLL_INTERVAL_MS` constant.

3. Add `const STATE_FLUSH_INTERVAL_MS = 5_000` -- how often to flush the slot ref to React state. 5 seconds provides smooth countdown display (~1 update per 5s for the timer).

4. Call `useVisibility()` (no station required -- slot is used for countdown timer in DashboardGrid).

5. **Slot tracking architecture:**
   - `slotRef = useRef<number | null>(null)` -- stores the latest slot from WS (high-frequency).
   - `[displaySlot, setDisplaySlot] = useState<number | null>(null)` -- React state, updated every 5 seconds.

6. **Initial fetch:**
   - `connection.getSlot().then(slot => { slotRef.current = slot; setDisplaySlot(slot); })` for immediate data.

7. **WebSocket subscription:**
   - `const subId = connection.onSlotChange((info) => { slotRef.current = info.slot; })` -- stores in ref only, NO setState.

8. **Throttled state flush:**
   - `const flushInterval = setInterval(() => { if (slotRef.current !== null) setDisplaySlot(slotRef.current); }, STATE_FLUSH_INTERVAL_MS)`.
   - This interval provides the periodic React state updates that the countdown timer needs.

9. **Visibility-aware lifecycle** (same pattern as useEpochState):
   - When `isActive` becomes false: unsubscribe WS, clear flush interval.
   - When `isActive` becomes true: re-fetch initial slot via RPC, re-subscribe WS, restart flush interval.
   - Register burst-refresh callback via `onResume` to immediately fetch + flush on tab return.

10. **Cleanup on unmount:** Remove WS subscription + clear flush interval.

**Return `{ currentSlot: displaySlot, loading, error }`** -- same interface, consumers don't know about the throttling.

**Do NOT:**
- Call `setDisplaySlot` inside the `onSlotChange` callback (causes slot-flood re-renders).
- Create a new Connection instance.
  </action>
