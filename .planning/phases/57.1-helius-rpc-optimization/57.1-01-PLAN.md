  <name>Task 1: Create useVisibility hook</name>
  <files>app/hooks/useVisibility.ts</files>
  <action>
Create a new "use client" hook at `app/hooks/useVisibility.ts` that combines Page Visibility API with modal context awareness.

**Implementation details:**

1. Import `useState`, `useEffect`, `useRef`, `useCallback` from React.
2. Import `useModal` from `@/hooks/useModal` to access `state.activeStation`.
3. Import `StationId` type from `@/components/modal/ModalProvider`.

4. Signature: `export function useVisibility(requiredStation?: StationId): { isActive: boolean; tabVisible: boolean; onResume: (cb: () => void) => (() => void) }`

5. **Tab visibility tracking:**
   - Initialize `tabVisible` state with `typeof document !== "undefined" ? !document.hidden : true` (SSR-safe).
   - Attach `visibilitychange` event listener in a useEffect. On visibility change, update `tabVisible` state.
   - When tab becomes visible (`!document.hidden` transitions from false to true), fire all registered burst-refresh callbacks.

6. **Modal awareness:**
   - Read `state.activeStation` from `useModal()`.
   - `isActive` is computed as: `tabVisible && (!requiredStation || state.activeStation === requiredStation || state.activeStation === null)`.
   - Explanation: A hook is active when (a) the tab is visible AND (b) either no station is required, or the matching station is the open modal, or NO modal is open (dashboard view -- DashboardGrid hooks need to run when no modal is open).

7. **Burst-refresh registration:**
   - `onResume` accepts a callback, stores it in a `resumeCallbacksRef` (array of callbacks in a useRef).
   - Returns a cleanup function that removes the callback from the array.
   - When visibility transitions to visible, iterate through all callbacks and invoke them.

8. **Stale closure prevention:**
   - Store the previous `tabVisible` value in a ref to detect the false->true transition (tab return).
   - Only fire burst-refresh callbacks on the actual transition, not on every render where `tabVisible` is true.

9. Add JSDoc explaining the three use cases:
   - `useVisibility()` -- pause when tab is hidden only (for hooks like useTokenBalances that run everywhere)
   - `useVisibility("swap")` -- pause when tab is hidden OR when a different station is the active modal
   - The `isActive === true when activeStation === null` rule ensures DashboardGrid hooks run when no modal is open

**Do NOT:**
- Import any external packages (zero-dependency per project policy).
- Use `document.hasFocus()` (doesn't capture all cases like minimize/app-switch).
- Use Enhanced WebSocket APIs (standard Solana WS only).
