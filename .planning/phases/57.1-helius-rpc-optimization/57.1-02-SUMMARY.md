# Phase 57.1 Plan 02: useEpochState + useCurrentSlot WebSocket Conversion Summary

**Converted two polling hooks to WebSocket subscriptions with visibility-aware pausing and ref-based throttling, saving ~718 credits/hr combined**

## Performance

- **Duration:** 2 min
- **Started:** 2026-02-24T11:45:14Z
- **Completed:** 2026-02-24T11:47:38Z
- **Tasks:** 2/2
- **Files modified:** 2

## Accomplishments
- useEpochState: replaced 10s setInterval polling with onAccountChange WebSocket subscription (360 -> ~1 credit/hr)
- useCurrentSlot: replaced 10s setInterval polling with onSlotChange WebSocket subscription + ref-based throttle (360 -> ~1 credit/hr)
- Both hooks integrate useVisibility() for pausing WS when tab hidden or non-relevant modal open
- Both hooks fire burst-refresh RPC fetch on tab return via onResume callback
- Both hooks report to Sentry (zero-dep captureException) after 3 consecutive failures
- Return types and exported interfaces are identical to pre-modification -- zero consumer breakage

## Task Commits

Each task was committed atomically:

1. **Task 1: Convert useEpochState to WebSocket subscription** - `6319c02` (feat)
2. **Task 2: Convert useCurrentSlot to throttled WebSocket subscription** - `39a8f2c` (feat)

## Files Modified
- `app/hooks/useEpochState.ts` - setInterval polling replaced with onAccountChange WS + useVisibility pausing + Sentry error reporting
- `app/hooks/useCurrentSlot.ts` - setInterval polling replaced with onSlotChange WS + slotRef throttle (5s flush) + useVisibility pausing + Sentry error reporting

## Decisions Made
- **Sentry threshold: 3 consecutive failures** -- First failure could be transient network blip, second could be retry during recovery, third suggests persistent problem. Reset counter on any success.
- **5-second state flush interval** -- The countdown timer only needs periodic updates for smooth display. 5 seconds means 12 React re-renders/min instead of 150 from raw slot notifications.
- **Extracted subscribe/unsubscribe as useCallback** -- Clean separation allows the visibility effect to call them without recreating the mount effect. Prevents subscription leaks on pause/resume cycles.
- **prevIsActiveRef for transition detection** -- Prevents redundant subscribe/unsubscribe calls when isActive hasn't actually changed (just re-rendered with same value).
