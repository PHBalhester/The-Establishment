# Phase 57.1 Plan 04: Visibility Pausing for Data Hooks Summary

**Visibility-gated polling for 4 data hooks -- useCarnageData (30s + WS pause), useStaking (modal-scoped), useTokenBalances (tab-only), useSolPrice (tab-only) -- saving ~630 credits/hr**

## Accomplishments

- useCarnageData polling interval reduced from 10s to 30s and gated by useVisibility("carnage"); vault WebSocket subscription pauses when inactive and resumes with burst-refresh on tab return
- useStaking polling gated by useVisibility("staking"); execute function and confirmed-refetch intentionally left ungated (user actions must always trigger refresh regardless of visibility state)
- useTokenBalances polling gated by useVisibility() (tab visibility only, no station); refresh() function intentionally left ungated (cross-instance BALANCE_REFRESH_EVENT sync must work for user-action-triggered refreshes from SwapStation and StakingStation)
- useSolPrice polling gated by useVisibility() (tab visibility only, no station); burst-refresh on tab return
- All 4 hooks preserve their exact same return types -- consumers are unaware of the pausing optimization
- Zero type errors introduced (28 pre-existing errors in test/script files, none in app/hooks/)

## Decisions Made

1. **Reduced useCarnageData poll interval from 10s to 30s:** CarnageFundState PDA only changes on Carnage execution (~4% chance per epoch, epochs are ~5min) or tax collection deposits. 30s is more than sufficient and saves 2/3 of polling credits.
2. **Vault WS subscription pauses with visibility:** When the tab is hidden or a different station modal is active, the WebSocket subscription to CarnageSolVault is removed. On resume, a burst-refresh re-fetches the balance via RPC and re-subscribes. This prevents zombie WebSocket connections from consuming resources.
3. **useStaking confirmed-refetch NOT gated:** The `status === "confirmed"` re-fetch effect is intentionally left outside visibility gating because a user's successful transaction must always trigger a data refresh, even in edge cases where the tab might be briefly hidden.
4. **useTokenBalances refresh() NOT gated:** The `refresh()` function dispatches BALANCE_REFRESH_EVENT for cross-instance sync. This must remain ungated because it's triggered by user actions (completing a swap or staking transaction), not by background polling.
5. **useSolPrice and useTokenBalances use station-less useVisibility():** These hooks are consumed by multiple stations (or the dashboard), so they only gate on tab visibility, not modal state. This ensures they still run when any modal is open.
