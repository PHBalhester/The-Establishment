  <action>
    Add two new builder functions to the existing swap-builders.ts for PROFIT pool swaps. These have a simpler account structure than SOL pool swaps (11 named accounts instead of 20, no tax distribution accounts) but require dual Transfer Hook resolution (both sides are Token-2022).

    **buildProfitBuyTransaction(params: ProfitBuyParams): Promise<Transaction>** -- CRIME/FRAUD -> PROFIT swap.

    Parameters interface:
    ```typescript
    interface ProfitBuyParams {
      connection: Connection;
      userPublicKey: PublicKey;
      amountInBaseUnits: number;     // CRIME or FRAUD amount (6 decimals)
      minimumOutput: number;         // minimum PROFIT tokens (post-slippage)
      isCrime: boolean;              // true = CRIME/PROFIT pool, false = FRAUD/PROFIT pool
      computeUnits?: number;         // default 200_000
      priorityFeeMicroLamports?: number;
    }
    ```

    Named accounts (11 total, matching SwapProfitBuy struct from swap_profit_buy.rs lines 168-220):
    ```
    user, swapAuthority, pool, poolVaultA, poolVaultB,
    mintA (CRIME or FRAUD), mintB (PROFIT),
    userTokenA (user's CRIME/FRAUD ATA), userTokenB (user's PROFIT ATA),
    ammProgram, token2022Program (TOKEN_2022_PROGRAM_ID)
    ```

    Key difference from SOL pools:
    1. No epochState, taxAuthority, stakePool, stakingEscrow, carnageVault, treasury, stakingProgram, systemProgram
    2. Only ONE token program: token2022Program (both sides are Token-2022)
    3. EIGHT remaining_accounts (4 per Token-2022 side), not 4

    Transfer Hook resolution for PROFIT pools (CRITICAL -- dual hooks):
    ```typescript
    // Side A hooks: for the CRIME/FRAUD token transfer
    const hookAccountsA = await resolveHookAccounts(
      connection,
      userTokenA,                    // source: user's CRIME/FRAUD account
      mintA,                         // CRIME or FRAUD mint
      poolVaultA,                    // dest: pool's CRIME/FRAUD vault
      swapAuthorityPda,              // authority
    );

    // Side B hooks: for the PROFIT token transfer
    const hookAccountsB = await resolveHookAccounts(
      connection,
      poolVaultB,                    // source: pool's PROFIT vault
      MINTS.PROFIT,                  // PROFIT mint
      userTokenB,                    // dest: user's PROFIT account
      swapAuthorityPda,              // authority
    );

    // Concatenate: AMM splits at midpoint (first 4 = side A, last 4 = side B)
    const allHookAccounts = [...hookAccountsA, ...hookAccountsB];
    ```

    Check if user's PROFIT ATA exists; if not, prepend createAssociatedTokenAccountInstruction with TOKEN_2022_PROGRAM_ID. Also check user's CRIME/FRAUD input token ATA exists (it should if they have tokens, but defensive check).

    Build instruction: `getTaxProgram().methods.swapProfitBuy(new BN(amountInBaseUnits), new BN(minimumOutput), isCrime).accountsStrict({...}).remainingAccounts(allHookAccounts).instruction()`

    **buildProfitSellTransaction(params: ProfitSellParams): Promise<Transaction>** -- PROFIT -> CRIME/FRAUD swap.

    Parameters:
    ```typescript
    interface ProfitSellParams {
      connection: Connection;
      userPublicKey: PublicKey;
      amountInBaseUnits: number;     // PROFIT amount (6 decimals)
      minimumOutput: number;         // minimum CRIME/FRAUD tokens (post-slippage)
      isCrime: boolean;              // true = CRIME/PROFIT pool, false = FRAUD/PROFIT pool
      computeUnits?: number;
      priorityFeeMicroLamports?: number;
    }
    ```

    Named accounts (11 total, matching SwapProfitSell struct -- identical fields to SwapProfitBuy).

    Hook resolution for sell is the REVERSE direction:
    ```typescript
    // Side A hooks: CRIME/FRAUD goes FROM pool TO user (output direction)
    const hookAccountsA = await resolveHookAccounts(
      connection,
      poolVaultA,                    // source: pool's CRIME/FRAUD vault
      mintA,                         // CRIME or FRAUD mint
      userTokenA,                    // dest: user's CRIME/FRAUD account
      swapAuthorityPda,              // authority
    );

    // Side B hooks: PROFIT goes FROM user TO pool (input direction)
    const hookAccountsB = await resolveHookAccounts(
      connection,
      userTokenB,                    // source: user's PROFIT account
      MINTS.PROFIT,                  // PROFIT mint
      poolVaultB,                    // dest: pool's PROFIT vault
      swapAuthorityPda,              // authority
    );

    const allHookAccounts = [...hookAccountsA, ...hookAccountsB];
    ```

    Build instruction: `getTaxProgram().methods.swapProfitSell(new BN(amountInBaseUnits), new BN(minimumOutput), isCrime).accountsStrict({...}).remainingAccounts(allHookAccounts).instruction()`

    Check/create user's output token ATA (CRIME/FRAUD for sell, using TOKEN_2022_PROGRAM_ID).

    Export both new functions and their parameter interfaces.
