  <action>
    Create the `app/lib/swap/` directory and implement three library modules:

    **quote-engine.ts** -- Client-side AMM math that mirrors on-chain `programs/amm/src/helpers/math.rs` exactly. Must implement:

    1. `calculateEffectiveInput(amountIn: number, feeBps: number): number` -- LP fee deduction. Formula: `Math.floor(amountIn * (10_000 - feeBps) / 10_000)`. Uses BPS_DENOMINATOR = 10_000.

    2. `calculateSwapOutput(reserveIn: number, reserveOut: number, effectiveInput: number): number` -- Constant-product formula: `Math.floor((reserveOut * effectiveInput) / (reserveIn + effectiveInput))`. Returns 0 if denominator is 0.

    3. `calculateTax(amountLamports: number, taxBps: number): number` -- Tax deduction: `Math.floor(amountLamports * taxBps / 10_000)`.

    4. Forward quote functions for all 4 swap types:
       - `quoteSolBuy(solAmountLamports, reserveWsol, reserveToken, buyTaxBps, lpFeeBps)` -- SOL -> CRIME/FRAUD. Order: (1) tax on SOL input first, (2) LP fee on remaining, (3) AMM output. Returns `{ outputTokens, lpFee, taxAmount, netInput, priceImpactBps }`.
       - `quoteSolSell(tokenAmountBaseUnits, reserveWsol, reserveToken, sellTaxBps, lpFeeBps)` -- CRIME/FRAUD -> SOL. Order: (1) LP fee on token input, (2) AMM output in SOL, (3) tax on SOL output. Returns `{ outputSol, lpFee, taxAmount, grossSolOutput, priceImpactBps }`.
       - `quoteProfitBuy(tokenAmountBaseUnits, reserveToken, reserveProfit, lpFeeBps)` -- CRIME/FRAUD -> PROFIT. No tax. Returns `{ outputProfit, lpFee, priceImpactBps }`.
       - `quoteProfitSell(profitAmountBaseUnits, reserveToken, reserveProfit, lpFeeBps)` -- PROFIT -> CRIME/FRAUD. No tax. Returns `{ outputTokens, lpFee, priceImpactBps }`.

    5. Reverse quote functions for all 4 types (given desired output, calculate required input):
       - `reverseQuoteSolBuy(desiredOutputTokens, reserveWsol, reserveToken, buyTaxBps, lpFeeBps)` -- Returns `{ inputSolNeeded, lpFee, taxAmount } | null` (null if impossible, e.g. desiredOutput >= reserveToken).
       - `reverseQuoteSolSell(desiredOutputSol, reserveWsol, reserveToken, sellTaxBps, lpFeeBps)` -- Returns `{ inputTokensNeeded, lpFee, taxAmount } | null`.
       - `reverseQuoteProfitBuy(desiredOutputProfit, reserveToken, reserveProfit, lpFeeBps)` -- Returns `{ inputTokensNeeded, lpFee } | null`.
       - `reverseQuoteProfitSell(desiredOutputTokens, reserveToken, reserveProfit, lpFeeBps)` -- Returns `{ inputProfitNeeded, lpFee } | null`.

    6. Price impact: `priceImpactBps = Math.floor((1 - (actualRate / spotRate)) * 10_000)` where spotRate = reserveOut/reserveIn and actualRate = output/effectiveInput.

    Reverse quote math:
    - Reverse AMM: `effectiveInput = Math.ceil((reserveIn * desiredOutput) / (reserveOut - desiredOutput))`
    - Reverse LP fee: `grossInput = Math.ceil(effectiveInput * 10_000 / (10_000 - feeBps))`
    - Reverse buy tax (input-side): `totalSol = Math.ceil(grossInput * 10_000 / (10_000 - taxBps))`
    - Reverse sell tax (output-side): `grossSolNeeded = Math.ceil(desiredOutputSol * 10_000 / (10_000 - taxBps))`

    All functions are pure (no RPC calls, no imports from React). Export all functions. Add JSDoc comments explaining the math and the on-chain source reference.

    **error-map.ts** -- Error code to human-readable message mapping. Implement:

    1. `SWAP_ERROR_MAP: Record<number, string>` -- Maps Anchor error codes to user-friendly messages. Tax Program errors 6000-6013 (from programs/tax-program/src/errors.rs, 14 variants starting at 6000). AMM errors 6000-6017 (from programs/amm/src/errors.rs, 18 variants starting at 6000). Note: AMM errors come through CPI so they appear with the AMM program ID in logs.

    2. `parseSwapError(error: unknown): string` -- Parses error objects/strings to extract error codes. Check patterns in this order:
       (a) Anchor `Error Number: (\d+)` pattern -> look up in SWAP_ERROR_MAP
       (b) Solana `custom program error: 0x([0-9a-fA-F]+)` pattern -> parse hex, look up
       (c) `Blockhash not found` / `block height exceeded` -> "Transaction expired. Please try again."
       (d) `insufficient funds` / `Insufficient` -> "Insufficient balance for this swap."
       (e) `Transaction too large` -> "Transaction is too large. This is an unexpected error."
       (f) `User rejected` / `rejected` -> "Transaction was cancelled."
       (g) Fallback: "Swap failed. Please try again or reduce the swap amount."

    Key user-facing messages (map specific codes to these):
    - 6002 (Tax SlippageExceeded): "Price moved beyond your slippage tolerance. Try increasing slippage or reducing the swap size."
    - 6004 (Tax InsufficientInput): "The swap amount is too small to produce any output after fees."
    - 6003 (Tax InvalidEpochState): "The protocol's epoch state is currently unavailable. Please try again in a moment."

    **wsol.ts** -- WSOL wrapping and unwrapping instruction builders. Implement:

    1. `getWsolAta(owner: PublicKey): Promise<PublicKey>` -- Returns the Associated Token Address for WSOL using `getAssociatedTokenAddress(NATIVE_MINT, owner, false, TOKEN_PROGRAM_ID)`. CRITICAL: WSOL uses TOKEN_PROGRAM_ID, not TOKEN_2022_PROGRAM_ID.

    2. `buildWsolWrapInstructions(connection: Connection, userPublicKey: PublicKey, amountLamports: number): Promise<TransactionInstruction[]>` -- Returns an array of instructions to wrap SOL to WSOL:
       (a) Check if WSOL ATA exists via `connection.getAccountInfo(wsolAta)`
       (b) If not, include `createAssociatedTokenAccountInstruction(userPublicKey, wsolAta, userPublicKey, NATIVE_MINT, TOKEN_PROGRAM_ID)`
       (c) `SystemProgram.transfer({ fromPubkey: userPublicKey, toPubkey: wsolAta, lamports: amountLamports })`
       (d) `createSyncNativeInstruction(wsolAta, TOKEN_PROGRAM_ID)`

    3. `buildWsolUnwrapInstruction(userPublicKey: PublicKey): Promise<TransactionInstruction>` -- Returns `createCloseAccountInstruction(wsolAta, userPublicKey, userPublicKey, [], TOKEN_PROGRAM_ID)` to close WSOL account and recover SOL.

    Import from `@solana/spl-token`: `getAssociatedTokenAddress`, `createAssociatedTokenAccountInstruction`, `createSyncNativeInstruction`, `createCloseAccountInstruction`, `NATIVE_MINT`, `TOKEN_PROGRAM_ID`.

    **hook-resolver.ts** -- Transfer Hook remaining_accounts resolution for Token-2022 transfers. Port the exact pattern from `scripts/e2e/lib/swap-flow.ts` lines 132-159.

    1. `resolveHookAccounts(connection: Connection, source: PublicKey, mint: PublicKey, dest: PublicKey, authority: PublicKey): Promise<AccountMeta[]>` -- Uses `createTransferCheckedWithTransferHookInstruction` from `@solana/spl-token` to build a dummy transfer instruction, then extracts remaining_accounts (keys after the first 4: source, mint, dest, authority).

    Implementation:
    ```typescript
    const transferIx = await createTransferCheckedWithTransferHookInstruction(
      connection, source, mint, dest, authority,
      BigInt(0),  // amount doesn't affect hook resolution
      6,          // TOKEN_DECIMALS
      [],         // no extra signers
      "confirmed",
      TOKEN_2022_PROGRAM_ID,
    );
    return transferIx.keys.slice(4).map(key => ({
      pubkey: key.pubkey,
      isSigner: key.isSigner,
      isWritable: key.isWritable,
    }));
    ```

    This returns exactly 4 accounts per mint (per MEMORY.md: HOOK_ACCOUNTS_PER_MINT = 4). For PROFIT pool swaps, callers must call this twice (once per side) and concatenate.
