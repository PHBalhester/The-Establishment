    **swap-builders.ts** -- Transaction builder functions for SOL buy and SOL sell swaps. These produce complete Transaction objects ready for signing. Port the exact account structure from `scripts/e2e/lib/swap-flow.ts` lines 443-478 (SOL buy) and mirror the on-chain `SwapSolSell` struct (20 named accounts) for sell.

    1. `buildSolBuyTransaction(params: SolBuyParams): Promise<Transaction>` -- Builds a SOL -> CRIME/FRAUD swap transaction.

       Parameters interface:
       ```typescript
       interface SolBuyParams {
         connection: Connection;
         userPublicKey: PublicKey;
         amountInLamports: number;
         minimumOutput: number;         // post-slippage minimum
         isCrime: boolean;
         computeUnits?: number;         // default 200_000
         priorityFeeMicroLamports?: number;  // default 0
       }
       ```

       Transaction assembly:
       (a) Build WSOL wrap instructions via `buildWsolWrapInstructions(connection, userPublicKey, amountInLamports)` from wsol.ts
       (b) Check if user's output token ATA exists; if not, prepend `createAssociatedTokenAccountInstruction` using TOKEN_2022_PROGRAM_ID for CRIME/FRAUD
       (c) Resolve Transfer Hook accounts via `resolveHookAccounts(connection, poolVaultB, tokenMint, userTokenB, swapAuthorityPda)`
       (d) Build swap instruction using `getTaxProgram().methods.swapSolBuy(new BN(amountInLamports), new BN(minimumOutput), isCrime).accountsStrict({...}).remainingAccounts(hookAccounts).instruction()`
       (e) Assemble Transaction: ComputeBudgetProgram.setComputeUnitLimit + optional priority fee + WSOL wrap instructions + optional ATA creation + swap instruction

       Named accounts (20 total, exactly matching SwapSolBuy struct):
       ```
       user, epochState, swapAuthority, taxAuthority, pool, poolVaultA, poolVaultB,
       mintA (NATIVE_MINT), mintB (CRIME/FRAUD), userTokenA (WSOL ATA), userTokenB (token ATA),
       stakePool, stakingEscrow, carnageVault, treasury,
       ammProgram, tokenProgramA (TOKEN_PROGRAM_ID), tokenProgramB (TOKEN_2022_PROGRAM_ID),
       systemProgram, stakingProgram
       ```

       Use addresses from `DEVNET_POOL_CONFIGS`, `DEVNET_PDAS_EXTENDED`, `TREASURY_PUBKEY`, `MINTS`, `PROGRAM_IDS` (all from shared/constants.ts).

    2. `buildSolSellTransaction(params: SolSellParams): Promise<Transaction>` -- Builds a CRIME/FRAUD -> SOL swap transaction.

       Parameters interface (same shape as buy but with different semantics):
       ```typescript
       interface SolSellParams {
         connection: Connection;
         userPublicKey: PublicKey;
         amountInBaseUnits: number;     // CRIME or FRAUD amount (6 decimals)
         minimumOutput: number;         // minimum SOL to receive AFTER tax
         isCrime: boolean;
         computeUnits?: number;
         priorityFeeMicroLamports?: number;
       }
       ```

       Transaction assembly:
       (a) Check if user's WSOL ATA exists; if not, create it (receives SOL output)
       (b) Resolve Transfer Hook accounts for the Token-2022 side (pool vaultB -> user, for the B-to-A direction)
       (c) Build swap instruction using `getTaxProgram().methods.swapSolSell(new BN(amountInBaseUnits), new BN(minimumOutput), isCrime).accountsStrict({...}).remainingAccounts(hookAccounts).instruction()`
       (d) Append WSOL unwrap instruction via `buildWsolUnwrapInstruction(userPublicKey)` to close WSOL account after swap
       (e) Assemble Transaction: compute budget + optional WSOL ATA creation + swap instruction + WSOL unwrap

       Named accounts (20 total, matching SwapSolSell struct):
       Same as buy, with same addresses. The AMM internally determines direction from the instruction discriminator.

       CRITICAL for sell: The hook resolution for sell direction is different.
       - For buy: AMM transfers tokens FROM pool vaultB TO user (output is Token-2022)
       - For sell: AMM transfers tokens FROM user TO pool vaultB (input is Token-2022)
       - Hook resolution source/dest must match the Transfer Hook's expected flow direction
       - For sell, resolve: `resolveHookAccounts(connection, userTokenB, tokenMint, poolVaultB, swapAuthorityPda)`

    Export both functions + the parameter interfaces.

    **useSwap.ts** -- React hook that orchestrates the entire swap lifecycle for all pool types.

    State machine states: `'idle' | 'quoting' | 'building' | 'signing' | 'sending' | 'confirming' | 'confirmed' | 'failed'`

    Interface:
    ```typescript
    interface UseSwapReturn {
      // Form state
      inputToken: TokenSymbol;
      outputToken: TokenSymbol;
      inputAmount: string;         // User-entered string
      outputAmount: string;        // Computed string
      setInputToken: (token: TokenSymbol) => void;
      setOutputToken: (token: TokenSymbol) => void;
      setInputAmount: (amount: string) => void;
      setOutputAmount: (amount: string) => void;
      flipTokens: () => void;

      // Quote data
      quote: SwapQuote | null;
      quoteLoading: boolean;

      // Swap config
      slippageBps: number;
      setSlippageBps: (bps: number) => void;
      priorityFeePreset: PriorityFeePreset;
      setPriorityFeePreset: (preset: PriorityFeePreset) => void;

      // Execution
      executeSwap: () => Promise<void>;
      status: SwapStatus;
      txSignature: string | null;
      errorMessage: string | null;

      // Reset
      resetForm: () => void;
    }
    ```

    SwapQuote type:
    ```typescript
    interface SwapQuote {
      outputAmount: number;       // Base units
      lpFee: number;              // Base units
      taxAmount: number;          // Base units (0 for PROFIT pools)
      priceImpactBps: number;
      minimumOutput: number;      // After slippage
      totalFeePct: string;        // e.g. "3.5%" for display
    }
    ```

    PriorityFeePreset: `'none' | 'low' | 'medium' | 'high' | 'turbo'` with microLamports mapping: none=0, low=1000, medium=10000, high=100000, turbo=1000000.

    Implementation details:
    1. Use `usePoolPrices()` for real-time reserves and `useEpochState()` for tax rates
    2. Debounce quoting at 300ms using a simple useRef+setTimeout pattern (no external library). When user types in input field, compute output. When user types in output field, reverse-compute input.
    3. On `executeSwap()`: (a) set status='building', (b) call the appropriate buildSol*Transaction, (c) set blockhash+feePayer, (d) status='signing', (e) call `wallet.signTransaction(tx)`, (f) status='sending', (g) `connection.sendRawTransaction(signed.serialize(), { skipPreflight: false, maxRetries: 2 })`, (h) status='confirming', (i) `connection.confirmTransaction({ signature, blockhash, lastValidBlockHeight }, "confirmed")`, (j) check `confirmation.value.err`, (k) status='confirmed' or 'failed', (l) refresh token balances
    4. Error handling: wrap entire executeSwap in try/catch. On error, call `parseSwapError(error)` from error-map.ts and set errorMessage.
    5. `flipTokens()`: swap inputToken/outputToken AND swap inputAmount/outputAmount for natural UX
    6. `resetForm()`: clear amounts, set status='idle', clear txSignature/errorMessage. Called automatically 10 seconds after 'confirmed' state.
    7. Default tokens: inputToken='SOL', outputToken='CRIME'
    8. Default slippage: 100 bps (1%)
    9. Default priority: 'medium'

    Import `useProtocolWallet` for signing, `usePoolPrices` for reserves, `useEpochState` for tax rates, `useTokenBalances` for post-swap refresh.
