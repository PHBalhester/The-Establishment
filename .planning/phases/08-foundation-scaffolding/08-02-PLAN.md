  <behavior>
**Three functions, all returning `Option<T>` (None on overflow/invalid):**

1. `calculate_effective_input(amount_in: u64, fee_bps: u16) -> Option<u128>`
   - Computes: `amount_in * (10_000 - fee_bps) / 10_000` using u128 checked arithmetic
   - fee_bps=100, amount=1000 -> 990
   - fee_bps=50, amount=1000 -> 995
   - fee_bps=0, amount=1000 -> 1000
   - fee_bps=10000, amount=1000 -> 0
   - fee_bps=10001, amount=1000 -> None (underflow on 10_000 - 10_001)
   - amount=u64::MAX, fee_bps=100 -> Some(valid u128 value)

2. `calculate_swap_output(reserve_in: u64, reserve_out: u64, effective_input: u128) -> Option<u64>`
   - Computes: `reserve_out * effective_input / (reserve_in + effective_input)` in u128, result cast to u64
   - reserve_in=1_000_000, reserve_out=1_000_000, effective=1000 -> 999 (truncated)
   - effective=0 -> 0 (valid, not error)
   - reserve_out=0 -> 0 (valid)
   - reserve_in=0, effective=0 -> None (denominator = 0)
   - reserve_in=0, effective=1000 -> 1_000_000 (gets all of reserve_out)
   - u64::MAX reserves with small input -> Some(valid)

3. `verify_k_invariant(reserve_in_before: u64, reserve_out_before: u64, reserve_in_after: u64, reserve_out_after: u64) -> Option<bool>`
   - Computes: `(after_in * after_out) >= (before_in * before_out)` in u128
   - Valid swap (1M/1M -> 1_001_000/999_001) -> Some(true)
   - Invalid swap (1M/1M -> 1_001_000/998_000) -> Some(false)
   - u64::MAX * u64::MAX -> Some(true) [fits in u128: (2^64-1)^2 < 2^128]

**Critical constraints:**
- ZERO imports from anchor_lang or solana_program (pure Rust only)
- ALL arithmetic uses checked_add/checked_mul/checked_div/checked_sub
- Integer division truncates (rounds down) -- correct per AMM_Implementation.md Section 8.2
- Returns Option<T>, never panics

**Part B: Proptest property-based tests** (10,000 iterations)

Strategy configuration:
```rust
use proptest::prelude::*;

fn reserve_strategy() -> impl Strategy<Value = u64> {
    prop_oneof![
        9 => 1u64..=1_000_000_000_000_000_000u64,  // Realistic: 1 lamport to 1B tokens (9 decimals)
        1 => prop::sample::select(vec![0u64, 1, u64::MAX, u64::MAX / 2]),  // Edge cases
    ]
}

fn swap_amount_strategy() -> impl Strategy<Value = u64> {
    prop_oneof![
        9 => 1u64..=100_000_000_000_000_000u64,  // Up to 100M tokens
        1 => prop::sample::select(vec![0u64, 1, u64::MAX, u64::MAX / 2]),
    ]
}

fn fee_bps_strategy() -> impl Strategy<Value = u16> {
    prop_oneof![
        7 => 0u16..=9999u16,       // Valid range
        2 => Just(50u16),           // Protocol PROFIT pool fee
        1 => Just(100u16),          // Protocol SOL pool fee
    ]
}
```

Proptest 1 -- k-invariant holds for all valid swaps (10,000 iterations):
```
For random (reserve_in, reserve_out, amount_in, fee_bps):
  1. Skip if amount_in == 0 or reserve_in == 0 or reserve_out == 0
  2. Calculate effective_input, skip if None
  3. Calculate output, skip if None
  4. Skip if output >= reserve_out (would drain pool)
  5. new_reserve_in = reserve_in + amount_in (pre-fee amount added to pool)
  6. new_reserve_out = reserve_out - output
  7. Assert verify_k_invariant returns Some(true)
```

Proptest 2 -- output never exceeds reserve_out (10,000 iterations):
```
For random (reserve_in, reserve_out, effective_input):
  If calculate_swap_output returns Some(output):
    Assert output <= reserve_out
```

Proptest 3 -- fee calculation is monotonic (10,000 iterations):
```
For random (amount_in, fee_bps_low, fee_bps_high) where fee_bps_low <= fee_bps_high:
  If both calculate_effective_input calls return Some:
    Assert effective_low >= effective_high (higher fee = less effective input)
```
