**useWelcomeGate.ts:**

Create a client-side hook at `app/hooks/useWelcomeGate.ts` that manages the welcome modal lifecycle:

```
'use client';

import { useState, useEffect, useCallback } from 'react';

const STORAGE_KEY = 'dr-fraudsworth-welcomed';
```

- `shouldShow` state: Initialize as `false` (SSR and initial client render agree -- no hydration mismatch)
- `useEffect` on mount: Check `localStorage.getItem(STORAGE_KEY)`. If absent (null), call `setShouldShow(true)`
- `dismiss` callback: Sets `localStorage.setItem(STORAGE_KEY, 'true')` and `setShouldShow(false)`
- Export `{ shouldShow, dismiss }` from the hook
- The hook does NOT manage the dialog ref or body scroll lock -- that belongs to the component

Create `app/components/onboarding/WelcomeModal.tsx` -- a standalone `<dialog>` component that renders the welcome "title card".

**Component structure:**

```
'use client';

import { useRef, useEffect, useCallback } from 'react';
import { useConnectWallet } from '@privy-io/react-auth';
import { useWelcomeGate } from '@/hooks/useWelcomeGate';
```

**Dialog ref + showModal lifecycle:**
- `dialogRef = useRef<HTMLDialogElement>(null)`
- `useEffect` watching `shouldShow`: When `shouldShow` becomes `true`, call `dialogRef.current?.showModal()`, then add class `'welcome-opening'` for entrance animation, then `document.body.classList.add('modal-open')` for scroll lock
- Listen for `animationend` on the opening animation to remove the `'welcome-opening'` class

**Animated close (follows ModalShell pattern):**
- `handleAnimatedClose(onComplete?: () => void)`:
  1. Check `dialog.open` -- bail if already closed
  2. Add class `'welcome-closing'` to dialog
  3. Listen for `animationend` (once): remove `'welcome-closing'`, call `dialog.close()`, remove `'modal-open'` from body, call `onComplete?.()`

**Event handlers:**
- `handleEnterFactory`:
  1. Call `dismiss()` from useWelcomeGate (sets localStorage)
  2. Call `handleAnimatedClose()` -- plays fade-out, then closes

- `handleConnectWallet` -- CRITICAL SEQUENCING (Privy inert issue):
  1. Call `dismiss()` from useWelcomeGate (sets localStorage)
  2. Remove `'modal-open'` from body synchronously
  3. Call `dialogRef.current?.close()` synchronously (instant, no animation -- Privy dialog will mask the disappearance)
  4. Call `connectWallet()` -- Privy's HeadlessUI dialog can now receive clicks because our dialog is no longer making everything inert
  - Do NOT use handleAnimatedClose for this path. The animation delay would leave the dialog open when Privy tries to render, triggering the inert trap.

- `handleCancel` (native Escape key):
  1. `e.preventDefault()` (prevent instant close, allow animation)
  2. Call `dismiss()` from useWelcomeGate
  3. Call `handleAnimatedClose()`
  4. Register via `dialog.addEventListener('cancel', handleCancel)` in a useEffect

**JSX content -- the "title card":**

The dialog element: `<dialog ref={dialogRef} className="welcome-modal" aria-labelledby="welcome-title">`

Inside the dialog, a single `.welcome-chrome` div containing:

1. **Corner bolts** (4x): Same `.modal-bolt` pattern from ModalShell, absolutely positioned at corners with 6px inset. These are decorative `<div>` elements with `aria-hidden="true"`.

2. **Decorative emblem** (aria-hidden): An inline SVG factory/gear emblem in brass tones. Keep it simple -- a gear cog or factory chimney silhouette, ~64px, using `currentColor` or hardcoded brass palette colors (#b8860b / var(--color-factory-accent)). This is the "crest" mentioned in CONTEXT.md. Claude's discretion on exact design -- keep it elegant and Victorian, not clip-art.

3. **Title** `<h2 id="welcome-title">`: "Welcome to Dr. Fraudsworth's Finance Factory"
   - Classes: `font-heading text-factory-heading` (or equivalent Cinzel + dark-on-paper color)
   - Should feel grand -- use `text-xl` or `text-2xl`, `font-bold`, `tracking-wide`

4. **Tagline** `<p>`: One italicized line capturing the value prop. Something in the vein of "Where honest friction forges real yield" or "A peculiar establishment where trading friction generates genuine returns." Keep it theatrical-Victorian but informative. Claude's discretion on exact wording per CONTEXT.md.
   - Classes: italic, muted text color, `text-sm`, `mb-4`

5. **Body copy** `<p>`: 2-3 sentences maximum. Explain: Three tokens (CRIME, FRAUD, PROFIT -- use `font-mono` for token names), taxes on trades fund a yield pool, staking PROFIT earns real SOL. No jargon. The tone is Dr. Fraudsworth as eccentric but honest inventor.
   - Classes: `text-sm`, `leading-relaxed`, muted-but-readable text color, `mb-6`

6. **Action buttons** in a flex container:
   - Container: `flex flex-col gap-3 sm:flex-row sm:justify-center`
   - **"Enter the Factory"** button: Primary visual weight. Use `brass-button` class plus extra styling to make it feel like the default action (slightly larger, accent border, or filled background). `onClick={handleEnterFactory}`
   - **"Connect Wallet"** button: Secondary visual weight. Use `brass-button` class with a more subdued/outlined appearance. `onClick={handleConnectWallet}`
   - Both buttons should have `type="button"` explicitly
   - Both should have clear hover/active states (inherited from `.brass-button`)

**Accessibility:**
- `aria-labelledby="welcome-title"` on dialog
- `aria-hidden="true"` on decorative elements (bolts, emblem)
- Focus-visible styling inherited from existing dialog focus-visible rules
- Tab order: Enter the Factory -> Connect Wallet (primary action first)

**Important constraints:**
- Do NOT import from ModalProvider or ModalShell -- this is fully standalone
- Do NOT add 'welcome' to any StationId type
- Do NOT use the `open` attribute on `<dialog>` in JSX -- always use `showModal()` programmatically
- The component should return `null` (or the dialog in a hidden state) when `shouldShow` is false. Since the dialog starts without `open`, it's naturally hidden -- but consider whether to render the dialog element at all when not needed. Recommendation: always render the `<dialog>` (for ref stability), but only call `showModal()` when `shouldShow` is true.
