Create the `scripts/deploy/lib/` directory and 3 modules:

**connection.ts** -- Standalone Anchor Provider + Program loading:
- `loadProvider(clusterUrl?: string, walletPath?: string)` function:
  - Reads CLUSTER_URL env var (default: `http://localhost:8899`)
  - Reads WALLET env var (default: `keypairs/devnet-wallet.json`)
  - Loads wallet keypair from JSON file using `Keypair.fromSecretKey`
  - Creates `anchor.AnchorProvider` with `confirmed` commitment
  - Returns the provider
- `loadPrograms(provider: AnchorProvider)` function:
  - Loads all 5 IDL files from `target/idl/` (amm.json, transfer_hook.json, tax_program.json, epoch_program.json, staking.json)
  - Creates `anchor.Program` instances for each using the Anchor type imports
  - Returns object with all 5 programs typed: `{ amm, transferHook, taxProgram, epochProgram, staking }`
- Import types from `../../../target/types/amm`, `../../../target/types/transfer_hook`, etc.
- Use `path.resolve(__dirname, ...)` for IDL paths to work from any cwd
- Pattern reference: See 33-RESEARCH.md "Pattern 1: Standalone Anchor Provider" and existing `scripts/init-localnet.ts` lines 25-48

**logger.ts** -- Step logging with tx signature file:
- `createLogger(logFilePath?: string)` factory function:
  - Default log file: `scripts/deploy/deploy-log-{timestamp}.txt`
  - Creates/truncates the log file on init
  - Returns an object with methods:
    - `step(current: number, total: number, name: string, status: "done" | "SKIPPED", sig?: string)` -- prints `[N/M] name... status` to terminal, appends tx signature to log file if provided
    - `info(message: string)` -- prints informational line (e.g., cluster URL, wallet address)
    - `error(message: string)` -- prints error in red
    - `section(title: string)` -- prints section header (e.g., `=== Protocol Initialization ===`)
    - `getLogPath()` -- returns the log file path
- Use ANSI color codes for terminal output (green for done, yellow for SKIPPED, red for errors, bold for section headers)
- Pattern reference: See 33-RESEARCH.md "Pattern 4: Step-by-Step Operator Logging"

**account-check.ts** -- On-chain account existence checking:
- `accountExists(connection: Connection, address: PublicKey): Promise<boolean>` -- returns true if account has data (info !== null && info.data.length > 0)
- `programIsDeployed(connection: Connection, programId: PublicKey): Promise<boolean>` -- returns true if account exists AND is executable
- `mintExists(connection: Connection, mintAddress: PublicKey): Promise<boolean>` -- returns true if account exists and is owned by Token-2022 program OR SPL Token program (for WSOL)
- Import `TOKEN_2022_PROGRAM_ID` and `TOKEN_PROGRAM_ID` from `@solana/spl-token`
- Pattern reference: See 33-RESEARCH.md "Pattern 2: Idempotent Check-Before-Init" and `scripts/init-localnet.ts` lines 74-80

  <action>
**build.sh** -- Compile all programs + verify IDs:
- Shebang: `#!/bin/bash`
- `set -e` for fail-fast
- Source Rust/Solana/Node environment:
  ```
  source "$HOME/.cargo/env"
  export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"
  export PATH="/opt/homebrew/bin:$PATH"
  ```
- Print header: `=== Dr. Fraudsworth Build Script ===`
- Step [1/3]: Run `anchor build` -- compiles all 5 programs
- Step [2/3]: Verify all 5 .so files exist in `target/deploy/`:
  - amm.so, transfer_hook.so, tax_program.so, epoch_program.so, staking.so
  - Print `OK: {name}.so` for each, or `ERROR: {name}.so not found` and exit 1
- Step [3/3]: Run `npx tsx scripts/verify-program-ids.ts` to verify keypair consistency
  - Check exit code, print error and exit 1 if failed
- Print `=== Build Complete ===`
- Pattern reference: See 33-RESEARCH.md "Build Script Core Logic"

**deploy.sh** -- Deploy all 5 programs to target cluster:
- Shebang: `#!/bin/bash`
- `set -e` for fail-fast
- Source Rust/Solana environment (same as build.sh)
- Accept cluster URL as first positional arg or CLUSTER_URL env var (default: `http://localhost:8899`)
- Accept wallet path from WALLET env var (default: `keypairs/devnet-wallet.json`)
- Print header with cluster URL and wallet path
- Check wallet balance with `solana balance --url "$CLUSTER_URL" --keypair "$WALLET"` and print it
- Auto-airdrop: If cluster URL contains "localhost" or "devnet" AND balance < 5 SOL, request airdrop:
  - `solana airdrop 2 --url "$CLUSTER_URL" --keypair "$WALLET"` (repeat until >= 5 SOL, max 3 attempts)
  - On localnet this is free; on devnet it's rate-limited but harmless
- Deploy all 5 programs using this program-to-keypair mapping (from Anchor.toml and keypairs/ directory):
  - amm -> keypairs/amm-keypair.json
  - transfer_hook -> keypairs/transfer-hook-keypair.json
  - tax_program -> keypairs/tax-program-keypair.json
  - epoch_program -> keypairs/epoch-program.json
  - staking -> keypairs/staking-keypair.json
- For each program:
  - Derive program ID with `solana-keygen pubkey "$keypair"`
  - Print `[N/5] Deploying {name} ({programId})...`
  - Run `solana program deploy "target/deploy/${name}.so" --program-id "$keypair" --keypair "$WALLET" --url "$CLUSTER_URL"`
  - Print `  done`
- After all deploys, verify each program is deployed:
  - `solana program show "$PROGRAM_ID" --url "$CLUSTER_URL"` for each
  - Print `OK: {name} ({programId})` or `ERROR` and exit 1
- Print `=== Deploy Complete ===`
- Pattern reference: See 33-RESEARCH.md "Pattern 3: Deploy with Canonical Keypairs" and "Deploy Script Core Logic"

Make both scripts executable: `chmod +x scripts/deploy/build.sh scripts/deploy/deploy.sh`
  </action>
