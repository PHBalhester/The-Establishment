  <name>Task 1: Create post-deployment verification script</name>
  <files>scripts/deploy/verify.ts</files>
  <action>
Create `scripts/deploy/verify.ts` -- verifies the full protocol deployment state and produces a deployment report.

**Structure:** Uses the shared library (connection, logger, account-check, pda-manifest). Loads mint keypairs from `scripts/deploy/mint-keypairs/` (saved by initialize.ts).

**Verification depth (per 33-RESEARCH.md recommendation):** Existence + key data checks, not just existence.

**Verification steps:**

1. **Load mint keypairs** from `scripts/deploy/mint-keypairs/` -- error if directory doesn't exist (means init hasn't run yet)

2. **Generate expected PDA manifest** using `generateManifest()` with program IDs from IDL and mint addresses from keypairs

3. **Verify Programs (5 checks):**
   For each of the 5 programs:
   - Call `programIsDeployed(connection, programId)` -- checks account exists AND is executable
   - Verify owner is BPF Loader Upgradeable (`BPFLoaderUpgradeab1e11111111111111111111111`)
   - Print `[N/M] Program: {name} ({programId})... OK` or `FAIL`

4. **Verify Mints (3 checks):**
   For each mint (CRIME, FRAUD, PROFIT):
   - Call `mintExists(connection, mintAddress)`
   - Fetch mint info and verify: decimals == 6, supply > 0
   - Verify mint has TransferHook extension (check the mint account data length matches getMintLen([ExtensionType.TransferHook]))
   - Print status

5. **Verify Transfer Hook state (4+ checks):**
   - WhitelistAuthority PDA exists
   - ExtraAccountMetaList for each mint exists (3 checks)

6. **Verify AMM state (5+ checks):**
   - AdminConfig PDA exists
   - Each of 4 pool PDAs exists
   - For each pool: fetch account data and verify it has non-zero vault balances (indicating seed liquidity was deposited)

7. **Verify Epoch state (3+ checks):**
   - EpochState PDA exists
   - CarnageFund PDA exists
   - CarnageSolVault has > 0 lamports (rent-exempt minimum)

8. **Verify Staking state (3+ checks):**
   - StakePool PDA exists
   - StakeVault exists (fetch and verify it has >= MINIMUM_STAKE balance for dead stake)
   - EscrowVault PDA exists

9. **Verify Whitelist entries (batch check):**
   - Check existence of whitelist entry PDAs for: all pool vaults, StakeVault, Carnage token vaults, admin T22 accounts
   - This is a batch of ~12-14 existence checks
   - Print summary: `Whitelist entries: X/Y verified`

**Deployment Report:**
After all checks, write `scripts/deploy/deployment-report.md` with:
```markdown
# Deployment Report
Generated: {timestamp}
Cluster: {clusterUrl}
Wallet: {walletAddress}

## Summary
- Programs: 5/5 deployed
- Mints: 3/3 created
- Pools: 4/4 initialized
- PDAs: X/Y verified
- Whitelist: X/Y entries

## Results
| Check | Address | Status | Details |
|-------|---------|--------|---------|
| AMM Program | zFW9... | OK | Deployed, executable |
| CRIME Mint | ... | OK | 6 decimals, supply > 0 |
...

## Transaction Log
See: {logFilePath}

## PDA Manifest
See: scripts/deploy/pda-manifest.md
```

**Exit code:** 0 if all checks pass, 1 if any check fails (with clear error messages for each failure).

**Important:** Handle the case where mint keypairs don't exist yet (init hasn't run). Print clear error: "Mint keypairs not found at scripts/deploy/mint-keypairs/. Run initialize.ts first."
