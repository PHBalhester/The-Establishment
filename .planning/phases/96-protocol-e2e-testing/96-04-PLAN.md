  <name>Task 1: Create soak verification script and record soak start state</name>
  <files>
    scripts/e2e/soak-verify.ts
  </files>
  <action>
    Create `scripts/e2e/soak-verify.ts` that operates in two modes:

    **Mode 1: `--start` (record baseline)**
    1. Read current EpochState from on-chain (epoch_number, last_transition_slot)
    2. Check Railway crank health endpoint responds 200
    3. Record baseline to `scripts/e2e/soak-baseline.json`: { startTime, startEpoch, healthOk, crankUrl }
    4. Print: "Soak baseline recorded. Run `--verify` after 24 hours."

    **Mode 2: `--verify` (check after 24h)**
    1. Load baseline from `soak-baseline.json`
    2. Read current EpochState: epoch_number, last_transition_slot
    3. Calculate elapsed time from baseline startTime
    4. Calculate expected epochs: elapsed_seconds / (750 slots * 0.4 sec/slot = 300 seconds per epoch)
    5. Actual epochs = current epoch_number - start epoch_number
    6. Check Railway /health endpoint responds 200
    7. Pass criteria (all must be true):
       - Elapsed time >= 24 hours (86400 seconds)
       - Actual epochs >= expected * 0.9 (10% tolerance for slot timing variance)
       - No gap in epoch numbers (epochs are sequential)
       - Health endpoint responds 200
    8. E2E-09 (priority fees): if crank is advancing epochs consistently (~1 every 5 min), then TXs are landing reliably. Log the average epoch interval.
    9. Output: PASS/FAIL with details (epoch count, expected vs actual, uptime, average interval)
    10. Append results to `scripts/e2e/e2e-run.jsonl`

    Use `loadDeployment()` for the epochState PDA address. Use the crank's Railway URL from env var (`CRANK_HEALTH_URL` or hardcode the Railway URL from deployment context).

    Run `--start` mode immediately to begin the soak.
  </action>
  <verify>
    `npx tsx scripts/e2e/soak-verify.ts --start` creates `soak-baseline.json` with current epoch number and health status. The file exists and contains valid JSON with startEpoch > 0.
  </verify>

    Soak baseline recorded. Script ready for `--verify` mode after 24 hours.
