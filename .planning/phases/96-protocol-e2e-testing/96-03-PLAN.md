<task type="auto">
  <name>Task 1: Build 50-wallet stress test script</name>
  <files>
    scripts/e2e/stress-test.ts
    scripts/e2e/lib/stress-wallet.ts
  </files>
  <action>
    Create `scripts/e2e/lib/stress-wallet.ts` with a `StressWallet` class that encapsulates a single bot wallet:
    - Constructor: takes Keypair, Connection, deployment config
    - `fund(provider, amount)`: receive SOL from funder
    - `createAccounts()`: create WSOL + CRIME + FRAUD + PROFIT token accounts (Token-2022 for CRIME/FRAUD/PROFIT, standard SPL for WSOL)
    - `executeRandomSwap()`: pick random pool (CRIME/SOL or FRAUD/SOL), random direction (buy or sell), random amount (0.001-0.01 SOL equivalent). Buy uses standard TX, sell uses v0 VersionedTransaction with ALT. Track per-wallet token balances to know when sells are possible (must buy first). Returns {success, txSig, error, pair, direction, amount}.
    - Use `Promise.allSettled` everywhere (individual failures must not crash other wallets)

    Create `scripts/e2e/stress-test.ts` orchestrator:
    1. Load deployment config via `loadDeployment()`
    2. Check devnet wallet balance >= 6 SOL (50 wallets * ~0.1 SOL + overhead)
    3. Generate 50 Keypairs
    4. Batch-fund wallets (10 per TX to fit size limit, with 1s delay between batches to avoid rate limits)
    5. Create token accounts in batches (5 wallets per batch, 2s delay between batches)
    6. Run parallel swap loop: each wallet starts with random 0.5-3s initial delay, then swaps every 1-5s (random). First swap per wallet MUST be a buy (need tokens before selling). Run for configurable duration (default: 10 minutes, env var STRESS_DURATION_MIN).
    7. Stagger using random delays to stay under Helius rate limits (~10 TPS)
    8. Collect results: total swaps attempted, successes, failures, error breakdown by type
    9. Verify: no wallet has unexpected token balance (all TXs accounted for)
    10. Log results to `scripts/e2e/stress-test-results.jsonl`

    Key implementation details:
    - Use `loadDeployment()` from Plan 01 for all addresses
    - ALT address from `deployments/devnet.json` `.alt` field for sell TXs
    - Use `resolveHookAccounts()` pattern from existing swap-flow.ts for transfer hook accounts
    - `skipPreflight: true` for v0 TXs (devnet v0 simulation bug), check `confirmation.value.err`
    - Use `Promise.allSettled` not `Promise.all` to prevent cascade failures
    - Each wallet tracks its own token balances (updated after each successful swap)
  </action>
  <verify>
    Dry-run with 3 wallets for 2 minutes: `STRESS_WALLETS=3 STRESS_DURATION_MIN=2 npx tsx scripts/e2e/stress-test.ts`. Should complete with at least some successful swaps and no crashes. Check `stress-test-results.jsonl` for logged results.
  </verify>
  <done>
    Stress test script created. Dry-run with 3 wallets completes successfully with swap results logged.
  </done>
</task>
