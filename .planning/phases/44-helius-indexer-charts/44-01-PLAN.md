<objective>
Build the Helius webhook receiver infrastructure: Drizzle ORM database connection, Anchor event parser for decoding on-chain events from raw transaction logs, and the Next.js API route handler that receives Helius raw webhooks, parses them, and stores events in Postgres.

Purpose: This is the data ingestion foundation. Without capturing swap events from the blockchain, we cannot build OHLCV candles or historical charts. Raw webhooks are required because Helius enhanced webhooks do NOT parse custom Anchor program events (they appear as "UNKNOWN").

Output: Working webhook endpoint that stores parsed swap/epoch/carnage events in Postgres with idempotent handling.
</objective>

**Create `app/db/connection.ts`** -- Drizzle ORM client singleton.

This file establishes the database connection using postgres.js (NOT node-postgres/pg). postgres.js is the Drizzle-recommended driver: no native bindings, works in serverless/edge, and supports automatic connection pooling.

```typescript
import { drizzle } from "drizzle-orm/postgres-js";
import postgres from "postgres";
import * as schema from "./schema";

// Singleton pattern: globalThis cache survives Next.js hot reloads in dev.
// Without this, each HMR cycle creates a new connection pool, exhausting Postgres connections.
const globalForDb = globalThis as unknown as {
  pgClient: ReturnType<typeof postgres> | undefined;
};

const connectionString = process.env.DATABASE_URL;

if (!connectionString) {
  throw new Error(
    "DATABASE_URL environment variable is required. Set it in .env.local for local dev or Railway environment for production."
  );
}

// postgres.js client: max 10 connections (Railway free tier has 10 connection limit)
const client = globalForDb.pgClient ?? postgres(connectionString, { max: 10 });

if (process.env.NODE_ENV !== "production") {
  globalForDb.pgClient = client;
}

export const db = drizzle(client, { schema });
```

**Create `app/lib/event-parser.ts`** -- Parses Anchor `emit!()` events from raw transaction log messages.

**Why this is needed:** Helius raw webhooks deliver the full transaction object including `logMessages`. Anchor events are base64-encoded in the `Program data:` log lines. We need to decode these using Anchor's `BorshCoder` (which reads the IDL discriminators and field layouts).

**How Anchor event parsing works:**
1. Each `emit!()` call produces a log line: `Program data: <base64-encoded-event>`
2. The first 8 bytes are the event discriminator (sha256("event:EventName")[0..8])
3. The remaining bytes are Borsh-serialized event fields
4. Anchor's `EventParser` walks logMessages, identifies program invocations, and decodes events

**Implementation approach:**

Use `@coral-xyz/anchor`'s `EventParser` class (already in dependencies). It requires:
- An `Idl` object (from our JSON IDL files)
- A program ID (to match against log program invocation markers)

```typescript
import { EventParser, BorshCoder, Idl } from "@coral-xyz/anchor";
import taxProgramIdl from "@/idl/tax_program.json";
import epochProgramIdl from "@/idl/epoch_program.json";
import { PROGRAM_IDS } from "@dr-fraudsworth/shared";

// Type the IDLs (Anchor IDL JSON files are typed as Idl)
const taxIdl = taxProgramIdl as unknown as Idl;
const epochIdl = epochProgramIdl as unknown as Idl;
```

**Define TypeScript interfaces for parsed events** (matching the IDL event structs):

```typescript
export interface ParsedTaxedSwap {
  type: "TaxedSwap";

  direction: string;   // "Buy" | "Sell"
  inputAmount: number; // u64 -> number (safe for lamport amounts under 2^53)
  outputAmount: number;
  taxAmount: number;
  taxRateBps: number;  // u16

  carnagePortion: number;
  treasuryPortion: number;
  epoch: number;       // u32

export interface ParsedTaxesUpdated {
  type: "TaxesUpdated";

  highTaxBps: number;
  flipped: boolean;

export interface ParsedCarnageExecuted {
  type: "CarnageExecuted";
  epoch: number;
  action: number;    // 0=None, 1=Burn, 2=Sell
  target: number;    // 0=CRIME, 1=FRAUD
  solSpent: number;
  tokensBought: number;
  tokensBurned: number;
  solFromSale: number;

export type ParsedCarnageEvent = ParsedCarnageExecuted;
```

**Create parse functions:**

Each function takes `logMessages: string[]` and returns an array of parsed events.

```typescript
function createTaxParser(): EventParser {
  const coder = new BorshCoder(taxIdl);
  return new EventParser(PROGRAM_IDS.TAX_PROGRAM, coder);
}

function createEpochParser(): EventParser {
  const coder = new BorshCoder(epochIdl);
  return new EventParser(PROGRAM_IDS.EPOCH_PROGRAM, coder);
}
```

**For `parseSwapEvents(logMessages: string[]): ParsedSwapEvent[]`:**

Use the tax program EventParser. For each parsed event:
- Check `event.name` for "TaxedSwap", "UntaxedSwap", "ExemptSwap"
- Extract fields from `event.data` (Anchor BN values need `.toNumber()` conversion)
- Map Anchor enum variants (pool_type, direction) to string labels
- **Important:** Anchor BN objects from event data need `.toNumber()` -- these are u64 values that are safe for JS Number since they represent lamport amounts (max ~18B SOL = 18e18 lamports, within 2^53)

**For `parseEpochEvents(logMessages: string[]): ParsedEpochEvent[]`:**

Use the epoch program EventParser. Parse "EpochTransitionTriggered" and "TaxesUpdated" events.

**For `parseCarnageEvents(logMessages: string[]): ParsedCarnageEvent[]`:**

Use the epoch program EventParser (Carnage events are emitted by the Epoch Program). Parse "CarnageExecuted" events.

**Key detail on Anchor enum handling:**

Anchor serializes Rust enums as objects like `{ solCrime: {} }` or `{ buy: {} }`. To extract the variant name, use `Object.keys(value)[0]`. For example:

```typescript
function enumVariant(value: unknown): string {
  if (typeof value === "object" && value !== null) {
    return Object.keys(value)[0] ?? "unknown";
  }
  return String(value);
}
```

This is a known pattern in this codebase (see MEMORY.md: "Handle cheapSide as both numeric (0/1) and object ({crime:{}}) enum").

**Create `app/app/api/webhooks/helius/route.ts`** -- Next.js App Router POST handler.

**Why a route handler (not an API route in pages/):** We're using App Router (Next.js 16). Route handlers live in `app/api/` and export named HTTP method functions.

**Helius raw webhook payload structure:**

Helius raw webhooks (`rawDevnet` type) deliver an array of transaction objects. Each transaction object contains:
- `signature` (string): TX signature
- `slot` (number): slot number
- `blockTime` (number | null): unix timestamp
- `meta.logMessages` (string[]): program log messages (where Anchor events live)
- `transaction.message.accountKeys` (array): accounts involved

The webhook POSTs an array `[]` of transactions (batch delivery).

**Implementation:**

```typescript
import { NextRequest, NextResponse } from "next/server";
import { db } from "@/db/connection";
import { swapEvents, epochEvents, carnageEvents } from "@/db/schema";
import {
  parseSwapEvents,
  parseEpochEvents,
  parseCarnageEvents,
  ParsedTaxedSwap,
  ParsedUntaxedSwap,
} from "@/lib/event-parser";
import { DEVNET_POOLS } from "@dr-fraudsworth/shared";

// Force Node.js runtime (not Edge) -- we need postgres.js driver
export const runtime = "nodejs";
// Disable response caching
export const dynamic = "force-dynamic";
```

**Authorization check:**

Check the `Authorization` header against `HELIUS_WEBHOOK_SECRET` env var. If the env var is not set, skip the check (allows local testing without a secret). If set and the header doesn't match, return 401.

```typescript
const webhookSecret = process.env.HELIUS_WEBHOOK_SECRET;
if (webhookSecret) {
  const authHeader = req.headers.get("authorization");
  if (authHeader !== webhookSecret) {
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
  }
}
```

**Main POST handler flow:**

1. Parse JSON body as array of transactions
2. For each transaction:
   a. Extract `signature`, `slot`, `blockTime`, `meta.logMessages`
   b. If no logMessages, skip (no events to parse)
   c. Run `parseSwapEvents(logMessages)` to get swap events
   d. Run `parseEpochEvents(logMessages)` to get epoch events
   e. Run `parseCarnageEvents(logMessages)` to get carnage events
3. For each parsed TaxedSwap or UntaxedSwap event, upsert into `swap_events` table:
   - Map pool type enum to pool PDA address using a lookup (SolCrime -> DEVNET_POOLS.CRIME_SOL.pool, etc.)
   - Direction: "Buy" -> "buy", "Sell" -> "sell"
   - Price derivation: For SOL pools: `sol_lamports / token_base_units`. For PROFIT pools: `input_amount / output_amount` (direction-dependent)
   - **Price derivation detail for TaxedSwap:**
     - Buy (SOL -> Token): price = inputAmount / outputAmount (SOL per token, pre-tax input)
     - Sell (Token -> SOL): price = outputAmount / inputAmount (SOL per token, post-tax output)
   - **Price derivation for UntaxedSwap:**
     - Buy (CRIME/FRAUD -> PROFIT): price = inputAmount / outputAmount
     - Sell (PROFIT -> CRIME/FRAUD): price = outputAmount / inputAmount
   - Use `onConflictDoNothing()` on tx_signature (idempotent -- skip duplicates)
   - Set timestamp from `blockTime` (convert unix seconds to Date) or fallback to `new Date()`
4. For each parsed EpochTransitionTriggered, upsert into `epoch_events`:
   - Use `onConflictDoNothing()` on epoch_number unique index
   - Note: TaxesUpdated events provide tax rates but epoch_events schema stores them together. We need to handle the case where EpochTransitionTriggered and TaxesUpdated come in the same TX (they often do during consume_randomness). If both are present, merge them into one row. If only EpochTransitionTriggered arrives, insert with null tax fields. If only TaxesUpdated, use `onConflictDoUpdate` to fill in tax rates.
   - **Simplification:** Since both events typically arrive in the same transaction, iterate all epoch events and build a merged record before inserting.
5. For each parsed CarnageExecuted event, upsert into `carnage_events`:
   - Map target (0/1) to "CRIME"/"FRAUD"
   - Map action (0/1/2) to "BuyOnly"/"Burn"/"BurnAndSell"
   - Use `onConflictDoNothing()` on epoch_number unique index
6. Return 200 OK with count of processed events

**ExemptSwap handling:** ExemptSwap events are Carnage-internal swaps. They don't represent user trades and should NOT be stored in swap_events (they'd create false price data). Log them for debugging but don't store them.

**Error handling:**
- Wrap the entire handler in try/catch
- On parse errors for individual transactions, log and continue (don't fail the whole batch)
- On DB errors, return 500 (Helius will retry)
- Always return a response (Helius expects 200 for success)

**Pool type to pool address mapping helper:**

```typescript
function poolTypeToAddress(poolType: string): string | null {
  const map: Record<string, string> = {
    solCrime: DEVNET_POOLS.CRIME_SOL.pool.toBase58(),
    solFraud: DEVNET_POOLS.FRAUD_SOL.pool.toBase58(),
    crimeProfit: DEVNET_POOLS.CRIME_PROFIT.pool.toBase58(),
    fraudProfit: DEVNET_POOLS.FRAUD_PROFIT.pool.toBase58(),
  };
  return map[poolType] ?? null;
}
```
