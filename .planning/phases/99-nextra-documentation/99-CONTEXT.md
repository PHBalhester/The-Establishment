<decisions>
## Implementation Decisions

### Workflow approach
- Page-by-page, front to back, in sidebar order (Welcome -> Laboratory -> How It Works -> Earning -> Launch -> Trust & Safety -> Reference)
- Each page: review together, decide what to cut/change, craft diagram prompts inline, rewrite, move to next
- Collaborative — user and Claude work through every page together

### Diagram style & tooling
- Custom PNG artwork generated by user from prompts crafted collaboratively
- Steampunk-themed art style: brass pipes, gears, gauges, Victorian blueprint aesthetic
- 9 diagrams total: 6 core (token flow, tax distribution, epoch lifecycle, carnage mechanics, staking rewards, bonding curve) + 3 hero (system architecture, conversion vault, soft peg)
- Diagram prompts crafted when we reach the relevant page, not batch-upfront
- Images placed in `docs-site/public/diagrams/` and referenced via `<img>` in MDX

### Page structure
- Keep existing 6-section, 16-page structure — rewrite content in-place
- Add one new page: "Authority & Governance" in the Trust & Safety section
- No section reorganization — current hierarchy covers the protocol well
- Total: 17 pages after adding governance

### Mainnet address strategy
- Use known vanity mint addresses now (cRiME, FraUd, pRoFiT)
- Use clear placeholders for unknown program IDs: `<MAINNET_AMM_PROGRAM_ID>` etc.
- Create tracking document at `Docs/address-placeholders.md` mapping every placeholder to file:line
- Post-Phase 100 sweep uses tracking doc to mechanically replace all placeholders — no separate phase needed

### Audience & tone
- Primary audience: memecoin/DeFi traders (understand AMMs, staking, token mechanics)
- Light steampunk character flavor in headers, intros, and transitions
- Body text: clear, factual, direct — never obscure important info for brand flavor
- Terminology: "rewards" or "staking rewards" everywhere — not "yield" or "APY" or "yield farming"
- Framing: "protocol" or "system" — not "game" or "gamble". "Factory" and "experiment" for brand flavor only
- No code snippets or Rust details — this is user documentation, not developer docs

### Claude's Discretion
- Exact prose and paragraph structure within each page
- Whether to use Nextra callout boxes, tables, or plain markdown for specific content
- Ordering of subsections within a page
- Glossary term additions/removals based on what the rewritten pages reference

</decisions>
