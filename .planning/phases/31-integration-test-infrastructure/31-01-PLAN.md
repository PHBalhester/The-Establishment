  <action>
Create the `tests/integration/helpers/` directory structure.

**constants.ts** must export:

1. **PDA Seeds** (as Buffer) -- must exactly match on-chain program constants:
   - Staking: `STAKE_POOL_SEED` ("stake_pool"), `ESCROW_VAULT_SEED` ("escrow_vault"), `STAKE_VAULT_SEED` ("stake_vault"), `USER_STAKE_SEED` ("user_stake")
   - Transfer Hook: `WHITELIST_AUTHORITY_SEED` ("authority"), `WHITELIST_ENTRY_SEED` ("whitelist"), `EXTRA_ACCOUNT_META_SEED` ("extra-account-metas")
   - AMM: `ADMIN_SEED` ("admin"), `POOL_SEED` ("pool"), `VAULT_SEED` ("vault"), `VAULT_A_SEED` ("a"), `VAULT_B_SEED` ("b"), `SWAP_AUTHORITY_SEED` ("swap_authority")
   - Tax: `TAX_AUTHORITY_SEED` ("tax_authority")
   - Epoch: `EPOCH_STATE_SEED` ("epoch_state"), `CARNAGE_FUND_SEED` ("carnage_fund"), `CARNAGE_SOL_VAULT_SEED` ("carnage_sol_vault"), `CARNAGE_CRIME_VAULT_SEED` ("carnage_crime_vault"), `CARNAGE_FRAUD_VAULT_SEED` ("carnage_fraud_vault"), `CARNAGE_SIGNER_SEED` ("carnage_signer")

2. **Token Constants**:
   - `TOKEN_DECIMALS = 6` (all three meme tokens use 6 decimals)
   - `MINIMUM_STAKE = 1_000_000` (1 PROFIT in base units)

3. **Pool Fee Constants**:
   - `SOL_POOL_FEE_BPS = 100` (1% for CRIME/SOL, FRAUD/SOL)
   - `PROFIT_POOL_FEE_BPS = 50` (0.5% for CRIME/PROFIT, FRAUD/PROFIT)

4. **Seed Liquidity Amounts** (reasonable test amounts):
   - `SOL_POOL_SEED_SOL = 10 * LAMPORTS_PER_SOL` (10 SOL)
   - `SOL_POOL_SEED_TOKEN = 10_000_000_000` (10,000 tokens at 6 decimals)
   - `PROFIT_POOL_SEED_A = 10_000_000_000` (10,000 tokens)
   - `PROFIT_POOL_SEED_B = 10_000_000_000` (10,000 tokens)

5. **Convenience PDA derivation helpers**:
   - `derivePoolPDA(mintA, mintB, ammProgramId)` -> returns [PublicKey, bump]
   - `deriveVaultPDAs(poolKey, ammProgramId)` -> returns { vaultA: [PK, bump], vaultB: [PK, bump] }
   - `deriveWhitelistEntryPDA(address, hookProgramId)` -> returns [PublicKey, bump]

Import `LAMPORTS_PER_SOL` and `PublicKey` from `@solana/web3.js`.

**test-wallets.ts** must export:

1. **`TestWallets` interface**:
   ```typescript
   interface TestWallets {
     trader: { keypair: Keypair; wsolAccount?: PublicKey; tokenAccounts: Map<string, PublicKey> };
     staker: { keypair: Keypair; profitAccount?: PublicKey };
     admin: { keypair: Keypair };
     attacker: { keypair: Keypair };
   }
   ```

2. **`createTestWallets(connection, admin, mints)`** async function:
   - Takes: Connection, admin Keypair (who mints tokens), and object with `{ crimeMint, fraudMint, profitMint }` PublicKeys
   - Creates 4 keypairs (trader, staker, admin/authority, attacker)
   - Airdrops SOL to each (10 SOL for trader, 5 SOL for staker, 5 SOL for admin, 2 SOL for attacker)
   - For trader: Creates Token-2022 accounts for CRIME, FRAUD, PROFIT. Mints 1000 of each token. Creates WSOL account with 5 SOL wrapped.
   - For staker: Creates Token-2022 account for PROFIT. Mints 1000 PROFIT.
   - For attacker: Creates Token-2022 account for CRIME (for unauthorized swap attempts). Mints 100 CRIME.
   - Returns `TestWallets` struct
   - Use `@solana/spl-token` helpers: `createAccount`, `mintTo`, `createWrappedNativeAccount` (from standard SPL Token, not Token-2022 for WSOL)
   - Whitelist the trader's token accounts in the Transfer Hook (they need to be whitelisted to transfer to pool vaults). Actually -- wait. The whitelist check requires at least ONE party (source or dest) to be whitelisted. Pool vaults will be whitelisted in protocol-init. So user accounts do NOT need whitelisting. Skip user account whitelisting.

Both files should use TypeScript with proper imports from `@coral-xyz/anchor`, `@solana/web3.js`, and `@solana/spl-token`.

IMPORTANT: Do NOT hardcode program IDs as strings. The helpers should accept program IDs as parameters or derive them from `anchor.workspace.*` at call time.
