<task type="auto">
  <name>Task 2: Run upload-metadata.ts and update on-chain URIs</name>
  <files>
    deployments/devnet.json
    .env.devnet
  </files>
  <action>
    1. Verify logos exist and are valid PNGs:
       - Check `assets/logos/crime.png`, `assets/logos/fraud.png`, `assets/logos/profit.png` exist
       - Verify each is a valid PNG (check file header bytes or use `file` command)
       - Log dimensions if possible (should be 512x512)

    2. If user requested description changes in the checkpoint, update metadata-templates.ts with their chosen descriptions before proceeding.

    3. Run upload-metadata.ts to upload all assets to Arweave:
       ```bash
       export PATH="/opt/homebrew/bin:$PATH"
       set -a && source .env.devnet && set +a
       npx tsx scripts/deploy/upload-metadata.ts --cluster devnet --keypair keypairs/devnet-wallet.json
       ```
       This will:
       - Upload 3 PNGs to Arweave (permanent)
       - Upload 3 metadata JSONs to Arweave (permanent, referencing image URIs)
       - Write URIs to deployments/devnet.json metadata section
       - Write CRIME_METADATA_URI, FRAUD_METADATA_URI, PROFIT_METADATA_URI to .env.devnet
       - Verify each upload by fetching the URI

    4. After upload succeeds, update the on-chain metadata URIs for the 3 existing devnet mints.
       The existing mints (CRIME=8NEgQ, FRAUD=76ddo, PROFIT=7X6xx) currently have Railway placeholder URIs.
       Use `tokenMetadataUpdateField` from `@solana/spl-token` to update each mint's URI to the new Arweave URI.

       Create a small inline script or add an `--update-onchain` flag to upload-metadata.ts that:
       - Reads the 3 metadata URIs from deployments/devnet.json
       - Reads the 3 mint addresses from deployments/devnet.json
       - Calls `tokenMetadataUpdateField(connection, payer, mintPubkey, payer, "uri", arweaveUri)` for each mint
       - Logs the transaction signature for each update

       If this is cleaner as a separate step in upload-metadata.ts (e.g., after all uploads, if mints exist on-chain, update their URIs), do that. The key is: for EXISTING mints (devnet), the on-chain URI must be updated. For FRESH deploys (mainnet), initialize.ts will read the URIs automatically.

    5. Verify the on-chain updates:
       ```bash
       source "$HOME/.cargo/env" && export PATH="/Users/mlbob/.local/share/solana/install/active_release/bin:$PATH"
       # Check each mint's metadata -- should show arweave.net URI, not Railway
       solana account 8NEgQ... --output json  # CRIME mint
       solana account 76ddo... --output json  # FRAUD mint
       solana account 7X6xx... --output json  # PROFIT mint
       ```
       Or use a quick TypeScript snippet to read and display the metadata URI from each mint.

    Important: This step costs real SOL (from devnet wallet for Irys funding, plus TX fees for on-chain updates). Expected total cost: < 0.01 SOL for Arweave + ~0.000015 SOL for 3 on-chain TXs. Ensure devnet wallet has sufficient balance.
  </action>
  <verify>
    - All 6 Arweave URIs are accessible via HTTP GET (curl each, expect 200)
    - Image URIs return PNG content (Content-Type: image/png)
    - Metadata URIs return valid JSON with correct name, symbol, image, external_url fields
    - deployments/devnet.json has `metadata` section with 3 arweave.net URIs
    - .env.devnet contains CRIME_METADATA_URI, FRAUD_METADATA_URI, PROFIT_METADATA_URI
    - On-chain mint metadata shows arweave.net URIs (not Railway)
  </verify>
  <done>
    All 6 files uploaded to Arweave with permanent URIs. deployment.json and .env.devnet updated. On-chain mint metadata updated from Railway placeholders to Arweave URIs.
  </done>
