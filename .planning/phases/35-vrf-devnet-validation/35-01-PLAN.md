  ```bash
    --keypair keypairs/devnet-wallet.json \
    --with-compute-unit-price 1
  ```
  The `--program-id` flag ensures it deploys to the same address (AH7yaWFUrWmXGDKEwr8w2EMEiH6PxaxE2vT9orjRzvod). The upgrade authority is the devnet wallet.

  **Step C: Close existing EpochState PDA**
  The EpochState PDA (DVV9ebobxXctrsPZpuSDTj4g85Cg2VmroLLq3chLuBDU) was initialized with an old genesis_slot. With SLOTS_PER_EPOCH now 750 instead of 4,500, the computed current_epoch would be enormously large. Close it so we can re-init fresh.

  Use `solana account close` or write a small TypeScript script that calls Solana's system program to close the account. The devnet wallet is the authority. If `solana account close` doesn't work for PDA accounts, try the alternative: since we have upgrade authority over the program, the simplest approach may be to accept the epoch jump (the program's trigger_epoch_transition handles arbitrary epoch advancement). If that fails (PDA accounts may not be closeable via CLI since they aren't owned by the System Program), fall back to Option B from research: accept the epoch jump.

  **Step D: Re-initialize EpochState (if closed successfully)**
  Run the initialization script targeting just the EpochState:
  ```bash
  source .env && npx tsx scripts/deploy/initialize.ts
  ```
  The idempotent init script will skip all already-initialized accounts and only initialize the EpochState (since it was closed). Verify by querying the account:
  ```bash
  solana account DVV9ebobxXctrsPZpuSDTj4g85Cg2VmroLLq3chLuBDU --url devnet
  ```
