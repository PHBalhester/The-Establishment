Create swap_profit_buy.rs for CRIME/FRAUD -> PROFIT swaps.

**Key differences from swap_sol_buy:**
- NO tax calculation, NO tax distribution, NO epoch_state/staking/carnage/treasury accounts
- Uses AMM swap_profit_pool discriminator (not swap_sol_pool)
- Direction = 0 (AtoB) - buying PROFIT with CRIME/FRAUD
- Single token_2022_program (both sides are Token-2022)
- Emits UntaxedSwap event instead of TaxedSwap

**Account struct (SwapProfitBuy):**
Per Tax_Pool_Logic_Spec.md Section 10.4:
```rust
pub struct SwapProfitBuy<'info> {
    #[account(mut)]
    pub user: Signer<'info>,

    #[account(seeds = [SWAP_AUTHORITY_SEED], bump)]
    pub swap_authority: AccountInfo<'info>,

    #[account(mut)]
    pub pool: AccountInfo<'info>,

    #[account(mut)]
    pub pool_vault_a: InterfaceAccount<'info, TokenAccount>,  // CRIME/FRAUD
    #[account(mut)]
    pub pool_vault_b: InterfaceAccount<'info, TokenAccount>,  // PROFIT

    pub mint_a: InterfaceAccount<'info, Mint>,  // CRIME/FRAUD
    pub mint_b: InterfaceAccount<'info, Mint>,  // PROFIT

    #[account(mut)]
    pub user_token_a: InterfaceAccount<'info, TokenAccount>,  // User's input
    #[account(mut)]
    pub user_token_b: InterfaceAccount<'info, TokenAccount>,  // User's output

    pub amm_program: AccountInfo<'info>,
    pub token_2022_program: Interface<'info, TokenInterface>,  // Both sides T22
}
```

**Handler flow:**
1. Build swap_authority PDA signer seeds
2. Build AMM swap_profit_pool instruction:
   - Discriminator: `[0xce, 0xa3, 0x0b, 0x22, 0xf1, 0x6c, 0x24, 0xa6]` (verified in RESEARCH.md)
   - Direction: 0 (AtoB = buying PROFIT)
   - Forward remaining_accounts for dual transfer hooks
3. Execute CPI with invoke_signed
4. Emit UntaxedSwap event

**AMM account order (from swap_profit_pool.rs SwapProfitPool struct):**
swap_authority, pool, vault_a, vault_b, mint_a, mint_b, user_token_a, user_token_b, user, token_program_a, token_program_b

Pass token_2022_program for BOTH token_program_a and token_program_b.
