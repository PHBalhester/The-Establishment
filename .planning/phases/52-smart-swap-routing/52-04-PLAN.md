    **NOTE on resolvePool:** The existing resolvePool function returns null for multi-hop pairs (SOL->PROFIT, CRIME->FRAUD, etc.). This is correct -- resolvePool finds the DIRECT pool. For multi-hop pairs, the route engine handles path resolution. Do NOT modify resolvePool.

    **app/hooks/useRoutes.ts -- New hook:**

    ```typescript
    export interface UseRoutesReturn {
      routes: Route[];                    // ranked routes (best first)
      selectedRoute: Route | null;        // currently selected route
      selectRoute: (route: Route) => void;
      routesLoading: boolean;             // true during computation
      refreshCountdown: number;           // seconds until next auto-refresh (0-30)
    }

    export function useRoutes(
      inputToken: TokenSymbol,
      outputToken: TokenSymbol,
      inputAmount: number,                // base units (0 = no routes)
      pools: PoolPricesMap,               // from usePoolPrices
      epochState: EpochStateData | null,  // from useEpochState
      slippageBps: number,
      enabled: boolean,                   // false when smart routing is OFF
    ): UseRoutesReturn;
    ```

    Implementation:
    1. **Route computation** (debounced 300ms, matching useSwap pattern):
       - When inputs change (inputToken, outputToken, inputAmount, pools, epochState, slippageBps), start debounce timer
       - After debounce fires:
         a. Convert pools data (PoolPricesMap from usePoolPrices) into PoolReserves format for route engine
         b. Convert epochState (EpochStateData from useEpochState) into EpochTaxState format
         c. Call computeRoutes(inputToken, outputToken, inputAmount, poolReserves, epochTaxState, slippageBps)
         d. For split-eligible pairs (SOL<->PROFIT): Also call computeOptimalSplit with the two multi-hop path quoters
         e. If split is recommended (shouldSplit=true), create a split Route object and add it to the routes array
         f. Sort all routes by outputAmount descending
         g. Set routes state

    2. **Split route construction:**
       - Split-eligible pairs: (SOL, PROFIT) and (PROFIT, SOL)
       - For SOL->PROFIT: pathA = SOL->CRIME->PROFIT, pathB = SOL->FRAUD->PROFIT
       - Compose pathAQuoter and pathBQuoter using quoteSolBuy + quoteProfitBuy
       - Call computeOptimalSplit(totalInput, pathAQuoter, pathBQuoter)
       - If shouldSplit: create a Route with isSplit=true, splitRatio=[splitRatioA, splitRatioB], and label like "SOL -> CRIME/FRAUD -> PROFIT (Split)"
       - The split Route's outputAmount is the splitOutput from computeOptimalSplit
       - The split Route's steps should contain all 4 steps (2 per leg): [legA_hop1, legA_hop2, legB_hop1, legB_hop2]

    3. **Auto-refresh (30s timer):**
       - After routes are computed, start a 30-second countdown interval (1-second ticks)
       - refreshCountdown state counts down from 30 to 0
       - When refreshCountdown reaches 0, re-trigger route computation with current inputs
       - Timer resets to 30 whenever inputs change (inputToken, outputToken, inputAmount change)

    4. **Route selection with flicker prevention:**
       - State: selectedRoute (Route | null)
       - When routes change: if the current selectedRoute.label exists in new routes AND its output is within 0.1% (10 bps) of the new best route, keep it selected
       - If current selection is gone OR a different route beats it by > 0.1%, auto-select routes[0]
       - selectRoute(route) sets selectedRoute manually (user clicked a different route)

    5. **enabled flag:**
       - When enabled=false, return { routes: [], selectedRoute: null, routesLoading: false, refreshCountdown: 30 }
       - Do not start any timers or computations when disabled

    6. **Cleanup:** Clear all timers (debounce + refresh interval) on unmount and when enabled toggles to false

    **Important:** The PoolPricesMap type from usePoolPrices is `Record<string, PoolData>` where keys are pool labels ("CRIME/SOL", etc.) and PoolData has reserveA, reserveB, loading. Convert this to PoolReserves by checking all 4 pools are loaded and extracting reserves. If any pool is still loading, set routesLoading=true and skip computation.

    **Modified executeSwap behavior:**
    When smart routing is ON and a route is selected:
    - If selectedRoute.hops === 1: Use the existing single-swap execution path (no change)
    - If selectedRoute.hops > 1 (multi-hop): Call buildMultiHopTransactions then executeMultiHopRoute
    - If selectedRoute.isSplit: Execute as 4 sequential transactions (2 per leg), using executeMultiHopRoute for each leg sequentially

    Create a new function `executeRoute` that handles all three cases:
    ```typescript
    const executeRoute = useCallback(async () => {
      if (!smartRouting || !routesResult.selectedRoute) {
        // Fallback to existing direct swap
        return executeSwap();
      }

      const route = routesResult.selectedRoute;

      if (route.hops === 1 && !route.isSplit) {
        // Single hop, use existing direct swap path
        return executeSwap();
      }

      // Multi-hop or split execution
      try {
        setStatus("building");
        setErrorMessage(null);
        setTxSignature(null);

        const priorityMicroLamports = PRIORITY_FEE_MAP[priorityFeePreset];

        if (route.isSplit) {
          // Split route: execute each leg as a multi-hop
          // Leg A: steps[0] + steps[1], Leg B: steps[2] + steps[3]
          const legA: Route = { ...route, steps: route.steps.slice(0, 2), isSplit: false };
          const legB: Route = { ...route, steps: route.steps.slice(2, 4), isSplit: false };

          // Build transactions for both legs
          const txsA = await buildMultiHopTransactions(legA, connection, wallet.publicKey, priorityMicroLamports);
          const txsB = await buildMultiHopTransactions(legB, connection, wallet.publicKey, priorityMicroLamports);

          // Sign all 4 TXs at once
          setStatus("signing");
          const allTxs = [...txsA, ...txsB];
          const allSigned = wallet.signAllTransactions
            ? await wallet.signAllTransactions(allTxs)
            : await Promise.all(allTxs.map(tx => wallet.signTransaction(tx)));

          // Send sequentially
          setStatus("sending");
          // ... send each, track partial failures
          // (Full implementation in executeMultiHopRoute, but for split we call it twice)
        } else {
          // Multi-hop (not split)
          const txs = await buildMultiHopTransactions(route, connection, wallet.publicKey, priorityMicroLamports);

          setStatus("signing");
          const result = await executeMultiHopRoute(txs, route, wallet, connection);

          if (result.status === "confirmed") {
            setStatus("confirmed");
            setTxSignature(result.signatures[result.signatures.length - 1]); // last sig
            refreshBalances();
            autoResetTimerRef.current = setTimeout(resetForm, AUTO_RESET_MS);
          } else if (result.status === "partial") {
            // Partial failure: set special status
            setStatus("failed");
            setTxSignature(result.signatures[result.signatures.length - 1] || null);
            setErrorMessage(`Hop ${(result.failedAt ?? 0) + 1} failed: ${result.error}. You now hold ${result.intermediateToken}.`);
          } else {
            setStatus("failed");
            setErrorMessage(result.error || "Multi-hop swap failed");
          }
        }
      } catch (error) {
        console.error("[useSwap] executeRoute error:", error);
        setStatus("failed");
        setErrorMessage(parseSwapError(error));
      }
    }, [smartRouting, routesResult.selectedRoute, wallet, priorityFeePreset, executeSwap, ...]);
    ```
