    **MultiHopStatus.tsx** -- Multi-hop execution progress and partial failure UI:

    Props interface:
    ```typescript
    interface MultiHopStatusProps {
      status: "building" | "signing" | "sending" | "confirming" | "confirmed" | "failed";
      route: Route | null;
      errorMessage: string | null;
      intermediateToken?: string;  // set on partial failure
      onRetry: () => void;         // retry hop 2 with fresh quote
      onKeep: () => void;          // dismiss, keep intermediate token
      txSignature: string | null;
    }
    ```

    Rendering logic:
    - status "building"/"signing"/"sending"/"confirming": Show spinner (animate-spin SVG circle) + status text:
      - "building": "Preparing route..."
      - "signing": "Approve in wallet..."
      - "sending": "Executing route..."
      - "confirming": "Confirming transaction..."
    - If route is multi-hop (hops > 1) and status is "sending" or "confirming": Show "Executing route..." with route.label underneath in small gray text
    - status "confirmed": Show green check + "Route completed!" + link to explorer (using txSignature)
    - status "failed" with intermediateToken: Show partial failure UI:
      - Yellow/amber warning icon
      - "Hop 1 succeeded, Hop 2 failed"
      - Error message text
      - "You now hold {intermediateToken}" in bold
      - Two buttons:
        - "Retry swap" (primary blue button): calls onRetry
        - "Keep {intermediateToken}" (secondary gray button): calls onKeep
    - status "failed" without intermediateToken: Show standard red error with errorMessage

    **Styling:** Match SwapStatus.tsx patterns (same container width, button styles, text sizes). Use animate-spin for spinner. Explorer link: `https://explorer.solana.com/tx/${txSignature}?cluster=devnet`

    **FeeBreakdown.tsx** -- Update to handle multi-hop fee aggregation:

    The existing FeeBreakdown receives a SwapQuote with lpFee, taxAmount, totalFeePct, priceImpactBps. For multi-hop routes, these are already aggregated by the route engine (the quote is set from the selected route in useSwap Task 2 of Plan 04).

    Changes needed:
    1. Add a `route: Route | null` prop (optional, undefined for non-routed swaps)
    2. When route is provided and route.hops > 1: show fee breakdown per hop in the expanded details:
       - For each step in route.steps: show "{step.pool}: LP {step.lpFeeBps}bps" + "Tax: {step.taxBps}bps" on a single line
    3. When route.isSplit: show "Split route" label with the two legs
    4. Keep the existing summary row (total LP fee, total tax, total fee %, price impact) unchanged -- it already works from the aggregated quote

    This is a MINOR modification -- add the route prop and a conditional section in the expanded view. Do NOT change the collapsed/summary behavior.
