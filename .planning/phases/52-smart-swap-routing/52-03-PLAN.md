<task type="auto">
  <name>Task 2: Create RouteSelector component</name>
  <files>app/components/swap/RouteSelector.tsx</files>
  <action>
    **RouteSelector.tsx** -- Collapsible route list:
    - Props interface:
      ```typescript
      interface RouteSelectorProps {
        routes: Route[];           // ranked routes (best first)
        selectedRoute: Route | null;
        onSelectRoute: (route: Route) => void;
        loading: boolean;
        outputDecimals: number;    // pass through to RouteCard
        refreshCountdown: number;  // seconds until next quote refresh (0-30)
      }
      ```
    - State: `expanded` boolean (default false -- only best route shown)
    - Behavior:
      - If routes is empty and not loading: render nothing
      - If loading: render a subtle "Finding best route..." text with animate-pulse
      - If routes has 1 route: render just the RouteCard (no expand/collapse UI)
      - If routes has 2+ routes:
        1. Always show the best route (routes[0]) as a RouteCard with isBest=true
        2. Below it, show a "See all routes ({routes.length})" button that toggles expanded state
        3. When expanded, show all remaining routes as RouteCard components
        4. Show a circular countdown timer next to "See all routes":
           - Small circle (16x16px) with SVG stroke-dasharray animation
           - Shows refreshCountdown number in the center
           - Use stroke color gray-500, animate clockwise
      - selectedRoute determines which RouteCard gets isSelected=true (compare by route.label since routes are recomputed)
      - If selectedRoute is null, auto-select routes[0] (call onSelectRoute(routes[0]) on mount/route change)

    **Auto-selection logic:**
    - When routes change and the currently selected route's label still exists in the new routes, keep it selected (anti-flicker from RESEARCH.md: keep current selection if within 0.1% of new best)
    - If the current selection no longer exists in new routes, select routes[0]
    - Use a useEffect for this auto-selection logic

    **IMPORTANT: Route flicker prevention (from RESEARCH.md):**
    - If the currently selected route is still within 0.1% (10 bps) of the new best route's output, do NOT auto-switch the selection
    - Only auto-switch when: (a) current selection is gone, or (b) a different route beats current by > 0.1%

    **Styling:**
    - Container: `mt-3` gap from output section
    - Expand/collapse button: text-sm text-gray-400, underline on hover
    - Transition for expand/collapse: use height animation or simple conditional render (conditional render is fine for v1)
    - Match dark theme from SwapForm.tsx
  </action>
