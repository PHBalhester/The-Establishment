  <behavior>
    The route engine is a stateless module that takes pool reserves, epoch tax state, and swap parameters (inputToken, outputToken, inputAmount, slippageBps) and returns an array of Route objects sorted by descending outputAmount.

    **Route Types (route-types.ts):**

    ```typescript
    export type TokenSymbol = "SOL" | "CRIME" | "FRAUD" | "PROFIT";

    export interface RouteStep {
      pool: string;           // Pool label: "CRIME/SOL", "FRAUD/SOL", "CRIME/PROFIT", "FRAUD/PROFIT"
      inputToken: TokenSymbol;
      outputToken: TokenSymbol;
      inputAmount: number;    // base units
      outputAmount: number;   // base units
      lpFeeBps: number;
      taxBps: number;         // 0 for PROFIT pool steps
      priceImpactBps: number;
    }

    export interface Route {
      steps: RouteStep[];
      inputToken: TokenSymbol;
      outputToken: TokenSymbol;
      inputAmount: number;    // base units (user's original input)
      outputAmount: number;   // base units (final output to user)
      totalLpFee: number;     // aggregate LP fees across all hops
      totalTax: number;       // aggregate tax across all hops (lamports)
      totalPriceImpactBps: number;
      totalFeePct: string;    // display string, e.g. "3.5%"
      hops: number;
      isSplit: boolean;
      splitRatio?: [number, number]; // e.g. [60, 40]
      label: string;          // e.g. "SOL -> CRIME -> PROFIT"
      minimumOutput: number;  // after slippage tolerance
    }

    export interface PoolReserves {
      "CRIME/SOL": { reserveA: number; reserveB: number };    // A=WSOL, B=CRIME
      "FRAUD/SOL": { reserveA: number; reserveB: number };    // A=WSOL, B=FRAUD
      "CRIME/PROFIT": { reserveA: number; reserveB: number }; // A=CRIME, B=PROFIT
      "FRAUD/PROFIT": { reserveA: number; reserveB: number }; // A=FRAUD, B=PROFIT
    }

    export interface EpochTaxState {
      crimeBuyTaxBps: number;
      crimeSellTaxBps: number;
      fraudBuyTaxBps: number;
      fraudSellTaxBps: number;
    }
    ```

    **Route Graph:**
    The protocol has exactly 4 pools forming a diamond graph:
    ```
          SOL
         /   \
       CRIME  FRAUD
         \   /
         PROFIT
    ```

    Build a static ROUTE_GRAPH adjacency list mapping each token to its direct neighbors and the pool/instruction used. The graph is bidirectional (each pool supports both directions).

    **Path Enumeration Logic:**
    For a given (inputToken, outputToken) pair:
    1. Find all single-hop paths (direct pool connecting input->output)
    2. Find all 2-hop paths (input->intermediate->output where intermediate is any token with pools to both input and output)
    3. Do NOT enumerate 3+ hop paths (topology doesn't support beneficial ones)

    **Quoting Logic:**
    For each enumerated path, quote it step-by-step using the existing quote-engine.ts functions:
    - SOL buy step: quoteSolBuy(inputLamports, reserveWsol, reserveToken, buyTaxBps, 100)
    - SOL sell step: quoteSolSell(inputTokens, reserveWsol, reserveToken, sellTaxBps, 100)
    - PROFIT buy step: quoteProfitBuy(inputTokens, reserveToken, reserveProfit, 50)
    - PROFIT sell step: quoteProfitSell(inputProfit, reserveToken, reserveProfit, 50)

    The output of step N becomes the input of step N+1.

    **Ranking:** Sort routes by outputAmount descending (best first).

    **Test Cases (input -> expected behavior):**

    Given these test reserves:
    - CRIME/SOL: reserveA=2_000_000_000 (2 SOL), reserveB=10_000_000_000 (10K CRIME)
    - FRAUD/SOL: reserveA=2_000_000_000 (2 SOL), reserveB=10_000_000_000 (10K FRAUD)
    - CRIME/PROFIT: reserveA=10_000_000_000 (10K CRIME), reserveB=10_000_000_000 (10K PROFIT)
    - FRAUD/PROFIT: reserveA=10_000_000_000 (10K FRAUD), reserveB=10_000_000_000 (10K PROFIT)

    Tax state: crimeBuyTax=400, crimeSellTax=800, fraudBuyTax=800, fraudSellTax=400

    1. computeRoutes("SOL", "CRIME", 100_000_000, reserves, taxState, 100)
       -> Returns exactly 1 route (direct SOL->CRIME), outputTokens matches quoteSolBuy exactly

    2. computeRoutes("SOL", "PROFIT", 100_000_000, reserves, taxState, 100)
       -> Returns exactly 2 routes (SOL->CRIME->PROFIT and SOL->FRAUD->PROFIT)
       -> Both routes have hops=2
       -> The route through the cheap side (CRIME, lower buy tax 400) has higher output and ranks first

    3. computeRoutes("CRIME", "FRAUD", 1_000_000, reserves, taxState, 100)
       -> Returns exactly 2 routes (CRIME->SOL->FRAUD and CRIME->PROFIT->FRAUD)
       -> CRIME->SOL->FRAUD route pays sell tax + buy tax; CRIME->PROFIT->FRAUD is untaxed but has 2x LP fee

    4. computeRoutes("CRIME", "SOL", 1_000_000, reserves, taxState, 100)
       -> Returns exactly 1 route (direct CRIME->SOL), outputSol matches quoteSolSell exactly

    5. computeRoutes("SOL", "CRIME", 0, reserves, taxState, 100)
       -> Returns empty array (zero input)

    6. Route.minimumOutput = floor(outputAmount * (10000 - slippageBps) / 10000)

    7. Route.label for multi-hop: "SOL -> CRIME -> PROFIT" (tokens joined by " -> ")

    8. Route.totalFeePct correctly sums LP fees and tax as percentage of input value
