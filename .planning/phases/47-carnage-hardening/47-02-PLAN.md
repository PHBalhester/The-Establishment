<objective>
Harden the fallback Carnage execution path with lock window enforcement and 75% slippage floor, and upgrade expire_carnage to emit CarnageFailed diagnostic events.

Purpose: The fallback path (execute_carnage) currently has the same 50% slippage floor as atomic and no lock window check. This plan adds the lock window gate (fallback cannot execute during the 50-slot atomic-only window), upgrades to 75% slippage (more lenient than atomic since we're in recovery mode), and makes expire_carnage emit CarnageFailed for off-chain monitoring when both paths fail.

Output: Modified execute_carnage.rs and expire_carnage.rs with lock window, tighter slippage, and diagnostic events.

    2. **Add lock window check** immediately AFTER the existing deadline check (line ~210-213). The fallback must be called AFTER the lock window expires but BEFORE the deadline:
       ```rust
       // Validate deadline hasn't expired FIRST (before any execution)
       require!(
           clock.slot <= ctx.accounts.epoch_state.carnage_deadline_slot,
           EpochError::CarnageDeadlineExpired
       );

       // Validate lock window has expired (atomic-only period is over).
       // During the lock window (0 to CARNAGE_LOCK_SLOTS after trigger),
       // only the atomic path bundled with consume_randomness can execute.
       // This prevents MEV bots from front-running the atomic execution.
       require!(
           clock.slot > ctx.accounts.epoch_state.carnage_lock_slot,
           EpochError::CarnageLockActive
       );
       ```

    3. **Replace slippage floor** from 50% to 75% using the same u128 pattern as atomic. Replace the slippage check block (around line ~411-427):

       From:
       ```rust
       if reserve_sol > 0 && reserve_token > 0 {
           let expected = (reserve_token as u128)
               .checked_mul(total_buy_amount as u128)
               .and_then(|n| n.checked_div(
                   (reserve_sol as u128).checked_add(total_buy_amount as u128)?
               ))
               .ok_or(EpochError::Overflow)? as u64;

           require!(
               bought >= expected / 2,
               EpochError::CarnageSlippageExceeded
           );
       }
       ```

       To:
       ```rust
       // 75% slippage floor (25% tolerance) for fallback Carnage path.
       // More lenient than atomic (85%) -- prioritize execution over optimal
       // price when in recovery mode. The fallback only runs after the lock
       // window expires, so the atomic path had first chance.
       // Source: Phase 47 CONTEXT.md
       if reserve_sol > 0 && reserve_token > 0 {
           let expected = (reserve_token as u128)
               .checked_mul(total_buy_amount as u128)
               .and_then(|n| n.checked_div(
                   (reserve_sol as u128).checked_add(total_buy_amount as u128)?
               ))
               .ok_or(EpochError::Overflow)? as u64;

           let min_output = (expected as u128)
               .checked_mul(CARNAGE_SLIPPAGE_BPS_FALLBACK as u128)
               .and_then(|n| n.checked_div(10_000))
               .ok_or(EpochError::Overflow)? as u64;

           require!(
               bought >= min_output,
               EpochError::CarnageSlippageExceeded
           );
       }
       ```

    4. **Update doc comments** at the top of the file:
       - Change "100-slot deadline" references to "300-slot deadline"
