**Step 1: Install dependencies**

Run from the `app/` directory (use full path for npm: `export PATH="/opt/homebrew/bin:$PATH"`):

```bash
cd app && npm install @privy-io/react-auth @solana/spl-token
```

This installs:
- `@privy-io/react-auth` (^3.x) -- Privy SDK for React. Handles social login, embedded wallets, AND external wallet detection via Wallet Standard.
- `@solana/spl-token` (^0.4.x) -- Provides `TOKEN_2022_PROGRAM_ID` constant needed for balance queries (CRIME, FRAUD, PROFIT are all Token-2022 tokens).

Do NOT install `@solana/wallet-adapter-*` packages. Privy's `toSolanaWalletConnectors()` replaces wallet-adapter entirely.

**Step 2: Create app/providers/providers.tsx**

Create `app/providers/` directory and `providers.tsx`:

```typescript
"use client";

import { PrivyProvider } from "@privy-io/react-auth";
import { toSolanaWalletConnectors } from "@privy-io/react-auth/solana";

const solanaConnectors = toSolanaWalletConnectors();

export default function Providers({ children }: { children: React.ReactNode }) {
  return (
    <PrivyProvider
      appId={process.env.NEXT_PUBLIC_PRIVY_APP_ID!}
      config={{
        appearance: {
          walletChainType: "solana-only",
          walletList: ["phantom", "solflare", "backpack", "detected_solana_wallets"],
        },
        embeddedWallets: {
          solana: { createOnLogin: "users-without-wallets" },
        },
        externalWallets: {
          solana: { connectors: solanaConnectors },
        },
        loginMethods: ["email", "sms", "google"],
      }}
    >
      {children}
    </PrivyProvider>
  );
}
```

**Why each config option:**
- `walletChainType: "solana-only"` -- We don't use EVM. This prevents WalletConnect/wagmi from loading (avoids turbopack issues).
- `walletList` -- Explicitly lists the 3 wallets from CONTEXT.md + `detected_solana_wallets` as catch-all for Wallet Standard compliant wallets.
- `createOnLogin: "users-without-wallets"` -- Only creates embedded wallets for social login users. External wallet users keep their existing wallet, no Privy account needed.
- `solana.connectors: toSolanaWalletConnectors()` -- This is what replaces @solana/wallet-adapter. It uses Wallet Standard under the hood to detect browser extension wallets.
- `loginMethods: ["email", "sms", "google"]` -- Per CONTEXT.md: phone (SMS), email, Google. No Twitter/X.

**Important:** Call `toSolanaWalletConnectors()` OUTSIDE the component (at module scope) so it's only called once, not on every render.

**Step 3: Update app/app/layout.tsx**

Import and wrap children with the Providers component. The layout.tsx stays a server component -- only the Providers component is "use client".

**Step 5: Check turbopack compatibility**

After installing Privy, run `npm run dev` to verify turbopack builds without module resolution errors. Privy's Solana-only config should NOT trigger the WalletConnect/wagmi turbopack issues (those are EVM-specific).

If turbopack fails with module resolution errors from Privy's internals, add the failing module names to `next.config.ts` turbopack.resolveAlias with the existing `./lib/empty.ts` stub pattern. Common candidates:
- `crypto` (if not already shimmed by Next.js)
- `stream` (if Privy's deps need it)
- `http` / `https` (if WalletConnect loads despite solana-only)

Do NOT add stubs preemptively -- only if turbopack actually fails.
