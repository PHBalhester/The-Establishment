**Step 2: Create app/hooks/useProtocolWallet.ts**

This hook wraps Privy's Solana hooks to provide a stable interface for all protocol operations. It must handle both external wallet users (connected via `connectWallet()`) and embedded wallet users (connected via `login()`).

```typescript
"use client";
```

**Interface:**
```typescript
export interface ProtocolWallet {
  publicKey: PublicKey | null;
  connected: boolean;
  ready: boolean; // true when Privy + wallets are initialized
  signTransaction: (tx: Transaction | VersionedTransaction) => Promise<Transaction | VersionedTransaction>;
  signAllTransactions: (txs: (Transaction | VersionedTransaction)[]) => Promise<(Transaction | VersionedTransaction)[]>;
  disconnect: () => void;
}
```

**Implementation details:**

1. Import from `@privy-io/react-auth`:
   - `usePrivy` -- for `authenticated`, `ready`, `logout`

2. Import from `@privy-io/react-auth/solana`:
   - `useWallets` -- for `wallets` array (both external + embedded show up here)
   - `useSignTransaction` -- for signing

3. Import from `@solana/web3.js`:
   - `PublicKey`, `Transaction`, `VersionedTransaction`

4. **Active wallet:** Take `wallets[0]` from `useWallets()`. Privy returns embedded wallets and connected external wallets in this array. First one is active.

5. **publicKey:** `new PublicKey(activeWallet.address)` when wallet exists, null otherwise.

6. **connected:** `true` when `authenticated && activeWallet !== null`. Note: for external wallet users who used `connectWallet()` without `login()`, `authenticated` may be false. Test this during verification. If external wallet users are NOT `authenticated`, adjust to: `connected = !!activeWallet` (wallet exists regardless of auth state).

7. **ready:** `privyReady && walletsReady` (both `usePrivy().ready` and `useWallets().ready`).

8. **signTransaction:**
   - Accept `Transaction | VersionedTransaction`
   - Serialize: For legacy `Transaction`, use `tx.serialize({ requireAllSignatures: false, verifySignatures: false })`. For `VersionedTransaction`, use `tx.serialize()`.
   - Call `privySignTx({ transaction: serializedBytes, wallet: activeWallet })`
   - Privy returns `{ signedTransaction }` as a `SolanaSignedTransaction` type
   - Deserialize back: For legacy, use `Transaction.from(signedBytes)`. For versioned, use `VersionedTransaction.deserialize(signedBytes)`.
   - **Important:** You need to detect the transaction type. The simplest approach: check `tx instanceof VersionedTransaction`. If true, it's versioned. Otherwise, legacy.
   - **Edge case:** Privy's `signTransaction` may accept `Transaction` objects directly (the hook's `SupportedSolanaTransaction` type may include web3.js Transaction). If so, skip manual serialization/deserialization. Test this. The RESEARCH.md notes say it accepts `Uint8Array`, but the actual SDK types may be broader. Use the simplest approach that works.

9. **signAllTransactions:** Iterate `signTransaction` over the array with `Promise.all()`. Privy does not have native batch signing.

10. **disconnect:** Call `logout()` from `usePrivy()`. This disconnects both external and embedded wallets.

**Step 3: Create app/hooks/useTokenBalances.ts**

This hook fetches CRIME, FRAUD, PROFIT (all Token-2022), and SOL balances for a given public key.

```typescript
"use client";
```

**Interface:**
```typescript
export interface TokenBalances {
  sol: number;
  crime: number;
  fraud: number;
  profit: number;
  loading: boolean;
  error: string | null;
  refresh: () => void;
}
```

**Implementation details:**

1. Import `TOKEN_2022_PROGRAM_ID` from `@solana/spl-token` -- this is critical. CRIME, FRAUD, PROFIT are Token-2022 tokens. Using `TOKEN_PROGRAM_ID` (classic SPL) will return zero results.

2. Import `MINTS` from `@dr-fraudsworth/shared` for mint addresses.

3. Import `getConnection` from `@/lib/connection` for RPC access.

4. Import `PublicKey`, `LAMPORTS_PER_SOL` from `@solana/web3.js`.

5. **Fetching pattern:**
   - Use `useEffect` triggered by `publicKey` changes
   - Fetch SOL balance and Token-2022 accounts in parallel:
     ```typescript
     const [solBalance, tokenAccounts] = await Promise.all([
       connection.getBalance(publicKey),
       connection.getParsedTokenAccountsByOwner(publicKey, {
         programId: TOKEN_2022_PROGRAM_ID,
       }),
     ]);
     ```
   - Parse token accounts: iterate `tokenAccounts.value`, extract `account.data.parsed.info.mint` and `account.data.parsed.info.tokenAmount.uiAmount`
   - Match against known mints (MINTS.CRIME.toBase58(), etc.)

6. **refresh():** A `useCallback` that re-fetches all balances. Exposed so UI can trigger manual refresh.

7. **Auto-refresh:** Optionally set up a 30-second polling interval when publicKey is non-null. Use `setInterval` in the useEffect and clean up on unmount. This keeps balances reasonably fresh without WebSocket complexity. (The 30s interval matches what Phase 41 will use for protocol data.)

8. **Error handling:** Catch RPC errors, set `error` state with message. Don't crash the UI.

9. **When publicKey is null:** Return zeros for all balances, loading=false, error=null.
