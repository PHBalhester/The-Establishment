**CU Profiling Pattern (use for EVERY swap test):**
1. Build the instruction via `taxProgram.methods.swapXxx(...).instruction()`
2. Create a Transaction with `ComputeBudgetProgram.setComputeUnitLimit({ units: 1_400_000 })` + the instruction
3. Call `connection.simulateTransaction(tx)` to get `unitsConsumed`
4. Log: `console.log(\`[CU] swap_sol_buy (CRIME): ${unitsConsumed} CU\`)`
5. Then execute the real transaction with `ComputeBudgetProgram.setComputeUnitLimit({ units: Math.ceil(unitsConsumed * 1.1) })`

**Key implementation details:**

For SOL buy/sell:
- Use swap amount of 100_000_000 (0.1 SOL) — matches smoke test
- SOL buy: `swapSolBuy(amount, minOutput=0, isCrime=true/false)` with AtoB direction
- SOL sell: `swapSolSell(amount, minOutput=0, isCrime=true/false)` with BtoA direction
- The trader needs CRIME/FRAUD tokens for sells — either use tokens received from the buy test (run buy before sell in the same describe block), or mint fresh tokens to the trader in protocol-init
- Hook accounts: resolve for the T22 side only (CRIME or FRAUD). WSOL side (SPL Token) has no hooks.
- `tokenProgramA`/`tokenProgramB` must match the canonical ordering (WSOL vs T22 mint)

For PROFIT buy/sell:
- `swapProfitBuy(amount, minOutput=0, isCrime=true/false)` — CRIME/FRAUD input, PROFIT output
- `swapProfitSell(amount, minOutput=0, isCrime=true/false)` — PROFIT input, CRIME/FRAUD output
- Both sides are Token-2022 — need TWO sets of hook accounts
- For dual-hook remaining_accounts: resolve hooks for BOTH mints, concatenate (input hooks first, output hooks second)
- The pool is PureT22Pool so `tokenProgramA` and `tokenProgramB` are both TOKEN_2022_PROGRAM_ID
- Verify escrow balance is UNCHANGED (PROFIT swaps are untaxed)

For tax distribution test:
- Calculate expected tax: `tax_amount = sol_amount * tax_rate_bps / 10000` where default buy rate is 400 bps (4%)
- Read EpochState to get actual current tax rate (it's initialized with defaults)
- Staking portion: `tax_amount * 75 / 100` (truncation matches on-chain math)
- Carnage portion: `tax_amount * 24 / 100`
- Treasury portion: `tax_amount - staking - carnage` (remainder)
- Verify `StakePool.total_rewards_deposited` increased by staking portion via `stakingProgram.account.stakePool.fetch(stakePool)`

**What NOT to do:**
- Do NOT call AMM directly — all swaps go through Tax Program (CPI-only access control)
- Do NOT hardcode 400 bps — read actual rate from EpochState
- Do NOT skip hook accounts for T22 transfers — will cause "Account not found" errors
- Do NOT use `anchor test` — these run via `scripts/run-integration-tests.sh`
