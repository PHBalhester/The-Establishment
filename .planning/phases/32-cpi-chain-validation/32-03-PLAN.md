**Implementation details per test:**

For each test:
1. Generate or derive the wrong authority
2. Build the full instruction with all correct accounts EXCEPT the authority
3. The authority is the only thing wrong -- this isolates the access control check
4. Catch the error and verify it's a ConstraintSeeds error (Anchor error code 2006)

**Error assertion pattern:**
```typescript
try {
  await program.methods
    .protectedInstruction(args)
    .accountsStrict({
      authority: wrongAuthority,
      ...correctAccounts,
    })
    .signers([wrongKeypair]) // Only if authority is a regular keypair
    .rpc();
  expect.fail("Should have failed with seeds constraint violation");
} catch (err: any) {
  // Anchor error 2006 = ConstraintSeeds
  const isConstraintSeeds =
    err.error?.errorCode?.code === "ConstraintSeeds" ||
    err.message?.includes("seeds") ||
    err.message?.includes("2006") ||
    err.logs?.some((log: string) =>
      log.includes("seeds constraint") || log.includes("ConstraintSeeds")
    );
  expect(isConstraintSeeds, `Expected ConstraintSeeds error, got: ${err.message}`).to.be.true;
}
```

**PDA derivation for "wrong program" tests:**

```typescript
// Derive swap_authority from Epoch Program (wrong -- should be Tax Program)
const STAKING_AUTHORITY_SEED = Buffer.from("staking_authority");
const [wrongPDA] = PublicKey.findProgramAddressSync(
  [SWAP_AUTHORITY_SEED],
  epochProgram.programId // Wrong program!

```

**Note on `swap_exempt`:** This is trickier because `swap_exempt` is called via CPI from Epoch Program. To test it directly, you'd need to call it with a fake carnage_authority. Since `swap_exempt` uses `Signer` for carnage_authority AND `seeds::program = epoch_program_id()`, a random keypair will fail the seeds check. The test should call `taxProgram.methods.swapExempt(...)` directly with a random signer as carnage_authority.
