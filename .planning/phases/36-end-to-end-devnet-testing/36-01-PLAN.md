    **e2e-logger.ts** -- Incremental crash-safe file logger:
    - `LogEntry` interface with: timestamp, phase (string like "swap" | "staking" | "carnage" | "epoch"), status ("pass" | "fail" | "known_issue" | "skip"), message, txSignature (optional), details (optional object for snapshots/state diffs)

    - Silent console -- ALL output goes to the log file only. No `console.log` in this module.

    - `E2EUser` interface with: keypair, wsolAccount (PublicKey), crimeAccount (PublicKey), fraudAccount (PublicKey), profitAccount (PublicKey)

      5. Create WSOL account using `createWrappedNativeAccount` with `TOKEN_PROGRAM_ID` (NOT Token-2022)
      6. Add 200ms `sleep` between RPC calls for Helius rate limiting
      7. Return `E2EUser` object

    - Import `sleep` from `../lib/vrf-flow` (reuse existing helper)

    Build the SOL buy swap flow module and wire it into the orchestrator. Then execute the full E2E swap flow on devnet.

    **swap-flow.ts** -- Swap execution + tax verification:

    1. `resolveHookAccounts(connection, source, mint, dest, authority, amount)` helper:
       - Uses `createTransferCheckedWithTransferHookInstruction` from `@solana/spl-token`
       - Returns `transferIx.keys.slice(4)` mapped to `{ pubkey, isSigner, isWritable }`
       - See 36-RESEARCH.md Pattern 3 for exact implementation

    2. `captureBalanceSnapshot(connection, manifest)` helper:
       - Captures SOL balances of: EscrowVault, CarnageSolVault, Treasury (wallet.publicKey)
       - Returns typed snapshot object with lamport values
       - Uses `connection.getBalance()` for each

    3. `executeSolBuySwap(provider, programs, manifest, user, logger, poolName)` main function:
       - `poolName` defaults to "CRIME/SOL"
       - Steps:
         a. Read EpochState to get current tax rates using `readEpochState` from `scripts/vrf/lib/epoch-reader.ts`
         b. Capture pre-swap balance snapshot (escrow, carnage vault, treasury)
         c. Determine swap amount: 0.1 SOL (100_000_000 lamports) input, minimum output 0 (slippage tolerance for testing)
         d. Resolve Transfer Hook remaining_accounts for the output transfer (pool vault B -> user token B)
            - Source: pool vault B address from manifest
            - Mint: CRIME mint from manifest
            - Dest: user's CRIME account
            - Authority: SwapAuthority PDA from manifest
            - Amount: BigInt(0) (amount doesn't matter for hook account resolution, just for the instruction)
         e. Build `taxProgram.methods.swapSolBuy()` instruction with all ~20 accounts:
            - user: testUser.publicKey
            - epochState: manifest.pdas.EpochState
            - swapAuthority: manifest.pdas.SwapAuthority
            - taxAuthority: manifest.pdas.TaxAuthority
            - pool: manifest.pools[poolName].pool
            - poolVaultA: manifest.pools[poolName].vaultA (WSOL vault)
            - poolVaultB: manifest.pools[poolName].vaultB (token vault)
            - mintA: NATIVE_MINT (from @solana/spl-token)
            - mintB: manifest.mints.CRIME (or FRAUD depending on poolName)
            - userTokenA: user.wsolAccount
            - userTokenB: user.crimeAccount (or fraudAccount)
            - stakePool: manifest.pdas.StakePool
            - stakingEscrow: manifest.pdas.EscrowVault
            - carnageVault: manifest.pdas.CarnageSolVault
            - treasury: provider.wallet.publicKey
            - ammProgram: manifest.programs.AMM
            - tokenProgramA: TOKEN_PROGRAM_ID
            - tokenProgramB: TOKEN_2022_PROGRAM_ID
            - systemProgram: SystemProgram.programId
            - stakingProgram: manifest.programs.Staking
         f. Add `.remainingAccounts(hookAccounts)` and `.signers([user.keypair])`
         g. Send with `.rpc()` -- wrap in try/catch with up to 3 retries on transient errors
         h. Wait 200ms then capture post-swap balance snapshot
         i. Log the TX signature to logger

    4. `verifyTaxDistribution(pre, post, amountIn, taxBps, logger)` function:
       - Calculate expected total tax: `Math.floor(amountIn * taxBps / 10_000)`
       - Calculate deltas: escrow, carnage vault, treasury
       - Verify ratios: staking ~75%, carnage ~24%, treasury ~1%
       - Allow small rounding tolerance (1-2 lamports) per tax_math.rs micro-tax rules
       - Log pass/fail with detailed breakdown to logger
       - Return verification result object

    5. `runSwapFlow(provider, programs, manifest, user, logger)` orchestrator function:
       - Calls executeSolBuySwap for CRIME/SOL pool
       - Calls verifyTaxDistribution with results
       - Logs overall swap flow result (pass/fail)
       - Returns boolean success

    **Exports -- Plans 02 and 03 depend on these:**
    - `export` the following functions from swap-flow.ts so they can be imported by staking-flow.ts and carnage-flow.ts:
      - `executeSolBuySwap` -- used by Plan 02 (staking-flow.ts) to generate tax revenue between epochs, and by Plan 03 (carnage-flow.ts) for post-Carnage health checks
      - `runSwapFlow` -- used by the orchestrator
      - `resolveHookAccounts` -- used by Plan 02 (staking-flow.ts) for PROFIT transfer hook resolution during staking
    - Also export the helper types: `BalanceSnapshot` (return type of captureBalanceSnapshot) and `TaxVerificationResult` (return type of verifyTaxDistribution)

    **Wire into devnet-e2e-validation.ts:**
    - Import `runSwapFlow` from `./lib/swap-flow`
    - Call after user setup in `main()`
    - Log result to logger

    - If swap fails, investigate the error, log it as a failure, and note the issue. Common issues to watch for:
      - "IncorrectProgramId" -- wrong token program for WSOL vs Token-2022
      - "Transfer hook program not found" -- missing remaining_accounts
      - Compute budget exceeded -- add `ComputeBudgetProgram.setComputeUnitLimit({ units: 200_000 })` as a pre-instruction

    **Important details:**
    - The `isCrime` boolean parameter for `swapSolBuy` determines which pool: true = CRIME, false = FRAUD. Check the Anchor IDL for the exact method signature.
    - `NATIVE_MINT` is imported from `@solana/spl-token` -- it's the WSOL mint address
