**instructions/swap_sol_pool.rs** -- Create the full swap instruction file. This is the core of Phase 11.

**1. SwapDirection enum** (at the top of the file):
```rust
#[derive(AnchorSerialize, AnchorDeserialize, Clone, Copy, PartialEq, Eq)]
pub enum SwapDirection {
    AtoB,  // variant 0: Token A in, Token B out
    BtoA,  // variant 1: Token B in, Token A out
}
```

**2. SwapSolPool account struct** (position-based naming, per 11-RESEARCH.md Pattern 2):
- `pool: Account<'info, PoolState>` -- mut, PDA validated via seeds = [POOL_SEED, pool.mint_a.as_ref(), pool.mint_b.as_ref()], bump = pool.bump, constraint initialized and not locked
- `vault_a: InterfaceAccount<'info, TokenAccount>` -- mut, constraint vault_a.key() == pool.vault_a
- `vault_b: InterfaceAccount<'info, TokenAccount>` -- mut, constraint vault_b.key() == pool.vault_b
- `mint_a: InterfaceAccount<'info, Mint>` -- constraint mint_a.key() == pool.mint_a
- `mint_b: InterfaceAccount<'info, Mint>` -- constraint mint_b.key() == pool.mint_b
- `user_token_a: InterfaceAccount<'info, TokenAccount>` -- mut
- `user_token_b: InterfaceAccount<'info, TokenAccount>` -- mut
- `user: Signer<'info>` -- authority for user-to-vault transfers
- `token_program_a: Interface<'info, TokenInterface>` -- constraint matches pool.token_program_a
- `token_program_b: Interface<'info, TokenInterface>` -- constraint matches pool.token_program_b

Use `AmmError::PoolNotInitialized` for the initialized constraint, `AmmError::PoolLocked` for the locked constraint, `AmmError::VaultMismatch` for vault constraints, `AmmError::InvalidMint` for mint constraints, and `AmmError::InvalidTokenProgram` for token program constraints.

**Do NOT** add ownership constraints on user token accounts (per 11-RESEARCH.md Open Question 3 -- the token program validates authority during transfer).

**3. Handler function** following strict CEI (Checks-Effects-Interactions) ordering:

```
pub fn handler(
    ctx: Context<SwapSolPool>,
    amount_in: u64,
    direction: SwapDirection,
    minimum_amount_out: u64,
) -> Result<()>
```

**CHECKS phase:**
1. Set reentrancy guard: `pool.locked = true` (Anchor constraint already verified `!pool.locked`)
2. Validate zero amount: `require!(amount_in > 0, AmmError::ZeroAmount)`
3. Direction-based account selection -- match on direction to bind:
   - `reserve_in`, `reserve_out` (u64 from pool state)
   - References to the correct vault, mint, user token account, and token program for input/output sides
   - `input_decimals`, `output_decimals` (from mint accounts)
4. Compute effective_input via `calculate_effective_input(amount_in, pool.lp_fee_bps)` -- map None to AmmError::Overflow
5. Compute amount_out via `calculate_swap_output(reserve_in, reserve_out, effective_input)` -- map None to AmmError::Overflow
6. Slippage check: `require!(amount_out >= minimum_amount_out, AmmError::SlippageExceeded)`
7. Compute lp_fee for event: `amount_in - u64::try_from(effective_input)?` (use checked_sub, map error to Overflow). Note: effective_input is u128 -- convert to u64 first (safe because effective_input <= amount_in which is u64).

**EFFECTS phase (reserve updates):**
8. `new_reserve_in = reserve_in.checked_add(amount_in).ok_or(AmmError::Overflow)?`
9. `new_reserve_out = reserve_out.checked_sub(amount_out).ok_or(AmmError::Overflow)?`
10. Verify k-invariant: `verify_k_invariant(reserve_in, reserve_out, new_reserve_in, new_reserve_out)` -- require `Some(true)`, else `AmmError::KInvariantViolation` or `AmmError::Overflow`
11. Write new reserves to pool state based on direction:
    - AtoB: pool.reserve_a = new_reserve_in, pool.reserve_b = new_reserve_out
    - BtoA: pool.reserve_b = new_reserve_in, pool.reserve_a = new_reserve_out

**INTERACTIONS phase (token transfers):**
12. Build pool PDA signer seeds: `&[POOL_SEED, pool.mint_a.as_ref(), pool.mint_b.as_ref(), &[pool.bump]]`
    - **IMPORTANT:** You must save `pool.mint_a` and `pool.mint_b` into local variables (as `[u8; 32]` via `.to_bytes()`) BEFORE the mutable borrow of `pool` for reserve updates. Anchor's `Account` borrows prevent accessing `pool.mint_a` after `pool.reserve_a` is mutated. Save these as `let mint_a_key = pool.mint_a; let mint_b_key = pool.mint_b; let pool_bump = pool.bump;` at the top of the handler before any mutations.
13. Transfer input (user -> vault): Route through `transfer_t22_checked` or `transfer_spl` based on whether input token program is Token-2022.
    - Use `is_t22()` helper (define as a private function in this file): `fn is_t22(key: &Pubkey) -> bool { *key == anchor_spl::token_2022::ID }`
    - For T22 input: pass `ctx.remaining_accounts` as hook_accounts, `&[]` as signer_seeds (user signs directly)
    - For SPL input: `&[]` as signer_seeds (user signs directly)
14. Transfer output (vault -> user): Route through `transfer_t22_checked` or `transfer_spl` based on whether output token program is Token-2022.
    - For T22 output: pass `ctx.remaining_accounts` as hook_accounts, signer_seeds with pool PDA
    - For SPL output: signer_seeds with pool PDA

**POST-INTERACTION phase:**
15. Clear reentrancy guard: `pool.locked = false` (note: pool is still `&mut` from account struct)
    - **IMPORTANT:** After transfers, you need to re-access pool mutably. Since `pool` was destructured from `ctx.accounts.pool` and we need to clear `locked`, keep the `pool` reference alive across transfers. The cleanest approach: hold `pool` as `&mut ctx.accounts.pool` throughout the handler, save needed immutable values (mint keys, bumps, reserves, token program keys) into local variables before the match block.
16. Get Clock: `let clock = Clock::get()?;`
17. Emit SwapEvent with all fields. For `direction` field, use `direction as u8`. For `input_mint`/`output_mint`, use the mint keys saved in local variables based on direction.
