2. **src/errors.rs** - Create error codes:
   - `AlreadyUpdated` - "Cumulative already updated for this epoch"
   - `Overflow` - "Arithmetic overflow"
   - `NotInitialized` - "Stake pool not initialized"

3. **src/state.rs** - Create StubStakePool account:
   - `cumulative_epochs: u64` - Count of finalize calls
   - `last_epoch: u64` - Last epoch finalized (double-call protection)
   - `total_yield_distributed: u64` - Placeholder for yield tracking
   - `initialized: bool`
   - `bump: u8`
   - Implement `LEN` constant: 8 + 8 + 8 + 1 + 1 = 26 bytes

4. **src/lib.rs** - Create program with two instructions:

   a. **initialize** instruction:
      - Accounts: payer (Signer, mut), stake_pool (init, payer, space, seeds=[b"stake_pool"], bump)
      - System program
      - Handler: Set all fields to 0/false, initialized=true, save bump

   b. **update_cumulative** instruction (CPI-gated):
      - Accounts:
        - `epoch_authority: Signer` with constraint:
          ```rust
          seeds = [b"staking_authority"],
          bump,
          seeds::program = epoch_program_id(),
          ```
        - `stake_pool: Account<StubStakePool>` (mut, seeds=[b"stake_pool"], bump)
      - Instruction data: `epoch: u32`
      - Handler:
        - Require stake_pool.initialized
        - Require epoch as u64 > stake_pool.last_epoch (prevent double-finalization)
        - Increment cumulative_epochs
        - Set last_epoch = epoch as u64
        - Increment total_yield_distributed (placeholder)
        - Emit CumulativeUpdated event

   Add `epoch_program_id()` function returning the actual Epoch Program ID:
   ```rust
   pub fn epoch_program_id() -> Pubkey {
       pubkey!("AH7yaWFUrWmXGDKEwr8w2EMEiH6PxaxE2vT9orjRzvod")
   }
   ```

   Add CumulativeUpdated event:
   ```rust
   #[event]
   pub struct CumulativeUpdated {
       pub epoch: u32,
       pub cumulative_epochs: u64,
   }
   ```

5. **Anchor.toml** - Add stub-staking to [programs.localnet] section with a placeholder ID. Generate a new program ID using `solana-keygen grind --starts-with StUb:1` or use a placeholder like "StubStak1ng111111111111111111111111111111111".

CRITICAL: The seed for epoch_authority in UpdateCumulative MUST be `b"staking_authority"` - this must match what Epoch Program will use when signing the CPI.
