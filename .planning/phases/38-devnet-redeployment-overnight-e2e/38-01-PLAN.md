     ```bash
     set -a && source .env && set +a
     bash scripts/deploy/deploy.sh
     ```
     The `.env` file contains `CLUSTER_URL` pointing to the Helius devnet RPC. `deploy.sh` reads `CLUSTER_URL` from env. It uses `solana program deploy --program-id <keypair>` which handles both fresh deploy and upgrade-in-place natively. The wallet at `keypairs/devnet-wallet.json` is the upgrade authority.

     Note: Transfer Hook program has NOT changed in Phase 37, but deploy.sh deploys all 5. This is fine -- deploying an unchanged program is a no-op upgrade.

  5. Verify all 5 programs are deployed and executable:
     ```bash
     set -a && source .env && set +a && npx tsx scripts/deploy/verify.ts
     ```
     This runs 34 verification checks. All must pass.

  WHY: The Phase 37 fixes are critical security improvements (P0 PDA validation, independent tax rolls, BPF stack overflow fix). Without deploying them, the overnight E2E runner would test the old, broken code.

     const { createE2EUser } = require('./scripts/e2e/lib/user-setup');
