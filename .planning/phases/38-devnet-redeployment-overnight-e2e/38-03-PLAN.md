
  3. Launch the overnight runner in the background using nohup+disown (the proven pattern from Phase 32):
     ```bash
     set -a && source .env && set +a
     nohup npx tsx scripts/e2e/overnight-runner.ts > scripts/e2e/overnight-stdout.log 2>&1 &
     disown
     echo "Runner PID: $!"
     ```
     Save the PID for monitoring.

  4. Wait 2-3 minutes and verify the runner started successfully by checking the first few lines of output:
     ```bash
     head -30 scripts/e2e/overnight-stdout.log
     ```

  5. Check the JSONL log has at least 1 entry:
     ```bash
     wc -l scripts/e2e/overnight-run.jsonl
     head -1 scripts/e2e/overnight-run.jsonl | python3 -m json.tool
     ```
     Should show 1+ lines with a valid EpochRecord JSON object.

     ```bash
     set -a && source .env && set +a
     npx tsx -e "
     const fs = require('fs');
     const { OvernightReporter } = require('./scripts/e2e/lib/overnight-reporter');
     const records = fs.readFileSync('./scripts/e2e/overnight-run.jsonl', 'utf-8')
       .split('\n')
       .filter(l => l.trim())
       .map(l => JSON.parse(l));
     const reporter = new OvernightReporter(records);
     const report = reporter.generate();
     fs.writeFileSync('./Docs/Overnight_Report.md', report, 'utf-8');
     ```
