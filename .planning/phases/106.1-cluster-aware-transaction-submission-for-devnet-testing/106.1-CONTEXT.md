# Phase 106.1: Cluster-aware transaction submission for devnet testing - Context

**Gathered:** 2026-03-26
**Status:** Ready for planning

<domain>
## Phase Boundary

Make all frontend transaction submission paths work on devnet by centralizing the skipPreflight override in useProtocolWallet, without any changes to the mainnet send path. Devnet currently fails because Phantom's signAndSendTransaction silently drops devnet transactions, and v0 (versioned) transactions fail simulation with "Blockhash not found". useProtocolWallet already has cluster-aware branching (mainnet: walletSendTransaction, devnet: signTransaction + sendRawTransaction) — the missing piece is forcing skipPreflight:true on devnet centrally so all callers benefit.

</domain>

<decisions>
## Implementation Decisions

### skipPreflight Strategy
- **Centralized in useProtocolWallet** — force `skipPreflight: true` on devnet regardless of what callers pass. Callers express their mainnet preference; the wallet layer handles the devnet override.
- **Remove redundant local check in executeAtomicRoute** (multi-hop-builder.ts:426-432) — it already calls wallet.sendTransaction via useProtocolWallet, so the central override covers it. Remove the local `isDevnet` logic.
- **Mainnet path untouched** — on mainnet, callers' opts flow through to wallet-adapter's sendTransaction as-is. No override, no forcing skipPreflight:false. Zero mainnet changes.

### Callsite Scope
- All 5 callsites go through useProtocolWallet and automatically get the fix:
  - useSwap.ts single-hop (line 784)
  - useSwap.ts multi-hop via executeAtomicRoute
  - useStaking.ts (line 580)
  - BuyForm.tsx, SellForm.tsx, RefundPanel.tsx (bonding curve — already shipped, no testing needed)
- **No per-callsite changes needed** — the central useProtocolWallet fix handles everything

### Confirmation & Error Surfacing
- **Existing flow is sufficient** — all paths (useSwap, useStaking) already use pollTransactionConfirmation + parseSwapError/parseStakingError to catch on-chain errors after landing
- **No extra devnet logging** — no console.warn for skipPreflight active
- **Keep 90s timeout** — blockhash expiry (~60s) typically triggers first; 90s is a safe ceiling for slow devnet

### Testing
- **All 8 swap directions + staking** on devnet: SOL<->CRIME, SOL<->FRAUD, CRIME<->PROFIT, FRAUD<->PROFIT (both directions = 8), plus stake/unstake PROFIT
- **Railway devnet deployment** (dr-fraudsworth-production.up.railway.app) — test on the real environment, not locally
- Bonding curve forms (BuyForm, SellForm, RefundPanel) get the fix for free but are NOT tested — bonding curve is complete and the protocol is live on mainnet

### Claude's Discretion
- maxRetries default on devnet (whether to standardize or leave caller-controlled)

</decisions>
