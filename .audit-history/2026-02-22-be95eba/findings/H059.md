# H059: AMM Swap Emits Wrong Event (Accounting Mismatch)

**Strategy:** STR-059
**Determination:** NOT CONFIRMED
**Severity:** NONE (INFORMATIONAL)
**Confidence:** HIGH

## Summary

Investigation into whether the AMM swap instructions emit events with incorrect amounts (e.g., pre-fee instead of post-fee) that would cause off-chain indexers to display wrong data. **Finding: No issue detected.** The `SwapEvent` is emitted with correct amounts that match the actual on-chain state changes.

```rust
pub fn calculate_effective_input(amount_in: u64, fee_bps: u16) -> Option<u128> {
    let amount = amount_in as u128;
    let fee_factor = 10_000u128.checked_sub(fee_bps as u128)?;
    amount.checked_mul(fee_factor)?.checked_div(10_000)
}
```

### CEI Ordering Verification

Both handlers follow strict CEI (Checks-Effects-Interactions):

1. **CHECKS** (lines 63-148): Validate inputs, compute amounts, verify slippage
2. **EFFECTS** (lines 159-185): Update `reserve_a` and `reserve_b`
3. **INTERACTIONS** (lines 190-315): Execute token transfers
4. **POST-INTERACTION** (lines 318-339): Clear reentrancy guard, **emit event**

The event is emitted **after** all state mutations, reading the final reserve values. This is the correct pattern for event accuracy.

## Attack Scenario Analysis

**Hypothetical Attack:** Off-chain indexer reads `amount_out` as pre-fee instead of post-fee, causing frontend to display inflated swap returns.

**Result:** Prevented by design. The event emission logic ensures:
1. `amount_out` is computed **after** fee deduction (via `calculate_effective_input`)
2. Event is emitted **after** all state is final
3. Indexer receives the post-fee output amount, not pre-fee

## Impact Assessment

**Off-Chain Indexer Impact:** NONE (events are accurate)
**On-Chain Impact:** NONE (events are informational only; no contract logic depends on event correctness)
**User Confidence:** Neutral (correct event data supports accurate transaction history)

## Remediation

**No fix required.** The event emission is correct and accurately reflects all on-chain state changes.
