# H121: Carnage Burn Amount Calculation — No Error Found

**Strategy:** STR-121: Carnage Burn Amount Calculation Error

**Determination:** NOT A VULNERABILITY — Burn calculation is correct

**Severity:** INFO (Negative Finding)

**Confidence:** VERY HIGH (100%) — Code is straightforward, specification is clear, implementation matches specification exactly

---

## Summary

The hypothesis that "Carnage burns a portion of bought tokens, and if the burn percentage calculation has an error (e.g., burns 100% instead of 50%), the remaining sell step has no tokens" was investigated.

**Finding:** There is **no burn percentage calculation**. The Carnage mechanism is designed to **burn 100% of held tokens** when a Burn action is triggered by VRF. This is the correct and intended behavior per specification. The code implements this correctly.

The remaining sell step does not happen after a burn — the two actions are mutually exclusive within a single Carnage execution:
- **Path 1 (98%):** Burn held tokens → Clear holdings → Buy new target token (BurnThenBuy)
- **Path 2 (2%):** Sell held tokens → Clear holdings → Buy new target token (SellThenBuy)
- **Path 3 (no holdings):** Buy target token only (BuyOnly)

### 3. Mutation Analysis — The Two Actions Are Mutually Exclusive

**Source:** `/Users/mlbob/Projects/Dr Fraudsworth/programs/epoch-program/src/instructions/execute_carnage_atomic.rs` lines 267-336

The code structure shows that Burn and Sell are in a `match` statement with different branches:

```rust
match action {
    CarnageAction::Burn => {
        if held_amount > 0 {
            // ... burn_held_tokens() ...
            tokens_burned = ...;  // LINE 279
        }
    }
    CarnageAction::Sell => {
        if held_amount > 0 {
            // ... execute_sell_swap() ...
            sol_from_sale = ...;  // LINE 321
        }
    }
    CarnageAction::None => {
        // BuyOnly path - no holdings to handle
    }
}
```

**The hypothesis assumes:** "If burn percentage has an error, remaining sell step has no tokens"

**Reality:**
- A single Carnage execution has **ONE action** (Burn XOR Sell XOR None)
- If action = Burn: holdings are burned to zero, execution moves to the buy step
- If action = Sell: holdings are sold to SOL, execution moves to the buy step
- If action = None: no holdings exist, execution moves to the buy step
- **There is no "remaining sell step after burn"** — sell and burn never happen in the same execution

### 4. Arithmetic Safety Verified

The burn amount is used directly without percentage calculations:
- No multiplication: `amount * percentage / 100` — this pattern does NOT appear
- No division: `amount / 2` or similar — this pattern does NOT appear
- No truncation: The full `u64` value is passed to Token-2022 burn instruction
- Arithmetic: Only `checked_add` for statistics accumulation (line 564, 569) — safe
