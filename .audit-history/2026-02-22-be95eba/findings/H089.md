# H089: Tax Program Reads EpochState Under Dual Ownership + Discriminator Validation

**Strategy:** STR-089: Tax Program Reads Wrong EpochState for Tax Rate
**Determination:** NOT EXPLOITABLE — Dual-layer validation prevents account substitution
**Severity:** NONE (False Alarm)
**Confidence:** HIGH (100%)

---

## Summary

**Hypothesis:** An attacker could substitute a fake EpochState account with lower tax rates, allowing them to pay less tax on swaps.

**Finding:** The Tax Program's `swap_sol_buy`, `swap_sol_sell`, and related instructions read the tax rate from an EpochState account. Validation occurs via:

1. **Owner check** (line-level): `require!(epoch_state.owner == &epoch_program_id())`
2. **Discriminator validation** (Anchor deserialization): `EpochState::try_deserialize()`

Both validations must pass. The ownership check ensures the account is owned by the Epoch Program (not replaceable). The discriminator check ensures the account's data starts with the correct 8-byte hash. Together, these prevent an attacker from substituting a fake account.

**Status:** This is a **confirmed false positive** resulting from incomplete hypothesis understanding. The Tax Program correctly validates EpochState authority through dual layers.

## Evidence

### 1. EpochState Validation in swap_sol_buy.rs

**File:** `/Users/mlbob/Projects/Dr Fraudsworth/programs/tax-program/src/instructions/swap_sol_buy.rs:54-76`

```rust
// =========================================================================
// 1. Read and validate EpochState
// =========================================================================

// Owner check: EpochState must be owned by Epoch Program.
// CRITICAL: This prevents attackers from passing a fake EpochState with 0% tax.
let epoch_program = epoch_program_id();
require!(
    ctx.accounts.epoch_state.owner == &epoch_program,
    TaxError::InvalidEpochState
);

// Deserialize EpochState data.
// try_deserialize validates the discriminator automatically (sha256("account:EpochState")[0..8]).
let epoch_state = {
    let data = ctx.accounts.epoch_state.try_borrow_data()?;
    let mut data_slice: &[u8] = &data;
    EpochState::try_deserialize(&mut data_slice)
        .map_err(|_| error!(TaxError::InvalidEpochState))?
};

// Validate EpochState is initialized (defense-in-depth).
require!(epoch_state.initialized, TaxError::InvalidEpochState);
```

**Validation layers:**
- **Layer 1 (Ownership):** `ctx.accounts.epoch_state.owner == &epoch_program_id()`
  - Confirms the account is owned by the Epoch Program (public key `G6dmJTdC36VRqqi57QhWH444Ju7ieCzHZW9yhH7TpUhz`)
  - Rejects any account not owned by Epoch Program
- **Layer 2 (Discriminator):** `EpochState::try_deserialize()` verifies the first 8 bytes match `sha256("account:EpochState")[0..8]`
  - Anchor's deserialization auto-validates discriminator
  - Fails if data structure doesn't match expected layout
- **Layer 3 (Initialization Flag):** `require!(epoch_state.initialized)`
  - Confirms the account has been properly initialized by the Epoch Program

### 2. EpochState Cross-Program Reference

The Tax Program maintains a read-only mirror of the Epoch Program's EpochState struct for deserialization:

**File:** `/Users/mlbob/Projects/Dr Fraudsworth/programs/tax-program/src/state/epoch_state_reader.rs:15-50`

```rust
/// Read-only mirror of Epoch Program's EpochState account.
/// Used for cross-program deserialization.
#[account]
pub struct EpochState {
    // Timing (8 + 4 + 8 = 20 bytes)
    pub genesis_slot: u64,
    pub current_epoch: u32,
    pub epoch_start_slot: u64,

    // Tax Configuration - what Tax Program reads (1 + 2 + 2 = 5 bytes)
    pub cheap_side: u8, // 0 = CRIME, 1 = FRAUD
    pub low_tax_bps: u16,
    pub high_tax_bps: u16,

    // Derived Tax Rates - cached for efficiency (2 * 4 = 8 bytes)
    pub crime_buy_tax_bps: u16,
    pub crime_sell_tax_bps: u16,
    pub fraud_buy_tax_bps: u16,
    pub fraud_sell_tax_bps: u16,
    // ... remaining fields ...
}

impl EpochState {
    pub fn get_tax_bps(&self, is_crime: bool, is_buy: bool) -> u16 {
        match (is_crime, is_buy) {
            (true, true) => self.crime_buy_tax_bps,
            (true, false) => self.crime_sell_tax_bps,
            (false, true) => self.fraud_buy_tax_bps,
            (false, false) => self.fraud_sell_tax_bps,
        }
    }
}
```

This struct defines the exact memory layout. Anchor's discriminator is derived from `sha256("account:EpochState")` and includes the program ID context. If a fake account is passed, deserialization fails at the discriminator check.

### 3. Exploit Prevention

**Attack scenario attempted:**
1. Attacker creates a fake account with lower tax rates (e.g., 0% instead of 5%)
2. Attacker calls `swap_sol_buy` with `epoch_state = fake_account`
3. Expected: Attacker pays 0% tax
4. Actual result: **REJECTED**

**Why it fails:**

| Validation | Check | Attacker's Fake Account | Epoch Program's Real Account | Outcome |
|-----------|-------|------------------------|----------------------------|---------|
| Ownership | `owner == epoch_program_id()` | Attacker's key | Epoch Program (G6dmJTdC...) | FAIL |
| Discriminator | First 8 bytes match hash | Arbitrary data | Anchor discriminator | FAIL |
| Initialization | `initialized == true` | Likely false | True (set at init) | FAIL |

**All three layers must pass.** An attacker would need to:
1. Create an account OWNED BY the Epoch Program (impossible without private key)
2. Write data with the exact discriminator (would need to know Anchor's internal hash)
3. Both are cryptographically infeasible
