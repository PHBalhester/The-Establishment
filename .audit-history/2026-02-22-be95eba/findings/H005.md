# H005: Treasury Address is Compile-Time Constant — No Update Mechanism Exists

**Strategy:** STR-005
**Determination:** NOT VULNERABLE
**Severity:** N/A (False Positive)
**Confidence:** HIGH

## Summary

The hypothesized `update_treasury` instruction **does not exist** in the Tax Program. The treasury address is a compile-time constant defined via `treasury_pubkey()` function with feature-flag gating. There is no admin capability to redirect tax revenue at runtime.

### Treasury Address Implementation

**File:** `/Users/mlbob/Projects/Dr Fraudsworth/programs/tax-program/src/constants.rs:130-145`

```rust
/// Treasury wallet address for protocol revenue.
/// Feature-gated: devnet uses test wallet, mainnet requires explicit address.
///
/// MAINNET: Replace Pubkey::default() with the actual mainnet treasury
/// address before launch. See Docs/mainnet-checklist.md.
#[cfg(feature = "devnet")]
pub fn treasury_pubkey() -> Pubkey {
    Pubkey::from_str("8kPzhQoUPx7LYM18f9TzskW4ZgvGyq4jMPYZikqmHMH4").unwrap()
}

#[cfg(not(feature = "devnet"))]
pub fn treasury_pubkey() -> Pubkey {
    // MAINNET PLACEHOLDER: Must be set before mainnet deployment.
    // Using default (all zeros) to make it obvious if accidentally deployed.
    Pubkey::default()
}
```

This is a **pure function** that returns a hardcoded value based on the `devnet` feature flag. It is evaluated at compile time or as a constant during runtime. There is no state storage for the treasury address — it exists only in the compiled bytecode.

## Root Cause (of False Positive)

The strategy hypothesis assumed the existence of an `update_treasury` instruction based on the **devnet placeholder pattern** and the **admin capability map** documented in ARCHITECTURE.md:

> **File:** `.audit/ARCHITECTURE.md:80-81`
> ```
> | Tax Program | Update treasury address | Treasury update only (devnet placeholder exists) |
> ```

This documentation is **misleading**. The "devnet placeholder" refers to the feature-flag-gated hardcoded address, not a configurable state variable. The phrase "treasury update only" was likely intended to mean "treasury address must be updated in source code before mainnet deployment," not "there exists an instruction to update treasury."

## Why This Design is Secure

1. **No Runtime Modification:** Treasury address cannot be changed without redeploying the Tax Program with updated source code.

2. **Admin Key Compromise Impact:** Even if all admin keys are compromised:
   - AMM admin can only create pools (not redirect treasury)
   - Transfer Hook authority can only modify whitelist (not redirect treasury)
   - There is no admin key that controls treasury routing

3. **Mainnet Deployment Protection:** The `Pubkey::default()` placeholder for mainnet builds ensures that:
   - If deployed without setting the treasury, tax revenue goes to `11111111111111111111111111111111` (System Program)
   - This would cause immediate, obvious failure (transactions rejected or funds unrecoverable)
   - The failure is **fail-loud** rather than silent theft

4. **Immutability Guarantee:** The deploy-and-lock model means once deployed with the correct treasury address, it is permanently fixed.

## Attack Scenario (Hypothetical — Not Possible)

The strategy proposed:
1. Compromise admin key → ❌ **No admin key controls treasury**
2. Call `update_treasury` with attacker address → ❌ **Instruction does not exist**
3. All subsequent trades send 5% to attacker → ❌ **Cannot occur**

## Actual Risk: Deployment Error

The **real risk** is not runtime compromise but **deployment misconfiguration**:

**Risk:** If the Tax Program is deployed to mainnet without replacing `Pubkey::default()` with a real treasury address, the 1% treasury share of all tax revenue would be lost.

**Severity:** CRITICAL (if deployed incorrectly)

**Mitigation Status:** Already flagged as **Finding UA-001 (HIGH)** in `.audit/context/07-upgrade-admin.md:239-266`:

> **Impact**: If the Tax Program is deployed to mainnet without the `devnet` feature flag and without setting the correct treasury address, the 1% treasury share of all tax revenue will be sent to `Pubkey::default()`.

**Recommendation (Already Documented):**
- Add compile-time assertion to fail builds where `treasury_pubkey() == Pubkey::default()` on mainnet
- Use mainnet checklist (`Docs/mainnet-checklist.md:18`) to verify treasury address before deployment

## Conclusion

The strategy hypothesis **STR-005** is based on a misinterpretation of the ARCHITECTURE.md documentation. The Tax Program uses a **compile-time constant** for the treasury address with **no runtime update capability**. The phrase "Update treasury address" in the admin capability map refers to source code modification before deployment, not a privileged instruction.

The actual security concern (deployment with wrong treasury address) is already documented as **UA-001 (HIGH)** with appropriate remediation guidance.

**No new finding required.** The protocol is **NOT VULNERABLE** to runtime treasury redirect attacks.
