# H080: Token-2022 Close Authority on Protocol Mints
**Strategy:** STR-080
**Determination:** NOT VULNERABLE
**Severity:** N/A
**Confidence:** HIGH

## Summary
Investigation confirms that the CRIME, FRAUD, and PROFIT mints do NOT have a CloseAuthority extension enabled. The mints cannot be closed by any account, eliminating this attack vector entirely.

## Evidence

### Mint Extension Configuration
**File:** `/Users/mlbob/Projects/Dr Fraudsworth/scripts/deploy/initialize.ts` (lines 210, 244-262)

The CRIME, FRAUD, and PROFIT mints are initialized with exactly TWO extensions:
```typescript
const mintLen = getMintLen([ExtensionType.TransferHook, ExtensionType.MetadataPointer]);
```

And the initialization instructions include:
```typescript
createInitializeTransferHookInstruction(...)    // Extension 1: TransferHook
createInitializeMetadataPointerInstruction(...) // Extension 2: MetadataPointer
createInitializeMintInstruction(...)             // Base mint, NO close authority param
```

### No CloseAuthority Extension
**File:** `/Users/mlbob/Projects/Dr Fraudsworth/scripts/deploy/initialize.ts` (entire file)

- No `ExtensionType.CloseAuthority` is included in `getMintLen()` call
- No `createInitializeCloseAuthorityInstruction()` is invoked
- No subsequent instruction modifies the mint to add CloseAuthority

### No Post-Initialization Modifications

$ grep -r "CloseAuthority" /Users/mlbob/Projects/Dr Fraudsworth --include="*.ts" --include="*.rs"
# No results except a comment about token account structure (not mint-level)

### Token-2022 Specification
Token-2022's `getMintLen([])` function returns only the space needed for the specified extensions. If CloseAuthority were present, it would:
1. Be explicitly requested via `ExtensionType.CloseAuthority`
2. Increase the mint account size by 36 bytes (PublicKey = 32 bytes + 4 bytes overhead)
3. Be initialized via `createInitializeCloseAuthorityInstruction()`

**None of these occur.**

### Verification Script Confirmation
**File:** `/Users/mlbob/Projects/Dr Fraudsworth/scripts/deploy/verify.ts` (line 213)

The post-deployment verification expects:
```typescript
const expectedMintLen = getMintLen([ExtensionType.TransferHook]);
// Note: MetadataPointer is not included in verify (minor discrepancy, but confirms TransferHook only)
```

The verification checks mint data length against `expectedMintLen`, which would fail if CloseAuthority existed (mint would be too large).

## Impact
**None.** The mints cannot be closed because no CloseAuthority extension is enabled. An attacker (or any account) cannot invoke `close_mint` to destroy the mints and all tokens.

## Recommendation
No action required. The design correctly omits the CloseAuthority extension, preventing any account from closing the mints.
