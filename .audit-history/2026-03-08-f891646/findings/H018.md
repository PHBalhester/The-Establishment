# H018 -- Mainnet Pubkey::default() Placeholders

**Auditor:** Hypothesis Investigator (Upgrade/Admin)
**Severity:** MEDIUM (pre-deployment) / CRITICAL (if missed at mainnet launch)
**Status:** CONFIRMED
**Confidence:** HIGH
**Category:** Upgrade/Admin -- Deployment Safety
**Affected Programs:** Tax Program, Bonding Curve, Conversion Vault

---

## Hypothesis

> 8+ `Pubkey::default()` placeholders across 3 programs will cause mainnet failures or security holes if not replaced.

## Verdict

**CONFIRMED.** There are exactly **7 mainnet-path `Pubkey::default()` placeholders** across 3 programs (plus 3 localnet-path placeholders, totaling 10). The feature-flag replacement mechanism (`#[cfg(feature = "devnet")]`) works correctly for devnet builds. However, there is **no compile-time guard** (e.g., `compile_error!`) that would prevent a mainnet build from silently using these zero-address placeholders. The failure modes range from protocol non-functionality to irrecoverable fund loss.

### Failure Mode Analysis

**Tax Program `treasury_pubkey()`:**
The treasury address is used in an `address = treasury_pubkey()` Anchor constraint on both `swap_sol_buy` (line 455) and `swap_sol_sell` (line 594). With `Pubkey::default()`, the constraint requires the caller to pass the System Program (`11111111...`) as the treasury account. The System Program is a valid, mutable-markable account. Tax SOL transfers via `system_program::transfer` to the all-zeros address would succeed but funds would be **permanently unrecoverable** (no private key exists for the zero address).

**Conversion Vault mints:**
The `initialize` instruction has constraints like `crime_mint.key() == constants::crime_mint()`. With `Pubkey::default()`, the only mint that would pass validation is `11111111...`, which is the System Program -- not a valid SPL mint. The `mint::token_program = token_program` constraint would fail first, making the vault **non-functional** (fails closed -- no fund loss, just bricked).

**Bonding Curve mints:**
The `initialize_curve` instruction has `token_mint.key() == crime_mint() || token_mint.key() == fraud_mint()`. With both returning `Pubkey::default()`, the constraint reduces to `token_mint.key() == 11111...`, which is the System Program. The preceding `mint::token_program` constraint would reject this. Curves would be **non-functional** (fails closed).

**Bonding Curve `epoch_program_id()`:**
The `distribute_tax_escrow` instruction derives the carnage fund PDA as `Pubkey::find_program_address(&[CARNAGE_SOL_VAULT_SEED], &epoch_program_id())`. With `epoch_program_id()` returning `Pubkey::default()`, the derived PDA would be wrong, causing the constraint to fail. Tax escrow lamports would be **permanently locked** in the escrow PDA with no valid recipient.

### Replacement Mechanism

The feature-flag system (`#[cfg(feature = "devnet")]`) is the replacement mechanism. Devnet builds use real addresses (verified in the code). Mainnet builds would use a different set of hardcoded addresses that currently hold `Pubkey::default()`.

**What is missing:**

1. **No `compile_error!` guard.** A mainnet build (no features) compiles successfully with all-zeros addresses. Adding `compile_error!("Mainnet mint addresses not configured")` to the `#[cfg(not(...))]` blocks would make accidental deployment impossible.

2. **No runtime assertion.** None of the placeholder functions assert that the returned pubkey is non-default. A `assert_ne!(pk, Pubkey::default())` would catch this at first use.

3. **The test `test_treasury_pubkey_is_valid` (tax-program/constants.rs:240-245) does NOT check for `Pubkey::default()`.** It merely calls `.to_string()` which always succeeds. The staking program's `test_epoch_program_id` (constants.rs:151-154) is the only test that explicitly asserts `!= "11111111111111111111111111111111"`.

## Severity Justification

Rated **MEDIUM** because:
1. The protocol is currently on devnet where the feature flag correctly provides real addresses.
2. The failure modes are mixed: 2 of 3 programs fail closed (non-functional, no fund loss). Only the Tax Program's treasury placeholder could cause irrecoverable fund loss.
3. The team has documented awareness (comments say "MUST be set before mainnet deployment", mainnet-checklist.md exists).

Would escalate to **CRITICAL** if mainnet deployment were imminent without compile-time guards.

## Recommendation

Add `compile_error!` to every mainnet placeholder function body:

```rust
#[cfg(not(any(feature = "devnet", feature = "localnet")))]
pub fn crime_mint() -> Pubkey {
    compile_error!("MAINNET: crime_mint() address not configured. Set real address before mainnet build.");
}
```

This makes it **impossible** to compile a mainnet build without explicitly setting each address. Zero runtime cost, zero risk of accidental deployment.

For functions that must remain callable at compile time (if used in const contexts), an alternative is a runtime `assert_ne!` on first call. But `compile_error!` is strictly superior when possible.
