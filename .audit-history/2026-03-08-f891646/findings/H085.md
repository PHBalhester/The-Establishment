# H085 -- Token-2022 Transfer Fee Interaction

**Status:** SAFE
**Severity:** N/A
**File:** `scripts/deploy/initialize.ts` (lines 232-285)

## Hypothesis

Are any TransferFeeConfig extensions configured on CRIME/FRAUD/PROFIT mints? If so, swap math and reward calculations would be silently wrong (fees deducted at the token-program level before the program sees them).

## Evidence

### 1. Extension Allocation

At initialize.ts:232:
```typescript
const mintLen = getMintLen([ExtensionType.TransferHook, ExtensionType.MetadataPointer]);
```

Only two extensions are allocated space: **TransferHook** and **MetadataPointer**. `TransferFeeConfig` is NOT in the array. The mint account size is calculated for exactly these two extensions -- there is no space for a TransferFeeConfig extension.

### 2. Initialization Instructions

The mint creation transaction (lines 258-285) contains exactly four instructions per mint:

1. `SystemProgram.createAccount` -- allocates space for the two extensions only
2. `createInitializeTransferHookInstruction` -- configures the transfer hook
3. `createInitializeMetadataPointerInstruction` -- configures metadata pointer
4. `createInitializeMintInstruction` -- finalizes the mint

There is no `createInitializeTransferFeeConfigInstruction` call. This instruction does not appear anywhere in initialize.ts.

### 3. All Three Mints Use Same Template

Lines 248-285 show a single loop iterating over `[CRIME, FRAUD, PROFIT]` keypairs, all using the identical `mintLen` and the same four instructions. No mint receives special treatment.

### 4. Post-Init Immutability

After initialization, Token-2022 extensions cannot be added retroactively -- the account space is fixed at creation. Even if someone wanted to add TransferFeeConfig later, it would be impossible without creating a new mint.

### 5. Mint Authority Burned

Per ARCHITECTURE.md line 22 and INV-9: "initialize.ts burns all 3" mint authorities. Even if a hypothetical extension allowed authority-based reconfiguration, the burned authority prevents it.

## Conclusion

SAFE. No TransferFeeConfig extension is configured on any of the three mints. Only TransferHook and MetadataPointer are allocated and initialized. The mint account sizes are calculated for exactly these two extensions, and Token-2022 does not allow adding extensions after mint creation.
