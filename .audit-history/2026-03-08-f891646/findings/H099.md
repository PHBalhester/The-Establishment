# H099 -- Bonding Curve: Linear vs Quadratic Mismatch

**Status:** FALSE POSITIVE (terminology confusion, not a bug)
**Severity:** Informational
**Component:** `programs/bonding_curve/src/math.rs`, `programs/bonding_curve/src/constants.rs`

## Hypothesis

The ARCHITECTURE.md says "linear price curves" but the arithmetic audit mentions a "quadratic formula." Determine whether there is an actual mismatch between design intent and implementation.

## Findings

### The Price Function Is Linear

The price function is explicitly defined in `math.rs`:

```
P(x) = P_START + (P_END - P_START) * x / TOTAL_FOR_SALE
```

Where:
- `P_START = 900` (lamports per human token)
- `P_END = 3,450` (lamports per human token)
- `TOTAL_FOR_SALE = 460,000,000,000,000` (460M tokens in base units)

This is a **linear** function: price increases linearly with tokens sold. `get_current_price()` confirms this directly.

### The Cost Integral Is Quadratic (By Definition)

The cost to buy `N` tokens starting from position `x1` is the integral of the linear price function:

```
SOL = integral from x1 to x1+N of P(x) dx
    = P_START * N + (P_END - P_START) * N * (2*x1 + N) / (2 * TOTAL_FOR_SALE)
```

This integral contains the term `N * (2*x1 + N)`, which expands to `2*x1*N + N^2` -- a **quadratic** expression in N. This is mathematically inevitable: the area under a linear curve is always a quadratic function.

### The Inverse (SOL -> Tokens) Uses the Quadratic Formula

`calculate_tokens_out()` solves the integral equation for `dx` (tokens out) given a known SOL input `S`. Since the integral is quadratic in `dx`, the solution requires the quadratic formula:

```
dx = [-coef + sqrt(coef^2 + 2 * b_num * S * D * b_den)] / b_num
```

This is the standard positive-root solution of `ax^2 + bx - c = 0`.

### Consistency Check

| Aspect | Description | Consistent? |
|--------|-------------|-------------|
| Price curve shape | Linear: P(x) = mx + b | Yes -- matches "linear price curves" in ARCHITECTURE.md |
| Cost function | Quadratic integral of linear price | Yes -- mathematically correct |
| SOL-to-tokens solver | Quadratic formula (inverse of quadratic integral) | Yes -- correct inverse |
| Rounding direction | `calculate_tokens_out` floors (user gets fewer tokens), `calculate_sol_for_tokens` ceils (user pays more SOL) | Yes -- both protocol-favored |
| Overflow analysis | All intermediates verified to fit u128 | Yes -- documented in comments |

### Why the Terminology Seems Contradictory

- "Linear price curve" refers to the **price function** P(x) -- this is linear.
- "Quadratic formula" refers to the **algebraic solution** for the inverse of the cost integral -- this involves a quadratic equation.

Both statements are correct. There is no mismatch.

## Verdict

**FALSE POSITIVE.** The price curve is linear (P(x) = ax + b). The cost integral is naturally quadratic, and inverting it requires the quadratic formula. Architecture and implementation are fully consistent. The apparent contradiction is a terminology confusion between the price function and its integral.
