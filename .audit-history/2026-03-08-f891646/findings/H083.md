# H083 -- Staking: claim Without Unstaking

**Status:** SAFE
**Severity:** N/A
**File:** `programs/staking/src/instructions/claim.rs`
**Related:** `programs/staking/src/helpers/math.rs` (update_rewards)

## Hypothesis

Can `claim` be used to extract more than earned rewards, or does it leave stale state?

## Evidence

### 1. Reward Calculation (update_rewards called first)

At line 92, `update_rewards(pool, user)` is called before any state reads. This function (math.rs:36-67):

1. Computes `reward_delta = pool.rewards_per_token_stored - user.rewards_per_token_paid`
2. Computes `pending = (balance * reward_delta) / PRECISION` (floor division, favors protocol)
3. Adds `pending` to `user.rewards_earned`
4. Sets `user.rewards_per_token_paid = pool.rewards_per_token_stored` (checkpoint update)

This is the standard Synthetix cumulative-reward pattern. The user only earns rewards proportional to their staked balance and the delta since their last checkpoint.

### 2. rewards_earned Reset

At line 116: `user.rewards_earned = 0;`

This occurs in the EFFECTS section (before the INTERACTIONS transfer at line 140). The full amount is captured at line 98 (`let rewards_to_claim = user.rewards_earned`) and then zeroed. The transfer uses the captured value.

### 3. Transfer Amount

Lines 140-152 transfer exactly `rewards_to_claim` lamports from escrow to user via direct lamport manipulation (program-owned PDA, no CPI). The amount transferred equals the rewards_earned that was just zeroed.

### 4. No Staked Balance Modification

The `claim` handler never modifies `user.staked_balance` or `pool.total_staked`. Staking position is preserved.

### 5. CEI Pattern

State updates (lines 113-135) occur before external interactions (lines 137-152). Even if the lamport transfer could somehow be re-entered (it cannot -- direct lamport manipulation on program-owned accounts), the rewards_earned is already 0.

## Conclusion

SAFE. The `claim` instruction correctly:
- Finalizes pending rewards via checkpoint pattern before reading
- Transfers only the earned amount
- Resets `rewards_earned` to 0 before transfer (CEI)
- Does not modify staked balance
- Uses checked arithmetic throughout
