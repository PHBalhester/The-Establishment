# H071 — No Timelock on Admin Actions
**Status:** CONFIRMED
**Severity:** INFORMATIONAL
**Confidence:** HIGH

## Summary
No admin operation in the protocol has a timelock, delay, or two-step confirmation mechanism. All admin-gated instructions take immediate effect upon a single signature. This is mitigated by the fact that (a) mint authorities are burned, (b) several admin roles can be irreversibly burned, and (c) the protocol intends to revoke program upgrade authority before mainnet.

## Evidence
Every admin-gated instruction executes immediately with a single signer:

**AMM Program:**
- `initialize_pool` — `has_one = admin` on AdminConfig PDA, immediate effect
- `burn_admin` — `has_one = admin`, irreversibly sets admin to `Pubkey::default()`, immediate

**Bonding Curve Program (CRITICAL — no authority verification at all):**
- `initialize_curve` — `authority: Signer<'info>` with NO on-chain check (any signer accepted)
- `fund_curve` — same pattern, any signer
- `start_curve` — same pattern, any signer
- `prepare_transition` — same pattern, any signer
- `withdraw_graduated_sol` — same pattern, any signer (receives ~1000 SOL per curve)
- `close_token_vault` — same pattern, any signer (receives rent)

**Transfer Hook:**
- `initialize_authority` — first-caller-wins, no delay
- `add_whitelist_entry` — stored authority check, immediate
- `burn_authority` — stored authority check, irreversible, immediate

**Staking:**
- Only user-facing cooldown exists (12h unstake cooldown after claim). No admin timelock.

**Epoch Program:**
- `force_carnage` — gated by `cfg(feature = "devnet")` admin key, immediate

Grep for timelock/delay patterns returned zero results across all programs:
```
grep -r "timelock|time_lock|delay.*admin|pending_admin|two_step" programs/ --type rust
# No matches
```

## Recommendation
For the AMM admin (the only legitimate stored admin), this is acceptable given the burn capability. The bonding curve authority gap is a separate critical issue (covered by existing findings). For mainnet, revoking upgrade authority eliminates the most impactful admin vector (program replacement). No timelock is strictly necessary given the protocol's design of burning admin keys post-deployment, but a two-step process for `burn_admin` and `burn_authority` would prevent accidental irreversible burns.
