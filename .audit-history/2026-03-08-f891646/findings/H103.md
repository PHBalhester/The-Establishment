# H103 -- Tax Program: Treasury Address Validation

**Status:** SAFE (devnet) / MAINNET RISK (known placeholder)
**Severity:** Informational (flagged in A-9 already)
**Target:** `programs/tax-program/src/instructions/swap_sol_buy.rs`, `programs/tax-program/src/constants.rs`

## Hypothesis

Check treasury address source. Hardcoded constant or PDA field? Verify it's correct.

## Analysis

### Treasury Address Source

The treasury is a **feature-gated hardcoded address** defined in `constants.rs:135-145`:

```rust
#[cfg(feature = "devnet")]
pub fn treasury_pubkey() -> Pubkey {
    Pubkey::from_str("8kPzhQoUPx7LYM18f9TzskW4ZgvGyq4jMPYZikqmHMH4").unwrap()
}

#[cfg(not(feature = "devnet"))]
pub fn treasury_pubkey() -> Pubkey {
    Pubkey::default()  // MAINNET PLACEHOLDER
}
```

### Validation in swap_sol_buy.rs

The treasury account is validated via Anchor's `address` constraint (swap_sol_buy.rs:453-456):

```rust
#[account(
    mut,
    address = treasury_pubkey() @ TaxError::InvalidTreasury,
)]
pub treasury: AccountInfo<'info>,
```

This is a compile-time constant check. The constraint is evaluated at deserialization time -- if the passed account does not match `treasury_pubkey()`, the instruction fails with `TaxError::InvalidTreasury`. An attacker cannot substitute a different treasury address.

### Devnet Address Verification

The devnet treasury (`8kPzhQoUPx7LYM18f9TzskW4ZgvGyq4jMPYZikqmHMH4`) matches the project's devnet wallet listed in CLAUDE.md and MEMORY.md. Correct.

### Transfer Mechanism

Treasury receives SOL via `system_instruction::transfer` CPI (swap_sol_buy.rs:197-209):
- User signs the transaction (user is the `from` account).
- Amount is `treasury_portion` from `split_distribution(tax_amount)` -- 5% remainder after staking (71%) and carnage (24%).
- Zero-amount guard: `if treasury_portion > 0` (line 196).

### Mainnet Risk (Pre-existing, A-9)

The mainnet build (`cfg(not(feature = "devnet"))`) returns `Pubkey::default()` (all zeros). This is an intentional placeholder per constants.rs:143 comment. If deployed to mainnet without setting the real address:
- The `address = treasury_pubkey()` constraint would require passing the all-zeros pubkey.
- SOL transfers to `Pubkey::default()` would succeed (it's a valid system account).
- Treasury funds would be irrecoverable (no one has the private key for the all-zeros address).

This is already tracked as assumption A-9 in ARCHITECTURE.md.

### swap_sol_sell.rs Consistency

Not fully read but based on the symmetric design pattern, the sell path uses the same `treasury_pubkey()` function and `address` constraint. The constants are shared.

## Verdict

**No vulnerability on devnet.** Treasury address is hardcoded via `treasury_pubkey()` and validated by Anchor's `address` constraint at deserialization time. The devnet address matches the project wallet.

**Mainnet placeholder risk** (A-9): `Pubkey::default()` in the non-devnet build would send treasury funds to an unrecoverable address. This is a pre-existing known issue documented in ARCHITECTURE.md assumption A-9, not a new finding.
