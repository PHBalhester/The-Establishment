# H097 -- Staking: unstake Authority Check

**Status:** SAFE
**Severity:** N/A (properly enforced)
**Component:** `programs/staking/src/instructions/unstake.rs`

## Hypothesis

Does `unstake` properly verify the caller owns the UserStake account? Does it transfer tokens to the correct recipient?

## Evidence

### Ownership enforcement -- three layers

**Layer 1: PDA seeds include signer (line 52)**
```rust
seeds = [USER_STAKE_SEED, user.key().as_ref()],
```
The UserStake PDA is derived from the user's pubkey. An attacker cannot pass their own signer key and resolve to someone else's UserStake PDA.

**Layer 2: Anchor constraint (line 54)**
```rust
constraint = user_stake.owner == user.key() @ StakingError::Unauthorized,
```
Explicit `has_one`-equivalent check comparing the stored `owner` field against the signer.

**Layer 3: Redundant runtime check (lines 113-116)**
```rust
require!(
    ctx.accounts.user_stake.owner == ctx.accounts.user.key(),
    StakingError::Unauthorized
);
```
Belt-and-suspenders check in the handler body.

### Transfer destination validation (lines 59-66)
```rust
#[account(
    mut,
    token::mint = profit_mint,
    token::authority = user,
    token::token_program = token_program,
)]
pub user_token_account: InterfaceAccount<'info, TokenAccount>,
```
The destination token account is constrained to:
- Correct mint (`profit_mint`)
- Authority == `user` (the signer)
- Correct token program

An attacker cannot redirect unstaked tokens to a third party's account because the `token::authority = user` constraint requires the destination account to be owned by the signer.

### Transfer execution (lines 199-209)
The CPI transfers from `stake_vault` to `user_token_account` using the `stake_pool` PDA as signer. The amount transferred equals the validated `amount` variable (which may be adjusted upward for dust prevention but never exceeds `staked_balance`).

## Conclusion

**No vulnerability.** Ownership is triply enforced (PDA derivation, Anchor constraint, runtime check). The transfer destination is validated to belong to the signer via `token::authority = user`. Tokens can only flow to the rightful owner.
