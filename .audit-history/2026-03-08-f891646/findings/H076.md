# H076 -- Bonding Curve: Sell with 0 Tokens

**Status:** Confirmed Safe
**Severity:** Informational
**Confidence:** High

## Summary

A sell with `tokens_to_sell = 0` is explicitly rejected by the non-zero check at Step 3. The instruction correctly guards against zero-amount sells.

## Evidence

**sell.rs (line 137) -- Step 3:**
```rust
require!(tokens_to_sell > 0, CurveError::ZeroAmount);
```

This is checked before any SOL computation or state mutation occurs. The instruction aborts immediately.

**Defense in depth if the check were absent:**

1. **Step 7 (line 166):** `calculate_sol_for_tokens(x2, 0)` would return `sol_gross = 0` (integral over zero-width interval).
2. **Step 8 (line 174-179):** `tax = (0 * 1500 + 9999) / 10000 = 0` (ceil of 0).
3. **Step 9 (line 184-186):** `sol_net = 0 - 0 = 0`.
4. **Step 10 (line 191):** Slippage check: `0 >= minimum_sol_out` -- only passes if caller sets `minimum_sol_out = 0`.
5. **Step 11 (line 200-232):** Token-2022 `transfer_checked` of 0 tokens -- Solana permits this but it is a no-op.
6. **Steps 12-13 (lines 241-248):** Lamport transfers of 0 -- harmless no-ops.
7. **Step 14 (line 254):** `tokens_sold` unchanged (`x2 = x1 - 0 = x1`).
8. **Step 15 (lines 262-273):** Cumulative counters unchanged (adding 0).

Without the explicit check, a 0-token sell would be a harmless no-op (no SOL extracted, no state change) but would still emit misleading events and consume compute. The explicit `ZeroAmount` check is the correct approach.

## Recommendation

None required. The explicit `tokens_to_sell > 0` check is clean and positioned correctly before any computation.
