# H072 — Bonding Curve 15% Sell Tax Flash Loan Prevention
**Status:** NOT VULNERABLE
**Severity:** LOW
**Confidence:** HIGH

## Summary
The 15% sell tax (`SELL_TAX_BPS = 1500`) is unconditionally applied to every sell instruction on the bonding curve. It cannot be bypassed, skipped, or reduced. The tax makes flash loan arbitrage economically infeasible — any round-trip buy+sell within the same transaction loses at minimum 15% of the SOL value, far exceeding any potential arbitrage profit from a linear bonding curve.

## Evidence

**Tax is hardcoded and unconditional** (`programs/bonding_curve/src/instructions/sell.rs:174-179`):
```rust
let tax = sol_gross
    .checked_mul(SELL_TAX_BPS)          // SELL_TAX_BPS = 1_500
    .ok_or(CurveError::Overflow)?
    .checked_add(BPS_DENOMINATOR - 1)   // ceil rounding (protocol-favored)
    .ok_or(CurveError::Overflow)?
    / BPS_DENOMINATOR;                  // BPS_DENOMINATOR = 10_000
```

**Constants** (`programs/bonding_curve/src/constants.rs`):
- `SELL_TAX_BPS: u64 = 1_500` (15%)
- `BPS_DENOMINATOR: u64 = 10_000`

**No bypass paths:**
1. No admin override or configurable tax rate — hardcoded constant
2. No whitelist/exemption mechanism for sell tax
3. Ceil rounding ensures even dust amounts pay at least 1 lamport tax
4. Tax is computed BEFORE slippage check, so user cannot avoid it
5. Tax is transferred to `tax_escrow` PDA (lines 247-248) before state update

**Flash loan break-even analysis:**
- A flash loan attacker buys tokens at price P, pushing the curve up, then sells immediately
- Linear bonding curve: buy pushes price up, sell brings it back down to the same point
- Gross SOL returned from sell = exactly what was paid on buy (same integral, same curve)
- Net SOL returned = gross - 15% tax = 85% of original buy
- **Loss per round-trip: exactly 15% of SOL invested**
- No external price oracle or AMM pool exists during bonding curve phase, so there is no external venue to arbitrage against
- Break-even requires: external profit > 15% of position size (impossible within the bonding curve alone)

**CEI compliance:**
- Token transfer (user -> vault) happens BEFORE SOL transfers (steps 11-13)
- State update (`tokens_sold = x2`) happens AFTER all transfers (step 14)
- Solvency assertion provides defense-in-depth (step 16)

## Recommendation
No action required. The 15% sell tax is a robust anti-flash-loan mechanism. The hardcoded constant, ceil rounding, and lack of any exemption path make it impossible to bypass.
