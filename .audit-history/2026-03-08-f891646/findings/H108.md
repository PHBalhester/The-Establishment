# H108 -- Bonding Curve: CurveState Account Size (192 bytes)

**Status:** REJECTED (Not a vulnerability)
**Severity:** Informational
**Component:** `programs/bonding_curve/src/state.rs`

## Hypothesis

CurveState account size (192 data bytes / 200 total) may not fit all fields, or the Anchor space macro may be incorrect.

## Analysis

### Field-by-Field Verification

| Field | Type | Size |
|-------|------|------|
| Discriminator | [u8; 8] | 8 |
| token | Token (enum) | 1 |
| token_mint | Pubkey | 32 |
| token_vault | Pubkey | 32 |
| sol_vault | Pubkey | 32 |
| tokens_sold | u64 | 8 |
| sol_raised | u64 | 8 |
| status | CurveStatus (enum) | 1 |
| start_slot | u64 | 8 |
| deadline_slot | u64 | 8 |
| participant_count | u32 | 4 |
| tokens_returned | u64 | 8 |
| sol_returned | u64 | 8 |
| tax_collected | u64 | 8 |
| tax_escrow | Pubkey | 32 |
| bump | u8 | 1 |
| escrow_consolidated | bool | 1 |
| **Total** | | **200** (8 disc + 192 data) |

Manual sum: 8 + 1 + 32 + 32 + 32 + 8 + 8 + 1 + 8 + 8 + 4 + 8 + 8 + 8 + 32 + 1 + 1 = **200 bytes**.

### Constant and Allocation

- `CurveState::LEN = 8 + 192 = 200` -- matches.
- `initialize_curve.rs` uses `space = CurveState::LEN` for allocation.

### Test Coverage

Two unit tests provide compile-time guarantees:
1. `curve_state_len_is_200` -- asserts `CurveState::LEN == 200`.
2. `curve_state_serialized_size_matches_len` -- serializes a fully-populated CurveState via Borsh and asserts the serialized byte count equals `LEN - 8` (192 bytes).

The Borsh serialization test is particularly strong because it catches Borsh-level alignment or encoding surprises that a manual byte count might miss (e.g., enum variant encoding differences).

### Borsh Enum Encoding Note

Both `Token` and `CurveStatus` are simple enums with <= 256 variants. Borsh encodes these as a single `u8` tag byte, which matches the 1-byte size assumed in the layout. This is correct.

## Verdict

**No finding.** All 16 fields fit exactly within the 192 data bytes (200 total with discriminator). The `LEN` constant is correct. Two unit tests provide serialization-level verification. The allocation in `initialize_curve.rs` uses `CurveState::LEN` directly.
