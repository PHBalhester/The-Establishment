# S004: Pubkey::default() Compile Guard Implementation

**Determination:** CONFIRMED VIABLE
**Severity:** INFORMATIONAL (recommendation)
**Confidence:** HIGH

## Hypothesis

Can `compile_error!()` be added to mainnet placeholder function bodies without breaking devnet builds?

## Evidence

### Current Pattern (3 programs, 8+ placeholders)

All three feature-gated programs use the same pattern:

**Tax Program** (`programs/tax-program/src/constants.rs:140-145`):
```rust
#[cfg(not(feature = "devnet"))]
pub fn treasury_pubkey() -> Pubkey {
    Pubkey::default()
}
```

**Bonding Curve** (`programs/bonding_curve/src/constants.rs:128-132`):
```rust
#[cfg(not(any(feature = "devnet", feature = "localnet")))]
pub fn crime_mint() -> Pubkey {
    Pubkey::default()
}
```
Plus: `fraud_mint()`, `epoch_program_id()` (3 placeholders total in BC).

**Conversion Vault** (`programs/conversion-vault/src/constants.rs:30-34`):
```rust
#[cfg(not(feature = "devnet"))]
pub fn crime_mint() -> Pubkey {
    Pubkey::default()
}
```
Plus: `fraud_mint()`, `profit_mint()` (3 placeholders total in CV).

**Tax Program** also has `treasury_pubkey()` (1 placeholder).

**Total: 8 Pubkey::default() placeholders across 3 programs.**

### Feature Flag Structure

All three Cargo.toml files define:
```toml
[features]
default = []
devnet = []
```

Bonding Curve and Conversion Vault also define `localnet = []`.

The build pipeline (`scripts/deploy/build.sh:79-86`) selectively rebuilds with `--features devnet`:
```bash
anchor build -p epoch_program -- --features devnet
anchor build -p tax_program -- --features devnet
anchor build -p conversion_vault -- --features devnet
anchor build -p bonding_curve -- --features devnet
```

### compile_error! Compatibility Analysis

The proposed guard:
```rust
#[cfg(not(any(feature = "devnet", feature = "localnet")))]
pub fn crime_mint() -> Pubkey {
    compile_error!("MAINNET: crime_mint() must be set before deployment");
}
```

**This will NOT compile.** `compile_error!()` is a macro that expands at compile time regardless of whether the function is called. The `#[cfg]` gate means the function body only exists in the default (no features) build. But `anchor build` (the first step in `build.sh:69`) builds with NO features before the devnet rebuild at line 81-85.

The first `anchor build` compiles all programs with default features (no `devnet`, no `localnet`). This means the `compile_error!()` would fire on EVERY build, including devnet deployments.

### Correct Implementation

There are two viable approaches:

**Option A: cfg + compile_error at module level (not in function body)**
```rust
#[cfg(not(any(feature = "devnet", feature = "localnet", feature = "mainnet")))]
compile_error!("Must build with --features devnet, localnet, or mainnet");
```
This requires adding a `mainnet` feature and ensuring one feature is always selected. The build script would need `--features mainnet` for production.

**Option B: Runtime assertion with const evaluation**
```rust
#[cfg(not(any(feature = "devnet", feature = "localnet")))]
pub fn crime_mint() -> Pubkey {
    panic!("MAINNET: crime_mint() not configured")
}
```
This compiles but panics at runtime. Less safe than compile-time but compatible with current build flow.

**Option C (Recommended): Restructure build.sh to always require a feature**
Remove the bare `anchor build` and always build with an explicit feature:
```bash
anchor build -- --features devnet    # or localnet, or mainnet
```
Then `compile_error!()` in the default (no-feature) path works as intended because that path is never compiled.

### Impact of Current State

The 8 `Pubkey::default()` placeholders silently produce all-zeros addresses. If accidentally deployed to mainnet:
- Treasury tax distributions (5% of all swap taxes) sent to the zero address (burned permanently)
- Conversion vault mint address checks compare against zero (all conversions fail)
- BC epoch_program_id derives wrong carnage vault PDA (tax escrow distribution fails)

## Recommendation

**Option C is the correct fix.** Modify `build.sh` to require an explicit `--devnet`, `--localnet`, or `--mainnet` flag and never run bare `anchor build`. Then add `compile_error!()` guards in the default feature path. This provides compile-time safety with zero runtime cost.

Estimated effort: 1-2 hours. Changes to `build.sh`, 3 Cargo.toml files, and 8 function bodies.
