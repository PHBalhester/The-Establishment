| **Hypothesis**   | Verify that the `has_one admin` constraint fix for pool creation actually works. |

## Evidence

### 1. AdminConfig Account Struct

`programs/amm/src/state/admin.rs:12-18`:
```rust
pub struct AdminConfig {
    /// The admin pubkey authorized to create pools.
    pub admin: Pubkey,
    pub bump: u8,
}
```

The `admin` field stores the authorized pubkey, set once during `initialize_admin` (which requires ProgramData upgrade authority).

### 2. InitializePool Accounts Struct

`programs/amm/src/instructions/initialize_pool.rs:205-214`:
```rust
#[account(
    seeds = [ADMIN_SEED],
    bump = admin_config.bump,
    has_one = admin @ AmmError::Unauthorized,
)]
pub admin_config: Account<'info, AdminConfig>,

/// The admin signer. Must match admin_config.admin.
pub admin: Signer<'info>,
```

Three constraints work together:
1. **`seeds = [ADMIN_SEED]`** -- ensures the AdminConfig account is the canonical PDA (not a spoofed account).
2. **`bump = admin_config.bump`** -- uses stored bump for canonical derivation.
3. **`has_one = admin @ AmmError::Unauthorized`** -- Anchor verifies `admin_config.admin == admin.key()`. If the signer does not match the stored admin pubkey, the transaction fails with `AmmError::Unauthorized`.

Additionally, `admin` is declared as `Signer<'info>`, so the matching pubkey must actually sign the transaction.

### 3. Burn Admin Prevents Future Pool Creation

`programs/amm/src/instructions/burn_admin.rs` sets `admin_config.admin = Pubkey::default()`. Since `Pubkey::default()` (all zeros) is not signable, `has_one = admin` permanently blocks pool creation after burn.

## Conclusion

**FIXED.** The `has_one = admin` constraint on `AdminConfig` in the `InitializePool` accounts struct correctly restricts pool creation to the admin pubkey stored in `AdminConfig`. The fix matches the model pattern established by `initialize_admin` (ProgramData-gated) and `burn_admin` (irreversible disable). No bypass vector exists.

This confirms the Audit #1 H125 recheck verdict: the access control gap for pool creation has been fully resolved.
